## 摘要
多模态大语言模型（MLLM）推理可分为两个硬件需求截然不同的阶段：视觉编码属于计算密集型，而语言生成属于内存带宽密集型。我们证明，在标准 Transformer KV 缓存机制下，模态边界（即视觉编码器与语言模型之间）是所有保持标准阶段式执行的切分点中跨设备传输量最小的。在此处进行切分可将传输复杂度从 $O(L * s_ctx)$ 字节（阶段级解耦下的 GB 级 KV 缓存）降低至 $O(N_v * d)$ 字节（MB 级嵌入），实现了 $O(L)$ 数量级的降低（L 为 Transformer 深度）。该结论在不同注意力机制（MHA/GQA）、动态视觉分辨率及模型规模下均成立，且优势随模型深度增加而扩大。一个直接的推论是，现有的阶段级解耦系统受限于高带宽互连（如 NVLink），而模态级解耦则支持在通用 PCIe 上进行跨层级异构服务。闭式成本模型显示，在阶段可分离的工作负载下，异构部署具有最优成本效益（预测节省 31.4%，实测节省 40.6%）。我们开发了 HeteroServe，这是一种具备模态级切分和跨层级调度的阶段感知运行时系统，并在 LLaVA-1.5-7B 和 Qwen2.5-VL 上对比 vLLM v0.3.0 进行了评估。在相同的 4xA100 硬件上，引擎优化使吞吐量提升了高达 54%。在固定预算下，异构集群（3.8 万美元）相比同构基准（6.4 万美元）在不增加延迟的前提下，将 Tokens/$ 提升了 37%。

## Abstract
Multimodal large language model (MLLM) inference splits into two phases with opposing hardware demands: vision encoding is compute-bound, while language generation is memory-bandwidth-bound. We show that under standard transformer KV caching, the modality boundary (between vision encoder and language model) minimizes cross-device transfer among all partition points that preserve standard stage-based execution. Partitioning here reduces transfer complexity from $O(L * s_ctx)$ bytes (GB-scale KV caches under stage-level disaggregation) to $O(N_v * d)$ bytes (MB-scale embeddings), an O(L) reduction where L is the transformer depth. The result holds across attention mechanisms (MHA/GQA), dynamic vision resolutions, and model scales, and the advantage grows as models deepen. A direct implication is that existing stage-level disaggregation systems are constrained to high-bandwidth interconnects (e.g., NVLink), whereas modality-level disaggregation enables cross-tier heterogeneous serving over commodity PCIe. A closed-form cost model shows that heterogeneous deployment is cost-optimal under phase-separable workloads (predicts 31.4% savings; observed 40.6%). We build HeteroServe, a phase-aware runtime with modality-level partitioning and cross-tier scheduling, and evaluate it on LLaVA-1.5-7B and Qwen2.5-VL against vLLM v0.3.0. On identical 4xA100 hardware, engine optimizations raise throughput by up to 54%. Under a fixed budget, a heterogeneous cluster (\$38k) improves Tokens/\$ by 37% over a homogeneous baseline (\$64k) without degrading latency.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心矛盾**：多模态大语言模型（MLLM）推理存在硬件需求失配。视觉编码阶段是**计算密集型**（消耗算力，不怎么用带宽），而语言生成阶段是**内存带宽密集型**（消耗 HBM 带宽，算力利用率低）。
*   **现有方案缺陷**：目前的解耦推理系统（如 EPD、Cauchy）多在“阶段级”（如 Prefill 和 Decode 之间）进行切分。这导致跨设备传输的数据量与模型深度 $L$ 成正比，产生 GB 级的 KV 缓存传输需求，迫使系统必须使用昂贵的 NVLink 或 InfiniBand 互连，限制了廉价消费级 GPU 的参与。
*   **研究动机**：探索是否存在一个更优的切分点，能够大幅降低通信开销，从而利用异构 GPU（如用廉价的 RTX 4090 做视觉，昂贵的 A100 做语言）实现高性价比推理。

### 2. 方法论
*   **核心思想**：提出**模态级切分（Modality-Level Partitioning）**。在视觉编码器输出与语言模型输入之间进行切分。
*   **关键技术细节**：
    *   **传输优化理论（定理 1）**：证明了在标准 KV 缓存机制下，模态边界是传输量最小的切分点。传输复杂度从阶段级的 $O(L \cdot s_{ctx})$ 降低到模态级的 $O(N_v \cdot d)$（$L$ 为层数，$s_{ctx}$ 为上下文长度，$N_v$ 为视觉 Token 数，$d$ 为隐藏层维度）。这实现了 $O(L)$ 数量级的通信缩减（实测缩减 12x–196x）。
    *   **HeteroServe 系统架构**：
        *   **跨层级调度**：将视觉任务分配给高算力/低成本的消费级 GPU（Pool C），将语言任务分配给高带宽的中心级 GPU（Pool D）。
        *   **嵌入向量传输协议**：通过通用 PCIe 链路仅传输 MB 级的视觉嵌入向量（Embedding），延迟几乎可忽略。
        *   **跨类型工作窃取（Work Stealing）**：当消费级 GPU 完成视觉编码处于闲置时，利用其预装载的 LLM 权重辅助处理部分语言解码任务，以回收剩余算力。
    *   **成本模型**：推导了闭式成本模型 $\Delta cost = \frac{\rho(1-\gamma)}{\rho+1}$，其中 $\rho$ 是视觉与语言的时间比，$\gamma$ 是两类 GPU 的价格比，用于指导最优部署。

### 6. 主要结论与发现
*   **通信优势**：模态级切分将跨设备传输从 GB 级降至 MB 级，使得在 PCIe 链路上进行异构部署成为可能。

### 7. 优点
*   **理论扎实**：通过数学证明（定理 1）确定了最优切分点，而非单纯的工程尝试。
*   **打破硬件壁垒**：成功证明了消费级 GPU 可以通过 PCIe 有效参与数据中心级的 MLLM 推理任务。
*   **系统完整性**：HeteroServe 不仅实现了切分，还通过“工作窃取”机制解决了异构计算中的负载不均衡问题。
*   **通用性**：方案适用于 MHA/GQA 等多种注意力机制和动态分辨率模型。

### 8. 不足与局限
*   **显存限制**：消费级 GPU（如 4090）的显存（24GB）限制了其能“窃取”的语言解码任务规模，对于超大规模模型（如 70B+），消费级 GPU 可能无法预装载 LLM 权重。
*   **模型覆盖**：虽然理论上支持音频、视频等多模态，但实验仅限于视觉-语言模型（VLM）。
*   **价格敏感性**：成本模型的有效性高度依赖于 GPU 的市场价格比，若未来硬件定价策略改变，性价比优势可能波动。
*   **应用限制**：主要针对高吞吐量的服务场景，对于极低延迟要求的单请求场景，异构传输仍可能引入微小的额外开销。
