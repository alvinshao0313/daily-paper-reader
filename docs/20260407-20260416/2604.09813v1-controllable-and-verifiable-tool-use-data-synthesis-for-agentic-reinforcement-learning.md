## 摘要
现有的合成工具使用语料库主要针对离线监督微调（SFT）设计，而强化学习（RL）则需要支持奖励可检查在线回放（rollouts）的可执行环境。我们提出了 COVERT，这是一个两阶段流水线：首先通过带有多级验证的自我演化合成生成可靠的基础工具使用轨迹，然后应用保持真值（oracle-preserving）的增强手段，系统性地增加环境复杂度。这些增强引入了干扰工具、间接或模糊的用户查询，以及噪声、多格式或错误的工具输出，同时严格保留原始工具调用和最终答案作为地面真值（ground truth）。这种设计使得标准情况可以通过参考匹配进行自动奖励计算，而对于错误检测等特殊行为，则可以通过轻量级的评判辅助验证，从而支持工具调用策略的 RL 优化。在 Qwen2.5-Instruct-14B 上，COVERT-RL 将 BFCL v3 的整体准确率从 56.5 提升至 59.9，将 ACEBench 的准确率从 53.0 提升至 59.3，且在通用能力基准测试上的性能退化极小；当在 SFT 之上叠加时，准确率进一步达到 62.1 和 61.8，证实了叠加增益。这些结果表明，保持真值的合成环境提供了一个实用的 RL 精炼阶段，与 SFT 互补，能够提高在模糊性和不可靠工具反馈下的工具使用鲁棒性。

## Abstract
Existing synthetic tool-use corpora are primarily designed for offline supervised fine-tuning, yet reinforcement learning (RL) requires executable environments that support reward-checkable online rollouts. We propose COVERT, a two-stage pipeline that first generates reliable base tool-use trajectories through self-evolving synthesis with multi-level validation, and then applies oracle-preserving augmentations that systematically increase environmental complexity. These augmentations introduce distractor tools, indirect or ambiguous user queries, and noisy, multi-format, or erroneous tool outputs, while strictly preserving oracle tool calls and final answers as ground truth. This design enables automatic reward computation via reference matching for standard cases and lightweight judge-assisted verification for special behaviors such as error detection, supporting RL optimization of tool-calling policies. On Qwen2.5-Instruct-14B, COVERT-RL improves overall accuracy on BFCL v3 from 56.5 to 59.9 and on ACEBench from 53.0 to 59.3, with minimal regressions on general-ability benchmarks; when stacked on SFT, it further reaches 62.1 and 61.8, confirming additive gains. These results suggest that oracle-preserving synthetic environments offer a practical RL refinement stage, complementary to SFT, for improving tool-use robustness under ambiguity and unreliable tool feedback.

### 1. 核心问题与研究动机
*   **研究背景**：大型语言模型（LLM）的工具使用能力通常通过合成数据的监督微调（SFT）来提升。然而，现有的合成语料库多为静态轨迹，缺乏强化学习（RL）所需的在线回放（rollout）和可验证奖励机制。
*   **核心问题**：现实中的工具调用环境复杂多位（包含干扰工具、模糊查询、噪声输出等），而现有的训练数据难以覆盖这些边缘情况。此外，在没有真实 API 环境的情况下，如何为 RL 提供可扩展且准确的奖励信号是一个重大挑战。
*   **研究目标**：构建一个可控、可验证的合成环境，支持通过强化学习优化工具调用策略，提升模型在复杂环境下的鲁棒性。

### 2. 方法论：COVERT 框架
COVERT 采用两阶段流水线构建 RL 训练环境：
*   **第一阶段：可靠基础轨迹生成**
    *   **自我演化合成**：从少量种子数据开始，利用 LLM 迭代生成新轨迹。
    *   **多级验证**：通过格式检查（JSON 结构）、规则检查（参数类型/枚举约束）和语义检查（LLM 评判工具选择与查询的匹配度）确保基础数据的极高可靠性。
*   **第二阶段：保真增强（Oracle-Preserving Augmentation）**
    *   **核心思想**：在修改环境组件（查询、工具集、输出）的同时，严格保留原始的“真值（Oracle）”工具调用和答案，作为 RL 的奖励基准。
    *   **增强类型**：
        *   **精确可验证类**：引入干扰工具、重写模糊查询、多格式/噪声输出。此类奖励通过参考匹配（Reference Matching）自动计算。
        *   **评判辅助类**：引入工具执行错误（如连接失败）或有问题的查询（如缺少参数）。此类奖励由轻量级 LLM 评判员根据预设的“真值”标签进行验证。
*   **强化学习优化**：
    *   使用 **GRPO** 算法进行策略优化。
    *   **奖励函数**：由格式奖励（$r_{fmt}$）和能力条件下的正确性奖励（$r_{cor}$）组成。

### 3. 实验设计
*   **模型选择**：以 Qwen2.5-Instruct-7B 和 14B 为基础模型。
*   **对比方法**：
    1.  Qwen2.5-Instruct（原始模型）。
    2.  TOUCAN-SFT（在 150 万条真实 MCP 轨迹上微调的模型）。

    4.  TOUCAN-SFT + COVERT-RL（叠加方案）。
*   **评测基准**：

*   **训练时长**：7B 模型单次 RL 运行约 **28 小时**；14B 模型约为其 1.6 倍（约 **45 小时**）。
*   **框架**：基于开源 RL 框架 VERL。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 7B 和 14B 两个参数量级，并在工具使用和通用能力两类共 10 多个基准测试上进行了验证。
*   **充分性**：实验设计了 SFT 与 RL 的叠加对比，证明了 RL 的增量价值。
*   **客观性**：通过在通用能力基准上进行测试，客观评估了 RL 训练是否导致“灾难性遗忘”。

### 6. 主要结论与发现
*   **性能提升**：COVERT-RL 显著提升了工具调用准确率。在 14B 模型上，BFCL v3 提升了 3.4%，ACEBench 提升了 6.3%。
*   **互补性**：RL 与 SFT 具有明显的叠加效应。SFT 提供结构化先验，而 RL 进一步精炼了语义正确性和对异常情况（如干扰项、错误输出）的鲁棒性。
*   **通用能力保持**：与 SFT 相比，COVERT-RL 在提升工具能力的同时，几乎没有造成通用推理和知识能力的退化。
*   **鲁棒性增强**：在“不相关查询检测”和“噪声输出处理”等复杂场景下，RL 带来的增益最为显著。

### 7. 优点
*   **保真增强设计**：巧妙地解决了合成数据中“奖励难计算”的问题，无需真实的 API 后端即可实现大规模 RL 训练。
*   **可控性强**：通过六大增强家族，可以针对性地训练模型处理特定的边缘情况（如参数缺失、工具报错）。
*   **训练效率**：采用 GRPO 算法和轻量级评判员，在有限的算力下实现了显著的性能增益。

### 8. 不足与局限
*   **长程规划限制**：目前的模拟环境主要针对单步或短程多步交互，对于需要复杂状态维护的长程智能体规划（Long-horizon planning）支持有限。
*   **评判员依赖**：部分增强场景依赖 LLM 评判员，虽然有 Oracle 信息辅助，但仍存在潜在的判定偏差，且未进行大规模的人类一致性研究。
*   **任务多样性**：虽然工具域广泛，但主要集中在函数调用，对于更复杂的交互协议（如多智能体协作）尚未深入探讨。
