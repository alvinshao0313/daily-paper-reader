## 摘要
大语言模型（LLM）智能体在复杂的交互式决策任务中展现出了强大的能力。然而，现有的 LLM 智能体通常依赖于日益增长的交互历史，导致计算成本高昂且可扩展性有限。在本文中，我们提出了 STEP-HRL，这是一个分层强化学习（HRL）框架，它通过仅以单步转移而非完整交互历史为条件，实现了步级学习。STEP-HRL 对任务进行分层构建，利用已完成的子任务来表示整个任务的全局进度。通过引入局部进度模块，它还迭代且有选择性地总结每个子任务内的交互历史，从而生成局部进度的紧凑摘要。这些组件共同为高层和底层策略生成了增强的步级转移。在 ScienceWorld 和 ALFWorld 基准测试上的实验结果一致表明，STEP-HRL 在性能和泛化能力方面显著优于基准模型，同时减少了 token 使用量。我们的代码可在 https://github.com/TonyStark042/STEP-HRL 获取。

## Abstract
Large language model (LLM) agents have demonstrated strong capabilities in complex interactive decision-making tasks. However, existing LLM agents typically rely on increasingly long interaction histories, resulting in high computational cost and limited scalability. In this paper, we propose STEP-HRL, a hierarchical reinforcement learning (HRL) framework that enables step-level learning by conditioning only on single-step transitions rather than full interaction histories. STEP-HRL structures tasks hierarchically, using completed subtasks to represent global progress of overall task. By introducing a local progress module, it also iteratively and selectively summarizes interaction history within each subtask to produce a compact summary of local progress. Together, these components yield augmented step-level transitions for both high-level and low-level policies. Experimental results on ScienceWorld and ALFWorld benchmarks consistently demonstrate that STEP-HRL substantially outperforms baselines in terms of performance and generalization while reducing token usage. Our code is available at https://github.com/TonyStark042/STEP-HRL.

#### 1. 核心问题与整体含义（研究动机和背景）
现有的 LLM 智能体在处理复杂决策任务时，通常采用“历史条件化”（History-conditioned）的建模方式，即策略的输出依赖于不断增长的完整交互历史（观察、行动、推理过程）。这种设计存在三大缺陷：
*   **计算成本高昂：** Transformer 的注意力机制随上下文长度呈平方级增长。
*   **冗余信息干扰：** 未经筛选的历史包含大量无关信息，会掩盖关键决策信号，降低推理质量。
*   **可扩展性受限：** 难以处理超长程任务，且信用分配（Credit Assignment）变得极其困难。
**STEP-HRL** 的核心动机是打破对完整历史的依赖，通过分层结构和进度摘要，将长程决策转化为高效的“步级”（Step-level）转移学习。

#### 2. 方法论
STEP-HRL 提出了一种增强步级转移的分层强化学习框架，其核心思想是将任务进度抽象化：
*   **分层结构：**
    *   **高层策略 ($\pi_h$)：** 负责将全局目标分解为一系列子任务（Subtasks）。已完成的子任务序列代表了全局进度。
    *   **底层策略 ($\pi_l$)：** 在当前子任务指导下执行具体动作。
*   **局部进度模块 ($\pi_p$)：** 这是本文的关键创新。它迭代地总结子任务内的交互历史，生成一个紧凑的文本摘要（Local Progress）。底层策略仅需根据“当前观察 + 当前子任务 + 局部进度摘要”即可做出决策，无需回溯完整历史。
*   **训练流程（两阶段）：**
    1.  **行为克隆 (BC)：** 利用专家演示数据（包含子任务和进度标注）对高层、底层和进度策略进行初始化。
    2.  **步级离线强化学习 (Offline RL)：** 采用类似 IQL/ILQL 的隐式值学习方法。通过优势加权回归（Advantage-Weighted Regression）优化策略，使其在离线数据中学习高价值动作。
*   **参数高效设计：** 三个策略共享同一个 LLM 主干参数，但配备独立的价值网络（Critic），以促进跨层级的知识迁移并减少开销。

实验设计非常充分且具有客观性：

*   **泛化能力强：** 在 Unseen 任务上的表现提升尤为明显，证明了步级抽象有助于模型掌握更本质的决策逻辑。

*   **推理稳定性：** 随着交互步数增加，传统方法延迟激增，而 STEP-HRL 保持了稳定的推理速度。

#### 7. 优点
*   **架构创新：** 成功将 LLM 智能体从“长上下文依赖”中解放出来，转向更符合经典 RL 定义的“状态转移”模式。
*   **效率与性能兼得：** 在提升任务成功率的同时，解决了 LLM 智能体落地中昂贵的计算成本问题。
*   **参数共享：** 巧妙地利用一个模型实现三种功能，既节省了显存，又通过多任务学习增强了模型对环境动力学的理解。

#### 8. 不足与局限
*   **对专家数据的依赖：** 构建步级数据需要高质量的子任务分解和进度标注，这在缺乏专家轨迹或任务边界模糊的领域可能难以实施。
