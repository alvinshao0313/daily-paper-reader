## 摘要
由大语言模型（LLM）驱动的自主 Web 智能体在完成复杂浏览器任务方面展现出潜力，但在处理长程工作流时仍面临挑战。一个关键瓶颈是现有技能表述中的具身鸿沟（grounding gap）：文本工作流技能提供自然语言指导但无法直接执行，而基于代码的技能虽可执行但对智能体而言是不透明的，无法提供用于错误恢复或自适应的步骤级理解。我们提出了 WebXSkill，这是一个通过可执行技能弥合该鸿沟的框架，每项技能都将参数化的动作程序与步骤级自然语言指导相结合，从而实现直接执行和智能体驱动的自适应。WebXSkill 分三个阶段运行：技能提取从现有的合成智能体轨迹中挖掘可重用的动作子序列并将其抽象为参数化技能；技能组织将技能索引到基于 URL 的图中以进行上下文感知检索；技能部署提供两种互补模式，即用于全自动多步执行的具身模式（grounded mode），以及技能作为分步指令供智能体通过其原生规划来遵循的引导模式（guided mode）。在 WebArena 和 WebVoyager 上，WebXSkill 相比基线分别将任务成功率提高了 9.8 和 12.9 个百分点，证明了可执行技能对 Web 智能体的有效性。代码已公开于 https://github.com/aiming-lab/WebXSkill。

## Abstract
Autonomous web agents powered by large language models (LLMs) have shown promise in completing complex browser tasks, yet they still struggle with long-horizon workflows. A key bottleneck is the grounding gap in existing skill formulations: textual workflow skills provide natural language guidance but cannot be directly executed, while code-based skills are executable but opaque to the agent, offering no step-level understanding for error recovery or adaptation. We introduce WebXSkill, a framework that bridges this gap with executable skills, each pairing a parameterized action program with step-level natural language guidance, enabling both direct execution and agent-driven adaptation. WebXSkill operates in three stages: skill extraction mines reusable action subsequences from readily available synthetic agent trajectories and abstracts them into parameterized skills, skill organization indexes skills into a URL-based graph for context-aware retrieval, and skill deployment exposes two complementary modes, grounded mode for fully automated multi-step execution and guided mode where skills serve as step-by-step instructions that the agent follows with its native planning. On WebArena and WebVoyager, WebXSkill improves task success rate by up to 9.8 and 12.9 points over the baseline, respectively, demonstrating the effectiveness of executable skills for web agents. The code is publicly available at https://github.com/aiming-lab/WebXSkill.

### 1. 核心问题与整体含义（研究动机和背景）
自主 Web 智能体在处理复杂、长程（long-horizon）工作流时表现脆弱，主要原因在于它们缺乏对交互知识的保留与重用，每次遇到重复流程（如购物结算、后台管理）都需从头规划。
现有研究在“技能”表述上存在**具身鸿沟（Grounding Gap）**：
*   **文本工作流技能：** 提供自然语言指导但不可直接执行，容易产生落地误差。
*   **代码化技能：** 可直接执行但对智能体是“黑盒”，在执行中途出错时，智能体无法理解步骤逻辑，难以进行错误恢复或自适应调整。
**WebXSkill 的核心目标**是构建一种既能直接执行又具备步骤级语义理解的技能框架，提升智能体在复杂网页环境中的效率与鲁棒性。

### 2. 方法论：核心思想与技术细节
WebXSkill 提出了一个三阶段框架，将参数化动作程序与步骤级自然语言指导相结合：

*   **技能提取（Skill Extraction）：**
    *   从低成本的合成智能体轨迹中挖掘可重用的动作子序列。
    *   利用 LLM 将具体动作值抽象为参数（如将特定搜索词变为 `query: str`）。
    *   为每个动作步骤添加语义注释（名称、描述、每步指导）。
    *   通过在线去重和环境验证（Validation）确保技能库的精简与可执行性。
*   **技能组织（Skill Organization）：**
    *   构建**基于 URL 的技能图谱（Skill Graph）**。将技能与通用的 URL 模式（如 `shopping/catalogsearch/*`）关联。
    *   在推理时，根据当前页面 URL 进行上下文感知检索，并结合 DOM 元素检查进一步过滤，确保推荐技能的可用性。
*   **技能部署（Skill Deployment）：** 提供两种互补模式：
    *   **具身模式（Grounded Mode）：** 将技能作为原子工具调用，由运行时自动执行动作序列，追求高效率。
    *   **引导模式（Guided Mode）：** 将技能转化为分步指令，智能体利用原生动作逐步跟随。这种模式保留了智能体的自主性，使其能在页面状态偏离预期时进行自适应调整。

### 6. 主要结论与发现
*   **性能提升：** WebXSkill 在 WebArena 和 WebVoyager 上分别比基线提升了最高 **9.8** 和 **12.9** 个百分点。
*   **模式适配：** 强模型（如 GPT-5）在具身模式下效率更高；而弱模型（如 Qwen）在引导模式下表现更好，因为分步指导弥补了其恢复执行错误的能力不足。
*   **技能迁移性：** 实验证明，在 WebArena 上提取的技能可以有效迁移至 WebVoyager 的真实网站中，尤其是引导模式展现了极强的跨环境适应力。
*   **效率：** 具身模式显著减少了完成任务所需的交互步数。

### 7. 优点（亮点）
*   **弥合鸿沟：** 首次系统性地结合了“可执行代码”与“步骤级语义指导”，解决了黑盒技能不可解释、不可干预的问题。
*   **上下文感知：** 引入基于 URL 的技能图谱，避免了在所有步骤盲目检索全量技能库，提高了检索精度。
*   **低成本获取：** 技能来源于合成轨迹而非昂贵的真实人工演示或大规模自主探索，具有较好的扩展性。

### 8. 不足与局限
*   **推理错误：** 失败分析显示，最大的失败原因仍是智能体的“后置推理”错误（即技能执行成功，但智能体在后续决策或提取最终答案时出错）。
*   **动态环境挑战：** 尽管有引导模式，但在极度动态或频繁变动的 UI 环境下，预定义的动作序列仍可能失效。
*   **依赖强模型：** 技能的提取和高质量参数化高度依赖于底层 LLM 的理解能力，弱模型可能无法生成高质量的技能库。
*   **应用限制：** 目前主要针对 Web 环境，对于需要极高实时性或非结构化 GUI（如移动端 App）的适配性尚待验证。
