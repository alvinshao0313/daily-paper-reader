## 摘要
从复杂的价值流图（VSM）仿真中提取具有可操作性的洞察可能具有挑战性、耗时且容易出错。大语言模型的最新进展为支持用户完成这一任务提供了新途径。虽然现有方法擅长处理原始数据以获取信息，但在结构上难以捕捉区分该领域相似数据源所需的细微情境差异。为解决这一问题，我们提出了一种解耦的两步式智能体架构。通过将编排与数据分析分离，该系统利用注入了领域专家知识的渐进式数据发现。这种架构允许编排层智能地选择数据源，并在保持精简内部上下文的同时，跨数据结构执行多跳推理。来自多个最先进大语言模型的结果证明了该框架的可行性：顶尖模型达到了高达 86% 的准确率，并在多次评估运行中表现出极高的鲁棒性。

## Abstract
Extracting actionable insights from complex value stream map simulations can be challenging, time-consuming, and error-prone. Recent advances in large language models offer new avenues to support users with this task. While existing approaches excel at processing raw data to gain information, they are structurally unfit to pick up on subtle situational differences needed to distinguish similar data sources in this domain. To address this issue, we propose a decoupled, two-step agentic architecture. By separating orchestration from data analysis, the system leverages progressive data discovery infused with domain expert knowledge. This architecture allows the orchestration to intelligently select data sources and perform multi-hop reasoning across data structures while maintaining a slim internal context. Results from multiple state-of-the-art large language models demonstrate the framework's viability: with top-tier models achieving accuracies of up to 86% and demonstrating high robustness across evaluation runs.

### 1. 核心问题与整体含义
*   **研究背景**：价值流图（VSM）仿真是优化生产流程的核心工具，但其生成的仿真结果极其复杂，包含海量的关键绩效指标（KPI）和多变量时间序列数据。
*   **核心问题**：传统的 LLM 或简单的检索增强生成（RAG）在处理此类数据时面临两大挑战：
    1.  **上下文腐烂（Context Rot）**：原始仿真数据量巨大，直接输入会导致模型忽略关键细节。
    2.  **语义模糊性**：仿真输出中存在大量语义相似但情境不同的 KPI，模型难以准确区分并定位正确的数据源。
*   **研究目的**：开发一种能够像领域专家一样，通过多步推理从复杂仿真模型中提取“可操作洞察”的智能体架构。

### 2. 方法论
论文提出了一种**解耦的两步式智能体架构（Decoupled Agentic Architecture）**，核心思想是将逻辑推理与重数据分析分离：
*   **编排智能体（Orchestration Agent）**：作为“大脑”，负责高层规划和多步推理。它不直接接触原始大数据，而是通过领域专家知识（描述数据含义的元数据）来决定需要调用哪些工具。
*   **总结子工作流（Summarization Subworkflow）**：作为“执行手”，专门负责处理具体的、被选中的数据片段。它接收编排智能体的指令和特定数据，完成分析后仅返回精炼的总结。
*   **关键技术细节（四种核心工具）**：
    1.  **节点发现工具**：列出 VSM 中的实体（如仓库、工位）。
    2.  **属性提取工具**：获取特定节点的静态参数（如安全库存、容量）。
    3.  **分类导航工具**：在 11 个仿真数据大类中定位具体的 KPI 数据表。
    4.  **总结工具**：触发子工作流，对选定的动态仿真日志进行针对性分析。

### 3. 实验设计
*   **数据集**：
    *   **开发集**：包含 12 个 VSM 上下文（8 个源自制药、汽车等真实工业场景，4 个通用场景）和 112 个“问题-答案”对。
    *   **评估集**：包含 3 个上下文和 20 个复杂问题（共 47 个测试三元组）。
*   **Benchmark（基准）**：采用 **LLM-as-a-Judge** 机制。首先通过两名人类专家达成共识建立基准评分，随后验证发现 GPT-4.1 的评分与人类专家高度相关（Pearson r=0.8999），故将其作为自动评估器。
*   **对比方法**：对比了四种不同规模的模型：
    *   **Ministral-3-14B**（边缘优化型）
    *   **Qwen3-30B-A3B**（混合专家模型 MoE）
    *   **GLM-5**（744B 参数，40B 激活）
    *   **Claude-Opus 4.6**（顶级闭源模型）

### 4. 资源与算力
*   **硬件环境**：本地模型（Ministral 和 Qwen）部署在两台 **NVIDIA RTX A6000** GPU 上，使用 vLLM 框架加速。
*   **API 服务**：较大模型（GLM-5, Claude）通过厂商 API 访问。
*   **算力消耗与碳足迹**：论文详细估算了实验产生的碳排放（约 100kg CO2eq），并说明通过 Climeworks 进行了碳抵消。

### 5. 实验数量与充分性
*   **实验规模**：针对评估集中的每个样本，每种模型配置均独立运行 **4 次**，以抵消 LLM 的非确定性影响。
*   **充分性评价**：实验设计较为充分。不仅涵盖了从 14B 到 700B+ 不同量级的模型，还通过人类专家验证了自动评估器的公正性。通过标准差（SD）分析了模型的鲁棒性，证明了实验结果的客观性。

### 6. 主要结论与发现
*   **性能分层**：模型表现与规模呈正相关。**Claude-Opus 4.6 表现最佳（85.96分）**，开源模型 GLM-5 紧随其后（83.88分）。
*   **稳定性差异**：顶级模型不仅准确率高，且运行稳定性极强（Claude 的标准差仅 0.86）；而小模型（14B）在编排任务中表现极不稳定（标准差 6.64）。
*   **失败分析**：约 15% 的错误源于编排智能体在导航时选择了错误的数据源，而非子工作流的分析错误，这说明领域知识的语义描述仍有优化空间。
*   **效率发现**：GLM-5 在保持高准确率的同时，平均每个样本消耗的 Token 最少，表现出极高的推理效率。

### 7. 优点
*   **架构优雅**：通过解耦设计有效解决了长上下文带来的“上下文腐烂”问题，使智能体能处理远超其上下文窗口的仿真数据集。
*   **渐进式发现**：模拟了人类专家的诊断过程（先看结构，再找指标，最后看日志），具有很强的可解释性。
*   **评估严谨**：引入人类专家共识来校验 LLM 裁判，确保了定量分析的可信度。

### 8. 不足与局限
*   **小模型适配性差**：目前的解耦架构对 14B 左右的小模型仍显复杂，导致其在编排阶段频繁失败。
*   **成本与延迟**：多步推理导致 Token 消耗量大（平均每样本约 3-4 万 Token），且多次调用 API 带来了较高的响应延迟。
*   **数据长度限制**：虽然架构支持解耦，但单个时间序列数据若超过子工作流模型的上下文窗口，目前尚无成熟的切片处理机制。
*   **闭环缺失**：目前仅实现了“洞察生成”（第二步），尚未实现自动修改参数并触发新仿真的“闭环优化”。
