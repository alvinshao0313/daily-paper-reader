## 摘要
大语言模型 (LLM) 为 Open RAN 中的代理式控制开启了新的可能性，允许运营商以自然语言表达意图，同时将低层执行委托给自主智能体。我们提出了 A1gent，这是一个将推理与实时执行解耦的代理式 RAN 控制栈。一个非实时 (non-RT) 代理式 rApp 将运营商目标编译为有类型的 A1 策略实例，而三个面向任务的近实时 (near-RT) 代理式 xApp 通过具有平面范围执行能力的确定性循环来强制执行这些策略——其中 E2 用于移动性和负载引导，O1 用于能量编排。这种代理式推理与执行的分离确保了 RAN 智能控制器 (RIC) 层级之间可审计的协调，并由编码的护栏和用于冲突治理的固定优先级动作合并器提供支持。随后，一个免训练的自适应策略微调器利用 KPI 记忆在无需重新训练的情况下优化有界参数，从而维持可预测的自适应。通过将意图驱动的规划与确定性的近实时执行相结合，A1gent 推动了 Open RAN 向可验证、自治且可复现的代理式智能迈进。

## Abstract
Large language models (LLMs) open new possibilities for agentic control in Open RAN, allowing operators to express intents in natural language while delegating low-level execution to autonomous agents. We present A1gent, an agentic RAN control stack that decouples reasoning from real-time actuation. A non-RT agentic rApp compiles operator goals into typed A1 policy instances, and three task-oriented near-RT agentic xApps enforce them through a deterministic loop with plane-scoped actuation - E2 for mobility and load steering, and O1 for energy orchestration. This agentic reasoning-execution split ensures auditable coordination between RAN intelligent controller (RIC) tiers, supported by encoded guardrails and a fixed-priority action merger for conflict governance. A training-free adaptive policy tuner then refines bounded parameters using KPI memory without retraining, sustaining predictable adaptation. By integrating intent-driven planning with deterministic near-RT execution, A1gent advances Open RAN toward verifiable, self-governing, and reproducible agentic intelligence.

随着移动网络向 AI 原生（AI-native）演进，运营商希望通过自然语言表达“意图”（Intent），而非微观管理每一个控制循环。然而，在 Open RAN 架构中引入大语言模型（LLM）面临两大挑战：
*   **确定性挑战**：LLM 的推理过程具有非确定性，难以满足近实时（near-RT）控制循环（毫秒级）的严格时延和稳定性要求。
*   **可审计性与安全性**：如何确保 LLM 的推理过程可追溯、可重放，并受到严格的安全护栏约束，防止网络崩溃。
**研究动机**：开发一个名为 **A1GENT** 的框架，通过解耦“推理”与“执行”，在利用 LLM 智能的同时，保证网络控制的确定性、安全性和可审计性。

### 2. 方法论：核心思想与关键技术
A1GENT 提出了一个代理化控制栈，其核心逻辑如下：
*   **推理与执行解耦（Reasoning-Execution Split）**：
    *   **非实时（Non-RT）层级**：部署代理化 **rApp**。它作为“策略编译器”，利用 LLM 将运营商的自然语言意图翻译成有类型的 A1 策略实例（数值化的目标和阈值）。
    *   **近实时（Near-RT）层级**：部署三个任务导向的代理化 **xApp**（QoE 消防员、负载均衡器、节能器）。它们在确定的控制循环中运行，仅根据 rApp 下发的 A1 策略执行具体动作（如切换、偏置调整）。
*   **关键技术细节**：
    *   **IaC（基础设施即代码）护栏**：通过 IaC 定义策略的硬性边界（如切换偏置范围、最小停留时间），确保 LLM 生成的参数不会超出安全范围。
    *   **固定优先级动作合并（Action Merger）**：当多个 xApp 产生冲突时，按“节能 > QoE > 负载”的固定优先级进行仲裁。
    *   **自适应策略微调器（APT）**：一个免训练模块，利用 KPI 记忆（历史性能数据）对 A1 策略中的软参数进行缓慢、可预测的微调，无需重新训练模型。
    *   **多接口协同**：通过 E2 接口处理移动性和负载，通过 O1 接口（经由 SMO）处理基站的休眠与唤醒。

*   **LLM 配置**：使用了 **Gemini 2.5 Flash** 作为意图翻译和 xApp 决策建议的推理引擎。

### 4. 资源与算力
*   **算力说明**：文中明确提到了使用 Gemini 2.5 Flash 模型。
*   **训练时长**：由于该框架强调“免训练”（Training-free）和“自适应微调”，因此**没有提及**大规模的 GPU 训练时长。其核心优势在于利用预训练 LLM 的推理能力和基于记忆的参数调整，而非在线强化学习。

### 5. 实验数量与充分性
*   **实验规模**：实验涵盖了 20 个具有不同移动模型（随机方向和车载模型）的用户设备（UE），并针对三种不同的网络阶段进行了详尽的性能评估。

### 6. 主要结论与发现
*   **性能提升**：在流量激增阶段，A1GENT 将下行吞吐量的底线（Floor）提升了 20%；在恢复阶段，增益达到 24%。

*   **稳定性**：切换停留时间（Dwell Time）有所增加，表明减少了无效的“乒乓切换”，网络运行完全在预设的安全护栏内。

### 7. 优点
*   **架构优雅**：通过 A1 接口作为“窄腰”，巧妙解决了 LLM 非确定性与电信级实时性要求之间的矛盾。
*   **可审计性强**：所有 LLM 的推理结果都转化为具体的 A1 策略值并记录在案，方便回溯和验证。
*   **部署友好**：免训练的 APT 机制降低了对实时算力和标注数据的依赖，更易于在实际网络中落地。
*   **冲突治理**：引入了明确的优先级机制和 IaC 约束，解决了多智能体协同中的安全隐患。
