## 摘要
大语言模型（LLM）正越来越多地被部署为能够改变外部环境的自主智能体的决策核心。然而，在模拟真实世界以客户为中心的问题解决场景的对话基准测试中，由于错误决策的级联效应，这些智能体经常失败。对于参数规模较小、上下文窗口有限且推理预算受限的开源 LLM 而言，这些挑战尤为突出，导致在智能体设置中错误累积增加。为了应对这些挑战，我们提出了故障感知元智能体（FAMA）框架。FAMA 分两个阶段运行：首先，它分析基准智能体的失败轨迹，以识别最普遍的错误；其次，它采用一种编排机制，激活专门针对这些失败而定制的最小化专业智能体子集，在决策步骤之前为工具使用智能体注入针对性的上下文。在开源 LLM 上的实验表明，在各种评估模式下，其性能比标准基准提升了高达 27%。这些结果强调，通过专业智能体针对性地策划上下文以解决常见故障，是构建模拟真实对话场景的可靠、多轮工具使用 LLM 智能体的一项有价值的设计原则。

## Abstract
Large Language Models are being increasingly deployed as the decision-making core of autonomous agents capable of effecting change in external environments. Yet, in conversational benchmarks, which simulate real-world customer-centric issue resolution scenarios, these agents frequently fail due to the cascading effects of incorrect decision-making. These challenges are particularly pronounced for open-source LLMs with smaller parameter sizes, limited context windows, and constrained inference budgets, which contribute to increased error accumulation in agentic settings. To tackle these challenges, we present the Failure-Aware Meta-Agentic (FAMA) framework. FAMA operates in two stages: first, it analyzes failure trajectories from baseline agents to identify the most prevalent errors; second, it employs an orchestration mechanism that activates a minimal subset of specialized agents tailored to address these failures by injecting a targeted context for the tool-use agent before the decision-making step. Experiments across open-source LLMs demonstrate performance gains up to 27% across evaluation modes over standard baselines. These results highlight that targeted curation of context through specialized agents to address common failures is a valuable design principle for building reliable, multi-turn tool-use LLM agents that simulate real-world conversational scenarios.

#### 1. 核心问题与整体含义（研究动机和背景）
在交互式工具使用（Tool Use）环境中，大语言模型（LLM）常被用作自主智能体的决策核心。然而，现有的开源模型（尤其是参数量较小的模型）在处理多轮对话和复杂任务时，容易产生决策错误的“级联效应”：一步错，步步错。
*   **研究动机**：开源模型受限于上下文窗口、推理预算和长程推理能力，在面对复杂 API 输出和严格领域策略时，错误累积严重。
*   **核心问题**：如何通过一种无需训练（Training-free）的方法，识别并针对性地修复开源模型在工具调用中的特定失败模式，从而在资源受限的情况下提升其可靠性。

#### 2. 方法论：核心思想与技术细节
FAMA 提出了一种**“元智能体”（Meta-Agentic）**的设计理念，即框架本身不直接参与环境交互，而是通过观察和诊断主智能体的行为来优化其决策上下文。
*   **核心思想**：分阶段运行。首先识别模型最容易犯的错误类型，然后动态地为该模型配置最精简的辅助代理子集。
*   **算法流程**：
    1.  **阶段 1（故障分析）**：运行基准智能体，收集失败轨迹。
    2.  **阶段 2（诊断与编排）**：
        *   **独立错误分析**：针对四类预定义错误（领域策略违规、复杂输出检索错误、上下文误解、执行不完全）进行专项诊断。
        *   **编排代理（Orchestrator）**：汇总诊断结果，确定失败的根本原因。
        *   **缓解代理（Mitigation Agent）**：根据根本原因，从代理池（如领域约束提取器 DCE、工具建议器 TSA、内存模块等）中挑选最小化的代理组合。
    3.  **阶段 3（执行）**：在选定的辅助代理支持下重新执行任务，通过注入针对性的上下文（Prior Context）引导主模型。

#### 6. 主要结论与发现
*   **动态优于静态**：盲目增加辅助代理（如 IRMA）有时会因上下文过载反而降低性能；FAMA 通过“按需分配”代理，实现了更高的成功率。
*   **性能提升显著**：在开源模型上，FAMA 相比基准方法实现了最高 **27%** 的性能提升。
*   **开源模型的瓶颈**：实验发现开源模型最核心的失败点在于“领域约束违规”和“上下文误解”，因此内存模块和领域约束提取器（DCE）是最常被推荐的组件。
*   **推理模型（Thinking Models）的局限**：带思维链的模型虽然推理强，但在多轮工具调用中容易因 Token 溢出导致失败，FAMA 在这种情况下表现更稳健。

#### 7. 优点
*   **故障驱动（Failure-Aware）**：不同于传统的通用优化，FAMA 能够针对特定模型的弱点“对症下药”。
*   **资源高效性**：通过选择最小代理子集，显著降低了多智能体系统的 Token 冗余和推理延迟。
*   **通用性与可扩展性**：框架不绑定特定模型或领域，可以轻松接入新的辅助代理来应对新出现的错误模式。

#### 8. 不足与局限
*   **依赖预定义代理池**：FAMA 的效果上限受限于现有的辅助代理能力，如果遇到代理池无法解决的新型错误，框架将失效。
*   **诊断成本**：虽然执行阶段高效，但“阶段 1”需要收集失败轨迹，且诊断过程依赖于更强的模型（如 GPT-4o），这在完全离线的私有化部署中可能存在限制。
*   **自动化程度**：目前辅助代理的编写仍需人工设计，尚未实现辅助代理的自动生成或自我进化。
