## 摘要
具身智能体（Embodied AI agents）日益依赖大语言模型（LLMs）进行规划，但每步调用 LLM 会带来严重的延迟和成本。在本文中，我们展示了具身任务表现出极强的规划局部性（plan locality），即下一步规划在很大程度上可以从当前规划中预测。基于此，我们提出了 AgenticCache，这是一个通过重用缓存规划来避免每步调用 LLM 的规划框架。在 AgenticCache 中，每个智能体查询频繁规划转换的运行时缓存，而后台的缓存更新器（Cache Updater）则异步调用 LLM 以验证和优化缓存条目。在四个多智能体具身基准测试中，AgenticCache 在 12 种配置（4 个基准测试 x 3 个模型）下平均将任务成功率提高了 22%，模拟延迟降低了 65%，Token 使用量减少了 50%。因此，基于缓存的规划重用为实现低延迟、低成本的具身智能体提供了一条切实可行的路径。代码可在 https://github.com/hojoonleokim/MLSys26_AgenticCache 获取。

## Abstract
Embodied AI agents increasingly rely on large language models (LLMs) for planning, yet per-step LLM calls impose severe latency and cost. In this paper, we show that embodied tasks exhibit strong plan locality, where the next plan is largely predictable from the current one. Building on this, we introduce AgenticCache, a planning framework that reuses cached plans to avoid per-step LLM calls. In AgenticCache, each agent queries a runtime cache of frequent plan transitions, while a background Cache Updater asynchronously calls the LLM to validate and refine cached entries. Across four multi-agent embodied benchmarks, AgenticCache improves task success rate by 22% on average across 12 configurations (4 benchmarks x 3 models), reduces simulation latency by 65%, and lowers token usage by 50%. Cache-based plan reuse thus offers a practical path to low-latency, low-cost embodied agents. Code is available at https://github.com/hojoonleokim/MLSys26_AgenticCache.

*   **研究动机**：具身智能体（Embodied AI）目前高度依赖大语言模型（LLM）进行高层规划。然而，传统的“感知-规划-行动”循环中，每一步都同步调用 LLM 会导致严重的**高延迟**（智能体需停顿等待响应）和**高昂成本**。
*   **核心发现**：研究者发现具身任务具有显著的“**规划局部性**”（Plan Locality），即智能体的下一步规划在很大程度上可以从当前状态和前一步规划中预测（例如：抓取物体后，下一步通常是移动到目标位置）。
*   **整体含义**：论文提出了 **AgenticCache** 框架，旨在通过缓存频繁的规划转换（Plan Transitions）来减少对 LLM 的实时调用，实现低延迟、低成本且高成功率的具身规划。

### 2. 方法论
*   **核心思想**：将规划过程解耦为“快速的本地缓存查询”和“异步的后台 LLM 更新”。
*   **关键技术细节**：
    *   **运行时缓存（Runtime Cache）**：存储 2-gram 模式的规划转换 $\langle P_i \to P_j \rangle$，并附带任务状态元数据（如步数、持有物品数等）。
    *   **评分机制**：使用复合评分 $S = C \cdot I$ 选择最佳候选规划。其中 $C$（Count）是局部转换频率，$I$（Importance）是全局确认率（由 LLM 验证）。
    *   **异步更新器（Cache Updater）**：在后台运行，不阻塞智能体行动。它负责：
        *   **确认（Confirmation）**：若 LLM 预测与缓存一致，增强该缓存条目。
        *   **修正（Correction）**：若 LLM 发现缓存规划错误，立即替换当前行动并更新缓存。
    *   **查询抑制**：当缓存与 LLM 达成一致或正在执行修正规划时，暂时停止冗余的 LLM 查询以节省 Token。
    *   **预填充（Warm-start）**：可选步骤，利用少量离线成功轨迹初始化缓存。

### 7. 优点
*   **架构创新**：将计算机体系结构中的“分支预测”和“缓存”思想巧妙引入具身智能规划。

*   **异步优势**：解决了具身智能体在等待 LLM 响应时的“死机”问题，显著提升了实时交互感。

### 8. 不足与局限
*   **多智能体冲突**：在极高协同要求的场景下，缓存重用可能导致智能体间出现短暂的资源竞争或步调不一致（虽然更新器最终会修正）。

*   **2-gram 限制**：目前的 2-gram 转换可能无法捕捉极长距离的逻辑依赖。
