## 摘要
具身智能体正从被动推理系统演变为与工具、机器人及物理环境交互的主动执行器。一旦被授予执行权限，核心挑战便在于如何在运行时保持动作的可治理性。现有方法将安全与恢复逻辑嵌入智能体循环内部，导致执行控制难以标准化、审计和适配。本文认为，具身智能不仅需要更强大的智能体，还需要更强大的运行时治理。我们提出了一个策略约束执行框架，将智能体认知与执行监督分离。治理功能被外部化到一个专门的运行时层，负责执行策略检查、能力准入、执行监控、回滚处理和人工干预。我们形式化了具身智能体、具身能力模块（ECMs）与运行时治理层之间的控制边界，并通过跨三个治理维度的 1000 次随机模拟实验进行了验证。结果显示，该框架拦截了 96.2% 的未授权动作，在运行时漂移情况下将不安全持续执行率从 100% 降低至 22.2%，并在完全符合策略的前提下实现了 91.4% 的恢复成功率，显著优于所有基准方法（p<0.001）。通过将运行时治理重新定义为一等系统问题，本文将策略约束执行定位为具身智能体系统的关键设计原则。

## Abstract
Embodied agents are evolving from passive reasoning systems into active executors that interact with tools, robots, and physical environments. Once granted execution authority, the central challenge becomes how to keep actions governable at runtime. Existing approaches embed safety and recovery logic inside the agent loop, making execution control difficult to standardize, audit, and adapt.   This paper argues that embodied intelligence requires not only stronger agents, but stronger runtime governance. We propose a framework for policy-constrained execution that separates agent cognition from execution oversight. Governance is externalized into a dedicated runtime layer performing policy checking, capability admission, execution monitoring, rollback handling, and human override.   We formalize the control boundary among the embodied agent, Embodied Capability Modules (ECMs), and runtime governance layer, and validate through 1000 randomized simulation trials across three governance dimensions. Results show 96.2% interception of unauthorized actions, reduction of unsafe continuation from 100% to 22.2% under runtime drift, and 91.4% recovery success with full policy compliance, substantially outperforming all baselines (p<0.001). By reframing runtime governance as a first-class systems problem, this paper positions policy-constrained execution as a key design principle for embodied agent systems.

*   **研究背景**：现有的具身智能系统通常将安全约束、恢复逻辑和决策限制硬编码在智能体内部（如 Prompt 或控制器逻辑中）。这种“耦合”设计导致执行控制难以标准化、难以在不同环境（如从模拟到现实）间迁移，且缺乏透明的审计机制。
*   **整体含义**：论文主张将具身智能体的“认知（规划）”与“执行监管（治理）”分离，将治理视为一种一等系统问题（First-class systems problem），从而实现“策略约束执行”。

### 2. 方法论：核心思想与技术细节
*   **核心思想**：引入一个独立的**运行时治理层（Runtime Governance Layer）**，介于智能体决策与物理执行底座之间。智能体仅负责“提议”动作，而治理层决定该动作是否“准许”执行。
*   **三实体模型**：
    1.  **具身智能体（Agent）**：负责目标解释、规划和能力选择。
    2.  **能力包（Capability Package）**：模块化的执行单元（如抓取、导航），包含元数据和风险等级。
    3.  **运行时治理层**：负责策略评估、监控和干预。
*   **六大关键组件**：
    1.  **能力准入（Admission）**：检查智能体是否有权调用特定能力。
    2.  **策略守卫（Policy Guard）**：根据环境配置文件（如模拟 vs 现实）动态调整执行参数（如限速、限力）。
    3.  **执行观察者（Watcher）**：运行时监控传感器流，检测漂移、超时或安全违规。
    4.  **恢复与回滚管理器**：当执行失败时，执行策略感知的恢复动作（如重试或回滚到安全状态）。
    5.  **人工覆盖接口（Human Override）**：高风险动作需人工审批，且支持随时接管。
    6.  **审计与遥测**：记录所有提议、决策和干预过程，用于事后追责。
*   **执行流程**：目标解释 -> 能力提议 -> 准入/策略检查 -> 受控启动 -> 运行时监控 -> 异常干预/恢复 -> 审计入库。

### 7. 优点
*   **架构解耦**：将复杂的安全逻辑从智能体模型中剥离，使得同一智能体可以无缝适配不同的安全策略和物理环境。
*   **环境敏感性**：通过“环境配置文件”实现了策略的灵活性，例如在模拟器中允许的操作在现实中可能被禁止或需要审批。
*   **闭环治理**：不仅有预检，还有运行时监控和结构化恢复，形成了完整的执行生命周期管理。

### 8. 不足与局限
*   **模拟器依赖**：所有验证均在模拟环境中完成，尚未在真实物理机器人上进行大规模部署验证，真实传感器的噪声可能影响 Watcher 的准确性。
*   **Watcher 漏报风险**：实验显示运行时违规检测率（RVDR）为 61.3%，意味着仍有约 38.7% 的细微违规未能被检测到，这取决于监控算法的灵敏度设置。
*   **端到端模型适配**：该框架假设能力是模块化的（Capability Packages），对于完全端到端的像素到动作（Pixel-to-Action）模型，如何定义治理边界仍是挑战。
*   **策略编写成本**：治理的效果高度依赖于人工定义的策略质量，策略冲突或定义模糊可能导致系统失效。
