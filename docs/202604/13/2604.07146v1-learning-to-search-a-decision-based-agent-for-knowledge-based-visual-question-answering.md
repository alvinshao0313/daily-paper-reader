## 摘要
基于知识的视觉问答 (KB-VQA) 要求视觉语言模型理解图像并利用外部知识，特别是针对稀有实体和长尾事实。大多数现有的检索增强生成 (RAG) 方法采用固定的流水线，按顺序检索信息、进行过滤，然后生成答案。这种设计使其难以适应多样化的提问类型。此外，它将检索与推理分离，使得模型难以决定何时搜索、如何优化查询或何时停止。结果，检索到的证据往往与问题对齐不佳。为了解决这些局限性，我们将 KB-VQA 重新表述为一个搜索智能体问题，并将解决过程建模为一个多步决策程序。在每一步中，智能体根据其当前的信息状态，从四个动作（回答、图像检索、文本检索和图像描述）中选择其一。我们进一步设计了一个自动化流水线来收集多步轨迹，记录智能体的推理过程、工具使用和中间决策。这些轨迹随后被用作微调的监督信号。在 InfoSeek 和 E-VQA 上的实验表明，我们的方法达到了最先进的性能，持续优于之前的基准模型，并证实了我们框架的有效性。

## Abstract
Knowledge-based visual question answering (KB-VQA) requires vision-language models to understand images and use external knowledge, especially for rare entities and long-tail facts. Most existing retrieval-augmented generation (RAG) methods adopt a fixed pipeline that sequentially retrieves information, filters it, and then produces an answer. Such a design makes it difficult to adapt to diverse question types. Moreover, it separates retrieval from reasoning, making it hard for the model to decide when to search, how to refine queries, or when to stop. As a result, the retrieved evidence is often poorly aligned with the question. To address these limitations, we reformulate KB-VQA as a search-agent problem and model the solving process as a multi-step decision-making procedure. At each step, the agent selects one of four actions-Answer, Image Retrieval, Text Retrieval, and Caption-based on its current information state. We further design an automated pipeline to collect multi-step trajectories that record the agent's reasoning process, tool usage, and intermediate decisions. These trajectories are then used as supervision for fine-tuning. Experiments on InfoSeek and E-VQA demonstrate that our method achieves state-of-the-art performance, consistently outperforming prior baselines and confirming the effectiveness of our framework.

*   **核心问题**：现有的检索增强生成（RAG）方法通常采用**固定的静态流水线**（检索 $\rightarrow$ 过滤 $\rightarrow$ 生成）。这种设计存在三大缺陷：
    1.  **缺乏灵活性**：无法根据问题的难易程度调整检索深度。

*   **研究目标**：将 KB-VQA 重新定义为一个**搜索智能体（Search Agent）问题**，通过多步决策实现动态、自适应的知识获取。

### 2. 方法论：核心思想与技术细节
*   **核心思想**：提出 **DBAgent** 框架，将求解过程建模为马尔可夫决策过程（MDP）。模型不再是被动接收检索结果，而是作为智能体在每一步根据当前信息状态自主选择动作。
*   **关键技术细节**：
    *   **状态表示 ($s_t$)**：包含原始图像 $I$、问题 $q$ 以及之前所有步骤累积的证据 $e_{1...t-1}$。
    *   **动作空间 ($A$)**：
        1.  `Answer`：信息充足时输出最终答案。
        2.  `Text Retrieval`：生成文本查询以检索维基百科段落。
        3.  `Image Retrieval`：基于图像相似度检索相关文档。
        4.  `Caption`：将视觉内容转化为任务导向的描述，作为中间语义表示以优化后续查询。
    *   **自动化轨迹构建**：设计了一个“失败感知”的自动化流水线。通过判断模型在不同阶段的失败原因（如实体识别错误或知识缺失），自动生成包含推理、工具调用和观察结果的多步决策轨迹。
    *   **训练策略**：使用监督微调（SFT）在这些轨迹上训练模型。损失函数仅针对决策相关的 Token 进行计算，屏蔽掉工具返回的证据 Token，以防止模型死记硬背检索内容。

### 6. 主要结论与发现
*   **性能领先**：DBAgent 在 InfoSeek 和 E-VQA 上均显著超越了现有 SOTA 模型（如在 InfoSeek 上比最强基准高出约 6%）。

*   **鲁棒性**：随着外部知识库规模增大（噪声增加），DBAgent 的性能下降速度远慢于传统 RAG 方法。

### 7. 优点（亮点）
*   **范式转变**：将静态 RAG 流程转变为动态智能体决策，更符合人类解决复杂问题的逻辑。
*   **多模态协同**：通过 `Caption` 动作巧妙地解决了视觉识别与文本检索之间的语义鸿沟。
*   **高质量数据生成**：提出了一套自动化的轨迹构建方法，解决了智能体训练数据匮乏的问题。

### 8. 不足与局限
*   **推理延迟**：由于采用多步迭代推理，其推理速度（Latency）明显高于单次生成的流水线模型。
*   **轨迹质量依赖**：训练数据的质量高度依赖于用于生成轨迹的基础模型和 Prompt 策略，可能存在偏差传播。
*   **动作空间受限**：目前仅预定义了四种动作，对于需要调用计算器、知识图谱或外部 API 的更复杂场景，动作空间仍需扩展。
*   **环境简化**：实验主要在相对干净的维基百科环境下进行，在真实世界极度嘈杂或矛盾的信息环境下的表现尚待验证。
