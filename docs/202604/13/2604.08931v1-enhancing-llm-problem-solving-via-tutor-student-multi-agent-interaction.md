## 摘要
人类认知的发展不仅取决于个人努力，还受到结构化社会交互的影响。在这种交互中，基于角色的交流（如导师与学习者之间）能够产生双方单独都无法实现的解决方案。受这些发展原则的启发，我们探讨了导师-学生多智能体系统是否能产生协同效应，从而推动大语言模型（LLM）超越其在现有框架下的能力。为了验证这一想法，我们采用了自动编程问题领域，其中两个由同一 LLM 实例化的智能体被分配了非对称角色：学生智能体负责生成并迭代优化解决方案，而导师智能体在无法获取标准答案的情况下提供结构化的评估反馈。在我们提出的框架（PETITE）中，我们的目标是通过互补角色的结构化交互，从单一模型中挖掘出更好的问题解决性能，而不是依赖更强大的监督模型或异构集成。我们在 APPS 编程基准测试上对模型进行了评估，并与 Self-Consistency、Self-Refine、Multi-Agent Debate 和 Multi-Agent Review 等前沿方法进行了对比。结果表明，我们的模型在消耗显著更少 token 的情况下，达到了相似或更高的准确率。这些结果表明，基于发展心理学的角色差异化交互结构，为通过结构化的类同伴交互增强 LLM 问题解决能力提供了一种有原则且资源高效的范式。

## Abstract
Human cognitive development is shaped not only by individual effort but by structured social interaction, where role-based exchanges such as those between a tutor and a learner, enable solutions that neither could achieve alone. Inspired by these developmental principles, we ask the question whether a tutor-student multi-agent system can create a synergistic effect by pushing Large Language Model (LLM) beyond what it can do within existing frameworks. To test the idea, we adopt autonomous coding problem domain where two agents instantiated from the same LLM assigned asymmetric roles: a student agent generates and iteratively refines solutions, while a tutor agent provides structured evaluative feedback without access to ground-truth answers. In our proposed framework (PETITE), we aim to extract better problem-solving performance from one model by structuring its interaction through complementary roles, rather than relying on stronger supervisory models or heterogeneous ensembles. Our model is evaluated on the APPS coding benchmark against state-of-the-art approaches of Self-Consistency, Self-Refine, Multi-Agent Debate, and Multi-Agent Review. The results show that our model achieves similar or higher accuracy while consuming significantly fewer tokens. These results suggest that developmentally grounded role-differentiated interaction structures provide a principled and resource-efficient paradigm for enhancing LLM problem-solving through structured peer-like interactions. ### 1. 核心问题与研究动机
*   **核心问题**：如何通过结构化的社会化交互（而非仅仅增加模型规模或计算量）来提升 LLM 的逻辑推理和问题解决能力？
*   **研究动机**：
    *   **发展心理学启发**：人类学习依赖于“最近发展区”（ZPD）和角色分化（如导师与学生）。研究表明，即使是水平相当的同伴，通过角色分化（一个生成，一个评估）也能产生显著的学习增益。
    *   **生成与评估的不对称性**：人类在写代码时常忽略自己的错误，但能轻易发现他人的错误。作者假设 LLM 也存在这种不对称性，即同一个模型在扮演“评估者”时能发现其作为“生成者”时忽略的逻辑漏洞。
    *   **现有方法缺陷**：现有的多智能体方法（如辩论或并行评审）往往 Token 消耗巨大，且容易陷入共识陷阱或无效循环。

### 2. 核心方法论：PETITE 框架
*   **核心思想**：将同一个 LLM 实例化为两个具有非对称角色的智能体——**学生（Coder）**和**导师（Tutor）**，通过串行迭代和结构化反馈进行协作。
*   **关键技术细节**：
    *   **角色分工**：
        *   **学生智能体**：负责根据题目要求编写代码，并根据导师反馈进行迭代修改。
        *   **导师智能体**：负责评估代码，识别逻辑错误、边缘案例和语法问题，提供改进指令，并做出“正确/错误”的决策。
    *   **早停机制（Early Stopping）**：这是 PETITE 效率的核心。导师在每轮评估后给出决策。一旦导师判定为“Correct”，流程立即终止。这避免了对简单问题的过度计算，实现了计算资源的自适应分配。
    *   **算法流程**：
        1. 初始化学生和导师的对话上下文。
        2. 学生生成初始代码。
        3. 导师评估代码并提供结构化反馈（优点、缺点、改进建议、决策）。
        4. 若决策为“Correct”或达到最大迭代次数，则停止；否则，学生根据反馈更新代码，进入下一轮。

### 3. 实验设计
*   **数据集**：**APPS 基准测试**（从 5000 多个编程问题中随机抽取 100 个）。涵盖三个难度等级：入门级（Introductory）、面试级（Interview）和竞赛级（Competition）。
*   **对比方法（Baselines）**：
    *   **Self-Consistency (SC)**：采样多个答案并投票。
    *   **Self-Refine (SR)**：模型自我反馈和自我修正。
    *   **Multi-Agent Debate (MAD)**：多个智能体通过辩论达成共识。
    *   **MARS**：多智能体评审系统（包含多个评审者和一个元评审者）。
*   **评估指标**：成功率（通过测试用例的比例）、改进量（最终与初始答案的差值）、Token 消耗量、效率比（成功率/Token 消耗）。

### 4. 资源与算力
*   **硬件**：使用单张 **NVIDIA RTX 4060 (8GB VRAM)** 显卡。
*   **模型配置**：基础模型为 **Qwen2.5-Coder-7B-Instruct**，采用 **4-bit 量化 (NF4)** 以实现高效推理。
*   **参数设置**：最大上下文 4096 tokens，生成上限 2048 tokens，采样温度 0.7-0.8。

### 5. 实验数量与充分性
*   **实验规模**：在 100 个具有代表性的 APPS 问题上进行了测试，涵盖了不同难度分布。
*   **消融/参数实验**：对比了不同迭代次数（1、2、3 次）对性能和成本的影响，并测试了不同温度设置。
*   **客观性**：所有对比方法均使用相同的基础模型和解码参数，确保了比较的公平性。虽然 100 个样本量不算巨大，但覆盖了不同难度等级，具有一定的统计说服力。

### 6. 主要结论与发现
*   **性能优异**：PETITE 在 2 次迭代时达到了最高成功率（31.62%），优于所有基线方法。
*   **极高的 Token 效率**：在达到同等或更高准确率的情况下，PETITE 的 Token 消耗远低于 MAD 和 MARS。例如，在面试级题目中，PETITE 比 MAD 节省了约 81% 的 Token。
*   **自适应性**：早停机制使得简单题目能快速结束，而复杂题目能获得更多迭代机会，这模仿了人类学习的节奏。
*   **角色分化的价值**：实验证明，即使模型能力相同，通过赋予不同的角色（导师 vs 学生），也能比单一模型的自我修正（Self-Refine）产生更好的效果。

### 7. 优点与亮点
*   **理论支撑扎实**：将成熟的发展心理学理论（ZPD、同伴教学法）成功转化为 AI 交互协议。
*   **资源友好**：针对 Token 效率进行了深度优化，非常适合在端侧设备或预算有限的场景下部署。
*   **结构简洁**：相比于复杂的并行评审或多轮辩论，PETITE 的串行导师模式更易于实现且更稳定，减少了共识漂移的风险。

### 8. 不足与局限
*   **导师依赖性**：系统的最终效果高度依赖导师智能体判断的准确性。如果导师误判“正确”，会导致过早停止；如果导师无法识别深层逻辑错误，反馈质量会下降。
*   **样本覆盖面**：实验仅在 100 个 APPS 问题上进行，虽然涵盖了难度梯度，但在更大规模或更多样化的编程任务（如多文件项目）上的泛化能力有待验证。
*   **同质化限制**：导师和学生使用相同的模型，这限制了视角的差异性。未来若引入异构模型（如更强的模型当导师）可能会有更大提升。
