## 摘要
大语言模型智能体接收来自多种来源的指令——包括系统消息、用户提示、工具输出等——每种来源都带有不同程度的信任和权威。当这些指令发生冲突时，模型必须可靠地遵循最高权限的指令，以确保安全性和有效性。目前的主流范式，即指令层级（IH），假设存在一组由固定角色标签（如系统 > 用户）定义的、数量较少（通常少于五个）的固定权限级别。这对于现实世界的智能体场景而言是不够的，因为在这些场景中，冲突可能源于更多的来源和上下文。在这项工作中，我们提出了多层级指令架构（ManyIH），这是一种用于解决具有任意多个权限级别的指令之间冲突的范式。我们推出了 ManyIH-Bench，这是首个针对 ManyIH 的基准测试。ManyIH-Bench 要求模型处理多达 12 个具有不同权限的冲突指令层级，包含 853 个智能体任务（427 个编程任务和 426 个指令遵循任务）。ManyIH-Bench 通过组合由大语言模型开发并经人工验证的约束条件，创建了涵盖 46 个真实世界智能体的真实且具有挑战性的测试用例。我们的实验表明，当指令冲突规模扩大时，即使是目前的最前沿模型表现也较差（准确率约为 40%）。这项工作强调了在智能体场景中，迫切需要专门针对细粒度、可扩展指令冲突解决的方法。

## Abstract
Large language model agents receive instructions from many sources-system messages, user prompts, tool outputs, and more-each carrying different levels of trust and authority. When these instructions conflict, models must reliably follow the highest-privilege instruction to remain safe and effective. The dominant paradigm, instruction hierarchy (IH), assumes a fixed, small set of privilege levels (typically fewer than five) defined by rigid role labels (e.g., system > user). This is inadequate for real-world agentic settings, where conflicts can arise across far more sources and contexts. In this work, we propose Many-Tier Instruction Hierarchy (ManyIH), a paradigm for resolving instruction conflicts among instructions with arbitrarily many privilege levels. We introduce ManyIH-Bench, the first benchmark for ManyIH. ManyIH-Bench requires models to navigate up to 12 levels of conflicting instructions with varying privileges, comprising 853 agentic tasks (427 coding and 426 instruction-following). ManyIH-Bench composes constraints developed by LLMs and verified by humans to create realistic and difficult test cases spanning 46 real-world agents. Our experiments show that even the current frontier models perform poorly (~40% accuracy) when instruction conflict scales. This work underscores the urgent need for methods that explicitly target fine-grained, scalable instruction conflict resolution in agentic settings.

#### 1. 核心问题与整体含义（研究动机和背景）
在复杂的智能体（Agent）场景中，大语言模型（LLM）会接收来自系统消息、用户输入、工具返回结果、子智能体反馈等多种来源的指令。这些指令往往存在冲突（例如：系统要求保持简洁，但工具输出要求详细总结）。
*   **现有局限**：目前的“指令层级（Instruction Hierarchy, IH）”范式通常依赖于固定的角色标签（如 `System > User > Tool`），层级极少（通常 < 5 层），且无法区分同一角色内的不同优先级。
*   **研究动机**：现实应用需要更细粒度、动态且可扩展的优先级管理机制。本文旨在打破“固定且少量层级”的瓶颈，提出一种能够处理任意多层级指令冲突的新范式。

#### 2. 方法论：核心思想与关键技术
论文提出了 **Many-Tier Instruction Hierarchy (ManyIH)** 范式，其核心思想是将指令的优先级与其在聊天模板中的“角色标签”解耦。
*   **权限提示接口 (Privilege Prompt Interface, PPI)**：通过在提示词中直接嵌入权限标识符来动态定义优先级。
    *   **序数接口 (Ordinal Interface)**：使用 `[[Privilege N]]` 标签，数值越小优先级越高（如 Privilege 1 覆盖 Privilege 2）。
    *   **标量接口 (Scalar Interface)**：使用 `[[z=V]]` 标签，数值越大优先级越高（如 z=95 覆盖 z=60）。
*   **元指令 (Meta-instruction)**：在提示词开头加入一段说明，告知模型如何解析这些标签并处理冲突（例如：“如果两个指令冲突，遵循权限数值较小的那个”）。
*   **冲突解决逻辑**：模型需要根据 PPI 提供的动态数值进行推理，识别哪些指令是“获胜指令（Active）”，哪些是“被抑制指令（Suppressed）”，并仅执行获胜指令。

#### 6. 主要结论与发现
*   **性能鸿沟**：即使是目前最强的模型（如 Gemini 3.1 Pro），在 ManyIH-Bench 上的准确率也仅为 **42.7%**。这与它们在标准两层级 IH 测试中 >99% 的表现形成鲜明对比。
*   **扩展性差**：随着指令层级（Tiers）的增加，所有模型的准确率均呈现明显的单调下降趋势。
*   **格式高度敏感**：仅改变权限的表达格式（序数 vs 标量），GPT-5.4 和 Opus 4.6 的准确率就下降了超过 **8%**。
*   **推理努力的局限**：虽然增加推理努力（Reasoning Effort）能提升性能，但即便在最高努力水平下，模型也无法完全解决多层级冲突问题。

#### 7. 优点
*   **前瞻性问题定义**：首次识别并定义了智能体场景中“固定层级瓶颈”的问题，提出了更符合现实需求的 ManyIH 范式。
*   **高质量 Benchmark**：ManyIH-Bench 结合了自动化验证和人工校验，任务设计严谨，且能独立控制指令难度与冲突复杂度。
*   **解耦设计**：PPI 的设计允许在不重新训练模型的情况下，在推理阶段灵活调整权限结构。

#### 8. 不足与局限
*   **安全性假设**：ManyIH 假设权限标签是由受信任的系统分配的。如果攻击者能够注入高权限标签（即“权限注入攻击”），该机制可能成为新的攻击面。
*   **评估闭环**：虽然 IF 子集经过人工验证，但部分 LLM 评估器（LLM-judge）在处理极度模糊的指令时仍可能存在偏差。
*   **应用限制**：目前仅在编程和通用指令遵循上做了验证，对于多模态智能体或极长上下文智能体的适用性尚待研究。
