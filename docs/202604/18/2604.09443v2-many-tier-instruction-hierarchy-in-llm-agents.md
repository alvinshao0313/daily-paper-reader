## 摘要
大语言模型（LLM）智能体接收来自多种来源的指令，包括系统消息、用户提示、工具输出、其他智能体等，每种来源都带有不同程度的信任和权威。当这些指令发生冲突时，智能体必须可靠地遵循最高权限的指令，以确保安全和有效。目前的主流范式——指令层级（Instruction Hierarchy, IH）——假设存在一组固定的、少量的权限级别（通常少于五个），并由僵化的角色标签（如系统 > 用户）定义。这对于现实世界的智能体场景是不够的，因为在这些场景中，冲突可能源于更多的来源和上下文。在这项工作中，我们提出了多层级指令层级（Many-Tier Instruction Hierarchy, ManyIH），这是一种用于解决具有任意多个权限级别的指令冲突的范式。我们推出了 ManyIH-Bench，这是首个针对 ManyIH 的基准测试。ManyIH-Bench 要求模型处理多达 12 个级别的具有不同权限的冲突指令，包含 853 个智能体任务（427 个编程任务和 426 个指令遵循任务）。ManyIH-Bench 结合了由 LLM 开发并经人工验证的约束条件，创建了涵盖 46 个现实世界智能体的真实且具有挑战性的测试用例。实验表明，当指令冲突规模扩大时，即使是目前的最前沿模型表现也较差（准确率约为 40%）。这项工作强调了在智能体场景中，迫切需要能够明确针对细粒度、可扩展指令冲突解决的方法。

## Abstract
Large language model agents receive instructions from many sources-system messages, user prompts, tool outputs, other agents, and more-each carrying different levels of trust and authority. When these instructions conflict, agents must reliably follow the highest-privilege instruction to remain safe and effective. The dominant paradigm, instruction hierarchy (IH), assumes a fixed, small set of privilege levels (typically fewer than five) defined by rigid role labels (e.g., system > user). This is inadequate for real-world agentic settings, where conflicts can arise across far more sources and contexts. In this work, we propose Many-Tier Instruction Hierarchy (ManyIH), a paradigm for resolving instruction conflicts among instructions with arbitrarily many privilege levels. We introduce ManyIH-Bench, the first benchmark for ManyIH. ManyIH-Bench requires models to navigate up to 12 levels of conflicting instructions with varying privileges, comprising 853 agentic tasks (427 coding and 426 instruction-following). ManyIH-Bench composes constraints developed by LLMs and verified by humans to create realistic and difficult test cases spanning 46 real-world agents. Our experiments show that even the current frontier models perform poorly (~40% accuracy) when instruction conflict scales. This work underscores the urgent need for methods that explicitly target fine-grained, scalable instruction conflict resolution in agentic settings.

随着大语言模型（LLM）被广泛应用于复杂的智能体（Agent）系统，它们需要同时处理来自系统消息、用户提示、工具输出、记忆文件及其他智能体的多源指令。现有的**指令层级（Instruction Hierarchy, IH）**范式存在“固定且少量层级的瓶颈”：它通常依赖于预定义的角色标签（如 `system > user > tool`），层级通常少于 5 层。
在现实场景中，这种粗粒度的划分无法处理更复杂的冲突（例如，两个具有不同信任等级的工具输出相互矛盾）。因此，论文提出了 **ManyIH**，旨在打破固定角色的限制，支持在推理阶段动态定义任意数量的特权层级，以确保智能体在复杂冲突中依然能遵循最高优先级的指令。

#### 2. 方法论：核心思想与关键技术
论文提出了 **Many-Tier Instruction Hierarchy (ManyIH)** 范式，其核心是将指令的“特权等级”与“消息角色”解耦。
*   **特权提示接口（Privilege Prompt Interface, PPI）：** 通过在提示词中加入特定的标签来显式标注每条指令的优先级。
    *   **序数接口（Ordinal Interface）：** 使用数字（如 `[[Privilege 1]]`）表示，数字越小优先级越高。
    *   **标量接口（Scalar Interface）：** 使用标量值（如 `[[z=95]]`）表示，数值越大优先级越高。
*   **元指令（Meta-instruction）：** 在提示词开头加入一段说明，告知模型如何解析这些标签并处理冲突（例如：“如果指令冲突，请遵循特权数字较小的指令”）。
*   **动态性：** 这种方法允许在推理时根据上下文动态分配优先级，而不需要重新训练模型。

*   **性能天花板：** 即使是最强的模型（Gemini 3.1 Pro），在 ManyIH-Bench 上的准确率也仅为 42.7%，远低于其在标准两层 IH 测试中 >99% 的表现。
*   **性能随层级增加而下降：** 随着指令层级从 6 层增加到 12 层，所有模型的准确率均出现显著且持续的下滑。

*   **推理长度不等于准确率：** 某些模型（如 Qwen 3.5-397B）虽然生成了极长的推理链，但准确率反而不如推理更简洁的 GPT-5.4。

#### 7. 优点
*   **前瞻性：** 首次识别并定义了智能体场景中“多层级指令冲突”这一关键挑战。
*   **解耦设计：** 提出的 PPI 接口具有高度的灵活性，不依赖于模型训练时的角色标签。
*   **高质量基准：** ManyIH-Bench 结合了自动化验证和人工审核，任务场景贴近现实（如 46 种不同类型的智能体）。

#### 8. 不足与局限
*   **合成冲突的局限：** 虽然 IF 子集经过人工验证，但部分冲突指令仍是合成的，可能无法完全覆盖极端复杂的现实人类意图。

*   **模型依赖性：** 实验显示模型对提示词格式高度敏感，这意味着 ManyIH 的有效性在很大程度上取决于模型对特定 PPI 格式的理解能力。
