## 摘要
由视觉语言模型（VLM）驱动的图形用户界面（GUI）智能体正迅速从被动辅助转向自主运行。然而，这种不受限制的动作空间使用户面临严重且不可逆的财务、隐私或社交损害。现有的安全保障措施依赖于提示工程、脆弱的启发式方法以及将 VLM 作为批判者（VLM-as-critic），缺乏形式化验证和用户可调的保证。我们提出了 CORA（共形风险控制 GUI 智能体），这是一种策略后、动作前的安全保障框架，可为执行有害动作提供统计学保证。CORA 将安全性重新定义为选择性动作执行：我们训练了一个 Guardian（守护者）模型来估计每个建议步骤的动作条件风险。我们并非简单地对原始分数设置阈值，而是利用共形风险控制（Conformal Risk Control）来校准满足用户指定风险预算的执行/放弃边界，并将被拒绝的动作路由到一个可训练的 Diagnostician（诊断者）模型，该模型对被拒绝的动作进行多模态推理，以推荐干预措施（如确认、反思或中止），从而最大限度地减轻用户负担。Goal-Lock（目标锁定）机制将评估锚定在明确且冻结的用户意图上，以抵御视觉注入攻击。为了严格评估这一范式，我们引入了 Phone-Harm，这是一个在真实场景下具有步骤级损害标签的移动端安全违规新基准。在 Phone-Harm 和公共基准上针对多种基线的实验验证了 CORA 改善了安全性-有用性-中断（safety--helpfulness--interruption）的帕累托前沿，为自主 GUI 执行提供了一种实用且具有统计学依据的安全范式。代码和基准测试可在 cora-agent.github.io 获取。

## Abstract
Graphical user interface (GUI) agents powered by vision language models (VLMs) are rapidly moving from passive assistance to autonomous operation. However, this unrestricted action space exposes users to severe and irreversible financial, privacy or social harm. Existing safeguards rely on prompt engineering, brittle heuristics and VLM-as-critic lack formal verification and user-tunable guarantees. We propose CORA (COnformal Risk-controlled GUI Agent), a post-policy, pre-action safeguarding framework that provides statistical guarantees on harmful executed actions. CORA reformulates safety as selective action execution: we train a Guardian model to estimate action-conditional risk for each proposed step. Rather than thresholding raw scores, we leverage Conformal Risk Control to calibrate an execute/abstain boundary that satisfies a user-specified risk budget and route rejected actions to a trainable Diagnostician model, which performs multimodal reasoning over rejected actions to recommend interventions (e.g., confirm, reflect, or abort) to minimize user burden. A Goal-Lock mechanism anchors assessment to a clarified, frozen user intent to resist visual injection attacks. To rigorously evaluate this paradigm, we introduce Phone-Harm, a new benchmark of mobile safety violations with step-level harm labels under real-world settings. Experiments on Phone-Harm and public benchmarks against diverse baselines validate that CORA improves the safety--helpfulness--interruption Pareto frontier, offering a practical, statistically grounded safety paradigm for autonomous GUI execution. Code and benchmark are available at cora-agent.github.io.

### 1. 核心问题与整体含义
*   **研究背景**：随着视觉语言模型（VLM）的发展，GUI 智能体正从简单的对话辅助转向能够直接在移动端执行点击、输入等环境级动作的自主操作。
*   **核心问题**：这种自主性带来了巨大的安全风险，如误转账、隐私泄露或误删数据。现有的安全防御手段（如提示词工程、简单的启发式规则或让 VLM 自我审查）要么容易被视觉注入攻击绕过，要么过于死板导致频繁干扰用户，且最关键的是**缺乏定量的统计安全保证**。
*   **整体含义**：论文旨在建立一个“安全盾牌”，在不修改底层策略模型的前提下，通过统计学方法（符合性风险控制）为智能体的动作执行提供用户可调的、有理论依据的安全边界。

### 2. 方法论
CORA 框架将安全问题重新定义为**选择性动作执行（Selective Action Execution）**，包含以下核心组件：
*   **Guardian (守护者 - System 1)**：一个轻量级的动作条件风险评估模型。它接收当前截图、用户目标和建议动作，输出一个 0 到 1 之间的风险分数 $s_t$。
*   **符合性风险控制 (CRC)**：这是核心数学工具。通过在离线校准集上运行，计算出一个动态阈值 $\hat{\tau}$。该阈值能保证在用户指定的风险预算 $\alpha$（例如 1% 的有害动作率）下，系统能自动过滤高风险动作。
*   **Diagnostician (诊断者 - System 2)**：当动作被 Guardian 拦截时，该多模态模型会介入进行深度推理，生成诊断报告并决定干预方式：**确认（Confirm）**、**反思（Reflect）**或**中止（Abort）**。
*   **Goal-Lock (目标锁定) 机制**：强制将初始用户意图冻结，并将屏幕上的任何文本视为“不可信证据”，从而有效抵御试图篡改智能体目标的间接提示注入攻击。

### 3. 实验设计
*   **数据集与基准 (Benchmark)**：
    *   **Phone-Harm (新提出)**：包含 150 个有害任务（Harm-150，涵盖误用、注入、误操作）和 150 个良性任务（Normal-150），提供步骤级的损害标签。
    *   **MobileRisk**：用于评估移动端风险检测和注入防御。
    *   **AndroidWorld**：用于评估在良性环境下的任务成功率和能力保持。
*   **对比方法**：
    *   **基础策略**：GPT-5, Gemini-3, UI-TARS-1.5, AutoGLM-VLM。
    *   **防御基线**：基于 Prompt 的自我检查、基于规则的启发式门控、VLM-as-critic（将 VLM 作为裁判）。

### 4. 资源与算力
*   **硬件环境**：使用了 1 台配备 **8 张 NVIDIA A100 (80GB)** GPU 的服务器。
*   **训练细节**：
    *   Guardian 和 Diagnostician 均基于 AutoGLM-Phone-9B 模型进行 LoRA 微调。
    *   训练时长：每个模型训练 3 个 epoch。
    *   推理延迟：CORA 整体平均每步延迟约 21.37 秒，相比原始策略（20.52 秒）增加极小，但在触发诊断干预时延迟会升至 34.13 秒。

### 5. 实验数量与充分性
*   **实验规模**：论文在三个不同的基准测试上进行了广泛实验，涵盖了从安全性到有用性的全方位评估。
*   **消融实验**：针对 CRC 组件、动作条件、诊断器设计和 Goal-Lock 机制分别做了详细的消融研究，验证了每个模块的必要性。
*   **客观性与公平性**：采用了严格的 **Train-Calibrate-Test** 隔离协议，确保校准集和测试集完全独立，且 Phone-Harm 基准是人工撰写的，避免了数据泄露风险。实验对比了当前最强的闭源和开源模型，具有较高的参考价值。

### 6. 主要结论与发现
*   **安全与效能的平衡**：CORA 在保持高任务达成率（GAR）的同时，能严格将有害动作率控制在预设预算内，其帕累托前沿（Pareto Frontier）显著优于所有基线。
*   **能力保留**：在良性任务（AndroidWorld）中，CORA 不仅没有降低成功率，反而通过“反思”机制帮助智能体纠正了点击偏差，将成功率从 30% 提升至 40%。
*   **防御有效性**：Goal-Lock 机制被证明是抵御视觉注入攻击的关键，能显著降低因屏幕恶意内容导致的误操作。

### 7. 优点
*   **统计学保证**：不同于以往“凭感觉”的安全设置，CORA 提供了严谨的数学证明来控制风险。
*   **用户可调性**：用户可以根据应用场景（如金融 vs 娱乐）自由调整风险预算 $\alpha$。
*   **即插即用**：作为事后控制器，它不需要重新训练底层的复杂策略模型，具有很强的通用性。

### 8. 不足与局限
*   **分布偏移挑战**：虽然论文提出了加权校准来应对应用和系统的变化，但在极端未见的 UI 环境下，风险控制的准确性可能仍受限于 Guardian 模型的泛化能力。
*   **序列依赖性**：目前的 CRC 主要针对单步风险，对于长程任务中累积的隐蔽风险（如多次小额转账累积成巨额损失）的建模仍有提升空间。
*   **延迟开销**：虽然平均延迟增加不多，但在高风险判定时的二次推理（System 2）仍会增加用户的等待感。
