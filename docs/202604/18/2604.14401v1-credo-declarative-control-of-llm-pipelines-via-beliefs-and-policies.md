## 摘要
智能体 AI 系统在需要于持续演进的环境中进行长期、有状态决策的领域正变得日益普遍。因此，正确性不仅取决于单个模型调用的输出，还取决于在纳入新证据或修正先前结论时如何进行最佳调整。然而，现有框架依赖于命令式控制循环、瞬时内存和嵌入提示词的逻辑，导致智能体行为不透明、脆弱且难以验证。本文介绍了 Credo，它将语义状态表示为信念，并使用在这些信念之上定义的声明式策略来规范行为。该设计通过数据库支持的语义控制平面，实现了自适应、可审计且可组合的执行。我们在一个决策控制场景中展示了这些概念，其中信念和策略以声明方式引导关键的执行选择（例如模型选择、检索、纠错性重新执行），从而在无需更改底层流水线代码的情况下实现动态行为。

## Abstract
Agentic AI systems are becoming commonplace in domains that require long-lived, stateful decision-making in continuously evolving conditions. As such, correctness depends not only on the output of individual model calls, but also on how to best adapt when incorporating new evidence or revising prior conclusions. However, existing frameworks rely on imperative control loops, ephemeral memory, and prompt-embedded logic, making agent behavior opaque, brittle, and difficult to verify. This paper introduces Credo, which represents semantic state as beliefs and regulates behavior using declarative policies defined over these beliefs. This design supports adaptive, auditable, and composable execution through a database-backed semantic control plane. We showcase these concepts in a decision-control scenario, where beliefs and policies declaratively guide critical execution choices (e.g., model selection, retrieval, corrective re-execution), enabling dynamic behavior without requiring any changes to the underlying pipeline code.

### 1. 核心问题与研究动机
*   **核心问题**：现有的 LLM 智能体（Agent）框架在处理复杂任务时面临两难境地：要么使用**命令式控制流**（如 LangChain），逻辑硬编码在代码中，缺乏灵活性；要么依赖**提示词内嵌逻辑**（如 ReAct），决策过程黑盒化、不可审计且难以验证。
*   **研究动机**：作者发现，在 LLM 流水线中，固定的配置（如检索深度、模型选择）往往导致资源浪费或精度不足。实验表明，检索策略的选择对准确率的影响甚至超过了模型规模。因此，需要一种既能像 SQL 一样声明式定义，又能像自主智能体一样根据运行时语义信号动态调整的系统。

### 2. 方法论
Credo 提出了一个**数据库驱动的语义控制平面**，将流水线的逻辑定义与物理执行决策分离。其核心包含三个层次：
*   **观察层：信念 (Beliefs)**
    *   将运行时的语义状态表示为类型化的“信念”，以 `(value, confidence)` 对的形式持久化存储。
    *   **静态信念**：在执行前通过轻量级模型（如 RouteLLM）评估查询复杂度。
    *   **动态信念**：在运行时从中间结果中提取（如使用 LLM 评估检索到的证据是否相关）。
*   **决策层：策略 (Policies)**
    *   使用声明式规则映射信念到执行参数。例如：`WHEN query_complexity >= 0.25 THEN REWRITE generate SET model = gemma3-27b`。
    *   支持**算子重写**（局部参数调整）和**流水线重写**（使先前结果失效并触发重新执行）。
*   **执行层：动作 (Actions)**
    *   提供内置算子（检索、生成、验证），这些算子不感知策略，仅接收控制平面下发的参数执行任务。
    *   所有决策路径和信念状态都记录在数据库中，实现完全的可追溯和可审计。

### 6. 主要结论与发现
*   **检索优于规模**：在金融问答场景下，检索策略的质量对结果的影响远大于模型参数量的增加。

*   **声明式控制的可行性**：通过 Credo 的信念和策略机制，可以在不修改底层流水线代码的情况下，实现复杂的纠错逻辑（如发现证据不相关时自动切换检索算法并重试）。

### 7. 优点
*   **解耦设计**：成功将“做什么”（逻辑流水线）与“怎么做”（物理执行策略）分离，类似于数据库中逻辑计划与物理计划的分离。
*   **可审计性**：所有中间语义状态（信念）和决策依据（策略触发）都持久化在数据库中，解决了 Agent 行为不可预测的痛点。
*   **灵活性**：用户可以像调整 SQL 索引一样，在不停止服务的情况下在线修改策略，动态优化流水线行为。

### 8. 不足与局限
*   **开销风险**：提取“信念”（尤其是使用 LLM 作为评判者时）本身会产生额外的成本和延迟，论文中虽提到会优先使用廉价提取器，但未给出详细的开销平衡分析。
*   **策略编写门槛**：虽然是声明式的，但编写高效的策略仍需要开发者对任务领域有深度理解，未来可能需要自动化的策略发现机制。
*   **实验覆盖面**：目前仅在金融问答单一领域展示，对于多模态或更长程的复杂 Agent 任务，其策略冲突解决和信念一致性维护仍待验证。
