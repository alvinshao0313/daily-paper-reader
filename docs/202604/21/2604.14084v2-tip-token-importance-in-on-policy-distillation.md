## 摘要
在线策略知识蒸馏（OPD）在教师模型的 Token 级监督下，利用学生模型自身的生成序列（rollouts）对其进行训练。并非所有 Token 位置都具有同等的重要性，但现有关于 Token 重要性的观点尚不完整。我们提出了一个直接的问题：在 OPD 中，哪些 Token 携带了最有用的学习信号？我们的回答是，信息丰富的 Token 来自两个区域：学生模型熵值较高的位置，以及学生模型熵值较低但教师与学生模型散度（divergence）较高的位置（即学生模型过度自信且错误的位置）。实验表明，学生模型的熵是一个强大的一阶代理指标：通过基于熵的采样保留 50% 的 Token，其训练效果即可匹配或超过全量 Token 训练，同时减少高达 47% 的峰值显存。但仅凭熵会遗漏第二个重要区域。当我们分离出低熵、高散度的 Token 时，仅在不到 10% 的 Token 上进行训练就几乎达到了全量 Token 的基准水平，这表明过度自信的 Token 携带了密集的修正信号，尽管它们在仅基于熵的规则下几乎不可见。我们通过 TIP（在线策略蒸馏中的 Token 重要性）整理了这些发现，这是一个基于学生模型熵和师生散度的双轴分类法，并从理论上解释了为什么熵虽然有用但在结构上是不完整的。这一观点启发了结合不确定性（uncertainty）和分歧（disagreement）的类型感知 Token 选择规则。我们在 MATH-500、AIME 2024/2025 以及用于长程智能体规划的 DeepPlanning 基准测试上，跨越 Qwen3、Llama 和 Qwen2.5 的三对师生模型验证了这一图景。在 DeepPlanning 中，仅在 Q3 区域（不到 20% 的 Token）进行训练就超过了全量 Token 的 OPD。我们的实验通过扩展 OPD 仓库（https://github.com/HJSang/OPSD_OnPolicyDistillation）实现，该仓库支持在有限的 GPU 预算下对大型模型进行显存高效的蒸馏。

## Abstract
On-policy knowledge distillation (OPD) trains a student on its own rollouts under token-level supervision from a teacher. Not all token positions matter equally, but existing views of token importance are incomplete. We ask a direct question: which tokens carry the most useful learning signal in OPD? Our answer is that informative tokens come from two regions: positions with high student entropy, and positions with low student entropy plus high teacher--student divergence, where the student is overconfident and wrong.   Empirically, student entropy is a strong first-order proxy: retaining $50\%$ of tokens with entropy-based sampling matches or exceeds all-token training while reducing peak memory by up to $47\%$. But entropy alone misses a second important region. When we isolate low-entropy, high-divergence tokens, training on fewer than $10\%$ of all tokens nearly matches full-token baselines, showing that overconfident tokens carry dense corrective signal despite being nearly invisible to entropy-only rules.   We organize these findings with TIP (Token Importance in on-Policy distillation), a two-axis taxonomy over student entropy and teacher--student divergence, and give a theoretical explanation for why entropy is useful yet structurally incomplete. This view motivates type-aware token selection rules that combine uncertainty and disagreement. We validate this picture across three teacher--student pairs spanning Qwen3, Llama, and Qwen2.5 on MATH-500 and AIME 2024/2025, and on the DeepPlanning benchmark for long-horizon agentic planning, where Q3-only training on $<$$20\%$ of tokens surpasses full-token OPD. Our experiments are implemented by extending the OPD repository https://github.com/HJSang/OPSD_OnPolicyDistillation, which supports memory-efficient distillation of larger models under limited GPU budgets.

*   **研究背景**：在线策略蒸馏（On-Policy Distillation, OPD）通过让学生模型生成序列并由教师模型进行 Token 级纠正来提升性能。然而，长序列中并非所有 Token 都对学习有同等贡献。
*   **核心问题**：在 OPD 过程中，哪些 Token 携带了最有价值的学习信号？
*   **研究动机**：现有的 Token 选择方法（如仅基于熵）是不完整的。作者试图通过识别“关键 Token”来提高蒸馏效率，并在有限的计算资源下实现更好的性能。

### 2. 方法论：TIP 分类法与 Soft-OR 评分
*   **核心思想**：提出 **TIP（Token Importance in on-Policy distillation）** 分类法，将 Token 重要性映射到两个维度：**学生熵 ($h_t$)**（代表学生的不确定性）和**师生散度 ($\delta_t$)**（代表学生与教师的分歧）。
*   **TIP 四象限分类**：
    *   **Q1 (高熵, 高散度)**：学生不确定且与教师分歧大，提供最密集的学习信号。
    *   **Q2 (高熵, 低散度)**：学生不确定但与教师一致，有助于稳定预测。
    *   **Q3 (低熵, 高散度)**：**盲点区域**。学生过度自信但错误，携带关键的修正信号。
    *   **Q4 (低熵, 低散度)**：学生已掌握，信号微弱，可忽略。
*   **关键技术细节**：
    *   **Soft-OR 评分公式**：为了同时捕捉高熵区域和 Q3 盲点，定义评分 $s_t = \hat{h}_t + \hat{\delta}_t - \hat{h}_t \cdot \hat{\delta}_t$（其中 $\hat{h}_t, \hat{\delta}_t$ 为归一化后的值）。
    *   **选择机制**：根据 $s_t$ 分数进行 Top-K 采样，仅对选中的 Token 计算损失并进行反向传播。
*   **理论证明**：论文证明了任何仅基于熵的规则都会在结构上忽略 Q3 区域（过度自信的错误），而 Soft-OR 能够有效恢复这一区域。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了三个主流模型家族（Qwen3, Llama, Qwen2.5），跨越了从 1.5B 到 70B 的参数规模。
*   **任务多样性**：不仅在标准的数学推理上验证，还扩展到了复杂的长程智能体规划任务。
*   **消融实验**：详细对比了不同保留比例（10%, 20%, 50%）以及不同筛选轴（仅熵、仅散度、Q3 区域）的影响。
*   **客观性**：实验结果提供了均值和标准差（mean@16），并进行了“Top vs Bottom”的对比实验，证明了筛选规则的有效性而非随机性。

### 6. 主要结论与发现
*   **熵是强代理但非唯一**：保留 50% 的高熵 Token 即可达到或超过全量训练的效果。
*   **Q3 区域的价值**：仅在不到 10% 的 Q3 Token（过度自信且错误）上训练，性能就能接近全量基准。在规划任务中，Q3 训练甚至超过了全量 OPD。
*   **Soft-OR 的优越性**：在数学任务中，Soft-OR 始终优于纯熵筛选，证明了结合“不确定性”与“分歧度”的必要性。
*   **效率提升**：在保持或提升性能的同时，显著降低了计算开销和显存占用。

### 7. 优点
*   **理论与实践结合**：不仅有直观的分类法，还提供了严谨的数学证明来解释“盲点”存在的原因。
*   **即插即用且低开销**：Soft-OR 评分所需的变量在标准 OPD 损失计算中已存在，几乎不增加额外计算负担。
*   **显著的资源节省**：对于显存受限的开发者，该方法提供了在有限硬件上蒸馏大模型的切实可行方案。

### 8. 不足与局限
*   **依赖教师分布**：Q3 区域的识别需要教师模型的完整输出分布（用于计算散度），这在某些仅提供 Top-1 结果的 API 场景下无法实现。
*   **归一化敏感性**：Soft-OR 依赖于 Batch 内的 Min-Max 归一化，可能会受到 Batch 中异常 Token 的影响。
*   **监督函数限制**：所有实验均基于逆 KL 散度（Reverse KL），该分类法在 Forward KL 或 JSD 监督下的表现尚未验证。
*   **超参数依赖**：虽然评分是无参的，但保留比例（Retention Ratio）仍需根据具体任务手动调整。
