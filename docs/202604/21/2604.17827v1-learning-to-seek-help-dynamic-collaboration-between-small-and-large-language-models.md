## 摘要
大语言模型（LLMs）具备强大的能力，但引发了成本和隐私方面的担忧；而小语言模型（SLMs）便于进行高效且私密的本地推理，但其能力有限。为了协同这些互补优势，我们引入了一个动态协作框架。在该框架中，SLM 学习在多步推理过程中主动决定如何向 LLM 发起请求，而 LLM 则提供自适应反馈，而非仅仅作为一个被动工具。我们进一步系统地研究了协作策略是如何受 SLM 和 LLM 的能力以及效率和隐私约束影响的。评估结果揭示了明显的缩放效应：更强大的 SLM 变得更加自立，而更强大的 LLM 则能实现更少且更具信息量的交互。此外，学习到的动态协作策略显著优于静态流水线和独立推理，并且能够稳健地迁移到未见过的 LLM。

## Abstract
Large language models (LLMs) offer strong capabilities but raise cost and privacy concerns, whereas small language models (SLMs) facilitate efficient and private local inference yet suffer from limited capacity. To synergize the complementary strengths, we introduce a dynamic collaboration framework, where an SLM learns to proactively decide how to request an LLM during multi-step reasoning, while the LLM provides adaptive feedback instead of acting as a passive tool. We further systematically investigate how collaboration strategies are shaped by SLM and LLM capabilities as well as efficiency and privacy constraints. Evaluation results reveal a distinct scaling effect: stronger SLMs become more self-reliant, while stronger LLMs enable fewer and more informative interactions. In addition, the learned dynamic collaboration strategies significantly outperform static pipelines and standalone inference, and transfer robustly to unseen LLMs.

论文探讨了如何有效结合**端侧小语言模型（SLM）**与**云端大语言模型（LLM）**的优势。

*   **核心问题**：现有的协作方式多为“静态流水线”（如固定重写或路由），SLM 无法根据自身能力和任务难度动态决定何时求助，且 LLM 通常被视为被动工具，缺乏双向反馈。
*   **整体含义**：提出一种动态协作框架，让 SLM 学习“主动寻求帮助”，在多步推理中自主判断是否需要 LLM 介入，并由 LLM 提供自适应反馈，从而在性能、效率和隐私之间取得平衡。

### 2. 方法论：核心思想与关键技术
核心思想是将大小模型协作建模为**异构智能体之间的动态博弈**。
*   **主动式 SLM（Proactive SLM）**：SLM 在接收查询后进行多步推理。在每一步，它会分析当前状态，决定是本地解决还是向 LLM 发起结构化请求（`<message>`）。
*   **反馈式 LLM（Feedback LLM）**：LLM 不再只是黑盒工具，它可以执行两种动作：(1) 提供有用信息；(2) 若 SLM 请求模糊，则识别并要求 SLM 提供更多上下文（反馈引导）。
*   **在线强化学习（Online RL）**：使用 **VAPO 算法**对 SLM 进行端到端优化。奖励函数 $R$ 由四部分组成：
    *   **质量奖励（$r_{quality}$）**：基于最终答案的准确性（EM 分数）。
    *   **反馈奖励（$r_{feedback}$）**：若 LLM 认为请求不合理并要求补充上下文，则给予负反馈。
    *   **效率惩罚（$r_{efficiency}$）**：根据交互轮数和任务难度给予惩罚，鼓励 SLM 优先本地解决简单问题。
    *   **隐私惩罚（$r_{privacy}$）**：若请求中包含敏感隐私信息，则给予重罚。
*   **隐私注入流水线**：为了训练隐私意识，作者构建了 **PrivQA 数据集**，通过 LLM 将公开 QA 数据改写为包含个人隐私信息的间接引用形式。

### 3. 实验设计
*   **数据集**：构建了 **PrivQA**，源自 NQ、TriviaQA、PopQA（通用 QA）以及 HotpotQA、2Wiki、MuSiQuE（多跳推理）。

*   **Benchmark 与对比方法**：
    *   **独立推理**：SLM with CoT, LLM with CoT。

    *   **工具调用**：Router-R1（将 LLM 视为检索工具）。
*   **评估指标**：EM（准确率）、BERTScore（语义相似度）、交互轮数、INScore（交互必要性得分）、隐私泄露率。

### 5. 实验数量与充分性
*   **实验规模**：论文进行了跨模型（5 种 SLM，3 种 LLM）、跨领域（6 个数据集）的大规模测试。
*   **充分性**：

    *   **消融实验**：针对 LLM 反馈的作用、效率惩罚权重、隐私惩罚权重分别做了详细的参数敏感性分析。

### 6. 主要结论与发现
*   **性能提升**：动态协作框架在响应质量上比 SLM CoT 提升了 **14.5% - 17.4%**，比静态流水线提升了 **2.8% - 9.9%**。
*   **缩放效应（Scaling Effect）**：更强的 SLM 会变得更自立（减少求助）；更强的 LLM 能引导 SLM 进行更简洁高效的交互。
*   **隐私保护**：通过隐私惩罚项，SLM 学会了在不牺牲性能的前提下，将隐私查询重构为客观事实查询，隐私泄露率比静态方法降低了 **24.3% - 32.4%**。
*   **策略迁移**：在较小 LLM 上训练的协作策略可以直接迁移到更强大的闭源 LLM（如 Qwen3-Max），且性能进一步提升。

### 7. 优点
*   **动态性**：打破了固定流水线的僵化，实现了按需求助，显著提高了交互效率。
*   **隐私意识**：通过 RL 将隐私保护内化为模型的一种推理本能，而非简单的规则过滤。
*   **双向反馈**：引入 LLM 的主动反馈机制，使协作过程更接近人类专家与助手的互动。
*   **数据集创新**：PrivQA 的构建为未来端云协作研究提供了宝贵的基准。

### 8. 不足与局限
*   **模型门槛**：实验发现 0.6B 级别的极小模型由于指令遵循能力太弱，无法直接通过 RL 学习协作策略，需要先进行监督微调（SFT）冷启动。
*   **计算开销**：虽然推理时高效，但在线 RL 训练过程（涉及大小模型频繁交互）对算力和时间的要求较高。
*   **隐私定义**：隐私保护主要依赖于 LLM 作为裁判的判断，可能存在对某些隐蔽隐私泄露识别不准的风险。
*   **应用限制**：目前主要集中在 QA 任务，在更复杂的开放式创作或多模态任务中的表现尚待验证。
