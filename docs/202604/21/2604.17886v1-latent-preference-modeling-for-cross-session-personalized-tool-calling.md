## 摘要
用户在向基于大语言模型（LLM）的智能体提出请求时，往往会省略关键细节，导致工具使用的输入信息不足。这对工具增强型智能体提出了根本性挑战，因为 API 执行通常需要完整的参数，这凸显了个性化工具调用的必要性。为了研究这一问题，我们引入了 MPT，这是一个包含 265 个多会话对话的基准测试，涵盖了三个挑战：偏好召回、偏好归纳和偏好迁移。我们还提出了 PRefine，这是一种测试时内存增强方法，将用户偏好表示为不断演化的假设。通过“生成-验证-细化”循环，它从历史记录中提取可重用的约束，在提高工具调用准确性的同时，仅消耗全历史提示词所需 token 数量的 1.24%。这些结果表明，智能体系统中稳健的个性化取决于能够捕捉用户选择背后原因的记忆，而不仅仅是选择本身。

## Abstract
Users often omit essential details in their requests to LLM-based agents, resulting in under-specified inputs for tool use. This poses a fundamental challenge for tool-augmented agents, as API execution typically requires complete arguments, highlighting the need for personalized tool calling. To study this problem, we introduce MPT, a benchmark comprising 265 multi-session dialogues that cover three challenges: Preference Recall, Preference Induction, and Preference Transfer. We also propose PRefine, a test-time memory-augmented method that represents user preferences as evolving hypotheses. Through a generate--verify--refine loop, it extracts reusable constraints from history and improves tool-calling accuracy while using only 1.24% of the tokens required by full-history prompting. These results indicate that robust personalization in agentic systems depends on memory that captures the reasons behind user choices, not just the choices themselves.

#### 1. 核心问题与整体含义

*   **研究动机**：用户在请求工具执行任务时，往往会省略必要的细节（如订票时不说明舱位等级）。现有的工具调用研究多关注单次会话或显式指令，忽略了从用户长期的、跨会话的历史行为中推断“潜在偏好”（Latent Preference）的能力。
*   **核心挑战**：如何从嘈杂、无序且分布在多个会话中的交互历史中，提取出抽象的、可重用的行为约束，并将其应用于新的、甚至不同领域的工具调用场景中。

#### 2. 方法论：PRefine
论文提出了 **PRefine**，一种基于记忆增强的测试时（test-time）方法，其核心是将用户偏好视为**不断演化的假设**。
*   **核心思想**：通过一个“生成-验证-细化”（Generate-Verify-Refine）的闭环，将原始对话历史压缩为抽象的偏好约束。
*   **算法流程**：
    1.  **生成（Generate）**：根据当前会话、执行的 API 调用和先前的记忆，生成候选的偏好假设（如“用户倾向于选择廉价选项”）。
    2.  **验证（Verify）**：验证器根据四个准则评估假设：**证据支持**（是否有多次行为支持）、**抽象质量**（是否超越了简单的槽位复述）、**可操作性**（是否能指导未来决策）、**时间一致性**（是否符合最近的行为模式）。
    3.  **细化（Refine）**：如果假设未通过验证，则根据反馈进行修正，直到形成稳定的记忆。
*   **关键特性**：**模式无关性（Schema-Agnostic）**。记忆存储的是抽象的文本约束，而非特定 API 的参数名，这使得偏好可以在不同工具（Schema）之间迁移。

#### 6. 主要结论与发现
*   **现有方法的局限**：传统的 RAG 和全量历史提示在“偏好召回”上表现尚可，但在需要逻辑推理的“偏好归纳”和“偏好迁移”上表现较差。
*   **高效性**：PRefine 仅使用全量历史记录 **1.24% 的 token**，就达到了更好或相当的工具调用准确率。
*   **推理胜过检索**：稳健的个性化依赖于捕捉用户选择背后的**原因**（抽象约束），而不仅仅是记录选择本身（原始数据）。
*   **泛化能力**：PRefine 提取的偏好在 API 参数名发生变化（动态模式）时依然有效。

#### 7. 优点
*   **极高的压缩率**：显著降低了长上下文带来的计算开销和 Token 成本。
*   **逻辑严密**：引入“假设验证”机制，避免了智能体产生过度泛化或错误的偏好推断。
*   **架构灵活**：Schema-Agnostic 的设计使其能够适应不断更新的工具库，具有很强的实用价值。

#### 8. 不足与局限
*   **偏好分类受限**：目前仅验证了“预算（Budget）”和“旅行人数（Travel）”两类偏好，尚未涵盖更复杂的个性化需求（如饮食禁忌、品牌偏好等）。
*   **数据合成性**：MPT 基准基于合成的 SGD 数据集，可能无法完全模拟真实世界中用户行为的极端噪声和不可预测性。
*   **推理延迟**：虽然节省了 Token，但“生成-验证-细化”的循环增加了推理时的往返次数（Round-trips），可能影响实时交互的响应速度。
