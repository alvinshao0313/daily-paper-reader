## 摘要
随着大语言模型（LLM）及其基准测试的不断增加，编排多个模型以提升任务性能的需求变得前所未有的紧迫。虽然像 Mixture-of-Agents (MoA) 这样的框架尝试协调 LLM，但它们在以下方面往往表现不足：(1) 选择相关的智能体，(2) 促进有效的智能体间通信，以及 (3) 高效地整合响应。在这项工作中，我们提出了 Graph-of-Agents (GoA)，这是一种用于建模多智能体 LLM 通信的新型基于图的框架。我们的方法首先进行节点采样，利用总结了每个模型的领域、任务专长和其他特征的模型卡片（model cards），仅选择最相关的智能体。接着，我们通过相互评估智能体的响应来确定相关性排序，从而在选定的智能体之间构建边。然后，执行从高相关性智能体到低相关性智能体的有向消息传递，以增强其响应；随后进行反向消息传递，以细化高相关性智能体的原始响应。最后，通过基于图的池化（如最大池化或平均池化）聚合更新后的响应，从而生成单一、统一的答案。我们在多种多领域基准测试（MMLU、MMLU-Pro、GPQA）和特定领域基准测试（MATH、HumanEval、MedMCQA）上评估了 GoA，智能体池包含跨越多个领域的 6 个 LLM。令人惊讶的是，GoA 仅使用 3 个选定的智能体就实现了卓越的性能，优于同时使用全部 6 个智能体的最新多智能体 LLM 基准模型。通过采用图结构，GoA 通过结构化消息传递提供了可扩展性和有效性，使其成为应对日益增长的 LLM 集合挑战的有力候选方案。代码开源地址：https://github.com/UNITES-Lab/GoA。

## Abstract
With an ever-growing zoo of LLMs and benchmarks, the need to orchestrate multiple models for improved task performance has never been more pressing. While frameworks like Mixture-of-Agents (MoA) attempt to coordinate LLMs, they often fall short in terms of (1) selecting relevant agents, (2) facilitating effective intra-agent communication, and (3) integrating responses efficiently. In this work, we propose Graph-of-Agents (GoA), a new graph-based framework for modeling multi-agent LLM communication. Our approach begins with node sampling, selecting only the most relevant agents by leveraging model cards that summarize each model's domain, task specialization, and other characteristics. Next, we construct edges between the selected agents by evaluating their responses against one another to determine relevance ordering. Directed message passing is then performed from highly relevant agents to less relevant ones to enhance their responses, followed by reverse message passing to refine the original responses of the more relevant agents. Finally, the updated responses are aggregated via graph-based pooling (e.g., max or mean pooling) to produce a single, unified answer. We evaluate GoA on diverse multi-domain benchmarks (MMLU, MMLU-Pro, GPQA) and domain-specific benchmarks (MATH, HumanEval, MedMCQA), with an agent pool of 6 LLMs spanning multiple domains. Surprisingly, GoA achieves superior performance using only 3 selected agents, outperforming recent multi-agent LLM baselines that utilize all 6 agents simultaneously. By adopting a graph structure, GoA offers both scalability and effectiveness through structured message passing-positioning it as a strong candidate for navigating the challenges of the ever-growing LLM zoo. Code is available at: https://github.com/UNITES-Lab/GoA.

### 1. 核心问题与研究动机
*   **研究背景**：随着大语言模型（LLM）数量的激增，如何有效地编排多个模型以协同解决复杂任务成为挑战。
*   **核心问题**：现有的多智能体协作框架（如 Mixture-of-Agents, MoA）存在三大瓶颈：
    1.  **代理选择难**：通常盲目使用所有可用代理，导致计算浪费和噪声引入。
    2.  **通信效率低**：多对一的聚合方式缺乏细粒度的点对点交互。
    3.  **集成成本高**：通过拼接所有响应进行推理，导致 Token 消耗和延迟随代理数量呈线性甚至指数增长。
*   **研究动机**：提出一种基于图的框架，通过结构化通信和选择性激活，实现更高效、可扩展的多模型协作。

### 2. 方法论
GoA 将多智能体协作建模为有向图 $G=(V, E)$，核心流程包括：
*   **节点采样 (Node Sampling)**：利用“模型卡片”（包含领域、任务专长、规模等元数据），由一个 Meta-LLM 根据查询（Query）从代理池中筛选出 Top-k 个最相关的代理作为图节点。
*   **边采样 (Edge Sampling)**：
    *   让选定的代理对彼此的初始响应进行评分，构建相关性矩阵。
    *   根据评分对代理进行排序，并设置阈值 $\tau$ 剪枝低相关性节点。
*   **消息传递 (Message Passing)**：
    1.  **源到目标 (Source-to-Target)**：高排名（更可靠）的代理将信息传递给低排名代理，帮助其改进响应。
    2.  **目标到源 (Target-to-Source)**：低排名代理更新响应后，反馈给高排名代理，实现共识优化。
*   **图池化 (Graph Pooling)**：
    *   **Max Pooling**：直接采用连接数最多（最核心）代理的响应。
    *   **Mean Pooling**：由 Meta-LLM 综合所有优化后的响应，生成最终答案。

### 3. 实验设计
*   **数据集/基准测试**：

*   **客观性**：通过固定代理池规模（对比 6 代理基线 vs 3 代理 GoA）证明了“少而精”的协作优于“多而杂”，实验设计逻辑严密，对比公平。

### 6. 主要结论与发现
*   **性能卓越**：GoA 仅使用 **3 个代理** 的表现就全面超越了使用 **6 个代理** 的 MoA 和其他多智能体基线。
*   **结构优势**：双向消息传递（先从强到弱，再从弱到强）是提升性能的关键，证明了图结构在处理复杂逻辑推理时的有效性。
*   **高效可扩展**：通过节点和边采样，GoA 解决了“代理爆炸”问题，在保持高精度的同时显著降低了计算成本。
*   **泛化性**：该框架不仅适用于开源小模型，在 GPT-4o 等顶级闭源模型上同样能进一步提升性能。

### 7. 优点
*   **结构化通信**：将模糊的智能体交互转化为清晰的图论操作（消息传递、池化），理论框架完备。
*   **高效率**：引入模型卡片进行前置筛选，避免了无关模型的干扰和资源浪费。
*   **即插即用**：完全基于 Prompt 接口，无需微调，兼容黑盒 API。
*   **理论统一**：论文证明了 MoA 是 GoA 在特定拓扑结构下的一个特例，体现了 GoA 的广义性。

### 8. 不足与局限
*   **Meta-LLM 依赖**：节点采样和最终聚合高度依赖 Meta-LLM 的判断能力，若 Meta-LLM 选错代理，整体性能会受限。
*   **超参数敏感性**：阈值 $\tau$ 和采样数 $k$ 需要根据不同任务进行微调，缺乏完全自动化的自适应机制。
*   **代理池多样性**：实验主要集中在 7B-8B 模型，对于更大规模模型（如 70B+）或极小模型（如 1B）混合协作的效果探讨较少。
