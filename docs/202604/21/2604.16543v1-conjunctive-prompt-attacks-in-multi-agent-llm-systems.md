## 摘要
大多数大语言模型（LLM）安全研究关注单智能体模型，但许多实际应用依赖于多个相互协作的智能体。在这些系统中，提示词分段和智能体间路由产生了单智能体评估所忽略的攻击面。我们研究了“合取提示攻击”（conjunctive prompt attacks），其中用户查询中的触发键和被攻陷远程智能体中的隐藏对抗模板在单独存在时均表现为良性，但当路由将它们结合在一起时，就会激活有害行为。我们考虑这样一种攻击者：他们既不改变模型权重，也不改变客户端智能体，而是仅控制触发词的放置和模板的插入。在星型、链型和有向无环图（DAG）拓扑结构中，路由感知优化在保持低误触发率的同时，显著提高了攻击成功率，优于非优化基准。现有的防御措施，包括 PromptGuard、Llama-Guard 变体以及工具限制等系统级控制，无法可靠地阻止此类攻击，因为没有任何单一组件在孤立状态下显现出恶意特征。这些结果揭示了智能体化大语言模型流水线中的结构性漏洞，并促使开发能够对路由和跨智能体组合进行推理的防御措施。代码可在 https://github.com/UCF-ML-Research/ConjunctiveAgents 获取。

## Abstract
Most LLM safety work studies single-agent models, but many real applications rely on multiple interacting agents. In these systems, prompt segmentation and inter-agent routing create attack surfaces that single-agent evaluations miss. We study \emph{conjunctive prompt attacks}, where a trigger key in the user query and a hidden adversarial template in one compromised remote agent each appear benign alone but activate harmful behavior when routing brings them together. We consider an attacker who changes neither model weights nor the client agent and instead controls only trigger placement and template insertion. Across star, chain, and DAG topologies, routing-aware optimization substantially increases attack success over non-optimized baselines while keeping false activations low. Existing defenses, including PromptGuard, Llama-Guard variants, and system-level controls such as tool restrictions, do not reliably stop the attack because no single component appears malicious in isolation. These results expose a structural vulnerability in agentic LLM pipelines and motivate defenses that reason over routing and cross-agent composition. Code is available at https://github.com/UCF-ML-Research/ConjunctiveAgents.

*   **核心思想**：利用多智能体系统的拓扑结构和路由逻辑，通过优化提示词的构造，确保触发键与恶意模板在特定智能体处“汇合”。
*   **关键技术细节**：
    *   **威胁模型**：攻击者不修改模型权重或客户端逻辑，仅控制用户输入中的触发键位置和远程智能体中的模板插入位置。

    *   **Gumbel-Softmax 采样**：将离散的决策（如触发键放在哪个分段、模板放在前缀还是后缀）转化为连续的概率分布，从而允许使用梯度下降进行优化。

### 6. 主要结论与发现
*   **结构性漏洞**：多智能体系统的路由机制是天然的攻击放大器。经过优化后，攻击成功率在所有拓扑下均显著提升（ASR_max 达到 1.0）。
*   **现有防御失效**：PromptGuard 和 Llama-Guard 等现有防御工具在面对优化后的攻击时表现极差（F1 分数大幅下降）。这是因为这些工具只检查孤立的消息，而无法识别跨智能体的组合威胁。
*   **拓扑敏感性**：DAG 拓扑在未优化时最易受攻击，因为复杂的路由路径增加了触发键与模板汇合的概率。

### 7. 优点与亮点
*   **新颖的视角**：首次系统性地研究了多智能体系统中“分布式触发”的攻击模式，而非传统的单点注入。
*   **实用的优化框架**：提出的可微副本模型解决了黑盒 MAS 环境下离散变量难以优化的难题。
*   **现实的威胁模型**：攻击者无需接触核心模型权重，仅通过供应链中的一个远程智能体模板即可实施攻击，具有极高的现实威胁。

### 8. 不足与局限
*   **路由模型简化**：实验中的路由模型是基于概率的抽象模型，而现实中的路由可能基于更复杂的语义分类器或向量检索，攻击在真实复杂路由下的表现有待验证。
*   **单一受损智能体假设**：研究假设只有一个远程智能体被攻陷，未探讨多个智能体协同攻击的更复杂场景。
*   **激活判定简单**：使用硬编码的标记（Marker）来判定攻击成功，虽然便于量化，但可能无法完全捕捉到更隐蔽的、非标记性的有害行为。
*   **应用限制**：攻击效果高度依赖于对系统拓扑的了解，如果系统拓扑动态变化剧烈，攻击的稳定性可能会下降。
