## 摘要
从复杂的价值流图（VSM）模拟中提取可操作的洞察具有挑战性、耗时且容易出错。大语言模型的最新进展为支持用户完成这一任务提供了新途径。虽然现有方法在处理原始数据以获取信息方面表现出色，但在结构上难以捕捉区分该领域相似数据源所需的细微情境差异。为了解决这一问题，我们提出了一种解耦的两步代理式架构。通过将编排与数据分析分离，该系统利用注入了领域专家知识的渐进式数据发现。这种架构允许编排层智能地选择数据源，并在保持精简内部上下文的同时，跨数据结构执行多跳推理。来自多个最先进大语言模型的结果证明了该框架的可行性：顶级模型实现了高达 86% 的准确率，并在多次评估运行中表现出极高的鲁棒性。

## Abstract
Extracting actionable insights from complex value stream map simulations can be challenging, time-consuming, and error-prone. Recent advances in large language models offer new avenues to support users with this task. While existing approaches excel at processing raw data to gain information, they are structurally unfit to pick up on subtle situational differences needed to distinguish similar data sources in this domain. To address this issue, we propose a decoupled, two-step agentic architecture. By separating orchestration from data analysis, the system leverages progressive data discovery infused with domain expert knowledge. This architecture allows the orchestration to intelligently select data sources and perform multi-hop reasoning across data structures while maintaining a slim internal context. Results from multiple state-of-the-art large language models demonstrate the framework's viability: with top-tier models achieving accuracies of up to 86% and demonstrating high robustness across evaluation runs.

### 2. 方法论：解耦的两步代理架构
论文提出了一种**解耦的代理架构**，将逻辑推理与数据分析分离：
*   **编排代理（Orchestration Agent）**：作为“大脑”，负责高层规划和多跳推理。它不直接接触原始模拟数据，而是通过四个专用工具逐步探索 VSM 结构和数据目录：
    1.  **节点发现工具**：识别 VSM 中的实体（如超市、工位）。
    2.  **属性提取工具**：获取特定节点的静态参数（如安全库存水平）。
    3.  **分类导航工具**：浏览模拟数据的分类索引及专家知识描述。
    4.  **摘要工具**：触发子工作流。
*   **摘要子工作流（Summarization Subworkflow）**：作为“执行者”，专门负责处理具体的原始数据。它接收编排代理的指令、领域专家知识和目标数据片段，进行分析后返回精简的摘要。
*   **核心思想**：通过“渐进式发现”维持精简的上下文窗口，避免一次性输入大量数据导致模型性能下降。

### 6. 主要结论与发现
*   **性能与规模正相关**：顶级模型 Claude-Opus 4.6 表现最佳，准确率达到 **85.96%**；GLM-5 以 83.88% 紧随其后。

*   **失败原因分析**：约 15% 的错误源于编排代理在导航时选择了错误的数据源，而非子工作流的分析错误。这表明领域知识的语义描述对代理的引导至关重要。

### 7. 优点
*   **架构优雅**：解耦设计有效解决了长文本处理中的“上下文腐烂”问题，使模型能处理兆字节级别的模拟数据。
*   **领域知识融合**：将专家经验注入工具描述中，使通用模型能够理解 VSM 的特定逻辑（如安全库存、瓶颈分析）。
*   **评估严谨**：通过人类专家校验 LLM 裁判，建立了一套可信的自动化评估流程。

### 8. 不足与局限
*   **小模型适配性差**：较小的模型（<30B）在复杂的编排和多步推理中表现不佳，限制了该方案在低算力环境下的部署。
*   **成本与延迟**：由于需要多次工具调用和多步推理，Token 消耗量大（平均每样本约 3-4 万 Token），导致推理成本和延迟较高。
*   **数据长度限制**：虽然架构支持解耦，但如果单个时间序列数据超过了子工作流模型的上下文窗口，该方法仍需进一步改进（如引入更复杂的数据预处理）。
