## 摘要
在长上下文场景下，预填充（Prefilling）的计算开销是大型语言模型（LLMs）和大型多模态模型（LMMs）的主要瓶颈。虽然 Token 剪枝可以缩短序列长度，但以往的方法依赖于启发式规则，这破坏了与 FlashAttention 等硬件高效算核的兼容性。在这项工作中，我们观察到 Token 会向“语义固定点”（semantic fixing points）演化，使得进一步的处理变得冗余。为此，我们提出了 Delta Attention 选择性停止（DASH），这是一种无需训练的策略，通过监控自注意力机制的逐层更新动态来选择性地停止已稳定的 Token。广泛的评估证实，DASH 在语言和视觉基准测试中具有良好的泛化性，在保持模型准确性和硬件效率的同时，显著提升了预填充速度。代码将发布在 https://github.com/verach3n/DASH.git。

## Abstract
Prefilling computational costs pose a significant bottleneck for Large Language Models (LLMs) and Large Multimodal Models (LMMs) in long-context settings. While token pruning reduces sequence length, prior methods rely on heuristics that break compatibility with hardware-efficient kernels like FlashAttention. In this work, we observe that tokens evolve toward \textit{semantic fixing points}, making further processing redundant. To this end, we introduce Delta Attention Selective Halting (DASH), a training-free policy that monitors the layer-wise update dynamics of the self-attention mechanism to selectively halt stabilized tokens. Extensive evaluation confirms that DASH generalizes across language and vision benchmarks, delivering significant prefill speedups while preserving model accuracy and hardware efficiency. Code will be released at https://github.com/verach3n/DASH.git.

随着大语言模型（LLMs）和多模态模型（LMMs）处理的上下文长度不断增加，**预填充（Prefilling）阶段**的计算开销成为了主要的性能瓶颈。虽然现有的 Token 剪枝技术可以减少序列长度，但它们大多依赖于启发式的重要性评分（如累积注意力权重），这通常需要访问完整的注意力矩阵，从而破坏了与 **FlashAttention** 等硬件加速算核的兼容性。
论文的核心动机是寻找一种**无需训练、硬件友好**的方法，通过识别并停止处理那些已经“稳定”的 Token，来降低长上下文推理的延迟。

### 2. 方法论：DASH (Delta Attention Selective Halting)
*   **核心思想**：论文观察到 Token 在通过 Transformer 层时会向“语义固定点”演化。当一个 Token 的更新幅度（$\Delta$）变得极小时，意味着它在当前上下文中已经稳定，进一步的处理是冗余的。
*   **关键技术细节**：
    *   **Delta 信号定义**：定义 $\Delta_{attn}^{(l)} = \|U^{(l)}_t\|_2$，其中 $U^{(l)}_t$ 是第 $l$ 层自注意力机制在残差相加前的输出。
    *   **选择性停止策略**：在预设的启动层 $l_s$（通常为总层数的 30%-50% 处），计算所有 Token 的 $\Delta_{attn}$。
    *   **一次性决策（Single-shot）**：根据 $\Delta_{attn}$ 分数进行排序，保留前 $(1-\rho)T$ 个 Token（$\rho$ 为剪枝率），其余 Token 被判定为“停止（Halted）”。
    *   **执行语义**：被停止的 Token 在后续所有层中直接跳过自注意力（Self-Attention）和前馈网络（FFN）的计算，其隐藏状态保持在 $l_s$ 层的值不变。
*   **算法流程**：输入序列 $\rightarrow$ 正常处理前 $l_s$ 层 $\rightarrow$ 在 $l_s$ 层计算 $\Delta_{attn}$ 并筛选活跃集 $\rightarrow$ 后续层仅对活跃集进行计算 $\rightarrow$ 输出。

### 6. 主要结论与发现
*   **语义固定点确实存在**：大多数 Token 在中间层之后更新量极小，且这些 Token 在后续层中吸引的注意力也极少。
*   **视觉 Token 饱和更快**：在多模态模型中，视觉 Token 比文本 Token 更早达到稳定状态，因此可以更激进地剪枝。

*   **硬件兼容性**：由于不依赖注意力矩阵的显式计算，DASH 完美支持 FlashAttention，实现了真正的推理加速。

### 7. 优点
*   **高效且实用**：无需重新训练模型，即插即用，且直接针对预填充阶段的 FLOPs 进行优化。
*   **理论支撑扎实**：通过“稳定性”这一动力学视角解释 Token 冗余，比纯启发式方法更具说服力。
*   **跨模态通用性**：一套逻辑同时适用于纯文本和多模态输入，无需针对模态设计特殊规则。

### 8. 不足与局限
*   **启动层依赖**：虽然有 PPL 代理指标，但最佳启动层 $l_s$ 在不同模型和任务间仍可能存在细微差异，需要少量调优。
*   **细粒度任务挑战**：在 OCRBench 和 ChartQA 等需要极高局部细节的任务中，虽然 DASH 领先于其他剪枝方法，但相比原始模型仍有一定性能下降。
*   **KV Cache 优化有限**：DASH 主要优化预填充阶段的计算（FLOPs），虽然缩短了 KV Cache 长度，但对解码阶段的显存带宽优化并非其核心讨论点。
