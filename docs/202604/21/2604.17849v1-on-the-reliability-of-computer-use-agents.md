## 摘要
计算机使用智能体在网页导航、桌面自动化和软件交互等现实任务中取得了快速进步，在某些情况下甚至超越了人类的表现。然而，即使在任务和模型保持不变的情况下，一次成功的智能体在重复执行同一任务时也可能失败。这提出了一个根本性的问题：如果一个智能体能够成功完成一次任务，是什么阻碍了它可靠地完成任务？在这项工作中，我们通过三个因素研究了计算机使用智能体不可靠性的来源：执行过程中的随机性、任务说明的歧义性以及智能体行为的可变性。我们在 OSWorld 基准测试上，通过对同一任务的重复执行以及捕捉不同设置下任务级变化的配对统计检验，对这些因素进行了分析。我们的分析表明，可靠性既取决于任务是如何说明的，也取决于智能体行为在多次执行中的变化情况。这些发现表明，有必要在重复执行的情况下评估智能体，允许智能体通过交互消除任务歧义，并倾向于选择在多次运行中保持稳定的策略。

## Abstract
Computer-use agents have rapidly improved on real-world tasks such as web navigation, desktop automation, and software interaction, in some cases surpassing human performance. Yet even when the task and model are unchanged, an agent that succeeds once may fail on a repeated execution of the same task. This raises a fundamental question: if an agent can succeed at a task once, what prevents it from doing so reliably? In this work, we study the sources of unreliability in computer-use agents through three factors: stochasticity during execution, ambiguity in task specification, and variability in agent behavior. We analyze these factors on OSWorld using repeated executions of the same task together with paired statistical tests that capture task-level changes across settings. Our analysis shows that reliability depends on both how tasks are specified and how agent behavior varies across executions. These findings suggest the need to evaluate agents under repeated execution, to allow agents to resolve task ambiguity through interaction, and to favor strategies that remain stable across runs.

### 1. 核心问题与研究动机
*   **核心问题**：为什么一个在特定任务上能够成功一次的智能体，在重复执行相同任务时却无法保持一致的成功？
*   **研究动机**：当前的评估指标（如 Pass@k）侧重于衡量智能体“解决问题的上限”，即多次尝试中只要成功一次即可。但在航空、医疗、自动驾驶等现实严苛场景中，**可靠性（即重复执行的一致成功率）**是部署的前提。研究者发现，即使模型和任务不变，智能体也经常表现出极大的不稳定性。

### 2. 方法论
论文将不可靠性的来源分解为三个维度，并提出了相应的评估框架：
*   **三个影响因素**：
    1.  **执行随机性 (Stochasticity)**：包括模型解码时的 Token 采样随机性和环境动态变化的干扰。
    2.  **任务说明歧义性 (Ambiguity)**：用户指令不明确，导致智能体理解与评估准则不一致。
    3.  **行为变异性 (Variability)**：即使指令明确，智能体也可能选择不同的执行路径，其中某些路径鲁棒性较差。
*   **核心指标与统计工具**：
    *   **Pass^k**：估计连续 $k$ 次执行均成功的概率（衡量可靠性的下限）。

    *   **Wilcoxon 符号秩检验**：用于捕捉任务成功次数的增量改进，即使未达到完全可靠。

### 6. 主要结论与发现
*   **确定性并非灵丹妙药**：强制确定性解码并不总能提高可靠性，有时反而降低了智能体应对环境微小变化的能力。
*   **歧义是主要障碍**：澄清任务指令能显著提升所有模型的可靠性。执行中的实时反馈（用户模拟器）比单纯的多次重试（Retry）更有效。

*   **环境敏感性**：即使是壁纸或图标等非功能性改变，也会导致前沿模型（如 Claude）的可靠性显著下降。

*   **实用性强**：提出的“用户模拟器反馈”和“指令澄清”方法为提升现有智能体的性能提供了直接可行的路径。

### 8. 不足与局限
*   **基准局限**：实验主要集中在 OSWorld，虽然该基准很全面，但仍可能存在特定环境的偏差。
