## 摘要
本文研究了多智能体具身问答（MA-EQA），旨在针对机器人团队在长时程内所观察到的内容进行查询。与现有的强调感知、通信或计算性能指标的边缘资源管理方法不同，MA-EQA 强调存储质量。为了应对这一范式转变，我们提出了一种基于生成对抗考试（GAE）的存储质量（QoM）模型，该模型利用前向仿真来评估存储检索，并使用所得的考试分数来计算 QoM 值。随后，我们提出了以存储为中心的功率分配（MCPA），在通信资源约束下最大化 QoM 函数。通过渐近分析发现，发射功率与 GAE 错误概率成正比，从而优先考虑高 QoM 的机器人。大量的实验表明，MCPA 在各种场景下的多种指标上均优于广泛的基准方法，取得了显著的提升。

## Abstract
This paper considers multi-agent embodied question answering (MA-EQA), which aims to query robot teams on what they have seen over a long horizon. In contrast to existing edge resource management methods that emphasize sensing, communication, or computation performance metrics, MA-EQA emphasizes the memory qualities. To cope with this paradigm shift, we propose a quality of memory (QoM) model based on generative adversarial exam (GAE), which leverages forward simulation to assess memory retrieval and uses the resulting exam scores to compute QoM values. Then we propose memory centric power allocation (MCPA), which maximizes the QoM function under communication resource constraints. Through asymptotic analysis, it is found that the transmit powers are proportional to the GAE error probability, thus prioritizing towards high-QoM robots. Extensive experiments demonstrate that MCPA achieves significant improvements over extensive benchmarks in terms of diverse metrics in various scenarios.

*   **研究动机**：
    *   **范式缺失**：现有的资源管理主要关注通信速率、感知覆盖范围或计算效率，但这些指标并不直接等同于“记忆的质量”。
    *   **任务特性**：具身问答（EQA）是开放式的，在收集数据时无法预知用户会问什么，导致难以定义明确的优化目标。

### 2. 核心方法论
*   **生成对抗考试（GAE）**：
    *   **核心思想**：利用“前向仿真”模拟问答过程。既然问题不可知，但模式可预测，则通过机器人上传的少量样本生成“模拟考题”。
    *   **流程**：机器人上传少量试点数据 -> 服务器利用视觉语言模型（VLM）生成考题和标准答案 -> 服务器用现有记忆尝试回答。如果现有记忆答不上来，说明该机器人的新数据价值极高。
*   **记忆质量（QoM）模型**：
    *   将 GAE 的得分（错误率）转化为显式的 QoM 函数。QoM 与机器人能提供的新知识量成正比。
*   **以存储为中心的功率分配（MCPA）**：
    *   **优化目标**：在总发射功率约束下，最大化全系统的 QoM 总和。
    *   **算法实现**：利用逐次优化（Successive Optimization）方法，将非凸问题转化为一系列凸子问题求解。
    *   **理论发现**：渐近分析表明，最优发射功率与 GAE 的错误概率成正比，即：系统会自动优先给那些拥有“服务器尚未掌握的知识”的机器人分配更多功率。

*   **实验规模**：所有定量结果均基于 **50 次随机模拟运行** 的平均值，每轮包含独立的信道实现。
*   **充分性**：实验涵盖了不同机器人数量（5 台和 10 台）、不同功率预算（100mW 到 300mW）以及多种评估指标（准确率、QoM、速率、激活机器人数量）。

### 6. 主要结论与发现
*   **性能领先**：MCPA 在问答准确率上达到 **95.35%**，远超 MaxRate（61.69%）和 MaxCov（86.09%）。
*   **质量胜过数量**：通信速率最高（MaxRate）或覆盖范围最广（MaxCov）并不意味着问答效果最好。只有当增加的通信/感知能带来“新知识”时，才对存储有益。
*   **自适应性**：MCPA 能够识别冗余信息（如两个机器人看到同一物体），并自动抑制冗余节点的功率以节省资源。

### 7. 优点与亮点
*   **范式创新**：首次提出了“以存储为中心”的通信设计，精准切中了具身智能长时程任务的痛点。
*   **GAE 机制**：巧妙地解决了开放式任务中目标函数难以定义的问题，通过“模拟考试”实现了端到端的性能优化。
*   **理论支撑**：不仅有算法，还通过数学证明给出了功率分配的闭式解趋势，具有较强的解释性。

### 8. 不足与局限
*   **计算开销**：虽然推理时间相对通信时间较短，但在实时性要求极高的场景下，GAE 过程（VLM 描述 + LLM 出题）仍可能产生不可忽视的初始延迟。
*   **试点数据依赖**：GAE 依赖于机器人上传的少量试点数据（Pilot Data），如果试点数据采样不具有代表性，可能会导致 QoM 估计偏差。
*   **场景单一**：实验主要集中在 CARLA 城市道路场景，对于室内、森林等更复杂或遮挡严重的具身环境，其泛化能力有待进一步验证。
