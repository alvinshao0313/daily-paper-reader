# DM 3-Nav: Decentralized Multi-Agent Multimodal Multi-Object Semantic Navigation 

Amin Kashiri, Atharva Jamsandekar, Yasin Yazıcıo˘ glu 

Abstract — We present DM 3-Nav, a fully decentralized multi-agent semantic navigation system supporting multimodal open-vocabulary goal specification and multi-object missions. In our setting, decentralization implies operation without a central coordinator, global map aggregation, or shared global state at runtime. Robots operate autonomously and coordinate through ad-hoc pairwise communication, exchanging local maps, goal status, and navigation intent without synchronization. An im-plicit task allocation mechanism combining intent broadcasting and distance-weighted frontier selection reduces redundant exploration while preserving decentralized operation. Evalu-ations on HM3DSem scenes using the HM3Dv0.2 and GOAT-Bench datasets demonstrate that DM 3-Nav matches or exceeds centralized and shared-map baselines while eliminating single points of failure inherent in centralized architectures. Finally, we validate our approach in a real-world office environment using two mobile robots, demonstrating successful deployment relying entirely on onboard sensing and computation. A video of our real-world experiments is available online. 1

I. I NTRODUCTION 

Multi-robot systems offer significant potential for search, inspection, and retrieval tasks that require locating multi-ple objects specified through diverse modalities, including category labels, language descriptions, or reference images. Achieving robust and scalable multi-object semantic navi-gation with robot teams in unknown environments requires coordination without centralized planning or shared global state. However, existing approaches address these require-ments only partially: some focus on multimodal or multi-object navigation but are limited to single-agent settings, while others introduce multi-agent coordination but rely on centralization and support only categorical goals. Semantic navigation has made substantial progress in re-cent years. Early work on object-goal navigation introduced semantic maps for category-level navigation [1]. Gadre et al. [2] showed that CLIP-based representations enable zero-shot navigation to object categories, and Yokoyama et al. [3] grounded vision-language semantics into persistent frontier maps for long-horizon planning. However, these approaches

The MultiON benchmark [4] formalized this task. Subsequent approaches addressed MON through reinforcement learning [5], [6], but require training and support only categorical goals. More recently, zero-shot object navigation (ZSON) methods have emerged. OneMap [7] builds reusable open-vocabulary se-mantic maps, and WTRP-Searcher [8] formulates frontier se-lection as a weighted traveling repairman problem. GOAT [9] further supports multimodal goals specified through language and images via instance-aware semantic maps. While these methods support open-vocabulary queries and multi-object sequences, they remain limited to single-agent operation. Semantic navigation has also been extended to multi-agent settings. Co-NavGPT [10] uses a centralized vision-language planner operating on a shared map. MCoCo-Nav [11] performs decentralized reasoning but relies on a globally aggregated semantic map, introducing implicit centraliza-tion. Moreover, both systems support only single categorical goals. In the related domain of multi-agent exploration, the literature provides foundations for decentralized coordina-tion. Frontier-based approaches achieve coordination through map-sharing [12], while RACER [13] and related methods [14] achieve fully decentralized exploration via efficient map representations and pairwise interactions. A complementary line of work studies collaborative SLAM, where the goal is to build globally consistent maps and trajectories. Centralized approaches such as CCM-SLAM [15] and COVINS [16] aggregate data at a central server, while decentralized alter-natives such as DOOR-SLAM [17], Kimera-Multi [18], and Swarm-SLAM [19] coordinate through peer-to-peer com-munication, using distributed pose graph optimization and inter-robot loop closure detection. Multi-robot active SLAM further couples mapping with exploration, using information-theoretic or frontier-based methods [20], [21] to jointly re-duce map uncertainty. While these methods provide effective solutions for multi-robot exploration and SLAM, they do not address decentralized object search or semantic navigation. In this paper, we present, to the best of our knowledge, the first multi-agent semantic navigation system that jointly supports multimodal goals (category, language, and image) and multi-object episodes under fully decentralized coor-dination. Our approach employs an implicit coordination mechanism that combines intent broadcasting and distance-weighted frontier selection to achieve task allocation without explicit negotiation, while each robot leverages an instance-aware semantic mapping framework based on GOAT [9]. P ROBLEM FORMULATION 

We consider a decentralized multi-agent semantic naviga-tion task where a team of n robots must cooperatively locate a set of target objects in an unknown environment. 

Task Definition. An episode E = ( S, P, G ) consists of a scene S, initial robot positions P = {p1, . . . , p n}, and a set of goal specifications G = {g1, . . . , g m}. Each goal gj

can be specified in one of three modalities: (1) a category label (e.g., “tv”), (2) a natural language description (e.g., “the black table next to the door”), or (3) a reference image of the goal object. Robots have no prior knowledge of the scene layout or object locations, and need to find the goals in any order. 

Observations and Actions. At each timestep t, robot i

receives an observation oit = ( Iit , D it, p it) consisting of an RGB image, depth image, and the robot’s pose. Based on this observation and its internal state, the robot selects a navigation action ait.

Decentralization Constraints. Robots operate without ac-cess to a centralized server or shared global map. Each robot maintains its own local representation of the environment and coordinates with others through ad-hoc pairwise exchanges, sharing local maps, goal status, and navigation intent when-ever communication between two robots is available. 

Success Criteria. A goal gj is considered found when any robot navigates to within a threshold distance of an object matching the goal specification and executes a termination action. Robots operate under a maximum step budget, after which the episode terminates regardless of how many goals have been found. 

Objective. The objective is to maximize the number of goals found while minimizing total navigation cost across all robots. We formalize this as Multi-agent SPL (MSPL) in Section IV-B, which combines success rate with the ratio of optimal to actual path length. III. M ETHODOLOGY 

We present a decentralized multi-agent navigation system capable of locating multiple goals specified through images, language descriptions, or category labels. Two robots coordinate to locate four distinct targets specified via language descrip-tions, reference images, and category labels. By dynamically exchanging information when within communication range, the agents achieve scalable autonomous exploration and task allocation without relying on a centralized planner or continuous connectivity. A. System Overview 

Each robot independently processes RGB-D observations to build a spatial-semantic map, detect and localize objects, and make navigation decisions. Unlike multi-agent methods that require a shared global map [11] or central coordination Fig. 2: Overview of the DM 3-Nav architecture. Each robot operates autonomously with its own perception, memory, and planning modules. Decentralized coordination is achieved by exchanging semantic maps, goal status, and navigation intent through local communications with in-range robots. [10], our approach coordinates through ad-hoc pairwise information exchange. When two robots have the opportunity to communicate, they share relevant map information and goal status, enabling implicit task allocation without global synchronization. Since coordination occurs only through lo-cal pairwise exchanges, the system scales naturally as robots are added—each robot’s computational and communication overhead depends only on its local neighborhood, not the total team size. 

B. Single-Agent Architecture 

For individual robot perception, mapping, and goal match-ing, we adopt the modular architecture from GOAT [9]. Each robot maintains a 2D top-down semantic map with geomet-ric channels (obstacle, explored, navigable) and semantic channels encoding detected object instances. An instance memory I stores multi-view appearance observations for each detected object, enabling robust goal matching across viewpoints. Goals are matched by category lookup, CLIP-based [25] similarity for language descriptions, or SuperGlue [26] feature matching for image goals. The robot navigates to matched instances or performs frontier-based exploration until a match is found. Modifications to the original GOAT design are detailed in Appendix I. 

C. Decentralized Coordination 1) Communication Protocol: Coordinating multiple robots without a central planner requires mechanisms for information sharing and implicit task allocation. Our approach relies on local communication between robots to coordinate exploration and assign goals. Robots communicate opportunistically in a fully decentral-ized, pairwise manner. To reduce communication overhead, each robot maintains a per-neighbor cooldown period τ : after sending a full map to robot rj , robot ri will not send another full map to rj until at least τ steps have elapsed. Between full exchanges, robots continue to share lightweight messages containing only location and task information, enabling fast coordination with minimal bandwidth. Since each robot builds its map in its own local coor-dinate frame, merging maps requires estimating the rigid transformation between frames. We identify candidate point matches between the two maps using two methods: ORB feature matching on the obstacle grid restricted to explored regions, and semantic landmark matching that pairs object centroids of the same category by comparing the local obstacle structure around each centroid using a rotation-invariant radial descriptor. Because candidate matches in-evitably include incorrect correspondences, a robust esti-mator is needed to recover the transformation. Several ap-proaches exist for occupancy grid map alignment. Spectral methods based on Hough spectrum cross-correlation [27] estimate rotation and translation from global map structure but assume environments with prominent linear features. ICP [28] requires a good initial alignment and is sensitive to partial overlap. Feature-based pipelines pair a keypoint detector with a robust estimator such as RANSAC [29] or MSAC [30] to reject outliers [31]. Within the feature-based family, we evaluated several detector-estimator combinations, including KAZE/MSAC [31] and AKAZE/RANSAC [32], and found that ORB with RANSAC performed most reliably in our setting, particularly under the limited map overlap that arises in our setup. Since the alignment stage of our pipeline is designed to be modular, the ORB/RANSAC combination can be replaced with alternative methods without affecting the rest of the system. The alignment is validated by computing the intersection-over-union of obstacle cells in the overlapping explored re-gions; alignments below a confidence threshold are rejected. If alignment fails, robots continue operating independently with their local maps; subsequent encounters may succeed as additional explored regions provide more feature corre-spondences. Once estimated, the transformation is cached per neighbor pair to accelerate future map merges and enable the lightweight location-only exchanges described above. During communication, robots can exchange:  

> •

Semantic map : The multi-layer map including occu-pancy, explored regions, and semantic channels. The neighbor’s map is warped into the receiving robot’s frame using the estimated transformation. Since both occupancy and semantic channels are stored as log-odds, merging selects the value with the greater absolute magnitude per cell, preserving the stronger evidence from either robot. Values are not summed to avoid double-counting evidence when both robots have ob-served the same region.  

> •

Location : The robot’s current position in its own frame, which the receiver transforms into its frame using the cached transformation.  

> •

Goal status : Which goals have been completed, allow-ing other robots to remove them from their task list.  

> •

Intent : The goal each robot is currently pursuing, used for cooperative task allocation as described below. The communication protocol requires no central com-ponent or global connectivity; each robot builds its own understanding of the team state by accumulating informa-tion from pairwise exchanges. When a robot broadcasts its intent to pursue a goal, other robots remove that goal from their planning. To resolve conflicts when multiple robots simultaneously target the same goal, if a robot receives an intent for a goal it is currently pursuing from a robot with a higher score, it yields and selects an alternative goal. Ties are broken by a unique priority assigned at initialization. This mechanism reduces duplicate effort without negotiation.

This formulation encourages robots to select frontiers where they are closer than others, naturally distributing the team across the environment without explicit negotiation. Fig. 3 illustrates this behavior: robots that have recently communicated select spatially separated frontiers, while Robot 4, lacking communication with Robots 1 and 3, targets a region they have already explored. Several methods exist for coordinating frontier selection in multi-robot exploration, including auction-based alloca-tion [33], distributed variants of the Hungarian method [34], and heuristics such as MinPos [35]. Auction-based methods require multiple rounds of bidding each time the frontier set changes, adding communication overhead and latency. Distributed Hungarian methods rely on iterative message passing that assumes a connected communication graph throughout convergence, an assumption that does not hold under intermittent ad-hoc connectivity. IV. E VALUATION 

We evaluate our method in Habitat simulator [36], [37] using HM3DSem scenes. Robots operate with a discrete action space: move forward (25cm), turn left/right (30 ◦), and 

stop . A goal is considered found when a robot executes stop 

within a dataset-specified distance (0.2 m - 1 m) of the target. For evaluations, we use a proximity-based communication model where robots can communicate within range rcomm =5m. 

A. Dataset 

We employ two task datasets to benchmark our method. 

HM3Dv0.2: We use the HM3Dv0.2 validation split to compare against existing multi-agent semantic navigation methods. This dataset consists of 1000 single-target cate-gorical goal episodes across 36 scenes, limited to 6 object categories: chair, bed, couch, potted plant, toilet, and TV. A goal is considered found if the agent stops within 0.2 m of the target. For comparison with the baselines on this dataset, we use two robots to match the configuration used in prior work [10], [11]; For this dataset, we use MaskRCNN [38] for object detection to ensure fair comparison with baselines that use similar closed-vocabulary detectors. 

GOAT-Bench. While HM3Dv0.2 enables comparison with existing multi-agent methods, it lacks open-vocabulary multimodal goals. To evaluate multimodal, multi-object per-formance, we use GOAT-Bench [24], which provides open-vocabulary navigation episodes with goals specified via cat-egory labels, language descriptions, or images. The dataset spans 312 object categories, and each episode contains 5-10 goals. We evaluate on the val seen split, which uses the same 36 scenes as HM3Dv0.2 with 10 episodes per scene. A goal is considered found if the agent stops within 1 m

of the target (following GOAT-Bench evaluations). We use a communication cooldown period of τ = 10 steps and evaluate with 1-4 agents. For the GOAT-Bench dataset, we use Detic [39] to support open-vocabulary detection. 

B. Metrics 

For single-goal episodes, we report Success Rate (SR), the proportion of episodes where the agent successfully locates the target, and Success weighted by Path Length (SPL), which additionally penalizes inefficient paths: SPL = S · d∗

max( d, d ∗)

where S ∈ { 0, 1} indicates success, d∗ is the length of the shortest navigable path from the agent’s starting position to the target, and d is the total distance traveled by the agent. However, SPL is designed for single-agent, single-object episodes and does not extend naturally to our setting, which involves multiple robots, multiple objects, and multiple valid instances per object. A goal specification such as ”chair” may be satisfied by any of several instances in the environment, so the optimal path depends on which instance each robot targets. This structure corresponds to the Generalized TSP (GTSP) [40], where goals define clusters of valid locations and visiting any node in a cluster satisfies the corresponding goal. We extend GTSP to the multi-agent setting with a min-max objective, minimizing the maximum distance traveled by any robot, because makespan reflects actual completion time under parallel execution. Unlike closed-tour variants, robots follow open paths and do not return to their starting positions. Based on this formulation, we propose Multi-object Multi-agent SPL (MSPL) , a metric that evaluates navigation efficiency for any combination of team sizes and goal counts. For multi-object episodes, SR is defined as the proportion of goals successfully located across all episodes (also known as “Progress” [4]). MSPL extends SPL as: MSPL = SR · d∗

max {d∗, max j dj }

where d∗ is the optimal makespan and max j dj is the longest distance traveled by any robot during execution. The max in the denominator ensures MSPL remains bounded in [0 , 1] ,consistent with single-agent SPL. Notably, MSPL reduces to standard SPL when n = 1 and m = 1 (single agent, single goal), making it a strict generalization. We formulate the optimal makespan computation as a Mixed Integer Linear Program (MILP) and solve it using Gurobi [41]. Since d∗ depends only on the scene geometry and goal locations, it can be computed offline once per episode for evaluation purposes and does not affect runtime performance. C. Baselines 

Co-NavGPT [10] uses a Vision Language Model (VLM) as a centralized global planner. Each robot builds a local semantic map, which is aggregated into a unified global map. The VLM receives this map along with encoded robot states and frontier regions, and assigns frontiers to each robot for coordinated exploration. However, this approach requires centralized map aggregation and a single VLM to coordinate all robots. 

MCoCo-Nav [11] employs multimodal chain-of-thought reasoning with VLMs to evaluate exploration value at each frontier through probabilistic scoring. Each robot runs its own VLM for decentralized planning. However, all robots share a global semantic map that integrates observations from the entire team, creating implicit coupling among agents. Both baselines are limited to single categorical goals per episode and rely on shared global state. In contrast, our method operates fully decentralized with only ad-hoc pairwise communication, and supports multimodal goals (category, language, image) across multi-object episodes, which we evaluate separately.

V. E XPERIMENTS 

To demonstrate feasibility beyond simulation, we deploy our system on two AgileX Scout mini mobile robots, each equipped with an NVIDIA Jetson Orin for onboard com-putation. Robots perceive their environment using an Intel RealSense D455 RGB-D camera for semantic mapping and a Livox MID-360 LiDAR for localization via Direct LiDAR-Inertial Odometry [42]. Consistent with our decentralized design, robots communicate through an ad-hoc WiFi network with no central infrastructure. We evaluate our approach in an office environment of approximately 200 m2. The robots are tasked with locating 8 targets specified through all three supported modalities: category labels (backpack, hoodie, cup, chair, plant), lan-guage descriptions (tennis racket, umbrella), and a reference image (kettle), as shown in Fig. 4c. Fig. 4a shows the robot platform, and Fig. 4b presents sample instances where the robots find a target. The two-robot team collaboratively explored the environment and successfully located all the targets in 14 minutes. We presented DM 3-Nav, a decentralized multi-agent se-mantic navigation system that supports multimodal goal specifications and multi-object episodes without prior train-ing. Our approach achieves coordination through ad-hoc pairwise communication, eliminating the need for centralized planning or global map synchronization. Experiments on HM3Dv0.2 and GOAT-Bench demonstrate that decentral-ized coordination matches or exceeds centralized baselines while providing architectural benefits in scalability and fault tolerance. We further validated the system’s real-world ap-plicability by deploying it on a team of mobile robots in an office environment, relying entirely on onboard sensing, localization, and computation. Building upon this foundation, future work will pursue several directions. First, we aim to optimize the system for edge deployment by integrating lightweight perception models to reduce computational overhead and accelerate the decision-making loop. Second, we plan to extend our framework to heterogeneous robot teams, such as combining UAVs and UGVs with complementary sensing capabilities or assigning specialized roles to different agents. Third, we intend to support more complex mission specifications through temporal logics, enabling constraints on goal order-ing, timing, and conditional dependencies beyond the current unordered goal sets. APPENDIX IMODIFICATIONS TO SINGLE -A GENT ARCHITECTURE 

Our single-agent architecture follows the GOAT frame-work [9] with several implementation differences. Table I shows that our implementation achieves 44.0% SR with predicted semantics and 85.7% SR with ground-truth semantics on GOAT-Bench, compared to 26.3% and 56.7% for the original GOAT. We document the key differences below. 

A. Probabilistic Map Accumulation 

The original GOAT implementation updates its spatial map by taking the element-wise maximum between the existing map and each new observation. While this preserves detections over time, it also means that a single false positive, whether from a noisy depth reading or an incorrect segmen-tation, permanently marks a cell as occupied or belonging to a semantic class, with no mechanism for correction. We replace this with a log-odds accumulation scheme for both occupancy and semantic channels. Each observation con-tributes positive evidence for detected occupancy or semantic class and negative evidence for free space, accumulated over time and clamped to a maximum magnitude. Cells are classified as occupied only when accumulated log-odds exceed zero, so isolated false detections decay and are even-tually overridden by consistent free-space observations. This representation also integrates naturally with our multi-robot map merging (Section III-C), where we select the value with greater absolute magnitude rather than summing, to avoid double-counting evidence from overlapping observations. 

B. Goal Region Selection 

When navigating to a detected object, the original imple-mentation selects the closest free cell to the object’s location. This approach can fail in several scenarios: the closest free cell may be unreachable, or it might be on the opposite side of a wall. We improve goal region selection through a three-step process: (1) dilate the goal region to identify nearby free cells, (2) cluster these free cells into connected components, and (3) select the cluster closest to the agent’s current viewpoint. This ensures the selected goal position is both reachable and observable from a feasible approach direction. 

C. Frontier-Based Exploration 

The original GOAT implementation marks explored re-gions using geometric heuristics, such as placing a fixed-radius circle around the agent’s position. This does not account for the agent’s actual field of view and can mark areas outside walls as explored. We replace this with a raycasting-based approach that marks only cells visible from the agent’s current position, accounting for field of view and obstacle occlusions. This provides an accurate representation of what the agent has actually observed. 

D. Local Planning 

We make several improvements to low-level path execu-tion. The original implementation selects waypoints from free cells within a fixed radius around the agent. We extend this by: (1) dilating obstacles when possible to maintain clearance from walls, (2) filtering out cells that are geo-metrically free but have obstacles along the path to them, and (3) adaptively reducing the planning radius when larger radii yield no valid waypoints, enabling finer movements in constrained spaces. 

E. Detection Verification 

To reduce false positive detections, we add a verification stage using YOLO as a secondary detector. For an object detected by the primary detector (MaskRCNN or Detic), we also compute a YOLO detection score and average it with the initial score when deciding whether to navigate to the object. We use YOLOv10 [43] for closed-vocabulary detection on HM3Dv0.2 and YOLO-World [44] for open-vocabulary detection on GOAT-Bench. This verification step improves the success rate by filtering out incorrect detections. 

F. Additional Modifications 

Stair detection. We detect stairs by identifying regions where expected ground plane points are missing in the point cloud, and mark these as non-navigable. 

Map scale switching. When no valid path exists in the local map, we switch to planning on the global map. Conversely, we use the local map when it suffices, reducing computational cost. 

Detection score thresholding. For categorical goals, rather than navigating to the highest-scoring detection im-mediately, we consider the detection confidence and navigate only when it exceeds a threshold. 

Segmentation refinement. We apply morphological ero-sion to segmentation masks to reduce boundary noise, and filter out points whose depth deviates significantly from the instance median to remove incorrectly included background regions.
