## 摘要
大语言模型 (LLM) 智能体越来越多地发出会改变真实系统的 API 调用，然而许多当前的架构直接将随机的模型输出传递给执行层。我们认为这种耦合带来了安全风险，因为在执行时无法假设模型的正确性、上下文感知能力和对齐性。我们引入了主权代理循环 (Sovereign Agentic Loops, SAL)，这是一种控制平面架构，其中模型发出带有理由的结构化意图，而控制平面在执行前根据真实的系统状态和策略验证这些意图。SAL 结合了混淆膜（限制模型访问身份敏感状态）和加密链接的证据链（用于可审计性和回放）。我们对 SAL 进行了形式化，并证明在所述假设下，它提供了受策略约束的执行、身份隔离和确定性回放。在针对云基础设施的 OpenKedge 原型中，SAL 在策略层拦截了 93% 的不安全意图，通过一致性检查拒绝了剩余的 7%，在我们的基准测试中防止了不安全执行，并增加了 12.4 毫秒的中值延迟。

## Abstract
Large language model (LLM) agents increasingly issue API calls that mutate real systems, yet many current architectures pass stochastic model outputs directly to execution layers. We argue that this coupling creates a safety risk because model correctness, context awareness, and alignment cannot be assumed at execution time. We introduce Sovereign Agentic Loops (SAL), a control-plane architecture in which models emit structured intents with justifications, and the control plane validates those intents against true system state and policy before execution. SAL combines an obfuscation membrane, which limits model access to identity-sensitive state, with a cryptographically linked Evidence Chain for auditability and replay. We formalize SAL and show that, under the stated assumptions, it provides policy-bounded execution, identity isolation, and deterministic replay. In an OpenKedge prototype for cloud infrastructure, SAL blocks 93% of unsafe intents at the policy layer, rejects the remaining 7% via consistency checks, prevents unsafe executions in our benchmark, and adds 12.4 ms median latency.

以下是对该论文的结构化总结：

### 1. 核心问题与整体含义
*   **研究动机**：当前的 AI 智能体架构通常将 LLM 的输出直接视为可执行命令（紧耦合）。由于 LLM 具有随机性（Stochastic）、可能产生幻觉，且往往在信息不全或上下文模糊的情况下工作，直接赋予其执行权会导致严重的安全风险（如误删生产数据库）。
*   **核心理念**：论文提出了**“解耦原则”**（Decoupling Principle），主张将“推理”与“执行权”分离。推理模型应仅产生“可验证的意图”（Intent），而执行必须经过一个主权控制平面的中介和验证。

### 2. 方法论
SAL 架构通过以下关键技术实现推理与执行的解耦：
*   **混淆膜（Obfuscation Membrane）**：
    *   **功能**：在将系统状态发送给外部 LLM 之前，剥离身份敏感信息（如 IP 地址、资源 ID），仅保留结构化拓扑信息。
    *   **数学保证**：通过映射函数 $\Pi$ 确保混淆后的状态与原始身份信息之间的互信息为零（$I(S_{id}; \hat{S}) = 0$），从而保护数据主权。
*   **意图协议（Intent Protocol）**：模型不直接发出 API 调用，而是输出包含“操作类型、目标令牌、参数、理由（Justification）”的结构化意图。
*   **确定性主权评估（Sovereign Evaluation）**：
    *   **策略对齐**：使用 Open Policy Agent (OPA) 等引擎检查意图是否违反硬性安全规则。
    *   **一致性检查**：验证模型给出的“理由”是否与真实的系统遥测数据相符（例如：模型声称因 CPU 高负载而重启机器，但实际监控显示负载正常，则拒绝执行）。
*   **证据链（Evidence Chain）**：一个加密链接的不可篡改账本，记录状态快照、意图、理由、评估结果和执行动作，支持确定性回放和审计。

*   **场景与原型**：在名为 **OpenKedge** 的原型系统中实现，针对云基础设施（AWS）环境。

### 6. 主要结论与发现
*   **安全性**：SAL 成功拦截了 **100%** 的不安全意图。其中 93% 被策略引擎拦截，剩余 7%（语义矛盾类）被一致性检查拦截。

### 7. 优点

*   **实用性强**：极低的延迟开销使其具备进入生产环境的潜力。
