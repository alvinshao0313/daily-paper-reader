## 摘要
大语言模型（LLM）正为游戏带来更丰富的对话和社交行为，但它们也暴露了一个现有游戏界面无法直接解决的控制问题：LLM 角色应如何在参与实时多人互动的过程中，既能保持在共享游戏世界中的可执行性，又能与其他活跃角色保持社交连贯性，并在需要时受玩家引导？我们将这一问题界定为“受限自主性”（bounded autonomy），这是一种针对实时多人游戏的控制架构，它围绕三个接口组织 LLM 角色控制：智能体间交互、智能体与世界的动作执行以及玩家对智能体的引导。我们通过概率回复链衰减、基于嵌入并带有回退机制的动作落地（grounding）流水线，以及“whisper”（一种轻量级软引导技术，允许玩家在不完全覆盖自主性的情况下影响角色的下一步行动）来实现受限自主性。我们在一个实时多人社交游戏中部署了该架构，并通过对交互稳定性、落地质量、whisper 干预成功率的分析以及形成性访谈研究了其行为。我们的结果表明，受限自主性如何使 LLM 角色交互在实践中变得可行，将可控性界定为实时多人游戏中 LLM 角色特有的运行时控制问题，并为未来围绕这种交互范式构建的游戏提供了具体的范例。

## Abstract
Large language models (LLMs) are bringing richer dialogue and social behavior into games, but they also expose a control problem that existing game interfaces do not directly address: how should LLM characters participate in live multiplayer interaction while remaining executable in the shared game world, socially coherent with other active characters, and steerable by players when needed? We frame this problem as bounded autonomy, a control architecture for live multiplayer games that organizes LLM character control around three interfaces: agent-agent interaction, agent-world action execution, and player-agent steering. We instantiate bounded autonomy with probabilistic reply-chain decay, an embedding-based action grounding pipeline with fallback, and whisper, a lightweight soft-steering technique that lets players influence a character's next move without fully overriding autonomy. We deploy this architecture in a live multiplayer social game and study its behavior through analyses of interaction stability, grounding quality, whisper intervention success, and formative interviews. Our results show how bounded autonomy makes LLM character interaction workable in practice, frames controllability as a distinct runtime control problem for LLM characters in live multiplayer games, and provides a concrete exemplar for future games built around this interaction paradigm.

### 2. 方法论：核心思想与关键技术
该架构围绕三个核心接口组织，运行在“客户端-服务器-AI服务”三层架构上：

*   **Converge（智能体间接口）：解决社交连贯性**
    *   **关系偏置仲裁**：当多个角色竞争注意力时，系统优先选择社交关系得分最高的目标进行回复。
    *   **概率回复链衰减**：为了防止角色间陷入无休止的对话，引入公式 $P_{reply}(s) = \max(0, 1 - (s-1) \times \alpha)$。其中 $s$ 是回复深度，$\alpha$ 是衰减率（部署中设为0.2）。随着对话加深，继续回复的概率线性下降，直到深度达到6时强制停止。
*   **Ground（智能体-世界接口）：解决动作执行**
    *   **行为束（Behavior Bundles）**：将动画、对话、导航等封装为可执行单元（共378个）。
    *   **嵌入式匹配**：使用 `all-mpnet-base-v2` 模型将LLM生成的意图描述映射到最接近的可执行行为束。
    *   **回退机制与过滤**：若匹配置信度低于阈值（0.3），则执行安全默认动作；同时根据角色当前情绪状态过滤掉矛盾的动作（如开心时过滤掉愤怒动作）。
*   **Whisper（玩家-智能体接口）：解决轻量级引导**
    *   **软引导技术**：玩家输入短语（如“赞美他的成就”），系统将其作为提示词引导LLM生成下一步行动，而非直接接管角色。这保留了角色的个性化表达空间。

### 6. 主要结论与发现
*   **社交稳定性**：概率衰减机制成功将平均对话深度从无限（或触及硬上限）降低到4.4层，并显著提升了角色的自主行为占比（从0.615升至0.773）。
*   **落地质量**：对话类动作匹配准确率高（Top-1 87%），但**非对话类动作（如肢体接触）准确率较低（63%）**，容易产生语义干扰（如将“拍肩膀”误匹配为更亲密的动作）。
*   **引导效果**：Whisper的干预对齐率达到86.7%，证明了“软引导”在不破坏自主性的前提下是有效的。
*   **玩家感知**：玩家将LLM角色视为“数字人格”或“孩子”，极其看重个性的连贯性。

### 7. 优点
*   **工程落地性强**：在真实商业部署的环境中解决LLM落地的“最后三公里”问题。
*   **机制优雅**：通过简单的线性概率衰减解决了复杂的智能体对话死循环问题。
*   **控制范式创新**：Whisper提供了一种介于“完全操控”和“完全自主”之间的中间地带，符合游戏交互需求。

### 8. 不足与局限
*   **响应延迟**：40秒的心跳周期对于快节奏社交互动来说延迟较高。
*   **语义重叠风险**：在动作池中，物理接触类动作的语义空间过于拥挤，导致匹配容易出错。
*   **评估偏差**：实验探针由研究者编写而非从真实玩家日志中完全随机采样，可能存在覆盖面不足的问题。
*   **单人评分**：Whisper的成功率由单名评估者判定，缺乏多专家交叉验证的客观性。
