## 摘要
视频大语言模型 (Video LLMs) 在视频理解任务上表现出色，但由于视觉 Token 数量庞大，面临着高昂的推理成本。我们提出了 KiToke，这是一种无需训练且与查询无关的 Token 压缩方法，在保留关键视觉信息的同时减少了时空冗余。我们的方法利用基于核的冗余度量全局估计 Token 多样性，实现了在极端 Token 预算下依然有效的内容自适应选择，并进一步引入了轻量级的时序区间构建和区间感知 Token 合并，以保持时序连贯性。与以往依赖局部或片段级启发式方法不同，KiToke 显式地捕捉整个视频的全局冗余，从而实现了更高效的 Token 利用。在多个视频理解基准测试和 Video LLM 骨干网络上的广泛实验表明，KiToke 始终优于现有的无需训练的压缩方法，尤其是在保留率低至 1% 的激进情况下，提升尤为显著。

## Abstract
Video Large Language Models (Video LLMs) achieve strong performance on video understanding tasks but suffer from high inference costs due to the large number of visual tokens. We propose KiToke, a training-free, query-agnostic token compression approach that reduces spatiotemporal redundancy while preserving critical visual information. Our method estimates token diversity globally using a kernel-based redundancy measure, enabling content-adaptive selection that remains effective under extreme token budgets, and further introduces a lightweight temporal interval construction with interval-aware token merging to maintain temporal coherence. Unlike prior methods that rely on local or segment-level heuristics, KiToke explicitly captures global redundancy across an entire video, leading to more efficient token utilization. Extensive experiments on multiple video understanding benchmarks and Video LLM backbones demonstrate that KiToke consistently outperforms existing training-free compression methods, with particularly large gains at aggressive retention ratios down to 1%.

### 1. 核心问题与整体含义（研究动机和背景）
视频大语言模型（Video LLMs）在理解长视频时需要处理海量的视觉 Token，这导致了极高的推理成本和内存占用。视频内容在空间和时间上存在大量冗余（如静态背景、缓慢动作）。现有的 Token 压缩方法通常依赖局部启发式规则或简单的聚类，在极端压缩倍率（如仅保留 1% 的 Token）下，往往会丢失关键信息或破坏视频的时序连贯性。本文旨在提出一种**无需训练（Training-free）且与查询无关（Query-agnostic）**的压缩方案，在大幅降低计算开销的同时，最大限度保留视频的关键视觉证据。

### 2. 方法论：核心思想与关键技术
KiToke 的核心思想是将 Token 压缩视为一个**全局多样性最大化**问题，并结合**内容自适应的时序分段**。

*   **基于核的全局多样性估计（KDE）：**
    *   利用高斯核函数计算每个 Token 与视频中所有其他 Token 的相似度，从而估算其核密度（Density）。
    *   密度越高代表冗余度越高。通过取密度的倒数得到多样性得分（Diversity Score）。
    *   采用**枢轴采样（Pivotal Sampling）**而非确定性的 Top-K 选择，以确保在极低预算下仍能覆盖视频的不同视觉区域。
*   **内容自适应的时序区间构建：**
    *   计算相邻帧之间的视觉差异，结合了“位置差异”（对应位置 Token 的变化）和“匹配差异”（考虑物体位移后的最佳匹配变化）。
    *   通过检测视觉差异相对于局部时序动态的突变（Deviation）来确定场景边界，从而生成不等长的时序区间。
*   **区间感知 Token 合并：**
    *   在每个时序区间内，将未选中的 Token 合并到最相似的已选中 Token 中。
    *   合并过程采用基于多样性得分的加权平均，确保更具代表性的特征在最终表示中占据更大权重。

### 6. 主要结论与发现
*   **性能卓越：** KiToke 在所有保留率下均优于现有方法。在 25% 保留率下几乎无损（甚至在某些指标上略有提升）；在 1% 的极端保留率下，其性能领先优势扩大，证明了全局多样性建模的有效性。
*   **高效性：** 相比于基于聚类的复杂方法，KiToke 的计算开销极低，能够显著降低 Prefill 阶段的延迟。
*   **泛化性：** 该方法在不同架构（如 Qwen3-VL 的 M-RoPE 编码）和不同 Token 布局（如 LLaVA-Video 的换行符 Token）下均表现稳健。

### 7. 优点与亮点
*   **全局视角：** 突破了以往方法仅在局部片段内去冗余的限制，能够识别视频远端重复出现的背景或物体。
*   **鲁棒的采样机制：** 引入随机采样避免了确定性选择可能导致的“视觉盲区”。
*   **内容感知：** 时序区间的构建能够灵活应对视频中的快动作和慢动作，避免了跨场景的错误合并。
*   **即插即用：** 无需任何微调即可部署到现有的视频 LLM 中。

### 8. 不足与局限
*   **特定架构的依赖：** 虽然在多个模型上有效，但在 LLaVA-Video 和 Qwen3-VL 等具有复杂时序位置编码的模型上，极端压缩带来的性能下降仍比 LLaVA-OneVision 更明显，说明压缩策略与模型特定架构的深度融合仍有提升空间。
*   **查询无关的局限：** 作为一个与查询无关的方法，它无法根据用户问题的侧重点动态调整 Token 权重（例如，如果问题关注视频角落的一个微小细节，该方法可能会因其全局显著性低而将其过滤）。
*   **超参数敏感度：** 虽然论文称超参数 $\alpha$ 相对稳定，但在处理风格迥异的视频流时，可能仍需微调以达到最优平衡。
