## 摘要
搜索智能体将语言模型（LM）与网络搜索相结合，在回答复杂用户查询方面正变得至关重要。由于昂贵的人工标注或繁琐的前提条件，为涉及多步检索和推理的深度研究任务构建训练数据集仍然具有挑战性。在这项工作中，我们介绍了 ORBIT，这是一个包含 2 万个推理密集型查询及其简短可验证答案的训练数据集，该数据集是使用一种不依赖付费 API 服务的节俭框架生成的。该模块化框架包含四个阶段：种子创建、问答对生成，以及两个验证阶段（自我验证和外部验证）。ORBIT 涵盖 15 个领域，每个训练对需要 4-5 个推理步骤，并需要通过全网进行外部搜索验证。我们使用 GRPO 算法在 ORBIT 上训练 Qwen3-4B 作为基础模型，并在维基百科问答任务上对其进行了评估。广泛的实验结果表明，ORBIT-4B 在 4B 参数以下的搜索智能体大语言模型（LLM）中表现强劲，证明了合成数据集的效用。我们的框架、代码和数据集已开源并公开发布。

## Abstract
Search agents, which integrate language models (LMs) with web search, are becoming crucial for answering complex user queries. Constructing training datasets for deep research tasks, involving multi-step retrieval and reasoning, remains challenging due to expensive human annotation, or cumbersome prerequisites. In this work, we introduce ORBIT, a training dataset with 20K reasoning-intensive queries with short verifiable answers, generated using a frugal framework without relying on paid API services. The modular framework relies on four stages: seed creation, question--answer pair generation, and two stages of verification: self and external. ORBIT spans 15 domains and each training pair requires 4--5 reasoning steps, with external search verification required from the complete web. We train Qwen3-4B as the base model on ORBIT using GRPO and evaluate it on Wikipedia question answering tasks. Extensive experiment results demonstrate that ORBIT-4B achieves strong performance among sub-4B LLMs as search agents, proving the utility of synthetic datasets. Our framework, code and datasets are open-sourced and available publicly.

*   **核心问题**：现有的搜索智能体（Search Agents）训练数据存在两难境地：要么过于简单（如 NQ、HotpotQA，仅需 1-2 步搜索），要么构建成本极高（依赖昂贵的人工标注或付费 API，如 GPT-4）。
*   **研究动机**：随着“深度搜索”（Deep Search）需求的增加，智能体需要处理涉及多步检索、推理和信息聚合的复杂查询。作者旨在提出一种**低成本（Frugal）**、**可扩展**且**可验证**的框架，在不依赖付费 API 的情况下生成高质量的复杂推理训练数据。

### 2. 方法论：核心思想与技术细节
ORBIT 采用了一个四阶段的模块化框架来生成 2 万条高质量数据：
1.  **种子创建（Seed Creation）**：手动确定 15 个领域（如科技、法律、谜题等），利用工具生成每个领域下约 100 个维基百科类别，并提取相关页面标题作为生成问题的“种子”。
2.  **问答对生成（QA Pair Generation）**：使用带有搜索功能的 **DeepSeek-V3.1**（通过 Selenium 自动化操作免费网页版）。核心思想是生成“反向问题”（Inverted Questions），即问题描述复杂但答案简短且唯一，这种形式更易于验证。
3.  **自我验证（Self-Verification）**：再次调用 DeepSeek-V3.1 对生成的问答对进行核查，要求其提供支持每个事实点的 URL 引用。随后使用 Qwen3-4B 过滤掉证据不足或不匹配的条目。
4.  **外部验证（External Verification）**：为消除模型自偏见，使用 **Qwen3-4B** 和 **gpt-oss-120b** 作为独立裁判。它们根据抓取的网页全文内容重新回答问题，并与原始答案比对，仅保留一致的条目。
*   **训练算法**：采用 **GRPO（Group Relative Policy Optimization）** 强化学习算法，以“准确率（Exact Match）”作为奖励信号，训练基础模型 Qwen3-4B。

### 3. 实验设计
*   **基础模型**：Qwen3-4B。
*   **对比基准（Benchmarks）**：
    *   **单跳 QA**：NQ, TriviaQA, PopQA。
    *   **多跳 QA**：HotpotQA, 2WikiMultihopQA, MuSiQue, Bamboogle, 以及更具挑战性的 **FRAMES**。
*   **对比方法**：Search-R1, Search-o1, ZeroSearch, InForage, InfoSeeker 等（主要针对参数量小于 4B 的模型）。
*   **评估指标**：Exact Match (EM) 准确率。

### 4. 资源与算力
*   **训练算力**：使用了 **4 张 NVIDIA H100 GPU**。
*   **训练时长**：训练 160 个步骤，耗时约 **1-2 天**。

*   **其他**：作者特别提到数据构建是在一台 2018 年的 Linux 笔记本上完成的，强调了方案的低成本属性。

### 5. 实验数量与充分性
*   **实验规模**：在 8 个主流维基百科问答数据集和 1 个复杂推理数据集（FRAMES）上进行了全面测试。
*   **消融实验**：
    *   测试了不同的**数据集混合比例**（如 NQ:HotpotQA:ORBIT 为 1:1:1 与 1:2:4 的对比）。
    *   测试了**搜索结果数量（Top-K）**对性能的影响（Top-5 vs Top-10）。
*   **充分性评价**：实验设计较为客观、公平。通过在相同基础模型（Qwen3-4B）上复现 Search-R1 和 InfoSeeker 的训练过程，确保了对比的直接性。

### 6. 主要结论与发现
*   **性能领先**：ORBIT-4B 在所有单跳和多跳任务中均显著优于同类 4B 以下模型，平均 EM 提升高达 9.0 分。
*   **合成数据的力量**：证明了即使在预算有限的情况下，通过精心设计的合成数据流水线也能显著提升小型模型的推理和搜索能力。
*   **数据混合策略**：实验发现，对于小型模型，将简单的单跳数据与复杂的 ORBIT 数据等比例混合（1:1:1）效果最好，这有助于模型建立基础的搜索逻辑。
*   **搜索深度**：ORBIT 问题的平均推理步骤为 4-6 步，远高于 HotpotQA（2 步）和 InfoSeeker（3-5 步）。

### 7. 优点
*   **极低成本**：完全避开了昂贵的付费 API（如 OpenAI API 或 Google Search API），对学术界和预算有限的团队非常友好。
*   **高质量验证**：通过“自我验证+外部裁判+网页全文抓取”的多重机制，确保了合成数据的准确性。
*   **开源贡献**：开源了包含 2 万条数据的 ORBIT 数据集、训练代码和模型，具有很高的复用价值。

### 8. 不足与局限
*   **生成效率低**：依赖网页自动化（Selenium）导致数据生成周期极长（数月），不适合需要快速迭代的场景。
*   **模型规模限制**：目前仅在 4B 以下的小模型上验证了效果，尚未探讨该数据集对 70B 或更大规模模型的提升作用。

*   **工具链单一**：训练时仅使用了搜索工具，未加入“网页全文阅读（Get Document）”等更高级的工具，限制了模型处理超长上下文的能力。
