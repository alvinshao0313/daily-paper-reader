## 摘要
由于其硬件支持以及在每个参数位数相对较少的情况下保留有用信息的能力，NVFP4 作为一种用于量化大语言模型的 4 位格式已变得越来越流行。然而，该格式并非没有局限性：最近的研究表明，NVFP4 受其误差分布的影响，导致每组 16 个值中接近最大值的部分产生大量量化误差。在这项工作中，我们利用这一见解设计了新的自适应分块缩放数据类型，能够适应其输入值的分布。对于 4 位量化，我们提出的 IF4 (Int/Float 4) 数据类型为每组 16 个值在 FP4 和 INT4 表示之间进行选择，然后像 NVFP4 一样使用 E4M3 缩放因子进行缩放。所选的数据类型使用缩放因子的符号位来标记（该位在 NVFP4 中目前未被使用），我们还将同样的见解应用于设计其他位宽的格式，包括 IF3 和 IF6。当用于量化语言模型时，我们发现 IF4 优于现有的 4 位分块缩放格式，在量化训练期间实现了更低的损失，并在训练后量化的许多任务中实现了更高的准确率。此外，我们设计并评估了一个 IF4 乘累加 (MAC) 单元，以证明 IF4 可以在下一代硬件加速器中高效实现。我们的代码可在 https://github.com/mit-han-lab/fouroversix 获取。

## Abstract
NVFP4 has grown increasingly popular as a 4-bit format for quantizing large language models due to its hardware support and its ability to retain useful information with relatively few bits per parameter. However, the format is not without limitations: recent work has shown that NVFP4 suffers from its error distribution, resulting in large amounts of quantization error on near-maximal values in each group of 16 values. In this work, we leverage this insight to design new Adaptive Block-Scaled Data Types that can adapt to the distribution of their input values. For four-bit quantization, our proposed IF4 (Int/Float 4) data type selects between FP4 and INT4 representations for each group of 16 values, which are then scaled by an E4M3 scale factor as is done with NVFP4. The selected data type is denoted using the scale factor's sign bit, which is currently unused in NVFP4, and we apply the same insight to design formats for other bit-widths, including IF3 and IF6. When used to quantize language models, we find that IF4 outperforms existing 4-bit block-scaled formats, achieving lower loss during quantized training and achieving higher accuracy on many tasks in post-training quantization. We additionally design and evaluate an IF4 Multiply-Accumulate (MAC) unit to demonstrate that IF4 can be implemented efficiently in next-generation hardware accelerators. Our code is available at https://github.com/mit-han-lab/fouroversix.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：在 4 比特量化（W4A4）领域，现有的主流格式如 **NVFP4**（NVIDIA 提出的 4 位浮点格式）虽然在硬件支持和精度平衡上表现良好，但在处理每组（通常为 16 个值）中的**近最大值（near-maximal values）**时存在显著的量化误差分布不均问题。
*   **研究背景**：随着大语言模型（LLM）规模的扩大，低比特量化训练和推理成为降低显存和提升速度的关键。然而，单一的浮点（FP4）或定点（INT4）格式难以同时适应模型中复杂多变的数值分布（如存在离群值或分布较均匀的情况）。
*   **研究动机**：利用 NVFP4 格式中分块缩放因子（Scale Factor）未被使用的符号位，设计一种能够根据数据分布自适应切换表示方式的新型数据类型，以进一步降低量化误差。

### 2. 方法论：核心思想与技术细节
*   **核心思想（IF4 数据类型）**：提出 **IF4 (Int/Float 4)**。对于每 16 个数值组成的一个块，算法会同时尝试用 **FP4** 和 **INT4** 进行量化，并选择均方误差（MSE）更小的那种表示方式。
*   **关键技术细节**：
    *   **标志位复用**：NVFP4 使用 FP8 (E4M3) 作为每 16 个值的缩放因子。由于量化值本身带有符号位，缩放因子通常固定为正。IF4 巧妙地利用缩放因子的**符号位**作为开关：符号位为 0 表示该块使用 FP4，为 1 表示使用 INT4。
    *   **6/7 缩放对齐法**：FP4 的最大可表示值为 6，而 INT4 的最大值为 7。为了让两者在同一动态范围内竞争，IF4 在使用 INT4 时先将数值缩放 7/6 倍再量化，反量化时再缩回。这确保了所有块共享相同的张量级缩放逻辑，且不损失动态范围。
    *   **扩展性**：该思路被推广至 **IF3**（3 位）和 **IF6**（6 位）格式。
*   **算法流程**：计算块内最大值 -> 生成 FP8 缩放因子 -> 分别计算 FP4 和 INT4 量化后的 MSE -> 选择最优格式并设置缩放因子符号位 -> 存储。

*   **实验规模**：论文涵盖了从 340M 到 397B 不同量级的模型，实验跨度极大，验证了方法在不同参数规模下的泛化性。

### 6. 主要结论与发现
*   **精度领先**：IF4 在几乎所有测试场景下均优于 NVFP4 和 MXFP4。在 Qwen 3.5-122B 模型上，IF4 的表现非常接近 BF16 基准。
*   **自适应优势**：IF4 能够自动在离群值较多时选择 FP4，在分布均匀（如经过 Hadamard 变换后）时选择 INT4，从而在各种张量分布下保持低误差。
*   **硬件可行性**：虽然 IF4 MAC 单元相比纯 NVFP4 增加了约 4.7% 的延迟和一定的面积开销，但在系统层面（受限于内存带宽而非计算）这种开销是可以接受的。
*   **内存效率**：IF4 在不增加任何额外存储开销（零比特开销）的前提下，提升了量化精度。

### 7. 优点（亮点）
*   **设计精妙**：通过复用缩放因子的符号位实现“免费”的自适应切换，极具工程巧思。
*   **解决痛点**：直接针对 NVFP4 在近最大值处误差大的缺陷进行改进，且不牺牲动态范围（优于 4/6 方案）。
*   **软硬结合**：不仅有算法层面的 PPL 提升，还提供了具体的硬件电路设计方案，证明了其在下一代 AI 芯片中的应用潜力。

### 8. 不足与局限
*   **随机舍入偏差**：在结合随机舍入（Stochastic Rounding）进行训练时，IF4 会引入微小的量化偏差（Bias），虽然实验显示对 340M 模型影响不大，但在更大规模的超长训练中是否存在累积效应仍需验证。
*   **硬件开销**：相比于极度简化的单格式 MAC 单元，IF4 的混合逻辑增加了电路复杂度和功耗（功耗增加约 27.8%），这可能影响其在极低功耗边缘设备上的竞争力。
*   **实验覆盖**：虽然测试了多种 LLM，但尚未在视觉模型（ViT）或多模态模型上进行广泛验证。
