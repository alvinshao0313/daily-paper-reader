## 摘要
自主智能体越来越多地与 Web 交互，但大多数网站仍是为人类浏览器设计的——这是新兴的“智能体化网络”（Agentic Web）必须解决的根本性不匹配。智能体必须重复浏览页面、检查 DOM 并逆向工程可调用路由，这一过程缓慢、脆弱，且在不同智能体之间冗余重复。我们观察到，每个现代网站在其用户界面背后都已经暴露了内部 API（有时称为影子 API），即驱动网站自身功能的第一方端点。我们提出了 Unbrowse，这是一个共享路由图，它将基于浏览器的路由发现转化为这些可调用第一方接口的集体维护索引。该系统从真实的浏览流量中被动学习路由，并通过直接 API 调用提供缓存路由。在针对 94 个域名的等效信息检索任务的单主机实时网络基准测试中，完全预热的缓存执行平均耗时 950 毫秒，而 Playwright 浏览器自动化耗时 3,404 毫秒（平均加速 3.6 倍，中位数加速 5.4 倍），缓存良好的路由可在 100 毫秒内完成。三路径执行模型（本地缓存、共享图或浏览器回退）确保了系统的自愿性和自我修正。通过 x402 协议建立的三层微支付模型对图查询收取单次查询搜索费（第 3 层），对发现文档收取一次性安装费（第 1 层），并为选择加入的网站所有者提供可选的单次执行费（第 2 层）。所有层级都基于理性采用的必要条件：只有当总费用低于浏览器重新发现的预期成本时，智能体才会使用共享图。

## Abstract
Autonomous agents increasingly interact with the web, yet most websites remain designed for human browsers -- a fundamental mismatch that the emerging ``Agentic Web'' must resolve. Agents must repeatedly browse pages, inspect DOMs, and reverse-engineer callable routes -- a process that is slow, brittle, and redundantly repeated across agents. We observe that every modern website already exposes internal APIs (sometimes called \emph{shadow APIs}) behind its user interface -- first-party endpoints that power the site's own functionality. We present Unbrowse, a shared route graph that transforms browser-based route discovery into a collectively maintained index of these callable first-party interfaces. The system passively learns routes from real browsing traffic and serves cached routes via direct API calls. In a single-host live-web benchmark of equivalent information-retrieval tasks across 94 domains, fully warmed cached execution averaged 950\,ms versus 3{,}404\,ms for Playwright browser automation (3.6$\times$ mean speedup, 5.4$\times$ median), with well-cached routes completing in under 100\,ms. A three-path execution model -- local cache, shared graph, or browser fallback -- ensures the system is voluntary and self-correcting. A three-tier micropayment model via the x402 protocol charges per-query search fees for graph lookups (Tier~3), a one-time install fee for discovery documentation (Tier~1), and optional per-execution fees for site owners who opt in (Tier~2). All tiers are grounded in a necessary condition for rational adoption: an agent uses the shared graph only when the total fee is lower than the expected cost of browser rediscovery.

*   **核心问题**：如何消除这种冗余的发现成本，使智能体能够像调用原生 API 一样高效地与现有网页交互？
*   **整体含义**：论文提出了 **Unbrowse** 系统，主张利用网站 UI 背后的“影子 API”（Shadow APIs，即驱动前端功能的内部第一方接口），通过构建一个集体维护的“共享路由图”，将网页交互从“视觉推理”转变为“直接 API 调用”。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：建立一个共享的知识公地，通过被动观察真实浏览流量来学习 API 路由，并利用经济激励机制鼓励开发者和智能体贡献这些路由。
*   **三路径执行模型**：
    1.  **路径 1（本地缓存）**：优先检查本地是否已有该任务的路由缓存。
    2.  **路径 2（共享图）**：若本地无缓存，则从 Unbrowse 共享索引中搜索并安装对应的“技能”（Skill）。
    3.  **路径 3（浏览器回退）**：若索引中也没有，则启动基于 Zig 语言编写的 **Kuri** 浏览器引擎进行实时捕获、逆向工程并发布新路由。
*   **关键技术细节**：
    *   **流量提取**：通过启发式过滤（如 Content-Type 为 JSON/XML）从网络流量中识别 API 端点。
    *   **技能封装**：遵循 AgentSkills.io 标准，将端点、参数架构、认证模式封装为可执行的 TypeScript 客户端。
    *   **经济模型（x402 协议）**：引入三层微支付体系：
        *   **Tier 1（安装费）**：一次性支付给路由贡献者。
        *   **Tier 2（执行费）**：可选，支付给选择加入该生态的网站所有者。
        *   **Tier 3（搜索费）**：支付给维护索引的平台。
*   **采用公式**：智能体采用共享路由的理性条件是：$f_{route} < c_{rediscovery}$。即：支付给系统的总费用必须低于智能体自行通过浏览器重新发现路由的预期成本（包括延迟、算力、Token 消耗及失败重试成本）。

### 5. 实验数量与充分性
*   **实验规模**：对 94 个域名进行了测试，每个任务运行三次以消除网络波动影响。

### 6. 主要结论与发现
*   **显著提速**：在预热缓存状态下，Unbrowse 的平均执行时间为 **950 毫秒**，而 Playwright 为 **3,404 毫秒**。平均提速 **3.6 倍**，中位数提速 **5.4 倍**。
*   **极端表现**：在某些重前端逻辑的网站上，提速最高可达 **30 倍**（API 响应 < 100ms，而页面渲染需数秒）。
*   **成本降低**：通过 API 调用替代浏览器运行，可减少约 90-96% 的计算成本和 Token 消耗。
*   **冷启动代价**：发现新路由的平均耗时约为 12.4 秒，但该成本仅需支付一次，通常在 3-5 次重复使用后即可实现盈亏平衡。

### 7. 优点
*   **架构范式转移**：从“为人类设计的网页”转向“为智能体设计的 API 交互”，极大地提升了 Agentic Web 的效率。
*   **经济激励闭环**：利用 x402 协议解决了开源/共享知识库长期存在的维护动力不足问题，通过微支付实现了贡献者、平台和网站所有者的利益对齐。
*   **实用性强**：系统作为 npm 包发布，支持 MCP 等主流智能体协议，能够直接集成到现有的 Agent 工作流中。

### 8. 不足与局限
*   **反爬虫挑战**：对于部署了高级 Bot 检测（如 Cloudflare 高级模式）的网站，非浏览器请求仍可能被拦截，提速效果会打折扣。
*   **动态认证难题**：对于需要复杂人类交互（如 OAuth 弹窗、验证码）的认证流程，系统仍需回退到浏览器模式。
*   **安全风险**：虽然凭证存储在本地，但恶意贡献者可能发布指向钓鱼端点的虚假路由，系统目前的防御主要依赖于事后信用降级。
*   **法律灰色地带**：对于未明确授权 API 访问的网站，这种“影子 API”调用在法律和道德边界上仍存在争议（尽管论文提出了 Tier 2 补偿方案）。
