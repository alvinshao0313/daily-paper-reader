## 摘要
多模态大语言模型 (MLLMs) 通过提升输入保真度实现了更强的视觉理解能力，但由此产生的视觉 Token 增长使得同时维持高空间分辨率和长时序上下文变得代价高昂。我们认为瓶颈不在于编码后的表示如何压缩，而在于编码器接收的像素量。为此，我们提出了 ResAdapt，这是一种输入端自适应框架，用于在编码前学习每帧应分配的视觉预算。ResAdapt 将轻量级分配器 (Allocator) 与未改变的 MLLM 主干网络耦合，使主干网络在接收算子转换后的输入时，仍能保留其原生的视觉 Token 接口。我们将分配过程建模为上下文老虎机 (Contextual Bandit)，并使用成本感知策略优化 (Cost-Aware Policy Optimization, CAPO) 训练分配器，该方法将稀疏的展开反馈转化为稳定的准确率-成本学习信号。在受预算控制的视频问答、时序定位和图像推理任务中，ResAdapt 改善了低预算运行点，且通常处于或接近效率-准确率前沿，在激进压缩下的推理密集型基准测试中提升最为明显。值得注意的是，在相同的视觉预算下，ResAdapt 支持多达 16 倍的帧数，同时提供超过 15% 的性能提升。代码可在 https://github.com/Xnhyacinth/ResAdapt 获取。

## Abstract
Multimodal Large Language Models (MLLMs) achieve stronger visual understanding by scaling input fidelity, yet the resulting visual token growth makes jointly sustaining high spatial resolution and long temporal context prohibitive. We argue that the bottleneck lies not in how post-encoding representations are compressed but in the volume of pixels the encoder receives, and address it with ResAdapt, an Input-side adaptation framework that learns how much visual budget each frame should receive before encoding. ResAdapt couples a lightweight Allocator with an unchanged MLLM backbone, so the backbone retains its native visual-token interface while receiving an operator-transformed input. We formulate allocation as a contextual bandit and train the Allocator with Cost-Aware Policy Optimization (CAPO), which converts sparse rollout feedback into a stable accuracy-cost learning signal. Across budget-controlled video QA, temporal grounding, and image reasoning tasks, ResAdapt improves low-budget operating points and often lies on or near the efficiency-accuracy frontier, with the clearest gains on reasoning-intensive benchmarks under aggressive compression. Notably, ResAdapt supports up to 16x more frames at the same visual budget while delivering over 15% performance gain. Code is available at https://github.com/Xnhyacinth/ResAdapt.

*   **核心问题**：多模态大语言模型（MLLM）在处理高分辨率图像和长视频时，视觉 Token 数量呈爆炸式增长，导致计算成本极高。
*   **研究动机**：现有的效率优化策略存在“干预过晚”的问题。模型侧方法（如 Token 剪枝/合并）在编码器已经支付了全额计算成本后才丢弃信息；输出侧方法（如迭代检索/放大）则引入了多轮推理延迟。
*   **整体含义**：本文提出 **ResAdapt** 框架，主张在**编码前**进行干预。通过一个轻量级的分配器（Allocator），根据查询（Query）内容动态调整每一帧的输入分辨率，从而在源头上控制像素量，实现空间预算与时间覆盖的动态平衡。

### 2. 方法论
*   **核心思想**：将每帧的视觉预算分配建模为一个“上下文老虎机”（Contextual Bandit）问题，利用强化学习（RL）训练一个查询感知的分配器。
*   **关键技术细节**：
    *   **分配器架构**：采用轻量级视觉编码器提取粗粒度特征，结合 Transformer 解码器进行时序建模和查询交叉注意力，输出每帧的缩放因子 $s_t$。
    *   **动作空间**：使用 Beta 分布参数化连续动作空间，将缩放因子映射到 $[s_{min}, s_{max}]$ 区间（如 0.2x 到 1.8x）。
    *   **成本感知策略优化 (CAPO)**：为了防止策略坍缩（即模型为了省资源而盲目降低分辨率），引入了**动态成本基准（Dynamic Pivot）**和**非对称奖励塑造**。它对“低成本且正确”的样本给予适度奖励，而对“高成本且错误”的样本给予重罚。
    *   **时序相似性正则化 ($L_{sim}$)**：针对视频冗余，惩罚对相邻相似帧同时分配高预算的行为，强制模型进行差异化感知。
*   **算法流程**：分配器先预测缩放比例 $\rightarrow$ 算子（如 Resize）调整原始图像 $\rightarrow$ 骨干 MLLM 处理调整后的 Token 序列并生成答案 $\rightarrow$ 根据答案正确性和视觉成本通过 CAPO 更新分配器。

### 3. 实验设计
*   **数据集/场景**：

*   **算力资源**：使用了 **32 台 NVIDIA H100 GPU** 进行训练。

*   **推理开销**：分配器本身非常轻量，仅占总推理 FLOPs 的不到 3%，其带来的 Token 减少能显著抵消其自身的计算开销。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 6 个视频问答基准和 3 个定位基准，总计进行了数十组对比实验。
*   **消融实验**：非常充分，包括奖励函数设计（CAPO vs 直接成本惩罚）、正则化项的作用、不同分布模型（Beta vs Normal）的对比、以及算子泛化性测试（将缩放策略迁移到丢帧策略）。
*   **客观性**：通过在不同版本的 Qwen 模型上进行测试，证明了分配器的跨架构泛化能力；同时在长短视频（32 帧 vs 128 帧）下均验证了有效性。

### 6. 主要结论与发现
*   **性能前沿**：ResAdapt 在减少 90% 以上视觉 Token 的情况下，依然能匹配或超越全分辨率基准。
*   **空间换时间**：在相同的计算预算下，ResAdapt 可处理 **16 倍**更多的帧数，在复杂长视频任务中获得超过 **15%** 的相对增益。
*   **主动感知 (Active Perception)**：模型自发学会了将像素预算集中在信息密集的帧（如文字出现、场景切换、关键动作），而无需显式的显著性标注。
*   **硬件友好**：由于是在输入端调整，生成的 Token 序列是规则的，完全兼容 FlashAttention 和 vLLM 等推理加速引擎。

### 7. 优点
*   **干预点超前**：在编码前优化像素，比编码后剪枝更节省计算资源。
*   **查询感知**：分配策略不是静态的，而是根据问题动态调整（例如：问时间则关注时钟，问动作则关注关键帧）。
*   **训练稳定性**：CAPO 有效解决了强化学习在处理准确率-成本权衡时的不稳定性问题。

### 8. 不足与局限
*   **固定开销**：对于极短的视频（如少于 32 帧），分配器的前置开销可能会抵消部分后端节省的成本。
*   **开环决策**：分配器在推理开始前就做出了预算决定，如果中途发现分配错误，目前无法动态修正（即缺乏“边想边看”的闭环机制）。
*   **细微特征丢失风险**：由于分配器依赖粗粒度特征，可能会漏掉极小且关键的视觉线索（如背景中微小的物体）。
