## 摘要
住房选择是一个高风险且在很大程度上不可逆的决策问题。我们将住房咨询作为住房选择的决策支持接口进行研究。现有的住房平台和许多基于大语言模型（LLM）的助手通常将这一过程简化为排序或推荐，导致推理过程不透明、多约束处理能力脆弱，且对事实性的保证有限。我们提出了 HabitatAgent，这是首个用于端到端住房咨询的、由大语言模型驱动的多智能体架构。HabitatAgent 包含四个专门的智能体角色：记忆、检索、生成和验证。记忆智能体通过约束提取、记忆融合和验证门控更新等内部阶段来维护多层用户记忆；检索智能体执行混合向量-图检索（GraphRAG）；生成智能体产生引用证据的推荐和解释；验证智能体应用多层验证和针对性修复。这些智能体共同为端到端住房咨询提供了一个可审计且可靠的工作流。我们在端到端正确性协议下，针对 100 个真实用户咨询场景（300 个多轮问答对）对 HabitatAgent 进行了评估。一个强大的单阶段基准模型（Dense+Rerank）达到了 75% 的准确率，而 HabitatAgent 达到了 95%。

## Abstract
Housing selection is a high-stakes and largely irreversible decision problem. We study housing consultation as a decision-support interface for housing selection. Existing housing platforms and many LLM-based assistants often reduce this process to ranking or recommendation, resulting in opaque reasoning, brittle multi-constraint handling, and limited guarantees on factuality.   We present HabitatAgent, the first LLM-powered multi-agent architecture for end-to-end housing consultation. HabitatAgent comprises four specialized agent roles: Memory, Retrieval, Generation, and Validation. The Memory Agent maintains multi-layer user memory through internal stages for constraint extraction, memory fusion, and verification-gated updates; the Retrieval Agent performs hybrid vector--graph retrieval (GraphRAG); the Generation Agent produces evidence-referenced recommendations and explanations; and the Validation Agent applies multi-tier verification and targeted remediation. Together, these agents provide an auditable and reliable workflow for end-to-end housing consultation.   We evaluate HabitatAgent on 100 real user consultation scenarios (300 multi-turn question--answer pairs) under an end-to-end correctness protocol. A strong single-stage baseline (Dense+Rerank) achieves 75% accuracy, while HabitatAgent reaches 95%.

#### 1. 核心问题与整体含义（研究动机和背景）
住房选择是一项高风险、高成本且基本不可逆的决策，具有极强的路径依赖和信息不对称性。现有的住房平台和基于大语言模型（LLM）的助手在处理此类咨询时面临四大挑战：
*   **用户偏好演变：** 用户需求往往模糊且在对话中不断变化。
*   **异构约束复杂：** 需同时处理地理位置、学区、交通、政策等硬性及关联性约束。
*   **推理不透明：** 传统的 Top-K 推荐缺乏可审计的解释。
*   **事实性错误代价高：** 房源信息的幻觉或实体混淆可能导致严重的决策失误。
论文旨在通过一种端到端的多智能体架构，将住房咨询从简单的“搜索与排序”提升为“可靠的决策支持”。

#### 2. 论文提出的方法论
HabitatAgent 采用闭环多智能体协作架构，包含四个核心角色：
*   **记忆智能体 (Memory Agent)：** 维护四层记忆结构（对话、实体、偏好、检索历史）。核心技术是**验证门控更新 (Verification-Gated Update)**，即只有通过验证的响应信息才会被写入长期记忆，防止错误累积。
*   **检索智能体 (Retrieval Agent)：** 采用**自适应混合向量-图检索 (GraphRAG)**。通过一个路由模型预测查询复杂度，简单查询走向量检索，复杂查询（涉及空间、关联约束）则调用属性知识图谱（Neo4j）进行 Cypher 过滤。
*   **生成智能体 (Generation Agent)：** 基于 14 种任务特定模板（如学区分析、投资对比等）进行任务感知生成。要求所有事实性陈述必须带有**证据引用标签 (Evidence-ref tags)**。
*   **验证智能体 (Validation Agent)：** 执行三层验证（事实一致性、实体准确性、合规性）。若验证失败，触发**故障类型感知修复 (Failure-Type-Aware Remediation)**，根据错误类型（实体缺失、约束冲突、事实错误）进行针对性修正而非盲目重新生成。

#### 6. 主要结论与发现
*   **性能飞跃：** HabitatAgent 的端到端准确率达到 **95%**，远超强基准 Dense+Rerank 的 75%。
*   **复杂约束处理：** 在复杂查询子集中，传统向量检索的约束满足率（CSR@5）仅为 8%，而 HabitatAgent 保持在 95%，证明了 GraphRAG 和自适应路由的有效性。
*   **关键组件：** 消融实验显示，**自适应路由**对准确率的影响最大（移除后下降 20%），说明在住房领域，结构化知识的精准检索是决策支持的基石。

#### 7. 优点
*   **闭环可靠性：** 通过“验证-修复-门控记忆”构建了一个能够自我纠错且防止污染的系统。
*   **高效检索：** 自适应路由平衡了图检索的精准度与向量检索的响应速度。
*   **高透明度：** 强制性的证据引用标签使得 AI 的推荐过程可追溯、可审计，符合高风险决策的需求。

#### 8. 不足与局限
*   **地理局限性：** 实验数据仅限于北京，不同城市的住房市场逻辑（如学区政策差异）可能需要重新适配知识图谱。
