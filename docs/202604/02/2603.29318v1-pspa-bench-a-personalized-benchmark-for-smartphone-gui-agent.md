## 摘要
智能手机 GUI 智能体通过直接操作应用界面来执行任务，在无需深度系统集成的情况下提供了实现广泛能力的途径。然而，现实世界中的智能手机使用具有高度个性化：用户采用多样化的工作流和偏好，这对智能体提出了挑战，要求其提供定制化协助而非通用解决方案。由于用户特定数据稀疏且缺乏细粒度的评估指标，现有的 GUI 智能体基准测试无法充分捕捉这一个性化维度。为了填补这一空白，我们提出了 PSPA-Bench，这是一个专门用于评估智能手机 GUI 智能体个性化能力的基准测试。PSPA-Bench 包含超过 12,855 条与现实世界用户行为一致的个性化指令，涵盖 10 个代表性的日常使用场景和 22 个移动应用，并引入了一种结构感知的过程评估方法，用于在细粒度水平上衡量智能体的个性化能力。通过 PSPA-Bench，我们对 11 种最先进的 GUI 智能体进行了基准测试。结果显示，当前方法在个性化设置下表现不佳，即使是最强大的智能体也仅取得了有限的成功。我们的分析进一步强调了推进个性化 GUI 智能体的三个方向：(1) 面向推理的模型始终优于通用大语言模型（LLMs），(2) 感知仍然是一项简单但关键的能力，(3) 反思和长期记忆机制是提高适应性的关键。总之，这些发现使 PSPA-Bench 成为个性化 GUI 智能体系统研究和未来进展的基础。

## Abstract
Smartphone GUI agents execute tasks by operating directly on app interfaces, offering a path to broad capability without deep system integration. However, real-world smartphone use is highly personalized: users adopt diverse workflows and preferences, challenging agents to deliver customized assistance rather than generic solutions. Existing GUI agent benchmarks cannot adequately capture this personalization dimension due to sparse user-specific data and the lack of fine-grained evaluation metrics. To address this gap, we present PSPA-Bench, the benchmark dedicated to evaluating personalization in smartphone GUI agents. PSPA-Bench comprises over 12,855 personalized instructions aligned with real-world user behaviors across 10 representative daily-use scenarios and 22 mobile apps, and introduces a structure-aware process evaluation method that measures agents' personalized capabilities at a fine-grained level. Through PSPA-Bench, we benchmark 11 state-of-the-art GUI agents. Results reveal that current methods perform poorly under personalized settings, with even the strongest agent achieving limited success. Our analysis further highlights three directions for advancing personalized GUI agents: (1) reasoning-oriented models consistently outperform general LLMs, (2) perception remains a simple yet critical capability, and (3) reflection and long-term memory mechanisms are key to improving adaptation. Together, these findings establish PSPA-Bench as a foundation for systematic study and future progress in personalized GUI agents.

### PSPA-Bench: 智能手机 GUI 智能体个性化基准测试论文总结

#### 1. 核心问题与整体含义（研究动机和背景）
智能手机 GUI 智能体（如操作 App 界面的 AI 助手）正从通用化向个性化转型。现实中，不同用户完成同一任务（如订外卖）的偏好和操作路径迥异。然而，现有的基准测试（如 AndroidWorld, Mobile-Bench）主要关注通用任务的成功率，面临两大挑战：
*   **数据稀疏性**：受隐私法规（如 GDPR）限制，难以获取大规模真实用户的个性化操作日志。
*   **评估粗糙**：现有的二元成功信号（成功/失败）无法衡量智能体对用户特定偏好的适配程度，也无法评估其在长期使用中的进化能力。
**PSPA-Bench** 旨在通过结构化的任务表示和细粒度的评估指标，系统性地衡量 GUI 智能体在个性化场景下的表现。

#### 2. 论文提出的方法论
论文的核心思想是利用**任务分解图（Task Decomposition Graph, TDG）**来显式表示个性化任务的结构。
*   **任务分解图 (TDG)**：将任务表示为有向无环图。节点分为**固定节点（Fixed Nodes）**（如启动 App 等通用步骤）和**灵活节点（Flexible Nodes）**（如选择特定价位、菜系等偏好相关步骤）。
*   **模板驱动的指令生成**：
    *   **模板构建**：将 TDG 中的灵活节点转化为参数化插槽（Slots）。
    *   **插槽实例化**：基于用户画像（Profile）动态填充插槽。用户偏好被建模为**长期偏好**（对数增长函数，模拟稳定习惯）和**短期偏好**（正弦函数，模拟周期性波动）的结合。
*   **细粒度评估指标**：
    *   **轨迹-图对齐（Trace-Graph Alignment）**：利用 LLM 将智能体的操作轨迹与 TDG 路径对齐。
    *   **核心指标**：
        *   **APR (A-process Ratio)**：所有单元指令的完成比例。
        *   **PPR (P-process Ratio)**：仅针对灵活节点（个性化步骤）的完成比例。
        *   **效率指标**：完成时间（CT）和单任务成本（CPT）。
        *   **增量指标（$\Delta$）**：衡量智能体在积累经验后性能的提升。

#### 3. 实验设计
*   **场景与数据**：涵盖 10 个日常场景（购物、旅游、餐饮等），涉及 22 个真实的第三方移动 App。构建了 100 个用户画像，生成了 12,855 条个性化指令。
*   **对比方法**：评估了 11 种最先进的方法，分为三类：
    1.  **基础 LLM**：GPT-4o-mini 等（配合 A11y 树或 SoM 视觉增强）。
    2.  **通用 Agent 框架**：ReAct, Self-Refine, Reflexion。
    3.  **专用 GUI Agent 框架**：AppAgent, M3A, Mobile-Agent V2, Mobile-Agent E。
*   **实验维度**：分为“即时目标”（单次任务执行）和“长期目标”（跨任务的经验学习与适配）。

#### 4. 资源与算力
*   **算力说明**：论文**未明确说明**具体的训练 GPU 型号或数量。由于评估对象多为基于 API 的大模型智能体，实验重点在于推理和交互。
*   **成本评估**：论文通过 CPT（Cost Per Task）指标量化了不同智能体消耗的 Token 费用，例如 Mobile-Agent E 在高复杂度任务下的 CPT 显著高于其他模型。

#### 5. 实验数量与充分性
*   **实验规模**：在 10 个场景下进行了大规模测试，涵盖了不同任务复杂度（低、中、高）和指令清晰度（低、中、高）。
*   **充分性**：实验设计较为全面，不仅对比了不同框架的性能，还通过消融实验验证了感知、推理、记忆和进化机制的作用。
*   **客观性**：引入了人类评估（Human Validation）来验证 LLM 自动评分与人类判断的一致性（相关性 $\rho=0.95$），确保了基准测试的公正性。

#### 6. 主要结论与发现
*   **现状严峻**：现有智能体在个性化设置下表现普遍较差，即使是表现最好的 Mobile-Agent E，其 PPR（个性化完成率）也仅在 0.4-0.6 之间。
*   **感知是基石**：接入 A11y（无障碍）树或视觉增强（SoM）能显著提升性能，盲目依赖纯文本 LLM 无法处理 GUI 任务。
*   **推理解决歧义**：具备强推理能力（如 Thinking 模型或 ReAct 框架）的 Agent 能更好地处理模糊指令和复杂偏好。
*   **记忆与进化至关重要**：具备持久化记忆和自我进化能力的 Agent（如 Mobile-Agent E）在长期任务中表现出明显的性能增量（$\Delta$APR 为正），而无记忆模型性能往往停滞甚至倒退。
*   **权衡挑战**：个性化程度越高，准确性与效率（时间/成本）之间的权衡越剧烈。

#### 7. 优点（亮点）
*   **结构化创新**：TDG 的引入解决了“个性化”难以量化定义的难题，将模糊的偏好转化为可追踪的图节点。
*   **动态偏好建模**：通过长短期偏好权重函数生成指令，比静态画像更贴近真实人类行为。
*   **细粒度评估**：APR/PPR 指标比传统的“成功率”更能反映 Agent 在执行过程中的细微偏差。
*   **隐私友好**：通过模板生成大规模合成数据，规避了采集真实用户隐私数据的法律风险。

#### 8. 不足与局限
*   **App 覆盖范围**：虽然涵盖了 22 个 App，但相对于庞大的移动生态系统仍显不足，且未涉及金融、医疗等专业领域。
*   **平台局限性**：目前主要针对 Android 系统，未测试 iOS 等其他平台的 UI 特性。
*   **合成数据偏差**：用户画像和 TDG 仍由专家手动构建或模板生成，可能无法完全捕捉真实用户行为中的非理性或极端情况。
*   **实时性挑战**：复杂的 Agent 框架（如多 Agent 协作）虽然准确率高，但响应延迟巨大，难以满足实时交互需求。
