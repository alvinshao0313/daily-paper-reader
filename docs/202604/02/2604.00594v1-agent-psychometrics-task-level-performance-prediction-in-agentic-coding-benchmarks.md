## 摘要
随着基于大语言模型（LLM）的编程从静态单步代码生成转向与工具和环境的多步智能体交互，理解哪些任务会给智能体带来挑战以及原因变得越来越困难。目前的实践加剧了这一问题：智能体的性能通常通过基准测试的综合通过率来衡量，但单一数字指标掩盖了基准测试中任务的多样性。我们提出了一个专门针对智能体编程范式的框架，用于预测单个任务的成功或失败。我们的方法利用从任务中提取的丰富特征（包括问题陈述、仓库上下文、解决方案和测试用例）增强了项目反应理论（IRT），并引入了一种将智能体能力分解为 LLM 能力和脚手架能力组件的新颖方法。这种参数化使我们能够汇总跨异构排行榜的评估数据，并准确预测未见基准测试以及未见 LLM-脚手架组合的任务级性能。我们的方法对基准测试设计者具有实际效用，他们可以更好地校准新任务的难度，而无需运行计算成本昂贵的智能体评估。

## Abstract
As the focus in LLM-based coding shifts from static single-step code generation to multi-step agentic interaction with tools and environments, understanding which tasks will challenge agents and why becomes increasingly difficult. This is compounded by current practice: agent performance is typically measured by aggregate pass rates on benchmarks, but single-number metrics obscure the diversity of tasks within a benchmark. We present a framework for predicting success or failure on individual tasks tailored to the agentic coding regime. Our approach augments Item Response Theory (IRT) with rich features extracted from tasks, including issue statements, repository contexts, solutions, and test cases, and introduces a novel decomposition of agent ability into LLM and scaffold ability components. This parameterization enables us to aggregate evaluation data across heterogeneous leaderboards and accurately predict task-level performance for unseen benchmarks, as well as unseen LLM-scaffold combinations. Our methods have practical utility for benchmark designers, who can better calibrate the difficulty of their new tasks without running computationally expensive agent evaluations.

### 2. 方法论
该论文将心理测量学中的**项目反应理论（Item Response Theory, IRT）**扩展到了智能体编程领域：
*   **核心公式**：采用 Rasch 模型（1PL IRT），预测智能体 $i$ 在任务 $j$ 上的成功概率：$P(y_{ij} = 1 | \theta_i, \beta_j) = \sigma(\theta_i - \beta_j)$。其中 $\theta_i$ 是智能体能力，$\beta_j$ 是任务难度。
*   **智能体能力分解**：创新性地将智能体能力 $\theta_i$ 分解为 **LLM 能力 ($\theta_m$)** 和 **脚手架能力 ($\theta_s$)** 的加法组合：$\theta_i = \theta_m + \theta_s$。这允许模型整合来自不同排行榜的异构数据。
*   **任务特征提取**：
    1.  **嵌入特征（Embeddings）**：使用 DeepSeek-R1-Distill-Qwen-32B 等模型对任务说明和参考方案进行向量化。
    2.  **LLM-as-a-judge 特征**：调用 Claude Opus 4.6 根据预设维度（如领域知识需求、逻辑推理复杂度、验证难度等）对任务打分。
    3.  **环境审计（Auditor Agent）**：运行一个自动化智能体探索任务仓库，提取代码库规模、修复定位难度等环境特征。
*   **预测流程**：首先在已有数据上训练 IRT 模型获取潜在参数，然后利用岭回归（Ridge Regression）建立任务特征与难度参数 $\beta_j$ 之间的映射关系。

### 6. 主要结论与发现
*   **特征有效性**：仅靠问题陈述不足以准确预测难度，加入**测试用例和参考方案**等智能体特有的工件能显著提升预测准确度（AUC-ROC 提升明显）。
*   **能力可分解性**：LLM 能力和脚手架能力具有很强的**加法独立性**（相关系数 $r=0.974$），这意味着可以独立评估脚手架的贡献。
*   **跨基准泛化**：模型能够以优于基准线的准确度预测全新基准测试的任务难度。
*   **应用价值**：通过预测难度，可以实现“自适应测试”，即仅需评估少量关键任务即可准确估算智能体能力，大幅节省计算资源。

### 7. 优点与亮点
*   **理论结合实际**：将成熟的心理测量学理论（IRT）成功引入前沿的智能体评估领域。
*   **解决数据孤岛**：通过 LLM-Scaffold 分解，解决了不同排行榜之间无法直接比较的问题。
*   **实用性强**：为基准测试设计者提供了一套自动化的难度校准工具，具有很高的工程参考价值。
*   **多模态特征**：不仅考虑文本，还通过“审计智能体”考虑了代码仓库的动态环境特征。

### 8. 不足与局限
*   **非因果性**：目前的预测是基于相关性的，无法证明某些特征（如测试用例的复杂性）是导致任务变难的直接原因。
*   **依赖已知组件**：对于全新的、从未在任何排行榜出现过的 LLM 或脚手架，该模型仍无法直接预测其能力。
*   **忽略交互效应**：模型假设 LLM 和脚手架是独立加成的，但在现实中，某些脚手架可能专门针对特定 LLM 进行了优化，这种“化学反应”在当前模型中被忽略了。
*   **黑盒模型依赖**：特征提取高度依赖于 Claude 或 GPT 等商业模型，这可能引入特定的模型偏见。
