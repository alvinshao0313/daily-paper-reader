## 摘要
文档理解和图形用户界面（GUI）交互是视觉语言模型（VLM）中价值最高的应用之一，但它们带来了极重的计算负担：细粒度的文本和微小的 UI 元素需要高分辨率输入，从而产生数以万计的视觉 Token。我们观察到这种开销在很大程度上是浪费的——在文档和 GUI 基准测试中，只有 22-71% 的图像块在像素层面是唯一的，其余部分则是同一图像中另一个图像块的精确副本。我们提出了 PixelPrune，它通过基于预测编码的压缩技术利用这种像素级冗余，在视觉 Transformer（ViT）编码器之前剪枝冗余图像块。由于它在任何神经计算之前的像素空间中运行，PixelPrune 同时加速了 ViT 编码器和下游大语言模型（LLM），覆盖了完整的推理流水线。该方法无需训练，不需要可学习参数，并支持像素级无损压缩（τ=0）以及受控的有损压缩（τ>0）。在三种模型规模以及文档和 GUI 基准测试上的实验表明，PixelPrune 在保持具有竞争力的任务准确率的同时，实现了高达 4.2 倍的推理加速和 1.9 倍的训练加速。代码可在 https://github.com/OPPO-Mente-Lab/PixelPrune 获取。

## Abstract
Document understanding and GUI interaction are among the highest-value applications of Vision-Language Models (VLMs), yet they impose exceptionally heavy computational burden: fine-grained text and small UI elements demand high-resolution inputs that produce tens of thousands of visual tokens. We observe that this cost is largely wasteful -- across document and GUI benchmarks, only 22--71\% of image patches are pixel-unique, the rest being exact duplicates of another patch in the same image. We propose \textbf{PixelPrune}, which exploits this pixel-level redundancy through predictive-coding-based compression, pruning redundant patches \emph{before} the Vision Transformer (ViT) encoder. Because it operates in pixel space prior to any neural computation, PixelPrune accelerates both the ViT encoder and the downstream LLM, covering the full inference pipeline. The method is training-free, requires no learnable parameters, and supports pixel-lossless compression ($τ{=}0$) as well as controlled lossy compression ($τ{>}0$). Experiments across three model scales and document and GUI benchmarks show that PixelPrune maintains competitive task accuracy while delivering up to 4.2$\times$ inference speedup and 1.9$\times$ training acceleration. Code is available at https://github.com/OPPO-Mente-Lab/PixelPrune.

*   **研究动机**：在文档理解和 GUI 交互等视觉语言模型（VLM）的高价值应用中，为了捕捉细微文本和 UI 元素，通常需要极高分辨率的输入。这导致视觉 Token 数量激增（动辄上万），给计算资源（尤其是 ViT 编码器和 LLM 推理）带来巨大压力。
*   **核心观察**：作者发现文档和 GUI 图像存在极高的**像素级冗余**。例如，文档中的大量白边和 GUI 中的纯色背景导致只有 22%–71% 的图像块是唯一的，其余均为重复内容。
*   **研究目标**：提出一种在进入 ViT 编码器之前就能剔除冗余 Token 的方法，从而实现全流水线（ViT + Projector + LLM）的加速。

### 2. 方法论
*   **核心思想**：借鉴传统图像压缩（如 PNG、JPEG-LS）中的**预测编码（Predictive Coding）**原理。在像素空间对图像块进行扫描，利用已访问的邻居块预测当前块，若预测准确（残差在阈值 $\tau$ 内），则认为该块冗余并将其剔除。
*   **关键技术细节**：
    *   **预测策略（Pred-2D）**：默认采用 2D 预测，即根据当前块的左侧（A）、上方（B）和左上方（C）邻居块进行逻辑判断（类似于中值边缘预测器），选择最可能的参考块。
    *   **匹配准则**：支持无损压缩（$\tau=0$，像素完全一致）和有损压缩（$\tau>0$）。对于有损压缩，使用最大像素差值（Max）作为距离度量。
    *   **位置保持（Position-Preserving）**：剔除 Token 后，保留剩余 Token 的原始 2D 坐标信息，确保 ViT 的 2D RoPE 和 LLM 的 MRoPE 能正确识别空间布局。
    *   **无需训练**：该方法在文档理解任务上可以实现“即插即用”，无需任何参数更新。

*   **对比基线（Baselines）**：
    *   **Full**：保留所有 Token 的完整模型。
    *   **Resize**：通过降低分辨率来匹配相同的 Token 预算。

### 6. 主要结论与发现
*   **显著加速**：在推理阶段实现了高达 **4.2 倍** 的首字延迟（TTFT）加速；在训练阶段实现了 **1.9 倍** 的速度提升，并减少了 **33.6%** 的显存占用。
*   **精度保持**：在文档任务中，PixelPrune 在减少 30%-50% Token 的情况下，精度几乎无损（差距通常在 0.5% 以内）。
*   **GUI 任务特性**：GUI 任务对位置更敏感，虽然免训练模式下有一定下降，但通过轻量级知识蒸馏或带剪枝训练，可以完全恢复精度。
*   **全阶段节省**：证明了在 ViT 之前进行剪枝比在 ViT 内部剪枝更高效，因为它可以节省 ViT 所有的计算量。

### 7. 优点
*   **全路径加速**：不同于以往只减少 LLM 输入的方法，PixelPrune 连同视觉编码器（ViT）一起加速，这在 NaViT 风格的高分辨率模型中至关重要。
*   **算法简洁高效**：基于经典的预测编码，无需额外训练复杂的神经预测器，计算复杂度为 $O(N)$ 且易于向量化。
*   **通用性强**：支持无损和有损模式，且在通用图像域也表现出良好的鲁棒性。

### 8. 不足与局限
*   **GUI 坐标敏感性**：在完全不训练的情况下，直接剪枝可能会轻微干扰模型对 GUI 绝对坐标的判断，需要微调来补偿。
*   **任务依赖性**：对于像 ChartQA 这种需要极高空间推理能力的复杂图表任务，在极高压缩率下精度下降比普通文档更明显。
*   **硬件优化空间**：虽然理论 FLOPs 降低很多，但实际加速比受限于 GPU 的算子实现（如 FlashAttention 对非规则序列长度的优化程度）。
