## 摘要
AI 智能体的兴起为可视化分析（VA）带来了根本性的转变，智能体正成为一个新的用户群体。目前基于计算机视觉和原始 DOM 访问的智能体方法无法准确且高效地执行 VA 任务。本文介绍了可视化分析上下文协议（VACP），这是一个旨在使 VA 应用程序具备“智能体就绪”能力的框架，它通过显式暴露应用程序状态、可用交互以及直接执行机制，扩展了通用协议。为了支持该上下文协议，我们提出了 VA 界面中 AI 智能体需求和知识表示的形式化规范。我们将 VACP 实例化为一个兼容主流可视化语法和 Web 框架的库，从而能够增强现有系统并开发新系统。我们在代表性 VA 任务上的评估表明，与当前的智能体方法相比，启用 VACP 的智能体在界面解释和执行方面实现了更高的成功率，同时降低了 token 消耗和延迟。VACP 弥合了以人为中心的可视化分析界面与机器可感知性之间的鸿沟，确保智能体能够可靠地作为协作用户在 VA 系统中发挥作用。

## Abstract
The rise of AI agents introduces a fundamental shift in Visual Analytics (VA), in which agents act as a new user group. Current agentic approaches - based on computer vision and raw DOM access - fail to perform VA tasks accurately and efficiently. This paper introduces the Visual Analytics Context Protocol (VACP), a framework designed to make VA applications "agent-ready" that extends generic protocols by explicitly exposing application state, available interactions, and mechanisms for direct execution. To support our context protocol, we contribute a formal specification of AI agent requirements and knowledge representations in VA interfaces. We instantiate VACP as a library compatible with major visualization grammars and web frameworks, enabling augmentation of existing systems and the development of new ones. Our evaluation across representative VA tasks demonstrates that VACP-enabled agents achieve higher success rates in interface interpretation and execution compared to current agentic approaches, while reducing token consumption and latency. VACP closes the gap between human-centric VA interfaces and machine perceivability, ensuring agents can reliably act as collaborative users in VA systems.

#### 1. 核心问题与研究背景
*   **研究动机**：随着 AI 智能体（AI Agents）开始作为“新用户”直接操作 Web 界面，传统的可视化分析（VA）系统面临严重的“界面不匹配”问题。
*   **核心痛点**：
    *   **视觉局限性**：基于计算机视觉（截图）的方法在处理高密度数据时精度不足，且容易产生幻觉。
    *   **DOM 局限性**：现代 VA 系统多使用 `<canvas>` 或复杂的 SVG，导致 DOM 树极其庞大且缺乏语义，超出了 LLM 的上下文窗口。
*   **研究目标**：提出一种形式化的上下文协议，使 VA 系统能够向 AI 智能体显式暴露其内部状态和交互能力，从而实现高效、准确的机器感知与协作。

#### 2. 方法论：VACP 框架
*   **核心思想**：将 VA 界面从“像素/代码层”提升到“语义/交互层”，通过结构化的 API 替代模糊的视觉模拟。
*   **知识表示四层模型**：
    *   **L1 像素层**：非结构化图像。
    *   **L2 DOM 层**：浏览器逻辑树。
    *   **L3 声明式语法层**：如 Vega-Lite 定义的规则。
    *   **L4 语义逻辑层**：开发者的设计意图和生产逻辑。
*   **VACP 三大核心组件**：
    1.  **应用状态表示 (State)**：将界面建模为时序语义图，节点代表组件，边代表关联（如联动过滤），并提供稳定 ID 供智能体引用。
    2.  **交互能力暴露 (Capabilities)**：动态列出当前可用的交互意图（Intent），如“刷选”、“重置”，并定义其参数范围。
    3.  **执行网关 (Gateway)**：智能体通过调用工具（Tool Call）直接触发交互，系统验证参数后执行并返回结构化反馈。
*   **技术实现**：基于 TypeScript 开发，利用 Anthropic 的 MCP（Model Context Protocol）作为传输层，支持 D3.js、Vega-Lite 和 Mosaic 等主流框架。

#### 6. 主要结论与发现

*   **效率显著优化**：S3 场景下的 Token 消耗和执行时间最低，证明了结构化协议比解析原始 DOM 或图像更高效。
*   **智能体行为模式**：在 S4（全能力）中，智能体表现出“语义查询 + 视觉验证”的类人策略，即先通过协议获取精确数据，再通过截图确认视觉一致性。
*   **模型差异**：Claude 4.5 在 Token 效率上表现最优，而 GPT-5.2 在复杂推理中表现稳健。

#### 7. 优点与亮点
*   **填补空白**：首次提出了针对可视化分析领域的智能体交互协议。
*   **解耦设计**：将“交互意图”与“物理操作”解耦，智能体无需模拟鼠标点击坐标，只需发送语义指令。
*   **细节按需获取 (Details-on-Demand)**：通过 `inspect_data()` 机制避免了将海量数据一次性塞入上下文窗口，解决了 Token 溢出问题。

#### 8. 不足与局限
*   **视觉直觉缺失**：纯语义协议（S3）可能导致智能体忽略一些人类容易察觉的视觉模式（如 Gestalt 格式塔原则下的聚类）。
*   **适配成本**：虽然提供了库，但对于高度自定义的复杂 VA 系统，开发者仍需投入精力进行语义映射。
*   **实时性挑战**：受限于 LLM 的推理延迟，目前尚难以支持极高频的实时同步协作。
