## 摘要
长视频理解对视觉语言模型（VLM）而言具有内在挑战性，因为视频帧数庞大。由于每个视频帧通常会扩展为数十或数百个 token，大语言模型（LLM）有限的上下文长度迫使 VLM 只能稀疏地感知视频帧，从而丢失了时序信息。为解决这一问题，我们探索了极致的视频 token 压缩，旨在 LLM 的最终层实现每帧仅对应一个 token。我们的核心见解是，先前方法广泛采用的基于启发式的压缩容易导致信息丢失，因此有必要将 LLM 层监督为可学习且渐进式的 token 级压缩模块（LP-Comp）。这种压缩使我们的 VLM 能够处理多出 2 到 4 倍的视频帧，并提升性能。为了进一步提高 token 效率，我们研究了帧级压缩，即通过 LLM 层的内部注意力分数选择与查询最相关的帧，称之为问题条件压缩（QC-Comp）。与以往研究的一个显著区别在于，我们通过将长视频分割成短片段并采用局部注意力，缓解了长上下文中 LLM 注意力的位置偏差（即过度集中在序列的开头和结尾）。总之，我们将 token 级和帧级压缩相结合，构建了一个名为 XComp 的长视频理解极致压缩模型，实现了显著更高的压缩率并支持更密集的帧采样。我们的 XComp 基于 VideoChat-Flash 进行微调，采用了一种高效的数据监督压缩微调阶段，仅需 2.5% 的监督微调数据，便将 LVBench 上的准确率从 42.9% 提升至 46.2%，并增强了多个其他长视频基准测试的表现。

## Abstract
Long video understanding is inherently challenging for vision-language models (VLMs) because of the extensive number of frames. With each video frame typically expanding into tens or hundreds of tokens, the limited context length of large language models (LLMs) forces the VLMs to perceive the frames sparsely and lose temporal information. To address this, we explore extreme video token compression towards one token per frame at the final LLM layer. Our key insight is that heuristic-based compression, widely adopted by previous methods, is prone to information loss, and this necessitates supervising LLM layers into learnable and progressive modules for token-level compression (LP-Comp). Such compression enables our VLM to digest 2x-4x more frames with improved performance. To further increase the token efficiency, we investigate frame-level compression, which selects the frames most relevant to the queries via the internal attention scores of the LLM layers, named question-conditioned compression (QC-Comp). As a notable distinction from previous studies, we mitigate the position bias of LLM attention in long contexts, i.e., the over-concentration on the beginning and end of a sequence, by splitting long videos into short segments and employing local attention. Collectively, our combined token-level and frame-level leads to an extreme compression model for long video understanding, named XComp, achieving a significantly larger compression ratio and enabling denser frame sampling. Our XComp is finetuned from VideoChat-Flash with a data-efficient supervised compression tuning stage that only requires 2.5% of the supervised fine-tuning data, yet boosts the accuracy from 42.9% to 46.2% on LVBench and enhances multiple other long video benchmarks.

### 论文总结：One Token per Highly Selective Frame (XComp)

#### 1. 核心问题与整体含义
长视频理解的主要瓶颈在于**视觉标记（Visual Tokens）的爆炸式增长**。现有的视觉语言模型（VLM）在处理长视频时，每帧通常被编码为数十甚至数百个 token，这迅速填满了大语言模型（LLM）的上下文窗口，迫使模型不得不进行稀疏采样，从而丢失了关键的时序细节。

本文的研究动机在于：现有的压缩方法（如简单的池化或启发式采样）是“非学习”的，容易在极高压缩率下丢失关键信息。作者旨在探索一种**极致压缩**方案，使 LLM 最终层能够实现“每帧仅一个 token”的表示，从而允许模型消化更密集的视频帧（提升 2-4 倍采样率）。

#### 2. 方法论
XComp 框架由两个核心组件组成，分别从 token 级和帧级进行压缩：

*   **LP-Comp（可学习且渐进式标记压缩）**：
    *   **核心思想**：不再依赖预定义的启发式规则，而是通过监督微调（SCT）让 LLM 层学会如何将每帧的多个 token 浓缩。
    *   **技术细节**：采用**渐进式（Progressive）**策略，根据余弦调度函数在 LLM 的各层之间平滑地减少 token 数量。
    *   **后缀保留（Suffix-Preservation）**：在压缩时优先保留每帧序列的最后一个 token。这是因为在 Decoder-only 的 LLM 中，因果注意力机制使得后面的 token 能够聚合前面 token 的信息。
*   **QC-Comp（问题条件帧压缩）**：
    *   **核心思想**：在推理阶段，根据用户的问题动态筛选最相关的视频帧。
    *   **分段局部注意力（Segmented Local Attention）**：为了克服 LLM 在长上下文中存在的“位置偏差”（即注意力容易集中在开头和结尾，忽略中间内容），作者将长视频切分为短片段，在片段内计算问题与视频帧的注意力分数，以此作为筛选依据。

#### 3. 实验设计
*   **数据集与基准**：在四个主流长视频理解基准上进行了评估：**LongVideoBench、MLVU、VideoMME (Long)** 和 **LVBench**。这些基准涵盖了几分钟到一小时以上的视频。
*   **对比方法**：
    *   **同规模模型**：VideoChat-Flash-2B（基座模型）、InternVL3-2B。
    *   **大规模模型（参考）**：Qwen2.5VL-3B、mPLUG-Owl3-7B、VideoChat-Flash-7B、Eagle2.5-8B 等。
    *   **闭源模型**：GPT-4V、GPT-4o、Gemini-1.5-Pro。
*   **消融实验**：对比了可学习压缩 vs. 训练前压缩、渐进式 vs. 步进式压缩、后缀保留 vs. 均匀丢弃、局部注意力 vs. 全局注意力。

#### 4. 资源与算力
*   **硬件环境**：使用了 **8 × NVIDIA H100 GPU**。
*   **训练时长**：监督压缩微调（SCT）阶段大约耗时 **24 小时**。
*   **数据效率**：该方法非常高效，仅使用了 VideoChat-Flash 原始监督微调（SFT）数据的 **2.5%**（约 7.2 万条数据）。

#### 5. 实验数量与充分性
作者进行了多维度的实验，包括：
*   在 4 个不同侧重点的长视频数据集上进行主实验。
*   针对 LP-Comp 和 QC-Comp 的各个设计决策进行了详尽的消融实验（约 6 组核心对比）。
*   验证了方法在不同基座模型（如 LLaVA-Next-Video）上的泛化性。
*   **评价**：实验设计逻辑严密，通过对比“微调后的基座模型”与“应用 XComp 的模型”，排除了性能提升仅来自额外训练的可能性。实验结果客观地展示了极致压缩在处理超长序列时的优势。

#### 6. 主要结论与发现
*   **极致压缩可行性**：通过学习，LLM 能够将一帧的信息压缩至单个 token 而不损失核心语义。
*   **性能提升**：XComp 在 2B 规模下达到了 SOTA 水平，在 LVBench 上将准确率从 42.9% 提升至 46.2%。
*   **正向反馈循环**：由于 token 效率极高，模型可以输入更多帧（如 1024 帧以上），且性能随帧数增加持续提升，而传统模型在帧数过多时往往会出现性能下降。
*   **位置偏差缓解**：分段局部注意力能有效解决长视频理解中的“迷失在中间”问题。

#### 7. 优点
*   **极高的 Token 效率**：实现了每帧 1 token 的极致表示，极大缓解了计算压力。
*   **数据高效性**：仅需极少量的微调数据即可赋予模型压缩能力。
*   **设计巧妙**：结合了 LLM 的因果特性（后缀保留）和长文本处理痛点（局部注意力）。

#### 8. 不足与局限
*   **模型规模限制**：主要实验集中在 2B 规模模型，虽然在 LLaVA-Next 上做了泛化验证，但在更大规模（如 70B+）模型上的表现尚待观察。
*   **任务覆盖**：主要针对多选题形式的 QA 任务，对于需要极高空间分辨率（如细微物体检测）的任务，每帧 1 token 可能会导致空间细节丢失。
*   **联合优化**：目前的帧筛选（QC-Comp）是在推理时进行的，尚未实现帧筛选与 token 压缩的端到端联合训练。
