## 摘要
基于大语言模型（LLM）的智能体可能会执行在语法上有效、经用户授权且语义上恰当的操作，但由于正确进行策略判断所需的事实信息在决策时处于隐藏状态，这些操作仍可能违反组织策略。我们将这种失败模式称为“策略不可见违规”：即合规性取决于智能体可见上下文中所缺失的实体属性、上下文状态或会话历史的情况。我们提出了 PhantomPolicy，这是一个涵盖八个违规类别的基准测试，包含平衡的违规案例和安全对照案例，其中所有工具响应仅包含纯净的业务数据，而不包含策略元数据。我们人工审查了由五个前沿模型生成的全部 600 条模型轨迹，并使用经人工审核的轨迹标签对其进行评估。与原始的案例级标注相比，人工审查修改了 32 个标签（5.3%），证实了进行轨迹级人工审查的必要性。为了展示在理想条件下基于世界状态感知的策略执行所能达到的效果，我们引入了 Sentinel，这是一个基于反事实图模拟的执行框架。Sentinel 将智能体的每个动作视为对组织知识图谱的拟议变更，通过执行推测性操作来具象化动作后的世界状态，并验证图结构的不变性，从而做出“允许/拦截/澄清”的决策。在针对人工审核的轨迹标签进行的测试中，Sentinel 的表现显著优于仅基于内容的 DLP 基准（准确率从 68.8% 提升至 93.0%），同时保持了高精确率，尽管在某些违规类别上仍有改进空间。这些结果表明，一旦将与策略相关的世界状态提供给执行层，安全防护水平将得到显著提升。

## Abstract
LLM-based agents can execute actions that are syntactically valid, user-sanctioned, and semantically appropriate, yet still violate organizational policy because the facts needed for correct policy judgment are hidden at decision time. We call this failure mode policy-invisible violations: cases in which compliance depends on entity attributes, contextual state, or session history absent from the agent's visible context. We present PhantomPolicy, a benchmark spanning eight violation categories with balanced violation and safe-control cases, in which all tool responses contain clean business data without policy metadata. We manually review all 600 model traces produced by five frontier models and evaluate them using human-reviewed trace labels. Manual review changes 32 labels (5.3%) relative to the original case-level annotations, confirming the need for trace-level human review. To demonstrate what world-state-grounded enforcement can achieve under favorable conditions, we introduce Sentinel, an enforcement framework based on counterfactual graph simulation. Sentinel treats every agent action as a proposed mutation to an organizational knowledge graph, performs speculative execution to materialize the post-action world state, and verifies graph-structural invariants to decide Allow/Block/Clarify. Against human-reviewed trace labels, Sentinel substantially outperforms a content-only DLP baseline (68.8% vs. 93.0% accuracy) while maintaining high precision, though it still leaves room for improvement on certain violation categories. These results demonstrate what becomes achievable once policy-relevant world state is made available to the enforcement layer.

*   **研究动机**：当LLM智能体集成到组织工具中时，它们执行的操作可能在语法上正确、语义上合理且符合用户指令，但却违反了组织政策。这是因为判断是否违规所需的关键事实（如接收者的真实权限、文档的隐藏敏感度、会话历史的累积风险）在决策时并未包含在智能体的可见上下文中。

### 2. 论文提出的方法论
论文提出了两个核心贡献：**PhantomPolicy 基准测试**和 **Sentinel 执行框架**。

#### A. PhantomPolicy 基准测试
*   **核心思想**：设计一个没有任何策略元数据（如“机密”标签）的纯净业务数据环境，迫使模型无法通过表面文本线索避开违规。
*   **分类**：涵盖 8 类违规（上下文边界、文本输出泄露、过度共享、受众限制、累积会话泄露、跨上下文数据流、高价值资源保护、时间有效性）。

#### B. Sentinel 执行框架
*   **核心思想：反事实图模拟（Counterfactual Graph Simulation）**。将智能体的动作视为对组织知识图谱（World State Graph）的拟议修改，在虚拟层模拟执行并验证。
*   **关键技术流程**：
    1.  **翻译（Translate）**：将工具调用转换为图突变（Mutation），如添加边（数据流向）。
    2.  **分支（Fork）**：创建当前世界状态图的 $O(1)$ 写时复制（Copy-on-write）副本。
    3.  **突变（Mutate）**：在副本上应用拟议动作。
    4.  **检查（Check）**：验证 7 个声明式图不变性（Invariants），如 `ActiveRecipient`（接收者是否活跃）、`InformationFlow`（数据流向是否符合权限）。
    5.  **决策（Decide）**：根据验证结果决定“允许”、“拦截”或“澄清”。
*   **算法优势**：验证复杂度为 $O(|M|)$（$M$ 为动作数量），与基础图的大小无关，延迟极低（<1ms）。

### 6. 论文的主要结论与发现
*   **违规普遍性**：在隐藏策略状态下，所有前沿模型在风险案例中的违规率高达 90%–98%。
*   **提示词局限性**：将政策放入提示词（Policy-in-prompt）虽能降低违规率，但效果极不稳定（25%–85% 不等），且无法处理隐藏事实。

*   **核心瓶颈**：智能体违规的主要瓶颈不是推理能力，而是**缺乏与策略相关的世界状态信息**。

### 7. 优点
*   **问题定义新颖**：首次系统化定义了“策略不可见违规”这一隐蔽且高危的失败模式。
*   **架构解耦**：Sentinel 实现了策略执行与模型推理的架构分离，符合安全工程中的权限分离原则。
*   **高效且可证明**：提出了具有 $O(|M|)$ 复杂度的验证算法，并提供了组合性（Composability）的理论证明。
*   **高质量基准**：PhantomPolicy 强调“纯净数据”，排除了模型通过文本捷径（Shortcuts）进行判断的可能性。

### 8. 不足与局限
*   **世界模型构建难度**：Sentinel 的有效性高度依赖于组织知识图谱的完整性和准确性（ETL 挑战）。

*   **领域局限**：基准测试目前集中在企业办公（邮件、文件共享）领域，尚未扩展到医疗、金融等更复杂的垂直领域。
*   **静态政策**：实验中的政策相对静态，未深入探讨动态变化政策下的实时同步问题。
