## 摘要
大语言模型（LLM）的最新进展为机器人提供了理解人类指令的上下文推理能力。然而，目前支持 LLM 的机器人通常依赖于云端模型或高性能计算基础设施，这限制了它们在网络环境不可靠或计算资源受限的机器人（如无人机和小型地面车辆）上的部署。因此，部署支持机载运行的微调小语言模型（SLM）提供了一个极具前景的替代方案。本文介绍了 Ro-SLM，这是一个通过蒸馏 LLM 的知识和推理能力来实现可靠的 SLM 驱动机器人操作的框架。Ro-SLM 从数据集合成开始，利用 LLM 生成多样化的任务指令，在极少人工协助的情况下产生相应的基准代码，并将指令增强到现实世界的应用场景中。随后，Ro-SLM 使用该数据集进行微调，其中 LLM 作为奖励函数来指导训练。在无人机操作任务上的广泛实验表明，Ro-SLM 将 SLM 的性能从无法支持机器人任务规划和代码生成提升到了接近 LLM 的水平。

## Abstract
Recent advances in large language models (LLMs) provide robots with contextual reasoning abilities to comprehend human instructions. Yet, current LLM-enabled robots typically depend on cloud-based models or high-performance computing infrastructure, which limit their deployment on robots under unreliable internet environments or with constrained computational resources, such as UAVs and small ground vehicles. Thus, deploying fine-tuned small language models (SLMs) that support onboard deployment offers a promising alternative. This paper introduces Ro-SLM, a framework that enables reliable SLM-driven robot operation by distilling LLMs' knowledge and reasoning. Ro-SLM starts from dataset synthesis by leveraging LLMs to generate diverse task instructions, produce corresponding ground truth code with minimal human assistance, and augment instructions into real-world application scenarios. Ro-SLM is then fine-tuned with the dataset, in which LLM serves as a reward function to guide the training. Extensive experiments on UAV operation tasks demonstrate that Ro-SLM improves the performance of SLM from being incapable of supporting robotic task planning and code generation to achieving performance that approaches LLM.

#### 1. 核心问题与整体含义（研究动机和背景）
当前，利用大语言模型（LLM）驱动机器人进行任务规划和代码生成已成为主流，但面临两大瓶颈：
*   **云端依赖性**：GPT-4 等模型依赖稳定的互联网连接，在山区、灾区等网络受限环境下无法使用。
*   **资源受限性**：本地部署数百 B 参数的模型需要极高的计算和电力资源，而无人机（UAV）和小型地面车辆的载荷及电池容量有限。
*   **SLM 的局限性**：虽然小语言模型（SLM，如 8B 以下）适合机载部署，但其原生推理能力较弱，难以准确理解复杂指令并生成可执行的机器人操作代码。

**Ro-SLM 的核心目标**：通过知识蒸馏和针对性微调，将 LLM 的推理能力迁移到 SLM 中，使其在无需联网的情况下，能在嵌入式设备上实现高性能的机器人任务规划。

---

#### 2. 方法论：核心思想与关键技术
Ro-SLM 框架分为两个主要阶段：

**阶段一：数据集合成（Dataset Synthesis）**
1.  **任务指令生成 ($F_1$)**：配置 LLM Agent 自动生成不同复杂度的机器人操作指令（如空间移动、旋转），通过 In-Context Learning 确保指令的可执行性。
2.  **基准代码生成 ($F_2$)**：利用纠错式代码生成框架将指令转化为代码。对于简单任务完全自动化；对于高复杂度任务（5步以上操作），引入**极少量人工协助**进行校验，确保 Ground Truth 的绝对准确。
3.  **场景增强 ($F_3$)**：利用 LLM 的世界知识，将抽象的指令（如“向上飞5米”）改写为现实应用场景描述（如“检查仓库顶部的通风口”），提高模型对人类自然语言的鲁棒性。
4.  **数据预处理**：移除代码中的所有注释，防止 SLM 过度拟合语言模式而非逻辑结构。

**阶段二：SLM 微调（Fine-tuning）**
1.  **监督微调（SFT）**：使用合成数据集对 SLM（如 Llama-3.1-8B）进行初步对齐，学习 API 调用和基本逻辑。
2.  **LLM 反馈优化（GRPO）**：引入群体相对策略优化（GRPO），将 **LLM 作为奖励函数**。LLM 不比较文本相似度，而是解释代码背后的机器人行为意图，判断 SLM 生成的代码在逻辑上是否与基准代码等效。

#### 6. 主要结论与发现
*   **性能飞跃**：Ro-SLM 将 8B 规模的 SLM 从“完全无法胜任任务”提升到了“接近 LLM（GPT-4o-mini）”的水平。
*   **超越 LLM 的潜力**：在“极高难度”任务中，Ro-SLM 的成功率甚至超过了原始 LLM。这归功于微调数据中包含的人工校验代码，纠正了 LLM 在长程推理中的幻觉。
*   **鲁棒性**：经过现实场景增强训练后，SLM 对人类非规范化指令的理解能力显著增强。
*   **去注释的重要性**：实验发现，保留注释会导致 SLM 性能下降，因为小模型容易被注释中的语言噪声干扰。

#### 7. 优点（亮点）
*   **端侧闭环**：真正实现了在资源受限设备上脱离云端的智能决策。
*   **高效蒸馏**：巧妙地利用 LLM 作为“逻辑裁判”（奖励函数）而非简单的“文本模仿对象”，解决了代码语义等效性的评估难题。
*   **低成本数据生产**：通过 LLM 自动合成+关键节点人工干预，平衡了数据质量与标注成本。

#### 8. 不足与局限
*   **泛化性限制**：数据合成仍具有较强的任务特定性，迁移到完全不同的机器人领域（如机械臂抓取）可能需要重新构建数据集。
*   **硬件门槛**：虽然是 SLM，但 8B 模型仍需约 16GB 显存，对于更小型的嵌入式设备（如仅有 4GB/8GB 内存的单片机）依然过大。
*   **单次推理风险**：Ro-SLM 采用单次生成模式，缺乏类似 Corrective GSCE 的多轮迭代纠错机制，在处理极其复杂的逻辑时容错率较低。
*   **人工依赖**：极高难度任务仍需专家介入校验数据，限制了其向超大规模数据集扩展的自动化程度。
