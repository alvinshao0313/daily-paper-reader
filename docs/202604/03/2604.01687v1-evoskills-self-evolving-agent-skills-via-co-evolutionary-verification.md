## 摘要
Anthropic 提出了大语言模型（LLM）智能体“技能”（skills）的概念，旨在解决简单工具调用无法处理的多步专业任务。工具是单一且自包含的功能，而技能则是相互依赖的多文件构件组成的结构化捆绑包。目前，技能生成不仅因人工编写而具有标签密集型特征，还可能面临人机认知失调的问题，这可能导致智能体性能下降，SkillsBench 上的评估结果证明了这一点。因此，我们的目标是使智能体能够自主生成技能。然而，现有的针对工具设计的自我演化方法由于技能复杂性的增加而无法直接应用。为了解决这些问题，我们提出了 EvoSkills，这是一个自我演化技能框架，使智能体能够自主构建复杂的多文件技能包。具体而言，EvoSkills 将迭代优化技能的技能生成器（Skill Generator）与协同演化的代理验证器（Surrogate Verifier）相结合，后者在无法访问真实测试内容的情况下提供具有信息量且可操作的反馈。在 SkillsBench 上，EvoSkills 在 Claude Code 和 Codex 上均取得了五个基准测试中的最高通过率，并对另外六个 LLM 表现出强大的泛化能力。

## Abstract
Anthropic proposes the concept of skills for LLM agents to tackle multi-step professional tasks that simple tool invocations cannot address. A tool is a single, self-contained function, whereas a skill is a structured bundle of interdependent multi-file artifacts. Currently, skill generation is not only label-intensive due to manual authoring, but also may suffer from human--machine cognitive misalignment, which can lead to degraded agent performance, as evidenced by evaluations on SkillsBench. Therefore, we aim to enable agents to autonomously generate skills. However, existing self-evolving methods designed for tools cannot be directly applied to skills due to their increased complexity. To address these issues, we propose EvoSkills, a self-evolving skills framework that enables agents to autonomously construct complex, multi-file skill packages. Specifically, EvoSkills couples a Skill Generator that iteratively refines skills with a Surrogate Verifier that co-evolves to provide informative and actionable feedback without access to ground-truth test content. On SkillsBench, EvoSkills achieves the highest pass rate among five baselines on both Claude Code and Codex, and also exhibits strong generalization capabilities to six additional LLMs.

#### 1. 核心问题与研究动机
*   **核心问题**：传统的 LLM 智能体通常依赖简单的“工具调用”（Tool Use），但在处理复杂的专业任务（如软件修复、科学分析）时，单一工具难以胜任。Anthropic 提出了“技能”（Skills）概念——即包含工作流指令、可执行脚本和领域参考的结构化多文件包。
*   **研究动机**：
    *   **标注密集**：目前技能主要靠人工编写，难以规模化。
    *   **人机认知失调**：人类设计的专家工作流往往不符合 LLM 的推理模式，导致性能反而下降（如在自然科学领域）。
    *   **现有方法局限**：现有的工具演化方法多针对单文件函数，无法处理复杂的多文件技能包，且高度依赖真实标签（Ground-truth）进行错误诊断。

#### 2. 方法论
EvoSkills 提出了一个**协同演化（Co-Evolutionary）框架**，核心思想是通过两个相互独立但协同工作的组件进行迭代优化：
*   **技能生成器（Skill Generator）**：负责根据任务指令和反馈，迭代生成并完善包含 `SKILL.md` 和配套脚本的多文件技能包。它维护一个持久的对话上下文，积累高保真反馈。
*   **代理验证器（Surrogate Verifier）**：这是一个独立的 LLM 会话（信息隔离），在无法接触真实测试用例的情况下，根据任务指令自主合成测试脚本和断言。
*   **协同演化流程**：
    1.  生成器产生初始技能并执行。
    2.  验证器对输出进行评估，若失败则提供结构化的诊断反馈（F）。
    3.  若验证器通过但真实环境（Oracle）反馈失败（仅返回 0/1 信号），验证器会**升级（Escalate）**其测试用例的难度和覆盖面。
    4.  通过“生成-验证-精炼”循环，技能质量稳步提升。

*   **自主性高**：无需人工参与，解决了技能开发的扩展性问题。

*   **成本问题**：多次迭代演化（最多 15 次重试）会消耗大量的 Token 和推理时间，成本高于单次生成。
*   **模型依赖**：虽然具有迁移性，但演化过程本身仍依赖于强大的基座模型（如 Opus 4.6 或 GPT-5.2）作为生成器和验证器。
*   **复杂环境限制**：对于极其动态或需要实时交互的环境，预先演化的静态技能包可能仍存在滞后性。
