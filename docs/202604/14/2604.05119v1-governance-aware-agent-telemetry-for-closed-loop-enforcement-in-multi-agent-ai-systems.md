## 摘要
企业级多智能体 AI 系统每小时产生数千次智能体间的交互，然而现有的可观测性工具仅捕获这些依赖关系，而不进行任何强制执行。OpenTelemetry 和 Langfuse 虽然收集遥测数据，但将治理视为下游分析关注点，而非实时执行目标。这导致了“只观察不行动”的鸿沟，使得策略违规行为往往在损害发生后才被检测到。我们提出了治理感知智能体遥测（GAAT），这是一种为多智能体系统实现遥测收集与自动化策略执行闭环的参考架构。GAAT 引入了：(1) 扩展了 OpenTelemetry 治理属性的治理遥测模式（GTS）；(2) 一个在低于 200 毫秒延迟下使用兼容 OPA 的声明式规则的实时策略违规检测引擎；(3) 一个具有分级干预功能的治理执行总线（GEB）；以及 (4) 一个具有加密溯源功能的受信任遥测平面。

## Abstract
Enterprise multi-agent AI systems produce thousands of inter-agent interactions per hour, yet existing observability tools capture these dependencies without enforcing anything. OpenTelemetry and Langfuse collect telemetry but treat governance as a downstream analytics concern, not a real-time enforcement target. The result is an "observe-but-do-not-act" gap where policy violations are detected only after damage is done.   We present Governance-Aware Agent Telemetry (GAAT), a reference architecture that closes the loop between telemetry collection and automated policy enforcement for multi-agent systems. GAAT introduces (1) a Governance Telemetry Schema (GTS) extending OpenTelemetry with governance attributes; (2) a real-time policy violation detection engine using OPA-compatible declarative rules under sub-200 ms latency; (3) a Governance Enforcement Bus (GEB) with graduated interventions; and (4) a Trusted Telemetry Plane with cryptographic provenance.

*   **核心问题**：现有的 AI 观测工具（如 OpenTelemetry, Langfuse）存在“**只观察不行动**”的鸿沟。它们能记录智能体间的交互，但通常将治理视为事后分析任务。在企业级 MAS 中，违规行为（如隐私泄露、违反数据驻留政策）发生在毫秒级，事后检测无法阻止损害。
*   **研究背景**：随着 EU AI Act 和 NIST AI RMF 等法规的出台，对高风险 AI 系统进行实时监控和强制执行已成为合规刚需。目前的工具缺乏跨智能体的血缘追踪（Lineage Tracking）和闭环执行能力。

### 2. 方法论
GAAT 提出了一个五层参考架构，核心思想是将治理属性嵌入遥测流，并实现从检测到干预的闭环。

*   **五层架构**：
    1.  **智能体执行层**：运行基于 LLM 的智能体。
    2.  **治理仪表层**：扩展 OpenTelemetry 提出 **GTS (Governance Telemetry Schema)**，捕获数据分类、管辖权、敏感度和血缘。
    3.  **遥测聚合层**：使用 Kafka 处理流数据，进行签名验证、重放检测（布隆过滤器）和遗漏检测（基于隐马尔可夫模型 HMM）。
    4.  **策略评估层**：使用 OPA (Open Policy Agent) 评估声明式规则，支持亚 200ms 的并行评估。
    5.  **执行动作层**：通过 **GEB (治理执行总线)** 实施分级干预（L0 允许、L1 告警、L2 标记、L3 重定向、L4 隔离）。
*   **关键技术细节**：
    *   **形式化模型**：定义了策略代数和置信度组合，证明了执行的单调性、升级终止性和冲突解决的确定性。
    *   **分级升级公式**：$escalation(v, H(a, t)) = \min(4, base(v) + \lfloor |H(a, t)|/k \rfloor)$，根据违规历史动态调整干预强度。
    *   **信任遥测平面**：利用 ECDSA P-256 签名和 Merkle 树审计日志确保遥测数据的完整性和不可篡改性。

### 6. 主要结论与发现
*   **预防率 (VPR)**：GAAT 在合成流中达到 **98.3%** 的违规预防率，在生产追踪中达到 **99.7%**，显著优于 NeMo Guardrails (78.8%)。

*   **核心发现**：现有的边界检查工具（如 NeMo）无法检测跨智能体委派链中的违规，而 GAAT 的**跨智能体血缘追踪**是识别此类复杂违规的关键。

### 7. 优点
*   **闭环设计**：首次实现了从遥测到实时强制执行的闭环，填补了行业空白。
*   **形式化保证**：通过数学证明和模拟验证了系统在极端情况下的收敛性和确定性。
*   **低侵入性与高性能**：基于 OpenTelemetry 扩展，易于集成且延迟极低，适合高并发企业场景。
*   **分级干预**：避免了非黑即白的二元处理，通过分级手段平衡了系统可用性与合规性。

### 8. 不足与局限
*   **规模限制**：目前的实测仅在 5 个智能体规模下进行，50 个智能体的结果基于模拟，超大规模集群的性能表现有待验证。

*   **框架依赖**：目前仅在 LangChain 上验证，虽然理论上支持 OpenTelemetry 的框架均可使用，但尚未在 AutoGen 或 CrewAI 上实测。
*   **对抗性风险**：如果智能体私钥被窃取或 CA 被攻破，加密信任平面将失效。
