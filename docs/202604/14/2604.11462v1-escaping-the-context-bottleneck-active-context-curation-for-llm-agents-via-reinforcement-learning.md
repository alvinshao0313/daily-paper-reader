## 摘要
大语言模型（LLMs）在处理长程任务时面临挑战，这是由于“上下文瓶颈”和“迷失中间（lost-in-the-middle）”现象，即来自冗长环境的累积噪声降低了多轮交互中的推理能力。为了解决这一问题，我们引入了一种共生框架，将上下文管理与任务执行解耦。我们的架构将一个轻量级、专门化的策略模型 ContextCurator 与一个强大的冻结基础模型 TaskExecutor 配对。通过强化学习训练，ContextCurator 主动降低工作记忆中的信息熵。它在积极剪枝环境噪声的同时，保留推理锚点（reasoning anchors），即对未来推导至关重要的稀疏数据点。在 WebArena 上，我们的框架将 Gemini-3.0-flash 的成功率从 36.4% 提高到 41.2%，同时将 Token 消耗降低了 8.8%（从 47.4K 降至 43.3K）。在 DeepSearch 上，它实现了 57.1% 的成功率（对比 53.9%），同时将 Token 消耗降低了 8 倍。值得注意的是，一个 7B 参数的 ContextCurator 达到了与 GPT-4o 相当的上下文管理性能，为自主长程智能体提供了一种可扩展且计算高效的范式。

## Abstract
Large Language Models (LLMs) struggle with long-horizon tasks due to the "context bottleneck" and the "lost-in-the-middle" phenomenon, where accumulated noise from verbose environments degrades reasoning over multi-turn interactions. To address this issue, we introduce a symbiotic framework that decouples context management from task execution. Our architecture pairs a lightweight, specialized policy model, ContextCurator, with a powerful frozen foundation model, TaskExecutor. Trained via reinforcement learning, ContextCurator actively reduces information entropy in the working memory. It aggressively prunes environmental noise while preserving reasoning anchors, that is, sparse data points that are critical for future deductions. On WebArena, our framework improves the success rate of Gemini-3.0-flash from 36.4% to 41.2% while reducing token consumption by 8.8% (from 47.4K to 43.3K). On DeepSearch, it achieves a 57.1% success rate, compared with 53.9%, while reducing token consumption by a factor of 8. Remarkably, a 7B ContextCurator matches the context management performance of GPT-4o, providing a scalable and computationally efficient paradigm for autonomous long-horizon agents.

*   **整体含义**：本文认为解决长程任务的关键不在于无限扩大上下文窗口，而在于**主动的上下文策展（Active Context Curation）**。

### 2. 方法论：ActiveContext 框架
*   **核心思想**：采用“共生架构”，将**上下文管理**与**任务执行**解耦。
    *   **ContextCurator（策展人）**：一个轻量级的专门化策略模型（如 7B 规模），负责将原始交互日志压缩、精炼为高效的工作记忆。
    *   **TaskExecutor（执行者）**：一个强大的冻结基础模型（如 GPT-4o 或 Gemini），负责基于精炼后的记忆进行高阶推理和决策。
*   **关键技术细节**：
    *   **主动信息熵减少**：Curator 不仅仅是做摘要，而是通过生成式重写，剔除结构/语义噪声，同时识别并保留“推理锚点（Reasoning Anchors）”——即那些对未来决策至关重要但当前看似无关的稀疏数据点。
    *   **多轮 GRPO（MT-GRPO）**：使用强化学习算法优化 Curator。将 TaskExecutor 视为环境的一部分，根据任务最终的成功/失败（远端奖励）来训练 Curator。
    *   **因果依赖链**：Curator 决策 $\rightarrow$ Executor 动作 $\rightarrow$ 环境反馈。通过这种方式，Curator 学习如何生成最能支持 Executor 成功的记忆状态。

### 3. 实验设计

    2.  **DeepSearch**：基于 RAG 的复杂搜索任务，侧重于从大量检索文档中过滤语义噪声。
*   **Benchmark 与对比方法**：
    *   **基准**：No Memory（无记忆）、Full Context（全量上下文）。
    *   **对比方法**：包括被动存储系统（Mem0）、基于工作流的方法（AWM）、轨迹提示方法（Synapse）以及 Training-Free GRPO 等。

### 6. 主要结论与发现
*   **性能提升**：在 WebArena 上，将 Gemini-3.0-flash 的成功率从 36.4% 提升至 41.2%；在 DeepSearch 上成功率从 53.9% 提升至 57.1%。
*   **效率飞跃**：在提升性能的同时，Web 任务 Token 消耗降低 8.8%，搜索任务 Token 消耗降低了 **8 倍**（从 46.7K 降至 6.6K）。
*   **以小博大**：经过 RL 微调的 **7B 模型** 在上下文管理能力上可以媲美甚至超越未针对性训练的 **GPT-4o**。
*   **信噪比是关键**：实验证明，单纯增加上下文长度并不能解决问题，提高工作记忆的信噪比才是提升智能体长程能力的核心。

### 7. 优点
*   **解耦设计**：将昂贵的推理与频繁的记忆管理分开，既降低了成本，又避免了单模型在多任务目标下的梯度干扰。
*   **RL 驱动的精准策展**：通过远端奖励引导，模型学会了识别“推理锚点”，解决了传统语义检索（RAG）容易遗漏非文本相似关键信息的问题。
*   **显著的经济性**：大幅减少了调用闭源高阶 API 时的 Token 成本，具有极高的工程实用价值。

### 8. 不足与局限
*   **跨模型泛化风险**：虽然文中测试了多种执行器，但 Curator 是针对特定执行器的反馈训练的，如果执行器发生剧烈变动，Curator 的策展策略可能需要重新对齐。
*   **训练成本**：尽管推理时高效，但 MT-GRPO 的在线强化学习训练过程仍需要较多算力支持。
*   **应用领域限制**：目前仅在 Web 和 Search 两个领域验证，对于需要极高实时性或物理交互的具身智能场景，该框架的延迟和表现尚待观察。
