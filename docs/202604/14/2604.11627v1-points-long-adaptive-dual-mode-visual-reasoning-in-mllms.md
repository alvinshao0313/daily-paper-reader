## 摘要
多模态大语言模型 (MLLMs) 近期在跨模态理解与生成方面展现出了卓越的能力。然而，视觉 Token 序列的快速增长——尤其是在长视频和流式场景中——对其可扩展性和实际部署构成了重大挑战。为此，我们推出了 POINTS-Long，这是一种受人类视觉系统启发、具备动态视觉 Token 缩放功能的原生双模式 MLLM。该模型支持两种互补的感知模式：专注模式 (focus mode) 和待机模式 (standby mode)，使用户能够在推理过程中动态权衡效率与准确性。在细粒度视觉任务中，专注模式保持了最优性能；而在长程通用视觉理解任务中，待机模式仅使用 1/40 至 1/10 的视觉 Token 即可保留 97.7% 至 99.7% 的原始准确率。此外，POINTS-Long 通过动态可拆卸的 KV 缓存 (KV-cache) 设计，原生支持流式视觉理解，从而能够高效维护超长视觉记忆。我们的工作为未来 MLLM 的设计提供了新的见解，并为自适应且高效的长程视觉理解奠定了基础。

## Abstract
Multimodal Large Language Models (MLLMs) have recently demonstrated remarkable capabilities in cross-modal understanding and generation. However, the rapid growth of visual token sequences--especially in long-video and streaming scenarios--poses a major challenge to their scalability and real-world deployment. Thus, we introduce POINTS-Long, a native dual-mode MLLM featuring dynamic visual token scaling inspired by the human visual system. The model supports two complementary perception modes: focus mode and standby mode, enabling users to dynamically trade off efficiency and accuracy during inference. On fine-grained visual tasks, the focus mode retains the optimal performance, while on long-form general visual understanding, the standby mode retains 97.7-99.7% of the original accuracy using only 1/40-1/10th of the visual tokens. Moreover, POINTS-Long natively supports streaming visual understanding via a dynamically detachable KV-cache design, allowing efficient maintenance of ultra-long visual memory. Our work provides new insights into the design of future MLLMs and lays the foundation for adaptive and efficient long-form visual understanding.

*   **核心问题**：多模态大语言模型（MLLM）在处理长视频或流式数据时，视觉 Token 数量随时间呈线性增长，导致计算量和内存消耗呈二次方增长，严重限制了模型的可扩展性和实时部署。

*   **核心启发**：受人类视觉系统启发，人类既能通过“专注模式”捕捉细节，也能通过“待机模式”维持全局感知。论文旨在构建一个原生支持双模式切换的 MLLM，实现效率与精度的动态平衡。

### 2. 核心方法论
*   **双路径 ViT 架构 (Dual-Path ViT)**：
    *   在原始视觉序列基础上增加 $n$ 个可学习的“待机 Token”（Standby tokens）。
    *   **并行 MLP**：为待机 Token 复制一套并行的 MLP 层（初始权重相同），原始 Token 走原路径，待机 Token 走新路径。
    *   **非对称注意力掩码**：原始 Token 仅关注自身以保持特征不变；待机 Token 则关注全序列以聚合全局信息。
*   **时空建模**：在 ViT 后几层引入因果时间注意力（Causal Temporal Attention），对相邻 8 帧的待机 Token 进行建模，捕捉视频的时间冗余。
*   **两阶段训练策略**：
    *   **阶段 1（视觉蒸馏与对齐）**：冻结原始模型，仅训练新增的待机 Token、并行 MLP 和投影层，强制模型将视觉信息压缩进极少数 Token 中。
    *   **阶段 2（LLM 模式适配）**：解冻 LLM，采用“双前向（2-forward）”训练策略。在每一步中同时计算待机模式（短序列）和专注模式（全序列）的损失并取平均，确保 LLM 能同时理解两种输入格式。
*   **流式推理机制**：提出“可拆卸 KV 缓存（Detachable KV-cache）”。当局部窗口满时，丢弃庞大的专注模式缓存，仅保留并迁移紧凑的待机模式缓存至“记忆库”，实现超长时记忆。

### 6. 主要结论与发现
*   **极高的压缩比**：待机模式仅需原始 1/40 到 1/10 的 Token，即可保留 97.7% - 99.7% 的原始性能。
*   **性能无损**：专注模式完全保留了原始模型的细粒度图像理解能力（甚至略有提升）。

*   **长视频优势**：由于 Token 极少，模型在处理超长视频时不会轻易触碰上下文长度限制，展现出更好的零样本扩展性。

### 7. 优点（亮点）
*   **人类启发式设计**：双模式切换逻辑符合认知科学，解决了 MLLM 领域长期存在的效率-精度矛盾。
*   **部署友好**：设计时充分考虑了 Flash-Attn 和 KV 缓存管理，能直接在 SGLang 等高性能框架中获得实际加速，而非仅停留在理论 FLOPs 的降低。
*   **流式创新**：可拆卸 KV 缓存为实时多模态助手提供了一种优雅的长短期记忆管理方案。

### 8. 不足与局限
*   **细粒度损失**：尽管待机模式表现优异，但在涉及空间感知和极细微物体识别（如硬币正反面、复杂计数）时，相比专注模式仍有一定差距。
*   **模式切换策略**：目前推理时的模式选择（专注或待机）仍需人工预设或根据任务类型指定，尚未实现模型自主、动态地在视频内部进行“跳读”或“精读”的智能切换。
*   **训练成本**：虽然推理高效，但两阶段的后训练过程仍需要较大的算力投入和高质量数据支撑。
