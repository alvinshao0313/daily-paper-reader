## 摘要
随着基于大语言模型（LLM）的智能体在各个领域的广泛应用，其复杂性引入了新的安全威胁。现有的红队测试方法大多依赖于修改用户提示词，这不仅缺乏对新数据的适应性，还可能影响智能体的性能。为了应对这一挑战，本文提出了 JailAgent 框架，该框架完全避免了对用户提示词的修改。具体而言，它通过三个关键阶段隐式地操纵智能体的推理轨迹和记忆检索：触发器提取（Trigger Extraction）、推理劫持（Reasoning Hijacking）和约束收紧（Constraint Tightening）。通过精确的触发器识别、实时自适应机制以及优化的目标函数，JailAgent 在跨模型和跨场景环境中展现出了卓越的性能。

## Abstract
With the widespread application of LLM-based agents across various domains, their complexity has introduced new security threats. Existing red-team methods mostly rely on modifying user prompts, which lack adaptability to new data and may impact the agent's performance. To address the challenge, this paper proposes the JailAgent framework, which completely avoids modifying the user prompt. Specifically, it implicitly manipulates the agent's reasoning trajectory and memory retrieval with three key stages: Trigger Extraction, Reasoning Hijacking, and Constraint Tightening. Through precise trigger identification, real-time adaptive mechanisms, and an optimized objective function, JailAgent demonstrates outstanding performance in cross-model and cross-scenario environments.

### 2. 方法论
JailAgent 采用了一个三阶段的流水线架构：

*   **触发器提取（Trigger Extraction）**：
    *   利用语法解析（spaCy）和 BERT 分词对输入进行语义分组。
    *   采用“由粗到细”的策略，通过计算**对数概率变化（Log-probability change）**和**步进式 KL 散度（Step-wise KL-guided importance estimation）**，识别出对模型输出贡献最大的高价值 Token 作为触发器。
*   **推理劫持（Reasoning Hijacking）**：
    *   **数据工厂（Data Factory）**：实时合成包含（上下文、毒性响应、良性响应）的三元组数据。
    *   **重排序器（Reranker）**：使用冻结的 SentenceTransformer 编码器配合轻量级评分头（MLP），通过成对排名损失（Pairwise Ranking Loss）进行快速微调，使模型在推理时倾向于选择被注入的毒性路径。
*   **约束收紧（Constraint Tightening）**：
    *   设计了四个联合优化的损失函数来约束触发器在语义空间中的特征：
        1.  **特性损失（Particularity Loss）**：使触发器远离良性数据簇。
        2.  **聚类损失（Clustering Loss）**：确保触发器 Token 的嵌入分布紧凑稳定。
        3.  **可分离性损失（Separability Loss）**：提高毒性样本在检索任务中的命中率。
        4.  **边界损失（Margin Loss）**：扩大毒性条目与良性条目之间的决策间距。
