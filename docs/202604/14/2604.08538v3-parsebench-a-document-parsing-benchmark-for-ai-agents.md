## 摘要
AI 智能体正在改变对文档解析的需求。关键在于语义正确性：解析输出必须保留自主决策所需的结构和含义，包括正确的表格结构、精确的图表数据、具有语义意义的格式化以及视觉定位（visual grounding）。现有的基准测试未能充分涵盖企业自动化中的这一场景，它们往往依赖于狭窄的文档分布和文本相似度指标，从而忽略了对智能体至关重要的失效情况。我们推出了 ParseBench，这是一个包含约 2,000 页经过人工验证的基准测试，涵盖了保险、金融和政府等领域的企业文档，并围绕五个能力维度进行组织：表格、图表、内容忠实度、语义格式化和视觉定位。通过对涵盖视觉语言模型、专用文档解析器和 LlamaParse 在内的 14 种方法进行评估，该基准测试揭示了一个碎片化的能力格局：没有任何一种方法能在所有五个维度上都表现出持续的强势。LlamaParse Agentic 获得了 84.9% 的最高综合评分，该基准测试也凸显了当前系统仍存在的能力差距。数据集和评估代码可在 https://huggingface.co/datasets/llamaindex/ParseBench 和 https://github.com/run-llama/ParseBench 获取。

## Abstract
AI agents are changing the requirements for document parsing. What matters is semantic correctness: parsed output must preserve the structure and meaning needed for autonomous decisions, including correct table structure, precise chart data, semantically meaningful formatting, and visual grounding. Existing benchmarks do not fully capture this setting for enterprise automation, relying on narrow document distributions and text-similarity metrics that miss agent-critical failures. We introduce ParseBench, a benchmark of ${\sim}2{,}000$ human-verified pages from enterprise documents spanning insurance, finance, and government, organized around five capability dimensions: tables, charts, content faithfulness, semantic formatting, and visual grounding. Across 14 methods spanning vision-language models, specialized document parsers, and LlamaParse, the benchmark reveals a fragmented capability landscape: no method is consistently strong across all five dimensions. LlamaParse Agentic achieves the highest overall score at 84.9%, and the benchmark highlights the remaining capability gaps across current systems. Dataset and evaluation code are available on https://huggingface.co/datasets/llamaindex/ParseBench and https://github.com/run-llama/ParseBench.

*   **研究动机**：随着 AI 智能体（Agents）在企业自动化（如金融、保险、政府）中的应用，传统的文档解析标准已从“人类可读”转向“可靠决策”。
*   **核心痛点**：现有的文档解析基准测试（如 PubTabNet, DocLayNet）存在两个缺陷：
    1.  **分布偏差**：多集中在学术论文或网页，缺乏高价值的企业级复杂文档。
    2.  **指标失效**：过度依赖文本相似度（如编辑距离），忽略了对智能体至关重要的“语义正确性”（如表格标题错位、图表数值错误、删除线丢失等），这些微小错误会导致智能体做出错误的决策。
*   **整体含义**：本文提出了 **ParseBench**，旨在通过多维度、语义化的评估体系，衡量解析器是否能为 AI 智能体提供“足以支持行动”的精确结构化数据。

### 2. 方法论
ParseBench 围绕五个核心维度构建，每个维度都有特定的评估逻辑：
*   **表格（Tables）**：引入 **Table Record Match (TRM)** 指标。它将表格视为记录包（Bag of Records），通过匹配“列标题-单元格值”对来评分，对列顺序不敏感，但对标题错误极其敏感。
*   **图表（Charts）**：评估**数据点匹配率**。要求解析器将图表转换为表格，并验证特定数据点（数值+标签）的准确性，允许一定的视觉估算容差。
*   **内容忠实度（Content Faithfulness）**：通过 14.7 万条规则检测文本的遗漏、幻觉和阅读顺序。
*   **语义格式化（Semantic Formatting）**：评估对具有语义意义的格式（如删除线、上标/下标、加粗标题层级）的保留能力。
*   **视觉定位（Visual Grounding）**：评估元素定位（Bounding Box）、分类和内容归属（Attribution）的综合准确率。
*   **标注流程**：采用“VLM 预标注 + 人工多轮验证”的流水线，确保 2,000 页文档的高质量标注。

### 3. 实验设计
*   **数据集规模**：包含 2,078 页人工验证的企业文档，涵盖保险（SERFF）、金融报告、政府文件等，共计 16.9 万条测试规则。
*   **对比方法（14 种）**：
    *   **通用 VLM**：GPT-5 Mini, Claude Haiku 4.5, Gemini 3 Flash。
    *   **开源/专用 VLM**：Qwen 3 VL (8B), Dots OCR 1.5。
    *   **专用文档解析器**：AWS Textract, Azure Doc Intelligence, Google Cloud Doc AI, Docling (IBM), Reducto 等。
    *   **LlamaParse**：分为 Cost Effective（成本优化型）和 Agentic（多步推理增强型）两个版本。
*   **评估场景**：跨越简单文本到具有嵌套标题、跨页表格、3D 图表等极端复杂情况。

### 4. 资源与算力
*   **算力设备**：对于开源模型（Qwen 3 VL, Dots OCR 1.5）和开源工具（Docling），实验使用了 **NVIDIA H100 GPU**。
*   **部署框架**：使用了 **vLLM** 推理框架和 **Modal** 无服务器 GPU 基础设施。
*   **商业 API**：其他方法通过官方 REST API 调用。文中未详细说明具体的训练时长，因为主要侧重于推理端的基准测试评估。

### 5. 实验数量与充分性
*   **实验规模**：对 14 种方法在 5 个维度上进行了全面测试，并进行了**质量-成本前沿（Quality-Cost Frontier）**分析。
*   **充分性**：实验涵盖了从基础 OCR 到复杂语义理解的各个层次。通过对“简单”与“困难”页面的分层分析，验证了模型在极端情况下的鲁棒性。
*   **客观性**：所有 VLM 使用标准化的 Prompt，商业 API 使用默认/推荐配置，确保了对比的公平性。

### 6. 主要结论与发现
*   **能力碎片化**：没有任何一种方法在所有维度上都表现完美。VLM 在内容理解上较强，但在图表解析和视觉定位上较弱；专用解析器在定位上表现更好，但往往忽略语义格式。
*   **LlamaParse 领先**：**LlamaParse Agentic** 以 **84.9%** 的综合得分位居第一，尤其在图表和语义格式化方面优势明显。
*   **成本不等于质量**：增加推理预算（如开启 VLM 的 Thinking 模式）收益递减。例如，Gemini 3.1 Pro 的成本是 Flash 版的 13 倍，但综合得分反而更低。
*   **图表是重灾区**：大多数传统解析器在图表维度得分接近 0，因为它们只输出原始 OCR 文本而非结构化数据。

### 7. 优点（亮点）
*   **语义化指标设计**：TRM 指标比传统的 TEDS 更符合下游 Agent 的使用逻辑。
*   **真实企业场景**：避开了简单的学术数据集，专注于保险、金融等对解析错误“零容忍”的领域。
*   **多维度覆盖**：首次将语义格式化（如删除线）和图表数据点提取纳入统一的 Agent 评估框架。
*   **开源贡献**：公开了数据集和评估框架，具有很高的复现价值。

### 8. 不足与局限
*   **样本规模限制**：虽然 2,000 页已具规模，但相比于海量的企业文档类型，覆盖范围仍有提升空间。
*   **领域偏差**：数据集偏向英语和西方企业文档，对多语言（尤其是非拉丁语系）和非西方格式的覆盖较少。
*   **指标局限性**：例如在数字检测中，基于频率的校验无法识别相同频率数字的互换（如 6 和 8 互换）。
*   **动态性不足**：基准测试是静态的，而文档格式和 VLM 模型迭代极快，需要持续更新。
