## 摘要
基于专家混合（MoE）的大语言模型（LLM）表现强劲，但面临极高的内存和计算开销。权重二值化提供了极高的效率，但现有的针对稠密 LLM 设计的二值化方法在处理 MoE 特有议题时面临挑战，包括跨专家冗余、任务无关的重要性估计以及量化引起的路由偏移。为此，我们提出了 MoBiE，这是首个专为基于 MoE 的 LLM 定制的二值化框架。MoBiE 基于三项核心创新：1. 使用联合 SVD 分解来减少跨专家冗余；2. 将全局损失梯度集成到局部 Hessian 指标中，以增强权重重要性估计；3. 引入由输入零空间引导的误差约束，以减轻路由失真。值得注意的是，MoBiE 在实现这些优化的同时不产生额外的存储开销，在效率和模型性能之间取得了平衡。广泛的实验表明，在多个基于 MoE 的 LLM 和基准测试中，MoBiE 始终优于最先进的二值化方法。例如，在 Qwen3-30B-A3B 上，MoBiE 将困惑度降低了 52.2%，将平均零样本性能提升了 43.4%，实现了超过 2 倍的推理加速，并进一步缩短了量化时间。代码可在 https://github.com/Kishon-zzx/MoBiE 获取。

## Abstract
Mixture-of-Experts (MoE) based large language models (LLMs) offer strong performance but suffer from high memory and computation costs. Weight binarization provides extreme efficiency, yet existing binary methods designed for dense LLMs struggle with MoE-specific issues, including cross-expert redundancy, task-agnostic importance estimation, and quantization-induced routing shifts. To this end, we propose MoBiE, the first binarization framework tailored for MoE-based LLMs. MoBiE is built on three core innovations: 1. using joint SVD decomposition to reduce cross-expert redundancy; 2. integrating global loss gradients into local Hessian metrics to enhance weight importance estimation; 3. introducing an error constraint guided by the input null space to mitigate routing distortion. Notably, MoBiE achieves these optimizations while incurring no additional storage overhead, striking a balance between efficiency and model performance. Extensive experiments demonstrate that MoBiE consistently outperforms state-of-the-art binary methods across multiple MoE-based LLMs and benchmarks. For example, on Qwen3-30B-A3B, MoBiE reduces perplexity by 52.2$\%$, improves average zero-shot performance by 43.4$\%$, achieves over 2 $\times$ inference speedup, and further shortens quantization time. The code is available at https://github.com/Kishon-zzx/MoBiE.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：虽然 MoE 模型通过稀疏激活提升了计算效率，但其巨大的参数量要求所有专家必须常驻内存（例如 120B 模型需 >240GB 显存），导致推理时的内存开销极高。
*   **研究动机**：现有的权重二值化（1-bit 量化）方法主要针对稠密模型设计，直接应用于 MoE 时效果极差。作者识别出 MoE 二值化的三大挑战：
    1.  **专家冗余**：独立处理各专家忽略了专家间的结构相似性，导致存储冗余。
    2.  **任务无关的重要性评估**：现有的 Hessian 指标仅关注局部重建误差，与下游任务损失脱节。
    3.  **量化诱导的路由偏移（Expert-shift）**：二值化带来的扰动会改变路由器的选择，使 Token 被派发给错误的专家，严重破坏模型性能。

### 2. 方法论：核心思想与关键技术
MoBiE 提出了三个核心组件来应对上述挑战：
*   **跨专家联合分解（CEJD）**：
    *   **核心思想**：利用专家间的相似性进行降维。
    *   **技术细节**：将同一层的所有专家权重矩阵水平拼接，进行联合 SVD 分解。提取一个共享的高精度基底（$U\Sigma$），仅对专家特有的投影矩阵（$V$）进行二值化。这在几乎不增加存储的情况下，保护了核心语义特征。
*   **全局损失对齐显著性（GLAS）**：
    *   **核心思想**：让量化过程“感知”下游任务的重要性。
    *   **技术细节**：将全局损失函数的梯度信息注入到局部 Hessian 矩阵的构造中。通过梯度加权的残差目标函数，重新定义权重的显著性得分，确保在二值化过程中优先保留对最终输出影响最大的权重。
*   **零空间引导的路由偏移抑制（NGES）**：
    *   **核心思想**：将量化误差“隐藏”在对输出无影响的方向上。
    *   **技术细节**：利用输入激活值的零空间（Null Space）特性，通过轻量级的行/列缩放向量（优化后的 $\beta_r, \beta_c$）来约束二值化误差。这些向量被融合进量化缩放因子中，不产生额外存储开销，有效缓解了路由器的选择偏差。

*   **实现复杂度**：相比于简单的线性量化，MoBiE 涉及 SVD 分解和迭代优化，量化过程的实现逻辑相对复杂。
