## 摘要
在自主软件工程（SWE）中，以往具有代表性的 ReAct 风格方法通常缺乏进行深度分析和处理复杂边缘情况所需的显式 System-2 推理。虽然近期的推理模型展示了扩展思维链（CoT）的潜力，但将其应用于多轮 SWE 任务会产生一个根本性的两难困境：保留完整的推理历史会导致上下文爆炸和“迷失在中间”（Lost-in-the-Middle）的性能退化，而丢弃历史则会迫使智能体在每一步都进行冗余的重复推理。为了应对这些挑战，我们提出了 SWE-AGILE，这是一种新型软件智能体框架，旨在弥合推理深度、效率与上下文限制之间的差距。SWE-AGILE 引入了动态推理上下文策略，通过维护详细推理的“滑动窗口”来保证即时连续性，防止冗余的重复分析，同时将历史推理内容压缩为简洁的推理摘要（Reasoning Digests）。实验表明，SWE-AGILE 仅使用 2.2k 条轨迹和 896 个任务，就在 SWE-Bench-Verified 上为 7B-8B 级模型树立了新的标杆。代码可在 https://github.com/KDEGroup/SWE-AGILE 获取。

## Abstract
Prior representative ReAct-style approaches in autonomous Software Engineering (SWE) typically lack the explicit System-2 reasoning required for deep analysis and handling complex edge cases. While recent reasoning models demonstrate the potential of extended Chain-of-Thought (CoT), applying them to the multi-turn SWE task creates a fundamental dilemma: retaining full reasoning history leads to context explosion and ``Lost-in-the-Middle'' degradation, while discarding it would force the agent to redundantly re-reason at every step. To address these challenges, we propose SWE-AGILE, a novel software agent framework designed to bridge the gap between reasoning depth, efficiency, and context constraints. SWE-AGILE introduces a Dynamic Reasoning Context strategy, maintaining a ``sliding window'' of detailed reasoning for immediate continuity to prevent redundant re-analyzing, while compressing historical reasoning content into concise Reasoning Digests. Empirically, SWE-AGILE sets a new standard for 7B-8B models on SWE-Bench-Verified using only 2.2k trajectories and 896 tasks. Code is available at https://github.com/KDEGroup/SWE-AGILE.

#### 1. 核心问题与整体含义（研究动机和背景）
在自动软件工程（SWE）任务中，智能体需要通过多轮交互（如阅读代码、运行测试、修改文件）来修复 Bug。目前存在两类主流方法：
*   **ReAct 风格方法**：仅生成浅层思考，缺乏深度分析（System-2 推理），难以处理复杂的边缘情况。
*   **长思维链（CoT）推理模型**：虽然具备深度推理能力，但在多轮任务中面临**“上下文困境”**——若保留所有历史推理，会导致上下文爆炸和“迷失在中间”的性能退化；若丢弃历史推理，智能体在每一步都必须从头开始重新分析全局状态，导致严重的计算冗余和效率低下。

**SWE-AGILE** 的提出正是为了在多轮 SWE 任务中平衡**推理深度、执行效率与上下文约束**。

#### 2. 方法论：核心思想与关键技术
SWE-AGILE 的核心思想是**“动态推理上下文”**，将上下文增长模式从线性增长转变为“锯齿状”可持续增长。

*   **动态推理上下文策略**：
    *   **滑动推理窗口**：保留最近 $N$ 步的详细推理过程（$r_t$），确保认知的连续性，避免重复分析。
    *   **推理摘要（Reasoning Digests）**：将窗口之外的历史推理内容压缩为简洁的摘要（$d_t$），仅保留关键决策信息。
    *   **响应结构**：模型每一步生成的输出 $Y_t$ 包含三个部分：详细推理 ($r_t$) $\rightarrow$ 推理摘要 ($d_t$) $\rightarrow$ 执行动作 ($a_t$)。
*   **轨迹快照训练（Trajectory Snapshot Training）**：
    *   为了解决训练与推理时的上下文不一致（推理时历史推理被隐藏，而标准训练通常可见），将长轨迹分解为离散的“快照”。每个快照只对当前目标进行损失计算，并对历史推理进行掩码处理。
*   **事后回填数据合成（Hindsight Backfill）**：
    *   利用强推理模型（如 Qwen3-235B）对成功的原始轨迹进行标注，根据“未来”的正确动作回填深度推理和摘要，从而获得高质量的 SFT 数据。
*   **压缩感知优化（RLVR with Compression Reward）**：
    *   在强化学习阶段（RLVR），引入**轨迹级压缩率奖励**。在保证任务成功的前提下，奖励模型生成更短的摘要，从而在 Deliberative（深思熟虑型）步骤中保持深度，在 Reflexive（反射型）步骤中保持简洁。
