## 摘要
我们引入了 Agent^2 RL-Bench，这是一个用于评估智能体强化学习（RL）后训练的基准测试——即大语言模型（LLM）智能体能否自主设计、实现并运行完整的 RL 流水线，以提升基础模型的能力。这一能力至关重要，因为 RL 后训练正日益成为驱动模型对齐和专业化的核心，然而现有的基准测试在很大程度上仍是静态的：仅靠监督微调（SFT）就能取得显著成果，导致交互式 RL 工程化能力未得到测试。Agent^2 RL-Bench 通过涵盖三个层级的六个任务解决了这一问题——从静态的基于规则的训练到带有轨迹采集的闭环在线 RL——每个层级都增加了前一层级所不具备的结构性要求。该基准测试提供了带有评分 API 的隔离工作区、记录每次提交和代码修订的运行时检测工具，以及生成结构化运行报告的自动化事后分析，从而实现了对智能体驱动的后训练行为的首次自动化诊断。通过对涵盖 5 个智能体系统和 6 个驱动 LLM 的多个智能体堆栈进行测试，我们发现智能体取得了显著的交互式增益——在 ALFWorld 上，一个纯 RL 智能体通过 SFT 预热和带有在线采样的 GRPO，性能从 5.97 提升至 93.28——但在其他任务上仅取得微小进展（DeepSearchQA：+2.75，处于评估噪声范围内）；此外，驱动模型的选择对交互式任务有巨大影响——在相同的框架下，更换驱动模型可使交互式改进从接近零变为提升 78 个百分点。更广泛地说，该基准测试揭示了在固定预算下，监督流水线在智能体驱动的后训练中占据主导地位，而在线 RL 仅在 ALFWorld 上作为最终最佳路径取得了成功。代码可在 https://github.com/microsoft/RD-Agent/tree/main/rdagent/scenarios/rl/autorl_bench 获取。

## Abstract
We introduce Agent^2 RL-Bench, a benchmark for evaluating agentic RL post-training -- whether LLM agents can autonomously design, implement, and run complete RL pipelines that improve foundation models. This capability is important because RL post-training increasingly drives model alignment and specialization, yet existing benchmarks remain largely static: supervised fine-tuning alone yields strong results, leaving interactive RL engineering untested. Agent^2 RL-Bench addresses this with six tasks across three levels -- from static rule-based training to closed-loop online RL with trajectory collection -- each adding a structural requirement that prior levels do not impose. The benchmark provides isolated workspaces with a grading API, runtime instrumentation that records every submission and code revision, and automated post-hoc analysis that generates structured run reports, enabling the first automated diagnostic of agent-driven post-training behavior. Across multiple agent stacks spanning five agent systems and six driver LLMs, we find that agents achieve striking interactive gains -- on ALFWorld, an RL-only agent improves from 5.97 to 93.28 via SFT warm-up and GRPO with online rollouts -- yet make only marginal progress on others (DeepSearchQA: +2.75 within evaluation noise), and that driver choice has a large effect on interactive tasks -- within the same scaffold, switching drivers changes interactive improvement from near-zero to +78pp. More broadly, the benchmark reveals that supervised pipelines dominate agent-driven post-training under fixed budgets, with online RL succeeding as the final best route only on ALFWorld. Code is available at https://github.com/microsoft/RD-Agent/tree/main/rdagent/scenarios/rl/autorl_bench.

### 2. 方法论
*   **核心思想**：通过三个递进的结构化层级（Levels）来测试智能体，从简单的静态任务过渡到复杂的闭环交互式 RL 工程。

    *   **三层级结构**：
        *   **L1 (静态规则)**：基于确定性规则验证（如 GSM8K, HumanEval），测试基础数据构建和 SFT。
        *   **L2 (静态法官)**：引入非确定性信号（如 AlpacaEval），要求针对 LLM Judge 进行优化。
        *   **L3 (交互式采样)**：核心挑战，要求智能体实现环境交互、轨迹采集和闭环在线 RL（如 ALFWorld, WebShop）。
    *   **基础设施**：
        *   **隔离工作区**：为智能体提供只读的基础模型和数据，以及可写的代码/输出目录。
        *   **评分 API**：允许智能体在 12 小时预算内多次提交模型进行评估，模拟真实的“开发-测试-迭代”循环。
        *   **运行时监测与诊断**：记录每一次代码修订和提交，自动生成结构化的事后分析报告（Run Reports），用于诊断智能体的失败模式。

### 6. 主要结论与发现
*   **交互式任务是分水岭**：智能体在 ALFWorld 上表现惊人（通过 GRPO 从 5.97 提升至 93.28），但在 DeepSearchQA 等任务上几乎停滞。
*   **驱动模型至关重要**：在相同框架下，仅更换驱动模型（如从 GPT-4o 换成 GPT-5.2），ALFWorld 的改进幅度可从 0.74pp 飙升至 78.35pp。
*   **SFT 仍是首选策略**：在固定预算下，智能体更倾向于构建复杂的监督学习流水线（SFT），而非不稳定的在线 RL。RL 仅在 ALFWorld 任务中证明是最终的最佳路径。
*   **对齐差距闭合**：强力智能体堆栈能显著提升未对齐基础模型的性能，使其接近甚至超过 Instruct 版本的表现。

### 7. 优点
*   **诊断性设计**：通过自动化报告揭示了智能体“如何失败”，而不仅仅是“是否失败”。
*   **结构化层级**：明确区分了静态自动化和动态 RL 工程，暴露了现有基准测试无法检测到的能力短板。
*   **真实工程模拟**：引入了多次提交和时间预算的概念，非常贴合实际的 AI 研发场景。

### 8. 不足与局限
*   **评估噪声**：DeepSearchQA 等任务的评估方差较大（±3pp），导致智能体难以通过微小的性能反馈进行有效迭代。
*   **实验覆盖度**：受限于成本，并非所有“框架-模型”组合都进行了全因子测试，且部分实验缺乏多次重复的统计显著性。
*   **内容过滤干扰**：实验中提到 Azure 的内容过滤器偶尔会干扰 AlpacaEval 的法官评分，可能对结果产生轻微偏差。
*   **任务局限**：目前仅涵盖 6 个任务，对于更复杂的 RL 场景（如多智能体博弈、超长程规划）尚待扩展。
