## 摘要
视觉语言模型 (VLM) 通过联合处理视觉和文本信息，彻底改变了多模态学习。然而，由于处理长序列视觉 Token 对计算和内存的高需求，它们面临着重大挑战。许多现有方法依赖于局部启发式方法，例如注意力分数或 Token 范数。然而，这些标准存在位置偏差和信息分散的问题，限制了它们在高剪枝率下保留核心内容的能力，并导致在视觉细节丰富的图像上性能下降。为了解决这些问题，我们提出了 SVD-Prune，这是一种基于奇异值分解 (Singular Value Decomposition) 的无需训练、即插即用的 Token 剪枝方法。它对视觉 Token 特征矩阵进行分解，并使用统计杠杆得分 (statistical leverage scores) 选择前 K 个 Token，从而确保仅保留对主导全局方差贡献最大的 Token。实验表明，在极端的视觉 Token 预算下，SVD-Prune 始终优于先前的剪枝方法，即使在仅剩 32 个和 16 个视觉 Token 的情况下也能保持强大的性能。

## Abstract
Vision-Language Models (VLM) have revolutionized multimodal learning by jointly processing visual and textual information. Yet, they face significant challenges due to the high computational and memory demands of processing long sequences of vision tokens. Many existing methods rely on local heuristics, such as attention scores or token norms. However, these criteria suffer from positional bias and information dispersion, limiting their ability to preserve essential content at high pruning ratios and leading to performance degradation on visually detailed images. To address these issues, we propose SVD-Prune, a trainingfree, plug-and-play token pruning method based on Singular Value Decomposition. It decomposes the vision token feature matrix and selects the top-K tokens using statistical leverage scores, ensuring only tokens contributing most to the dominant global variance are preserved. Experiments show that SVD-Prune consistently outperforms prior pruning methods under extreme vision token budgets, maintaining strong performance even with 32 and 16 vision tokens.

### 1. 核心问题与研究动机
*   **核心问题**：视觉语言模型（VLM）在处理图像时会产生大量的视觉 Token（如 LLaVA 默认产生 576 个），这导致了巨大的计算开销和内存占用，限制了其在边缘设备上的部署。
*   **研究背景**：现有的 Token 剪枝方法多依赖于注意力分数、Token 范数或跨模态相似度等局部启发式准则。
*   **研究动机**：作者发现现有方法存在两大缺陷：
    1.  **位置偏差**：由于 LLM 解码器的因果掩码（Causal Masking），注意力分数受 Token 位置影响严重，而非完全取决于语义重要性。
    2.  **信息分散**：局部准则难以捕捉全局视觉结构，导致在高剪枝率（极低 Token 预算）下性能大幅下降。

### 2. 方法论（SVD-Prune）
该方法是一种**无需训练、即插即用**的视觉 Token 剪枝技术，核心思想是利用**奇异值分解（SVD）**从全局方差的角度识别重要信息。其流程分为四个阶段：
1.  **全局模式提取**：对视觉编码器输出的特征矩阵 $F$ 进行 SVD 分解（$F = U\Sigma V^\top$）。$U$ 矩阵的列代表了 Token 空间的主要方向。
2.  **主导方差截断**：计算奇异值的平方和（方差），选择保留前 $k$ 个奇异值，使其累积解释方差达到预设阈值 $\epsilon$（通常在 0.7 到 0.95 之间）。这能过滤掉噪声和冗余信息。
3.  **统计杠杆得分计算**：利用前 $k$ 个左奇异向量计算每个 Token 的**杠杆得分（Leverage Score）**。得分越高，说明该 Token 对全局主导模式的贡献越大。
4.  **Token 筛选与重排**：根据得分保留前 $m$ 个 Token，并将其恢复到原始的视觉空间顺序，以保持位置编码的连贯性，最后输入 LLM 解码器。

*   **对比方法**：涵盖了编码器端剪枝（ToMe, HIRED, VisionZip, TRIM）和解码器端剪枝（FastV, PyramidDrop, SparseVLM）等多种主流方法。
*   **剪枝预算**：将原始 576 个 Token 分别压缩至 192、128、64、32 甚至 16 个。

### 6. 主要结论与发现
*   **极端压缩下的鲁棒性**：在仅保留 16 或 32 个 Token 时，SVD-Prune 的性能显著优于其他方法。例如在 32 个 Token 下，GQA 仅下降 8.38 分，TextVQA 仅下降 3.39 分。

### 7. 优点
*   **无需训练**：作为插件即可使用，无需昂贵的微调过程，适配性强。
*   **克服偏差**：通过 SVD 全局分解，有效规避了 LLM 注意力机制中的位置偏差和“注意力汇聚（Attention Sink）”效应。
