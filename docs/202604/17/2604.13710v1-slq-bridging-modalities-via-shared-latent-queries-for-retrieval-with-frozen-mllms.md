## 摘要
多模态大语言模型（MLLMs）展现出强大的推理能力和世界知识，但将其适配于检索任务仍然具有挑战性。现有方法依赖于侵入性的参数更新，如全量微调和 LoRA，这可能会破坏预训练的语义空间，并损害对推理至关重要的结构化知识。在这项工作中，我们认为将 MLLM 适配于检索应侧重于诱导预训练表示，而非重写它们。为此，我们提出了 SLQ，这是一个高效且有效的框架，通过一小组共享潜查询（Shared Latent Queries）将冻结的 MLLM 适配为检索器。这些查询被附加在文本和图像标记序列的末尾，利用模型原生的因果注意力作为全局聚合接口，在保持主干网络不变的情况下，在统一空间中生成紧凑的嵌入。此外，为了更好地评估超越表层模式匹配的检索能力，我们构建了 KARR-Bench，这是一个专为知识感知推理检索设计的基准测试。广泛的实验表明，SLQ 在 COCO 和 Flickr30K 上的表现优于全量微调和 LoRA，同时在 MMEB 上取得了具有竞争力的性能，并在 KARR-Bench 上获得了显著提升。结果表明，SLQ 在保留预训练表示的同时，为将 MLLM 适配到检索任务提供了一个高效且有效的框架。

## Abstract
Multimodal Large Language Models (MLLMs) exhibit strong reasoning and world knowledge, yet adapting them for retrieval remains challenging. Existing approaches rely on invasive parameter updates, such as full fine-tuning and LoRA, which may disrupt the pre-trained semantic space and impair the structured knowledge essential for reasoning. In this work, we argue that adapting MLLMs for retrieval should focus on eliciting pre-trained representations rather than overwriting them. To this end, we propose SLQ, an effective and efficient framework that adapts a frozen MLLM into a retriever through a small set of Shared Latent Queries. Appended to the end of both text and image token sequences, these queries leverage the model's native causal attention to serve as global aggregation interfaces, producing compact embeddings in a unified space while keeping the backbone unchanged. Furthermore, to better evaluate retrieval beyond superficial pattern matching, we construct KARR-Bench, a benchmark designed for knowledge-aware reasoning retrieval. Extensive experiments show that SLQ outperforms full fine-tuning and LoRA on COCO and Flickr30K, while achieving competitive performance on MMEB and yielding substantial gains on KARR-Bench. The results demonstrate that SLQ, which preserves pre-trained representations, provides an effective and efficient framework for adapting MLLMs to retrieval.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：如何高效地将多模态大语言模型（MLLM）转化为高性能的检索器，同时不损害其原有的推理能力和世界知识？
*   **研究背景**：现有的 MLLM 检索适配方法（如全量微调或 LoRA）属于“侵入式”更新。这种做法存在两个弊端：
    1.  **语义退化**：对比学习目标与生成式预训练目标不匹配，剧烈的参数更新会扭曲预训练的语义空间，导致“灾难性遗忘”。
    2.  **训练低效**：微调数十亿参数的模型需要巨大的计算开销和极大的 Batch Size。
*   **核心主张**：检索适配不应是“重写”模型，而应是“诱导（Elicit）”模型内部已有的对齐表示。

### 2. 论文提出的方法论：SLQ 框架
*   **核心思想**：保持 MLLM 主干网络完全**冻结**，通过引入一小组可学习的**共享潜查询（Shared Latent Queries, SLQ）**来聚合信息。
*   **关键技术细节**：
    *   **共享查询**：在图像和文本序列的末尾分别附加 $N$ 个（实验中 $N=20$）可学习的 Token。这些 Token 在不同模态间共享，旨在将它们投影到统一的嵌入空间。
    *   **因果注意力聚合**：利用 Decoder-only 模型原生的因果注意力机制，让末尾的查询 Token 能够“看到”前面所有的图像或文本信息，从而充当全局聚合接口。
    *   **嵌入提取与对齐**：提取最后层中查询 Token 对应的隐藏状态，进行均值池化（Mean Pooling）并归一化，得到最终的检索向量。
    *   **训练目标**：仅优化这组查询 Token 和温度参数 $\tau$，使用对称的 **InfoNCE 损失函数**进行对比学习。

### 3. 实验设计
*   **数据集与基准**：
    *   **标准检索**：MS-COCO (5K test)、Flickr30K (1K test)。
    *   **通用多模态嵌入**：MMEB（包含 36 个任务，涵盖分类、VQA、检索等）。
    *   **新基准 KARR-Bench**：作者专门构建了“知识感知推理检索基准”，包含 2,915 个图像-文本对，要求模型通过逻辑推理（如“2+7 条命的动物”指代猫）而非简单的视觉模式匹配进行检索。
*   **对比方法**：
    *   双塔模型：CLIP, BLIP, SigLIP, FLAME。
    *   基于 MLLM 的检索器：E5-V, VLM2VEC, MM-Embed, MMRet 等。
    *   微调策略对比：全量微调（Full FT）、LoRA。

### 4. 资源与算力
*   **硬件环境**：使用了 **NVIDIA H800 GPU** 集群。
*   **训练时长与效率**（以 InternVL3-8B 为例）：
    *   **SLQ**：仅需 **38.9 GPU 小时**（16 张 H800），可学习参数仅 **144K**。
    *   **LoRA**：需要 130.1 GPU 小时，参数量 20.2M。
    *   **全量微调**：需要 403.8 GPU 小时。
*   **结论**：SLQ 相比 LoRA 节省了约 70% 的训练时间，参数量降低了两个数量级。

### 5. 实验数量与充分性
*   **实验规模**：非常充分。涵盖了从 1B 到 8B 不同规模的多个主干网络（InternVL3, Qwen3-VL）。
*   **消融实验**：详细探讨了查询 Token 数量（1 到 32 的影响）、提取策略（线性头 vs 共享查询）、池化方式（Max vs Mean）以及不同 PEFT 方法（Prompt Tuning, Adapter）的对比。
*   **客观性**：通过 PCA 降维可视化了模态间隙（Modality Gap），并定量分析了对齐度（Alignment）和均匀度（Uniformity），证明了 SLQ 在保持表示空间紧凑性方面的优势。

### 6. 主要结论与发现
*   **性能优越**：SLQ 在 COCO 和 Flickr30K 上优于全量微调和 LoRA，在 MMEB 上达到了 SOTA 水平。
*   **能力保留**：在适配检索任务后，SLQ 几乎没有损失 MLLM 原有的 VQA 和推理能力（在 MMMU、OCRBench 等测试中表现稳定），而微调方法则出现了明显的性能下降。
*   **推理优势**：在 KARR-Bench 上，SLQ 的领先优势随模型规模增大而扩大，证明了冻结主干对保留复杂推理链条至关重要。

### 7. 优点（亮点）
*   **非侵入式设计**：极大地降低了适配门槛，保护了预训练模型的“大脑”不受对比学习目标的干扰。
*   **极高效率**：参数量极小（K 级别），训练速度极快，适合资源受限的场景。
*   **高质量基准**：KARR-Bench 的提出填补了多模态检索中“深层推理”评估的空白，其构建过程包含人工校验，质量较高。

### 8. 不足与局限
*   **域外适配限制**：由于主干网络完全冻结，对于极端的域外数据（如专业的医学影像、卫星遥感图像），SLQ 可能无法学习到新的视觉特征，此时可能需要结合轻量级视觉适配器。
*   **训练目标单一**：目前仅在 InfoNCE 损失下验证，尚未探索三元组损失（Triplet Loss）或知识蒸馏等其他训练范式。
*   **长文本处理**：虽然查询 Token 放在末尾，但对于超长文本，因果注意力的压缩能力是否存在瓶颈尚待进一步研究。
