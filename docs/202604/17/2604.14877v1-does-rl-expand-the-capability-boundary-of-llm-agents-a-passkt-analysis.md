## 摘要
强化学习是真正扩展了大语言模型（LLM）智能体的能力，还是仅仅提高了它们的可靠性？对于静态推理，最近的研究给出了后者的答案：基础模型与强化学习模型的 pass@k 曲线在大 k 值下趋于收敛。我们探究这一结论是否适用于智能体工具使用场景，因为在这种场景下，T 轮交互能够实现重采样无法恢复的组合策略。我们引入了 PASS@(k,T)，这是一种同时改变采样预算 k 和交互深度 T 的二维指标，旨在将能力扩展与效率提升区分开来。我们的主要发现是，与静态推理的结果相反，工具使用中的强化学习真正扩大了能力边界：强化学习智能体的通过率曲线高于基础模型，且差距在大 k 值下逐渐扩大而非收敛。这种扩展专门针对组合式、序列化的信息收集任务；在更简单的任务上，强化学习的表现与先前研究的预测一致。在训练数据匹配的情况下，有监督微调在同样的组合任务上反而导致了边界退化，从而确定了自主探索是能力提升的因果因素。机制分析表明，强化学习重新分配了基础策略分布的权重，使其向那些下游推理更常产生正确答案的子集倾斜，改进主要集中在智能体如何整合检索到的信息。这些结果调和了对 LLM 强化学习的乐观与悲观解读：针对不同的任务类型，两者都是正确的。

## Abstract
Does reinforcement learning genuinely expand what LLM agents can do, or merely make them more reliable? For static reasoning, recent work answers the second: base and RL pass@k curves converge at large k. We ask whether this holds for agentic tool use, where T rounds of interaction enable compositional strategies that re-sampling cannot recover. We introduce PASS@(k,T), a two-dimensional metric that jointly varies sampling budget k and interaction depth T, separating capability expansion from efficiency improvement. Our main finding is that, contrary to the static-reasoning result, tool-use RL genuinely enlarges the capability boundary: the RL agent's pass-curve pulls above the base model's and the gap widens at large k rather than converging. The expansion is specific to compositional, sequential information gathering; on simpler tasks RL behaves as prior work predicts. Under matched training data, supervised fine-tuning regresses the boundary on the same compositional tasks, isolating self-directed exploration as the causal factor. Mechanism analysis shows RL reweights the base strategy distribution toward the subset whose downstream reasoning more often yields a correct answer, with the improvement concentrated on how the agent integrates retrieved information. These results reconcile optimistic and pessimistic readings of RL for LLMs: both are correct, on different task types.

### 1. 研究动机与核心问题
*   **核心问题**：强化学习是真正扩展了 LLM 智能体的“能力边界”（即模型在理论上能解决的问题范围），还是仅仅提高了其解决已知问题的“效率/可靠性”？
*   **研究背景**：在静态数学推理领域，近期研究（如针对 DeepSeek-R1 的分析）认为 RL 只是重新分配了概率质量，并未增加模型在无限采样下能解决的问题数。
*   **研究假设**：作者认为在“智能体工具使用”场景下，结论可能不同。因为智能体与环境的 $T$ 轮交互允许产生复杂的组合策略，这是单纯增加采样次数 $k$ 无法替代的。

### 2. 方法论：PASS@(k, T) 框架
*   **核心思想**：引入二维评估指标 **PASS@(k, T)**，将智能体的能力分解为两个维度：
    *   **$k$（采样预算）**：独立尝试的次数，用于衡量采样效率和可靠性。
    *   **$T$（交互深度）**：每轮尝试中允许与环境交互的最大步数，用于衡量处理复杂组合任务的能力。
*   **关键定义**：
    *   **能力边界 (Capability Boundary)**：在 $k \to \infty$ 时，模型至少能解决一次的问题集合。
    *   **能力扩展 (Capability Expansion)**：RL 模型解决了一些基座模型在任何采样预算下都无法解决的问题。
    *   **效率提升 (Efficiency Improvement)**：RL 提高了模型解决基座模型本就能解决的问题的概率。
*   **训练算法**：使用 **GRPO**（群组相对策略优化）进行强化学习，并与相同数据量下的 **SFT**（有监督微调）进行严格对比。

### 3. 实验设计
*   **模型**：以 Qwen2.5-7B-Instruct 为基座。
*   **任务分类**：
    *   **Category A (MATH-500)**：纯数学推理，不使用工具（对照组）。
    *   **Category B (HotPotQA Comparison)**：简单对比任务，需要两次独立的检索。
    *   **Category C (HotPotQA Bridge)**：复杂组合任务，第二次检索依赖第一次检索的结果（核心测试组）。
*   **对比方法**：基座模型 ($\pi_{base}$)、匹配数据的 SFT 模型 ($\pi_{SFT}$)、强化学习模型 ($\pi_{RL}$) 以及带探索奖励的 RL 变体 ($\pi_{RL+explore}$)。
*   **评测设置**：每个问题在每个 $T$ 深度下进行 $n=64$ 次独立采样。

### 4. 资源与算力
*   **硬件**：使用单张 **NVIDIA RTX 6000 Ada (48GB)** GPU。
*   **训练时长**：$\pi_{RL}$ 训练 1000 步（10 个 epoch），耗时约 **55 小时**。SFT 训练非常快（约 12 分钟）。
*   **推理成本**：使用 vLLM 加速，全量评估耗时约 8 个 GPU 小时。

### 5. 实验数量与充分性
*   **实验规模**：涵盖 3 大类任务，每类 100 个测试题，总计产生数万条轨迹。
*   **充分性分析**：
    *   **消融实验**：对比了 SFT 与 RL 在相同数据下的表现，证明了“探索”而非“数据暴露”是关键。
    *   **机制分析**：通过困惑度（Perplexity）分解、策略多样性测量和跨策略交换（Cross-policy swapping）深入探究了模型改进的来源。
    *   **统计显著性**：通过 Bootstrap 自举法计算了 95% 置信区间，并进行了子样本稳定性校验（$n=32, 48$），证明 $n=64$ 的采样足以支撑结论。

### 6. 主要结论与发现
*   **RL 确实扩展了能力边界**：在复杂的组合任务（Category C）中，RL 模型的 pass 曲线随 $k$ 增大而持续领先基座模型，且差距在右尾（大 $k$ 处）扩大，这与静态推理中“曲线收敛”的现象截然相反。
*   **SFT 的负面效应**：在相同数据下，SFT 反而导致了能力边界的退化（在 Category C 中净损失 4 个问题），证明了模仿专家轨迹可能抑制模型的自主探索。
*   **机制揭秘（Reweighting）**：RL 并没有创造全新的搜索策略，而是**重新加权**了基座模型已有的策略分布，使其向那些能产生正确推理的子集倾斜。改进核心在于“如何整合检索信息”，而非“搜索什么”。
*   **任务依赖性**：在简单任务或纯推理任务上，RL 的表现符合“仅提升效率”的预期；只有在需要序列化信息整合的智能体任务中，RL 才展现出能力扩展。

### 7. 优点与亮点
*   **维度创新**：首次提出 PASS@(k, T) 二维指标，清晰地解耦了采样可靠性与交互深度对智能体能力的贡献。
*   **严谨的对比**：通过匹配 SFT 和 RL 的训练数据，排除了知识获取（Data Exposure）的干扰，孤立出学习信号（探索 vs 模仿）的影响。
*   **调和矛盾**：成功解释了为什么 RL 在数学推理中看似平庸，但在智能体任务中表现卓越，为 LLM 训练提供了理论指导。

### 8. 不足与局限
*   **模型规模限制**：主要实验基于 7B 模型，尚未在大规模模型（如 70B+）上验证该规律的缩放效应。
*   **检索环境单一**：使用了确定性的 BM25 检索而非真实的互联网环境，可能简化了环境反馈的复杂性。
*   **训练数据量较小**：仅使用了 200 个问题的微型训练集，虽然足以证明原理，但未探索大规模强化学习下的边界动态。
*   **泛化性**：尚未测试在 $T > T_{train}$（推理时交互深度超过训练深度）时的外推能力。
