## 摘要
自主 AI 智能体正迅速从实验工具转变为运营基础设施，预计到 2026 年底，80% 的企业级应用将嵌入 AI 副驾驶（copilots）。随着智能体获得执行现实世界动作的能力（如读取文件、运行命令、发起网络请求、修改数据库），一个根本性的安全漏洞也随之出现。目前主流的智能体安全方法依赖于提示词级护栏（prompt-level guardrails），即在与它们试图缓解的威胁相同的抽象层级上运行的自然语言指令。本文认为，对于具备执行能力的智能体而言，基于提示词的安全性在架构上是不充分的，并据此提出了 Parallax。这是一种基于四个原则的安全自主 AI 执行范式：认知-执行分离（Cognitive-Executive Separation），从结构上防止推理系统执行动作；具有分级确定性的对抗性验证（Adversarial Validation with Graduated Determinism），在推理和执行之间插入一个独立的、多层级的验证器；信息流控制（Information Flow Control），通过智能体工作流传播数据敏感度标签，以检测与上下文相关的威胁；以及可逆执行（Reversible Execution），捕获破坏前的状态，以便在验证失败时实现回滚。我们展示了 OpenParallax，这是一个使用 Go 语言编写的开源参考实现，并使用“假设受损评估”（Assume-Compromise Evaluation）对其进行评估，该方法通过完全绕过推理系统来测试智能体在完全受损情况下的架构边界。在涵盖 9 个攻击类别的 280 个对抗性测试案例中，Parallax 在默认配置下拦截了 98.9% 的攻击且误报率为零，在最高安全配置下拦截了 100% 的攻击。当推理系统受损时，提示词级护栏无法提供任何保护，因为它们仅存在于受损系统内部；而 Parallax 的架构边界则依然稳固。

## Abstract
Autonomous AI agents are rapidly transitioning from experimental tools to operational infrastructure, with projections that 80% of enterprise applications will embed AI copilots by the end of 2026. As agents gain the ability to execute real-world actions (reading files, running commands, making network requests, modifying databases), a fundamental security gap has emerged. The dominant approach to agent safety relies on prompt-level guardrails: natural language instructions that operate at the same abstraction level as the threats they attempt to mitigate. This paper argues that prompt-based safety is architecturally insufficient for agents with execution capability and introduces Parallax, a paradigm for safe autonomous AI execution grounded in four principles: Cognitive-Executive Separation, which structurally prevents the reasoning system from executing actions; Adversarial Validation with Graduated Determinism, which interposes an independent, multi-tiered validator between reasoning and execution; Information Flow Control, which propagates data sensitivity labels through agent workflows to detect context-dependent threats; and Reversible Execution, which captures pre-destructive state to enable rollback when validation fails. We present OpenParallax, an open-source reference implementation in Go, and evaluate it using Assume-Compromise Evaluation, a methodology that bypasses the reasoning system entirely to test the architectural boundary under full agent compromise. Across 280 adversarial test cases in nine attack categories, Parallax blocks 98.9% of attacks with zero false positives under its default configuration, and 100% of attacks under its maximum-security configuration. When the reasoning system is compromised, prompt-level guardrails provide zero protection because they exist only within the compromised system; Parallax's architectural boundary holds regardless.

*   **整体含义**：论文提出了一种名为 **Parallax** 的新范式，主张将“思考”（推理）与“行动”（执行）在架构上完全解耦，建立一个独立于 AI 推理能力的物理安全边界。

### 2. 方法论：核心思想与关键技术
Parallax 范式由四大核心原则组成，并体现在其开源参考实现 **OpenParallax** 中：
*   **认知-执行分离 (Cognitive-Executive Separation, CES)**：
    *   推理进程（Agent）运行在无权限的沙箱中，无法直接访问文件或网络。
    *   执行进程（Engine）负责具体动作，两者通过受限的 gRPC 接口通信。
*   **分级确定性的对抗性验证 (Adversarial Validation, AV)**：引入名为 **Shield** 的四层验证机制：
    *   **Tier 0 (确定性策略)**：基于 YAML 的硬编码规则（如禁止访问 SSH 密钥）。
    *   **Tier 1 (双重分类器)**：结合启发式模式匹配与微调后的 DeBERTa 分类器，检测混淆命令。
    *   **Tier 2 (预算受限的 LLM 评估)**：使用一个独立的、无上下文的 LLM 对可疑动作进行二次审判，并引入“金丝雀令牌”防止评估器被注入。
    *   **Tier 3 (人工介入)**：最终的兜底环节。
*   **信息流控制 (Information Flow Control, IFC)**：为数据打上敏感度标签（如 PUBLIC, RESTRICTED），追踪数据从读取到输出的全过程，防止智能体读取敏感文件后通过网络请求外泄。
*   **可逆执行 (Reversible Execution)**：通过 **Chronicle** 系统在执行破坏性动作（如删除、修改）前捕获状态快照，支持一键回滚。

### 3. 实验设计
*   **评估方法**：提出 **假设受损评估 (Assume-Compromise Evaluation)**。该方法不测试 LLM 是否会拒绝恶意请求，而是直接跳过推理层，将恶意工具调用注入执行边界，模拟 Agent 已被完全控制的最极端情况。
*   **数据集/场景**：
    *   **对抗性案例**：280 个测试用例，涵盖 9 大攻击类别（直接/间接注入、多步操纵、工具链攻击、编码混淆、多智能体攻击、针对验证器的攻击等）。
    *   **误报测试**：50 个合法的开发工作流动作（如 `npm install`）。
*   **对比基准 (Benchmark)**：

    *   **Config C**：Parallax 完整架构（默认配置与最高安全配置）。

### 4. 资源与算力
*   **模型使用**：Tier 1 使用了在 ONNX 运行时部署的 **DeBERTa** 分类器；Tier 2 评估使用了 **Claude Haiku 4.5**。

### 5. 实验数量与充分性
*   **实验规模**：共进行了 337 组实验（280 对抗 + 50 误报 + 7 合法人工介入案例）。
*   **充分性与客观性**：实验设计非常**充分且客观**。通过“假设受损”模式，排除了不同 LLM 自身安全对齐水平的干扰，直接测试了架构本身的鲁棒性。消融实验展示了不同验证层（Tier 0-3）对攻击拦截的贡献比例，证明了多层防御的必要性。

### 6. 主要结论与发现
*   **极高的拦截率**：在默认配置下，Parallax 拦截了 **98.9%** 的攻击，且**误报率为 0**。在最高安全配置下，拦截率达到 **100%**。
*   **提示词防御的脆弱性**：实验证明，当推理系统被攻破时，基于提示词的护栏无法提供任何保护，而 Parallax 的架构边界依然稳固。
*   **确定性防御的价值**：超过 73% 的攻击在 Tier 0 和 Tier 1（非 LLM 层）就被成功拦截，证明了传统安全原则在 AI 时代的有效性。

### 7. 优点
*   **架构级创新**：跳出了“用 AI 监督 AI”的怪圈，引入了操作系统级的权限隔离思想。
*   **动态工具加载**：通过“动态工具表面缩减”（DTSR），智能体在特定任务中只能看到必要的工具，从源头上缩小了攻击面。
*   **可回滚性**：Chronicle 系统为自主执行提供了“后悔药”，降低了误报带来的业务风险。
*   **语言/模型无关性**：该范式适用于任何推理引擎，具有极强的通用性。

### 8. 不足与局限
*   **外部副作用不可逆**：对于发送邮件、调用第三方支付 API 等外部动作，快照回滚机制无法生效。
*   **脚本内容不透明**：Shield 目前主要检查命令本身（如 `python script.py`），但对脚本内部生成的恶意逻辑静态分析不足。
*   **性能开销**：Tier 1 的分类器调用和 Tier 2 的二次 LLM 验证增加了执行延迟，可能不适用于对实时性要求极高的场景。
*   **分类器偏差**：DeBERTa 分类器在处理包含复杂结构化文本的合法请求时，仍存在一定的误报风险（最高安全模式下误报率达 36%）。
