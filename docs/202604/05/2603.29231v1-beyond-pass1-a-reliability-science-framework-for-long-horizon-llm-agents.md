## 摘要
现有基准测试衡量的是能力——即模型在单次尝试中是否成功——但生产部署需要的是可靠性，即在不同时长的任务中多次尝试的一致成功。我们表明，随着任务时长的增加，这些属性会发生系统性偏离，而短任务上的 pass@1 在结构上无法察觉这种偏离。我们为长程 LLM 智能体引入了一个可靠性科学框架，包含四个指标：可靠性衰减曲线 (RDC)、方差放大因子 (VAF)、平滑退化得分 (GDS) 和崩溃起始点 (MOP)。我们在一个包含 396 个任务的基准上评估了 10 个模型，涵盖 23,392 个回合，跨越四个时长区间和三个领域。主要发现：(1) 可靠性衰减具有领域分层特性——软件工程 (SE) 的 GDS 从 0.90 下降到 0.44，而文档处理则几乎持平（0.74 到 0.71）；(2) VAF 按能力层级分化——高 VAF 是能力的特征，而非不稳定的信号；(3) 能力和可靠性排名存在显著差异，在长程任务中出现了多位次的排名倒置；(4) 前沿模型的崩溃率最高（高达 19%），因为它们尝试了雄心勃勃的多步策略，而这些策略有时会陷入恶性循环；(5) 记忆支架普遍损害了所有 10 个模型的长程性能。这些结果促使我们将可靠性作为与能力并列的一等评估维度。

## Abstract
Existing benchmarks measure capability -- whether a model succeeds on a single attempt -- but production deployments   require reliability -- consistent success across repeated attempts on tasks of varying duration. We show these   properties diverge systematically as task duration grows, and that pass@1 on short tasks is structurally blind to   this divergence.   We introduce a reliability science framework for long-horizon LLM agents with four metrics: Reliability Decay Curve   (RDC), Variance Amplification Factor (VAF), Graceful Degradation Score (GDS), and Meltdown Onset Point (MOP). We   evaluate 10 models across 23,392 episodes on a 396-task benchmark spanning four duration buckets and three domains.   Key findings: (1) reliability decay is domain-stratified -- SE GDS drops from 0.90 to 0.44 while document processing   is nearly flat (0.74 to 0.71); (2) VAF bifurcates by capability tier -- high VAF is a capability signature, not an   instability signal; (3) capability and reliability rankings diverge substantially, with multi-rank inversions at long   horizons; (4) frontier models have the highest meltdown rates (up to 19%) because they attempt ambitious multi-step   strategies that sometimes spiral; and (5) memory scaffolds universally hurt long-horizon performance across all 10   models. These results motivate reliability as a first-class evaluation dimension alongside capability.

### 论文总结：超越 pass@1：长程 LLM 智能体的可靠性科学框架

#### 1. 核心问题与整体含义
该论文探讨了大型语言模型（LLM）智能体在从实验评估向生产环境迁移过程中面临的关键挑战：**可靠性与能力的脱节**。
*   **研究动机**：现有的基准测试（如 pass@1）主要衡量模型在短、原子化任务上的单次成功率（即“能力”）。然而，在实际生产中，智能体需要处理长程（Long-Horizon）任务（如重构代码库、合成技术报告），这些任务耗时长、步骤多。
*   **核心发现**：随着任务时长的增加，智能体的可靠性（多次调用的一致成功率）会发生系统性且超线性的衰减。现有的 pass@1 指标对这种衰减具有“结构性盲区”。

#### 2. 方法论
论文借鉴航空航天和软件系统的可靠性工程，提出了一个包含四个核心指标的科学框架：
*   **可靠性衰减曲线 (RDC)**：衡量 pass@k 随任务时长（短、中、长、极长）变化的函数。通过线性回归斜率（RDS）量化衰减速度。
*   **方差放大因子 (VAF)**：定义为长程任务成功率方差与短程任务成功率方差的比值。用于量化时长如何放大随机失败模式。
*   **平滑退化得分 (GDS)**：一种部分得分机制。将任务分解为多个子任务并赋予权重，即使智能体未完全完成任务，也能根据进度给出 0 到 1 之间的得分，比二元化的 pass/fail 更具信息量。
*   **崩溃起始点 (MOP)**：通过计算工具调用序列在滑动窗口内的**信息熵**来检测“行为崩溃”（如陷入死循环、幻觉工具输出）。当熵值超过阈值且出现突增时，判定为崩溃。

#### 3. 实验设计
*   **基准测试 (Benchmark)**：构建了包含 **396 个任务** 的数据集，分为 4 个时长区间（≤5min 到 ≥120min）和 3 个领域（软件工程 SE、网页研究 WR、多文件文档处理 DP）。
*   **评估对象**：10 个主流开源/权重开放模型（包括 DeepSeek V3, Llama 3.3 70B, Qwen3 32B, Kimi K2.5, MiniMax M2.5 等）。
*   **实验配置**：采用两种支架（ReAct 循环和 Memory 增强型循环），每个模型-任务对重复 $k=3$ 次实验，以估计方差。
*   **对比维度**：对比了不同模型规模、不同领域以及有无记忆支架对可靠性的影响。

#### 4. 资源与算力
*   **算力消耗**：论文未提及模型训练的算力，因为其重点在于推理评估。
*   **推理成本**：所有模型通过 OpenRouter API 访问。23,392 个 episode 的总推理成本约为 **$80 - $120**。
*   **基础设施**：使用了带有速率限制处理、自动配额监控和隔离执行环境（临时目录）的 Python 评估框架。

#### 5. 实验数量与充分性
*   **实验规模**：共完成了 **23,392 个 episode** 的测试，这是目前针对长程智能体可靠性最大规模的研究之一。
*   **充分性**：实验设计非常充分。它不仅覆盖了多个模型层级（从 8B 到 1T 参数），还通过 $k=3$ 的重复实验确保了统计显著性。
*   **客观性**：评估过程尽可能采用程序化验证（如运行测试用例、模式校验），减少了“LLM-as-judge”带来的主观偏差。

#### 6. 主要结论与发现
*   **可靠性衰减是普遍且超线性的**：所有模型的 pass@1 均随时长增加而下降，平均从短程的 76.3% 降至极长程的 52.1%。
*   **VAF 悖论**：高方差放大（VAF ≥ 2.37）是**能力**的标志而非不稳定的标志。只有足够强的模型才能在长任务中偶尔成功，而弱模型则表现为稳定的“全盘失败”。
*   **排名倒置**：短任务上的“能力排名”不等于长任务上的“可靠性排名”。例如，Llama 3.3 70B 在短任务中表现平平，但在极长任务中的可靠性排名显著上升。
*   **MOP 悖论**：前沿模型（如 DeepSeek V3）的崩溃率反而更高（达 19%），因为它们倾向于尝试更复杂、多步的策略，一旦出错更容易陷入熵增的循环。
*   **记忆支架有害论**：简单的片段式记忆支架在长程任务中普遍损害了 GDS，因为其带来的上下文开销和步骤负担超过了收益。

#### 7. 优点
*   **指标创新**：将可靠性工程引入 AI 评估，提供了比 pass@1 更细致的度量工具（尤其是 MOP 和 VAF）。
*   **实战导向**：揭示了生产环境中常见的“基础设施不可靠”问题（如 API 配额、超时），并提出了应对策略。
*   **大规模实证**：通过海量实验数据证明了“能力”与“可靠性”是两个需要独立优化的维度。

#### 8. 不足与局限
*   **模型覆盖**：主要集中在开源模型，未包含 GPT-4o 或 Claude 3.7 等闭源前沿模型。
*   **时长定义的局限性**：任务时长基于人类完成时间，这与模型处理的逻辑复杂度（工具调用步数）虽相关但并不完全等同。
*   **领域限制**：虽然涵盖了三个领域，但尚未涉及具身智能、多智能体协作等更复杂的长程场景。
*   **网页任务的动态性**：网页研究任务依赖实时网络，可能存在不可复现的环境噪声。
