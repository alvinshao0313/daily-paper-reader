## 摘要
从复杂的价值流图（VSM）仿真中提取具有可操作性的洞察可能具有挑战性、耗时且容易出错。大语言模型的最新进展为支持用户完成这一任务提供了新途径。虽然现有方法在处理原始数据以获取信息方面表现出色，但在结构上难以捕捉区分该领域相似数据源所需的细微情境差异。为解决这一问题，我们提出了一种解耦的两步式代理架构。通过将编排与数据分析分离，该系统利用注入了领域专家知识的渐进式数据发现。这种架构允许编排层智能地选择数据源，并在保持精简内部上下文的同时，跨数据结构执行多跳推理。来自多个最先进大语言模型的结果证明了该框架的可行性：顶级模型的准确率高达 86%，并在多次评估运行中表现出极高的鲁棒性。

## Abstract
Extracting actionable insights from complex value stream map simulations can be challenging, time-consuming, and error-prone. Recent advances in large language models offer new avenues to support users with this task. While existing approaches excel at processing raw data to gain information, they are structurally unfit to pick up on subtle situational differences needed to distinguish similar data sources in this domain. To address this issue, we propose a decoupled, two-step agentic architecture. By separating orchestration from data analysis, the system leverages progressive data discovery infused with domain expert knowledge. This architecture allows the orchestration to intelligently select data sources and perform multi-hop reasoning across data structures while maintaining a slim internal context. Results from multiple state-of-the-art large language models demonstrate the framework's viability: with top-tier models achieving accuracies of up to 86% and demonstrating high robustness across evaluation runs.

### 1. 核心问题与研究动机
*   **核心问题**：从复杂的价值流图（Value Stream Map, VSM）仿真结果中提取可操作的洞察（Actionable Insights）非常困难。VSM 仿真会产生海量的、语义高度相似的关键绩效指标（KPI）和时间序列数据，传统的大语言模型（LLM）在处理此类任务时面临“上下文腐烂”（Context Rot）和难以区分细微情境差异的问题。
*   **研究背景**：虽然 LLM 在代码生成和通用问答上表现出色，但在工业仿真领域，如何让模型在不超出上下文窗口的前提下，精准定位并分析特定的仿真数据，仍是一个挑战。

### 2. 方法论：解耦的两步式代理架构
该论文提出了一种**解耦的代理架构**，核心思想是将“逻辑推理”与“数据分析”分离：
*   **编排代理（Orchestration Agent）**：作为高层规划器，负责理解用户目标并导航仿真数据的分类体系。它不直接接触原始海量数据，从而保持精简的上下文。
*   **总结子工作流（Summarization Subworkflow）**：当编排代理确定了需要分析的具体数据项后，触发此子工作流。它接收特定数据、领域专家知识和分析指令，完成分析后将精炼的结论返回给编排代理。
*   **关键技术细节（四种工具）**：
    1.  **节点发现工具**：按类型过滤并列出 VSM 中的节点。
    2.  **属性提取工具**：获取特定节点的结构化属性（如安全库存、产能）。
    3.  **分类导航工具**：返回仿真数据分类体系中的可用数据元素描述。
    4.  **总结工具**：连接结构化规划与动态分析，执行具体的子工作流。
*   **渐进式数据发现**：通过上述工具，代理能够像人类专家一样，先找节点，再看属性，最后查对应的仿真日志，实现多跳推理。

*   **实验规模**：针对 4 种不同量级的模型进行了对比，每组评估均运行 **4 次** 以消除 LLM 输出的随机性影响。

### 6. 主要结论与发现
*   **性能分层**：模型表现与参数规模正相关。Claude-Opus 4.6 表现最佳（86分），GLM-5 紧随其后（83.9分），证明顶级开源/开放权重模型正在缩小与闭源模型的差距。
*   **稳定性**：模型越大，输出的方差（SD）越小。Claude 的标准差仅为 0.86，而最小的 Ministral 达到 6.64，说明小模型在复杂编排任务中极不稳定。
*   **效率**：GLM-5 在保持高分的同时，Token 消耗量最低，显示出极高的推理效率。
*   **失败原因**：约 15% 的错误源于编排代理在导航分类体系时的失误，而非数据总结错误，这通常是因为领域知识描述存在语义歧义。

### 7. 优点与亮点
*   **架构优雅**：解耦设计有效解决了长文本下的“上下文腐烂”问题，使系统能处理兆字节级别的仿真数据。
*   **领域驱动**：将专家知识（Domain Expert Knowledge）注入工具描述中，引导代理进行专业化推理。
*   **评估严谨**：建立了“人类专家-裁判模型”的校准机制，为工业仿真领域的 LLM 应用提供了可量化的评价标准。

### 8. 不足与局限
*   **小模型适配性差**：当前的解耦架构在参数量较小的模型上运行不可靠，限制了边缘侧部署的可能。
*   **成本与延迟**：多步代理调用导致 Token 消耗较高且响应延迟增加。
*   **数据长度限制**：虽然缓解了上下文问题，但如果单个时间序列数据本身超过了子工作流模型的窗口，该方法仍需进一步改进（如引入预处理或分段总结）。
*   **闭环缺失**：目前仅实现了“洞察生成”（第二步），尚未完成“自动优化参数并重新仿真”的闭环。
