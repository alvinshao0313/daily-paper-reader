## 摘要
在现实世界的工具集成推理（TIR）场景中，大语言模型（LLM）将推理与外部工具调用交替进行。低效的一个主要来源是工具调用在 LLM 请求之间产生了停顿，并导致 KV-Cache 被逐出，从而迫使重新计算。此外，外部工具返回的长且未经筛选的响应会使 KV-Cache 膨胀，导致每个解码步骤在加载不断增长的缓存上花费更多时间，随着上下文长度的增加，推理速度稳步下降。然而，现有的效率指标（如 token 计数和工具调用计数）无法捕捉真实的模型推理延迟。为解决这一问题，我们引入了 PTE（Prefill Token Equivalents，预填充 Token 等效值），这是一种硬件感知的 TIR 效率指标，它统一了内部推理和外部工具使用的成本，同时明确考虑了不可重用的 KV-Cache 和长工具响应场景。在高并发工业环境中的验证表明，与标准 token 计数相比，PTE 与实际墙钟延迟的一致性显著更好，同时在不同硬件配置下保持了一致的效率排名。我们在五个 TIR 基准测试上进行了广泛实验，量化了它们的 PTE 成本，并识别出 TIR 中出现的四种低效模式。我们还发现，具有较高 PTE 成本的轨迹往往具有较低的推理正确性，这表明单纯增加工具的使用并不能提高答案的质量。

## Abstract
In real-world Tool-Integrated Reasoning (TIR) scenarios, where LLMs interleave reasoning with external tool calls, a major source of inefficiency is that the toolcalls create pauses between LLM requests and cause KV-Cache eviction, forcing recomputation. Also, the long, unfiltered response returned by external tools inflates the KV-Cache, so each decode step spends more time loading the growing cache and thus becomes steadily slower as context length increases. However, existing efficiency metrics like token counts and toolcall counts fail to capture the real model inference latency. To address this, we introduce PTE (Prefill Token Equivalents), a hardware-aware TIR-efficiency metric that unifies internal reasoning and external tool-use costs while explicitly accounting for non-reusable KV-Cache and long-tool-response scenarios. Validation in a high-concurrency industrial setting indicates that PTE aligns significantly better with wall-clock latency than standard token counts, while maintaining consistent efficiency rankings across diverse hardware profiles. We conduct extensive experiments across five TIR benchmarks, quantify their PTE costs, and identify four inefficiency patterns that appear in TIR. We also discover that trajectories with higher PTE costs tend to have lower reasoning correctness, indicating that simply using more tools does not improve the quality of the answer.

## 1. 核心问题与整体含义
论文聚焦于**工具集成推理（Tool-Integrated Reasoning, TIR）**场景下的效率评估问题。
*   **研究动机**：在 TIR 中，LLM 需要交替进行内部推理和外部工具调用。现有的效率指标（如 Token 计数或工具调用次数）无法反映真实的硬件延迟。
*   **核心痛点**：
    1.  **KV 缓存失效**：工具调用产生的停顿常导致 KV 缓存被逐出，后续请求需重新计算（Prefill）。
    2.  **上下文膨胀**：外部工具返回的长响应会显著增加上下文长度，导致内存受限的解码（Decode）阶段变得极慢。
*   **研究意义**：旨在建立一个基于硬件第一性原理的统一指标，量化推理与工具使用的真实物理开销。

## 2. 方法论：PTE 指标
论文提出了 **PTE（Prefill Token Equivalents，预填充 Token 等效值）** 指标，将所有开销统一为“处理一个输入 Token”的等效成本。

*   **核心思想**：区分计算受限的 **Prefill 阶段**和内存受限的 **Decode 阶段**，并根据硬件特性对后者进行加权。
*   **关键公式**：
    $$PTE = \sum_{i=1}^{k} (D_{prefill,i} + \gamma \cdot L_{seq,i} \cdot D_{decode,i})$$
    *   $D_{prefill,i}$：第 $i$ 轮的预填充 Token 数（计算受限）。
    *   $D_{decode,i}$：第 $i$ 轮生成的 Token 数（内存受限）。
    *   $L_{seq,i}$：解码开始前的累积序列长度。
    *   **$\gamma$（硬件感知系数）**：代表内存带宽负载相对于计算吞吐量的相对惩罚。它由硬件的**运行强度（HOI）**、模型参数量、层数及注意力机制（如 GQA、MLA）共同决定。
*   **算法流程**：通过记录每一轮对话的输入/输出长度及累积上下文，结合特定硬件-模型对的 $\gamma$ 值，计算出整条推理轨迹的总 PTE。

## 6. 主要结论与发现
1.  **效率与准确率的背离**：准确率相近的模型，其 PTE 成本可能相差数倍甚至一个数量级。
2.  **四种低效模式**：
    *   **验证性工具使用**：模型先推理出答案再调用工具验证，导致上下文无谓增长。
    *   **工具混用**：在不同工具间频繁切换，产生碎片化工作流。
    *   **缺乏工具先验**：模型因不熟悉工具环境（如漏掉 `print`）导致无效调用。
    *   **工具格式崩溃**：模型对工具调用格式极其敏感，微小偏差导致解析失败。
3.  **“思考”的代价**：推理模型（如 Qwen3-Thinking）在复杂任务上溢价合理，但在简单任务上存在严重的“过度思考”和效率浪费。
4.  **负相关性**：高 PTE 成本的轨迹往往对应错误的答案，表明低效的工具使用通常伴随着逻辑混乱。

## 7. 优点
*   **视角独特**：超越了单纯的准确率评估，从硬件底层逻辑出发定义效率。
*   **硬件感知**：PTE 指标考虑了 KV 缓存失效这一 TIR 场景的核心痛点，具有极强的实战指导意义。
*   **诊断功能**：通过 PTE 成功识别并量化了四种具体的模型行为缺陷，为后续模型微调提供了方向。

## 8. 不足与局限
*   **外部开销缺失**：PTE 主要衡量 Transformer 的计算/内存成本，未包含工具本身的执行延迟（如网络 API 响应时间）。
*   **闭源模型限制**：由于计算 $\gamma$ 需要模型架构参数（层数、维度等），该指标难以直接应用于参数不透明的闭源模型（如 GPT-4o）。
*   **因果关系探讨**：虽然发现了高 PTE 与低准确率的相关性，但对于“是低效导致了错误”还是“逻辑混乱表现为低效”的因果链条仍需进一步研究。
