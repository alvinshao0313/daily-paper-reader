## 摘要
评估图形用户界面（GUI）智能体面临着独特的挑战：轨迹长、基于视觉且具有开放性，但评估必须兼具准确性和可解释性。现有方法通常对整个“动作-观察”序列进行单一的整体判断，这种策略在长时程任务中被证明是不可靠的，且仅提供二元结论，无法深入了解智能体失败的位置或原因。这种不透明性限制了评估作为智能体开发诊断工具的效用。我们提出了 GUIDE（GUI 理解与可解释诊断评估），这是一个将轨迹评估分解为三个连续阶段的框架，镜像了 GUI 任务的组合结构。轨迹分段（Trajectory Segmentation）将完整轨迹划分为语义连贯的子任务单元。子任务诊断（Subtask Diagnosis）在上下文中评估每个单元，给出完成判定，并生成带有改进建议的结构化错误分析。总体总结（Overall Summary）将每个子任务的诊断汇总为任务级的判断。通过在有界的子任务片段而非完整轨迹上运行，GUIDE 减轻了随着任务复杂度增加而导致现有评估器性能下降的上下文过载问题。我们在三个基准测试上验证了 GUIDE：包含 932 条轨迹的工业电子商务数据集、涵盖五个 Web 智能体任务且包含 1302 条轨迹的 AGENTREWARDBENCH，以及用于移动设备控制的 AndroidBench。在所有设置中，GUIDE 的表现均显著优于现有评估器——准确率比最强基线高出多达 5.35 个百分点——同时生成能直接指导智能体改进的结构化诊断报告。

## Abstract
Evaluating GUI agents presents a distinct challenge: trajectories are long, visually grounded, and open-ended, yet evaluation must be both accurate and interpretable. Existing approaches typically apply a single holistic judgment over the entire action-observation sequence-a strategy that proves unreliable on long-horizon tasks and yields binary verdicts offering no insight into where or why an agent fails. This opacity limits the utility of evaluation as a diagnostic tool for agent development. We introduce GUIDE (GUI Understanding and Interpretable Diagnostic Evaluation), a framework that decomposes trajectory assessment into three sequential stages mirroring the compositional structure of GUI tasks. Trajectory Segmentation partitions the full trace into semantically coherent subtask units. Subtask Diagnosis evaluates each unit in context, assigning a completion verdict and generating a structured error analysis with corrective recommendations. Overall Summary aggregates per-subtask diagnoses into a task-level judgment. By operating on bounded subtask segments rather than full trajectories, GUIDE mitigates the context overload that degrades existing evaluators as task complexity grows. We validate GUIDE on three benchmarks: an industrial e-commerce dataset of 932 trajectories, AGENTREWARDBENCH spanning five web agent tasks with 1302 trajectories, and AndroidBench for mobile device control. Across all settings, GUIDE substantially outperforms existing evaluators-achieving up to 5.35 percentage points higher accuracy than the strongest baseline-while producing structured diagnostic reports that directly inform agent improvement.

*   **核心挑战**：
    *   **上下文过载**：现有评估器（如 LLM-as-judge）通常采用“整体式”评估，即一次性处理整个动作序列和截图。随着轨迹变长，模型处理长上下文的能力下降，导致准确率大幅跌落（实验显示 WebJudge 在长任务中准确率下降近 18%）。
    *   **缺乏可解释性**：现有方法多给出二元（成功/失败）结论，无法告诉开发者智能体在何处失败、为何失败以及如何修复。
    *   **任务开放性**：同一目标有多种达成路径，传统的基于规则的匹配方法过于僵化。

### 2. 方法论：GUIDE 框架
GUIDE 借鉴了智能体执行中的“层次化分解”思想，将其应用于事后评估，分为三个阶段：
1.  **轨迹分段 (Trajectory Segmentation)**：
    *   **核心思想**：将完整的动作-观察序列自动划分为语义连贯的子任务单元。
    *   **技术细节**：仅使用任务描述和文本动作日志（不含截图以节省算力并避免 API 限制），由 LLM 识别子任务边界并生成子任务描述。
2.  **子任务诊断 (Subtask Diagnosis)**：
    *   **核心思想**：在局部上下文中独立评估每个子任务。
    *   **技术细节**：采用多模态输入（子任务描述、动作序列、关键帧截图）。输出结构化的诊断报告，包含：**判定结果**（成功/部分成功/失败）、**根因分析**（Root Cause）和**改进建议**（Corrective Recommendations）。
3.  **总体总结 (Overall Summary)**：
    *   **核心思想**：汇总所有子任务的诊断结果，生成最终的任务级判断。
    *   **技术细节**：由 LLM 综合推理，识别哪些中间失败是可恢复的，哪些是致命的，从而给出比简单“全对才算对”更合理的最终结论。

*   **诊断价值**：GUIDE 能够精确定位错误步骤并提供可操作的修复建议，这在现有评估器中是缺失的。

### 7. 优点与亮点
*   **层次化架构**：通过“分而治之”解决了长序列多模态理解的瓶颈。
*   **高可解释性**：生成的结构化诊断报告（JSON 格式）不仅对研究人员有用，甚至可以反馈给智能体进行自我修正。
*   **零样本部署**：无需训练，无需特定领域的规则工程，具有极强的通用性。
*   **轻量化分段**：巧妙地在分段阶段排除图像，既保证了效率又避免了长序列图像导致的 API 崩溃。

### 8. 不足与局限
*   **领域知识依赖**：作为零样本框架，在处理需要特定业务逻辑（如特定电商平台的内部结算规则）的任务时，可能会因为缺乏背景知识而误判。
*   **推理成本**：相比于单次调用的整体评估，GUIDE 需要多次调用 API（分段、每个子任务诊断、总结），在推理成本和延迟上可能更高。
*   **模型依赖**：评估质量高度依赖于底层 VLM 的推理能力，若底层模型存在视觉幻觉，评估结果也会受损。
