## 摘要
部署在智能体环境中的大语言模型（LLMs）必须在不同的任务实例中行使多种能力，其中“能力”是指在轨迹中执行一个或多个动作，这些动作对于成功解决环境中的一部分任务是必要的。许多现有方法要么依赖于并非针对模型在目标环境中实际能力缺陷的合成训练数据，要么直接在目标环境上进行训练，而模型需要在其中隐式地学习跨任务的能力。我们介绍了 TRACE（将智能体循环失败转化为面向能力的训练环境），这是一个用于特定环境智能体自我提升的端到端系统。TRACE 通过对比成功和失败的轨迹来自动识别缺失的能力，为每种能力合成一个针对性的训练环境（该环境会根据是否行使了该能力给予奖励），并通过强化学习（RL）在每个合成环境上训练一个 LoRA 适配器，在推理时路由到相关的适配器。实验表明，TRACE 在不同环境中具有泛化性，在 $τ^2$-bench（客服场景）上比基础智能体提升了 14.1 分，在 ToolSandbox（工具使用场景）上增加了 7 个满分，分别比最强的基准模型高出 7.4 分和 4 个满分。在给定相同采样次数的情况下，TRACE 的扩展效率高于基准模型，在 $τ^2$-bench 上分别比 GRPO 和 GEPA 高出 9.2 分和 7.4 分。

## Abstract
Large Language Models (LLMs) deployed in agentic environments must exercise multiple capabilities across different task instances, where a capability is performing one or more actions in a trajectory that are necessary for successfully solving a subset of tasks in the environment. Many existing approaches either rely on synthetic training data that is not targeted to the model's actual capability deficits in the target environment or train directly on the target environment, where the model needs to implicitly learn the capabilities across tasks. We introduce TRACE (Turning Recurrent Agent failures into Capability-targeted training Environments), an end-to-end system for environment-specific agent self-improvement. TRACE contrasts successful and failed trajectories to automatically identify lacking capabilities, synthesizes a targeted training environment for each that rewards whether the capability was exercised, and trains a LoRA adapter via RL on each synthetic environment, routing to the relevant adapter at inference. Empirically, TRACE generalizes across different environments, improving over the base agent by +14.1 points on $τ^2$-bench (customer service) and +7 perfect scores on ToolSandbox (tool use), outperforming the strongest baseline by +7.4 points and +4 perfect scores, respectively. Given the same number of rollouts, TRACE scales more efficiently than baselines, outperforming GRPO and GEPA by +9.2 and +7.4 points on $τ^2$-bench.

*   **研究动机**：LLM 智能体在复杂环境中往往因为缺乏某些特定能力（如结构化数据推理、工具调用精度等）而失败。
*   **现有痛点**：
    *   **直接强化学习（RL）效率低**：在目标环境中直接训练，奖励信号往往只针对最终结果，难以归因到具体的中间能力缺失，导致学习稀疏且采样效率低下。
    *   **合成数据缺乏针对性**：现有的合成数据生成方法通常是通用的，并未针对模型在特定任务中的实际“短板”进行优化。
*   **核心目标**：建立一个能够自动诊断失败原因、生成针对性练习题并进行专项训练的闭环系统。

### 2. 方法论：核心思想与技术细节
TRACE 系统包含四个关键步骤：
1.  **对比式能力识别（Contrastive Identification）**：
    *   分析智能体在目标环境中的成功和失败轨迹。
    *   利用 LLM 分析智能体，对比两类轨迹，识别出能显著区分成功与失败的“能力缺口”（如：是否正确解析了嵌套 JSON）。
    *   计算“对比差距”（Contrastive Gap）和“失败覆盖率”，优先选择高影响力的能力。
2.  **能力针对性环境合成（Environment Synthesis）**：
    *   为每种缺失能力自动编写一个合成的、可验证的训练环境。
    *   这些环境保留了目标环境的接口（工具 Schema、交互协议），但任务逻辑高度简化并聚焦于单一能力的练习。
    *   环境包含自动验证逻辑，能根据能力行使情况给出稠密奖励。
3.  **强化学习训练（RL Training）**：
    *   针对每个合成环境，使用 **GRPO**（组相对策略优化）算法训练一个独立的 **LoRA 适配器**。
    *   这种方式保证了基础模型权重不变，仅更新少量参数（约 5.3%）。
4.  **能力路由与组合（Routing）**：
    *   在推理时，使用基础模型作为路由器，根据任务描述选择最匹配的 LoRA 适配器。
    *   如果任务不属于任何已训练的能力范畴，则回退到基础模型。

### 5. 实验数量与充分性
*   **实验规模**：在两个主流智能体基准上进行了全面测试，涵盖了从单能力训练到多能力组合的对比。

### 6. 主要结论与发现
*   **性能卓越**：TRACE 在 $\tau^2$-Bench 上比基础模型提升了 **14.1%** 的成功率，在 ToolSandBox 上显著增加了满分案例。
*   **扩展效率高**：在相同的采样次数下，TRACE 的表现远优于直接在目标环境上训练的 GRPO（提升 9.2%）。
*   **训练优于提示**：显式训练模型行使能力的效果显著优于仅在提示词中加入能力说明（GEPA）。
*   **路由更稳健**：相比于将多种能力强行合并到一个模型中，通过路由切换适配器能更好地保留各专项能力的性能。

### 7. 优点与亮点
*   **自动化闭环**：实现了从“发现问题”到“生成练习”再到“解决问题”的全自动流程。
*   **诊断性强**：通过对比分析，能够清晰地指出模型在特定环境下的具体弱点，具有很强的可解释性。
*   **模块化**：使用 LoRA 适配器和路由机制，使得系统可以轻松扩展新能力，而不会产生灾难性遗忘。

### 8. 不足与局限
*   **推理开销**：虽然 LoRA 切换很快，但路由步骤（分类任务）依然会增加少量的推理延迟。
*   **依赖分析智能体**：系统第一步的质量高度依赖于用于分析轨迹的 LLM 的水平，如果分析智能体无法准确识别失败原因，后续训练将偏离目标。
*   **长尾能力覆盖**：目前的实验主要针对高频出现的 2-4 种核心能力，对于极其稀疏的长尾能力缺口，识别和合成环境的难度可能更大。
