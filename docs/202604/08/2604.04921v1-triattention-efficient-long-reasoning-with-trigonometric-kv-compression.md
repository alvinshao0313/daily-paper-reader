## 摘要
大语言模型（LLM）中的长程推理会产生严重的 KV 缓存内存瓶颈。领先的 KV 缓存压缩方法利用近期 RoPE 后（post-RoPE）查询的注意力分数来估计 KV 的重要性。然而，在 RoPE 过程中，查询会随位置旋转，导致代表性查询极少，进而引发 Top-Key 选择不佳和推理不稳定。为避免此问题，我们转向 RoPE 前（pre-RoPE）空间，观察到 Q 和 K 向量高度集中在固定的非零中心周围，且在不同位置保持稳定——即 Q/K 集中现象。我们证明了这种集中现象会导致查询优先关注特定距离的键（例如最近的键），而这些中心通过三角级数决定了哪些距离是首选的。基于此，我们提出了 TriAttention，通过利用这些中心来估计键的重要性。通过三角级数，我们利用这些中心所表征的距离偏好，根据键的位置对其进行评分，并利用 Q/K 范数作为重要性估计的额外信号。在 32K token 生成的 AIME25 测试中，TriAttention 在达到与全注意力（Full Attention）相当的推理准确率的同时，实现了 2.5 倍的吞吐量提升或 10.7 倍的 KV 内存缩减，而领先的基准模型在相同效率下仅能达到约一半的准确率。TriAttention 使得 OpenClaw 能够在单个消费级 GPU 上部署，而在全注意力模式下，长上下文会导致显存溢出（OOM）。

## Abstract
Extended reasoning in large language models (LLMs) creates severe KV cache memory bottlenecks. Leading KV cache compression methods estimate KV importance using attention scores from recent post-RoPE queries. However, queries rotate with position during RoPE, making representative queries very few, leading to poor top-key selection and unstable reasoning. To avoid this issue, we turn to the pre-RoPE space, where we observe that Q and K vectors are highly concentrated around fixed non-zero centers and remain stable across positions -- Q/K concentration. We show that this concentration causes queries to preferentially attend to keys at specific distances (e.g., nearest keys), with the centers determining which distances are preferred via a trigonometric series. Based on this, we propose TriAttention to estimate key importance by leveraging these centers. Via the trigonometric series, we use the distance preference characterized by these centers to score keys according to their positions, and also leverage Q/K norms as an additional signal for importance estimation. On AIME25 with 32K-token generation, TriAttention matches Full Attention reasoning accuracy while achieving 2.5x higher throughput or 10.7x KV memory reduction, whereas leading baselines achieve only about half the accuracy at the same efficiency. TriAttention enables OpenClaw deployment on a single consumer GPU, where long context would otherwise cause out-of-memory with Full Attention.

*   **现有问题**：目前的压缩方法（如 H2O, SnapKV, R-KV）主要在 **RoPE 后（post-RoPE）** 空间通过近期查询（Query）的注意力分数来评估 Key 的重要性。但由于 RoPE 会随位置旋转向量，导致观察窗口极小且不稳定，容易误删后续关键的 Token，破坏推理链。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：利用 pre-RoPE 空间的 Q/K 中心，通过**三角级数（Trigonometric Series）**预测注意力对距离的偏好，从而在不依赖实时注意力观察的情况下稳定评估 Key 的重要性。
*   **关键技术细节**：
    *   **三角级数评分 ($S_{trig}$)**：将注意力计算简化为仅取决于 Q-K 距离的函数。利用离线校准得到的 Q 中心代替未来查询，计算每个 Key 在不同距离下的预期得分。
    *   **范数辅助评分 ($S_{norm}$)**：针对少数集中度较低的注意力头，引入 Q/K 范数作为补充信号，识别那些虽然距离符合偏好但因范数过小而贡献微弱的 Token。
    *   **自适应加权**：使用平均合矢量长度（Mean Resultant Length, R）来量化集中度，自动平衡三角级数评分和范数评分的权重。
    *   **窗口化剪枝**：每生成 128 个 token 触发一次评分和剪枝，保留得分最高的 Top-B 个 Key。
*   **算法流程**：离线校准获取 Q/K 中心 -> 推理时结合距离偏好和范数进行评分 -> 归一化处理 GQA 结构 -> 剪枝并保留关键 KV 对。

### 6. 主要结论与发现
*   **性能卓越**：在 AIME25 任务中，TriAttention 在保持全注意力精度的前提下，实现了 **10.7 倍**的 KV 内存缩减或 **2.5 倍**的吞吐量提升。

*   **模型内在属性**：Q/K 集中现象是模型固有的，不仅存在于 GQA 架构，在 MLA（如 GLM-4）架构中甚至更明显，且不随任务领域改变。

### 7. 优点
*   **理论创新**：首次揭示并利用了 pre-RoPE 空间的 Q/K 集中现象，将注意力预测转化为数学上的三角级数问题。
*   **高稳定性**：解决了 post-RoPE 方法因旋转导致的观察窗口不稳定的痛点，特别适合长链条推理。
*   **通用性强**：对校准数据不敏感（甚至用 HTML 网页数据校准也能在数学任务上表现良好），易于迁移到不同模型。

### 8. 不足与局限
*   **离线依赖**：虽然对数据不敏感，但仍需一个前置的离线校准步骤来获取 Q/K 中心。
*   **内核优化**：目前的吞吐量提升主要来自 KV 减少带来的计算量缩减，尚未针对三角级数计算开发专门的硬件感知（Hardware-aware）算子内核，仍有进一步提速空间。
*   **应用范围**：虽然在推理和长文本任务表现优异，但在极短文本或对实时性要求极高且内存充足的场景下，压缩带来的额外计算开销可能抵消其优势。
