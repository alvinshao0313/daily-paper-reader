## 摘要
多智能体大语言模型（LLM）架构日益依赖响应级聚合（如多数投票，MAJ）来提升推理上限。然而，在开放环境中，智能体极易受到隐蔽的上下文破坏（如针对性提示注入）的影响。我们揭示了当前多智能体系统中一个关键的结构性漏洞：当受损智能体形成局部多数时，响应级聚合会失效。由于投票聚合的是已形成的结论，它无法察觉有缺陷的中间逻辑。为了克服这一系统性限制，我们提出了 Token 级轮询（RR）协作，即智能体在共享的自回归上下文中顺序交替生成。我们将此过程形式化为离散时间动力系统，证明了 Token 级交替将聚合从脆弱的最终投票计数（线性求和）转变为动态交织的逻辑链（非线性算子乘积）。通过这一理论视角，我们证明了诚实模型的修复拉力可以战胜对抗性破坏，即使受损智能体占据多数。我们在多种推理基准上进行了详尽的实证评估，结果表明，虽然 MAJ 在受损智能体达到多数时会崩溃，但 RR 在远超这一临界阈值的情况下仍能保持稳健的准确性。

## Abstract
Multi-agent large language model (LLM) architectures increasingly rely on response-level aggregation, such as Majority Voting (MAJ), to raise reasoning ceilings. However, in open environments, agents are highly susceptible to stealthy contextual corruption, such as targeted prompt injections. We reveal a critical structural vulnerability in current multi-agent systems: response-level aggregation collapses when corrupted agents form a local majority. Because voting aggregates fully-formed conclusions, it is blind to flawed intermediate logic. To overcome this systematic limitation, we propose the Token-Level Round-Robin (RR) Collaboration, where agents sequentially interleave generation within a shared auto-regressive context. We formalize this process as a discrete-time dynamical system, proving that token-level interleaving transitions aggregation from a brittle counting of final votes (a linear sum) to a dynamic, interwoven chain of logic (a non-linear operator product). Through this theoretical lens, we prove that the honest model's restorative pull can overpower adversarial corruptions, even when corrupted agents form a majority. We conduct an exhaustive empirical evaluation across diverse reasoning benchmarks and demonstrate that while MAJ collapses when corrupted agents reach a majority, RR maintains robust accuracy well beyond this critical threshold.

### 1. 核心问题与整体含义
*   **研究动机**：当前多智能体 LLM 系统（如多数投票 MAJ、多智能体辩论）普遍依赖“响应级聚合”，即让多个智能体独立生成完整答案后再进行汇总。这种机制基于一个脆弱假设：智能体的错误是相互独立的。
*   **核心挑战（共识陷阱）**：在开放环境中，智能体容易受到协同的上下文破坏（如针对性提示注入）。当受损智能体占据局部多数（比例 $\rho > 0.5$）时，传统的多数投票机制会因无法识别中间逻辑错误而彻底崩溃，导致系统输出错误的共识。
*   **研究目标**：打破多数投票的算术限制，使系统在诚实智能体处于少数的情况下仍能输出正确答案。

### 2. 方法论
*   **核心思想**：从宏观的“响应级聚合”转向微观的“Token 级协作”。
*   **关键技术：Token 级轮询（RR）协作**：
    *   **交替生成**：$N$ 个智能体不再独立工作，而是在一个共享的自回归上下文中轮流生成。每个智能体每次仅生成固定长度（$K$ 个 Token）的片段。
    *   **逻辑干预**：诚实智能体可以实时监测共享轨迹，在对抗性逻辑出现时立即介入并修正（例如生成“等等，之前的计算有误……”）。
    *   **真理吸引子（Truth Attractor）**：一旦正确的逻辑链被注入上下文，由于自回归模型的概率约束，即使是受损智能体也会被迫沿着逻辑惯性生成正确结论，而非遵循其受损的初始指令。
*   **理论框架**：
    *   将生成过程建模为**离散时间动力系统**。
    *   利用**李雅普诺夫稳定性（Lyapunov Stability）**证明：诚实算子的“修复拉力”具有更高的结构深度，而对抗性扰动在共享上下文中受到“谄媚瓶颈（Sycophancy Bottleneck）”的限制。
    *   证明了当 $K$ 足够小时，系统在受损比例超过 0.5 时仍能保持局部稳定性。

### 3. 实验设计
*   **数据集**：涵盖 5 类需要多步推理的客观验证任务：数学推理（AQUA-RAT, GSM8K, MATH500）、空间状态追踪（Track-7）和逻辑演绎（Logic-3/7）。
*   **模型**：使用了 Llama-3.3-70B、Llama-4-Scout-17B、Mistral-Small-3.1-24B、Qwen2.5-32B 和 Qwen-3-30B 等主流模型。
*   **威胁模型**：
    *   **中度（建议性）注入**：劝说模型选择错误答案。
    *   **强度（命令性）注入**：强制要求模型必须选择错误答案。
*   **对比方法**：基准线为标准的多数投票（MAJ）。

### 4. 资源与算力
*   **算力说明**：文中未详细列出具体的 GPU 型号、数量或训练时长。
*   **成本分析**：作者强调该方法具有**成本中性（Cost-Neutrality）**。因为 RR 机制生成的总 Token 数与 MAJ 相同，主要的额外开销仅在于智能体切换时的上下文预填充（Pre-fill），这在现代推理架构中是高度并行且计算受限的，相比解码过程开销极小。

### 5. 实验数量与充分性
*   **实验规模**：
    *   跨越了 5 种模型家族和 5 个数据集。
    *   测试了从 1c4t（1 个受损，4 个诚实）到 4c1t（4 个受损，1 个诚实）的不同腐败比例。
    *   进行了详尽的**消融实验**：包括测试时计算缩放（M 从 1 到 40）、最终发言者身份的影响、异构智能体组合（如 70B 诚实模型救助 8B 受损模型，反之亦然）以及 Token 块大小 $K$ 的影响。
*   **充分性评价**：实验设计非常全面，不仅涵盖了同质模型，还探讨了不同规模模型混合的情况，验证了理论推导的普适性。

### 6. 主要结论与发现
*   **性能恢复**：在受损智能体占多数（3c2t）的情况下，MAJ 准确率往往跌至接近 0%，而 RRMaj 能实现巨大的恢复（如在 Track7 任务中从 0.8% 恢复至 89.1%）。
*   **孔多塞陷阱（Condorcet Trap）**：在多数受损的情况下，增加 MAJ 的采样次数反而会加速系统崩溃；而 RRMaj 随着采样次数增加，准确率单调上升。
*   **能力不对称**：弱小的诚实智能体能有效救助强大的受损智能体，因为强大的模型对逻辑一致性的敏感度更高（逻辑惯性更强）。
*   **最优块大小**：$K$ 在 100-300 Token 之间效果最佳。过小会导致逻辑碎片化，过大会给对抗智能体留下足够的“语义跑道”来完成错误逻辑。

### 7. 优点
*   **理论深度**：不仅有实证，还引入了动力系统和李雅普诺夫稳定性理论，为多智能体协作提供了数学基础。
*   **实用性强**：无需重新训练模型，仅在推理端通过改变生成顺序即可实现，且不增加显著计算成本。
*   **突破限制**：成功打破了传统社会选择理论中“少数服从多数”在对抗环境下的鲁棒性瓶颈。

### 8. 不足与局限
*   **对抗演进**：尚未测试能够感知共享上下文并刻意破坏修正步骤的“自适应对手”。
*   **任务局限**：目前主要集中在有客观标准答案的推理任务上，对于主观性强、缺乏明确逻辑链的任务（如创意写作或开放式对话），“真理吸引子”效应可能减弱。
*   **推理延迟**：虽然计算量持平，但由于需要频繁切换模型上下文，在某些未优化的推理框架下可能会增加首字延迟（TTFT）。
