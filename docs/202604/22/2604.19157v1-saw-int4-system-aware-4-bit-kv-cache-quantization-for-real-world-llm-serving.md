## 摘要
KV 缓存内存是真实大语言模型（LLM）推理服务中的主要瓶颈，系统必须同时支持对延迟敏感的小批量请求和高吞吐量的并发工作负载。尽管许多 KV 缓存压缩方法提高了离线精度或压缩率，但它们往往违反了实际的推理服务约束，如分页内存布局、规则内存访问和融合注意力执行，从而限制了它们在部署中的有效性。在这项工作中，我们确定了在这些约束下仍然可行的 4 位 KV 缓存量化方法的最小集合。我们的核心发现是，一种简单的设计——带有块对角 Hadamard 旋转的逐 token INT4 量化——始终能实现最佳的精度-效率权衡。在多个模型和基准测试中，这种方法几乎恢复了朴素 INT4 损失的所有精度，而一旦考虑到推理服务的兼容性，更复杂的方法（如矢量量化和 Hessian 感知量化）仅能提供微乎其微的额外收益。为了使其具有实用性，我们实现了一个融合旋转-量化算子，该算子直接集成到分页 KV 缓存布局中，且引入的可测量端到端开销为零，在各种并发水平下都能匹配普通 INT4 的吞吐量。我们的结果表明，有效的 KV 缓存压缩从根本上是一个系统协同设计问题：在真实的推理服务约束下，轻量级的块对角 Hadamard 旋转是一种可行的方法，它在不牺牲推理效率的情况下提供了近乎无损的精度。

## Abstract
KV-cache memory is a major bottleneck in real-world LLM serving, where systems must simultaneously support latency-sensitive small-batch requests and high-throughput concurrent workloads. Although many KV-cache compression methods improve offline accuracy or compression ratio, they often violate practical serving constraints such as paged memory layouts, regular memory access, and fused attention execution, limiting their effectiveness in deployment.   In this work, we identify the minimal set of 4-bit KV-cache quantization methods that remain viable under these constraints. Our central finding is that a simple design--token-wise INT4 quantization with block-diagonal Hadamard rotation--consistently achieves the best accuracy-efficiency trade-off. Across multiple models and benchmarks, this approach recovers nearly all of the accuracy lost by naive INT4, while more complex methods such as vector quantization and Hessian-aware quantization provide only marginal additional gains once serving compatibility is taken into account.   To make this practical, we implement a fused rotation-quantization kernel that integrates directly into paged KV-cache layouts and introduces zero measurable end-to-end overhead, matching plain INT4 throughput across concurrency levels. Our results show that effective KV-cache compression is fundamentally a systems co-design problem: under real serving constraints, lightweight block-diagonal Hadamard rotation is a viable method that delivers near-lossless accuracy without sacrificing serving efficiency.

### 论文总结：SAW-INT4 —— 面向真实 LLM 推理服务的系统感知 4 位 KV 缓存量化

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）支持的上下文窗口不断延长，**KV 缓存（KV-Cache）**已成为推理服务中的主要内存瓶颈，直接限制了系统的吞吐量和并发处理能力。
*   **研究动机**：现有的 KV 缓存压缩技术（如 KIVI、Kitty、矢量量化等）虽然在离线测试中表现出高压缩比或高精度，但在实际生产环境（如 vLLM、SGLang）中往往难以落地。
*   **核心矛盾**：实际部署受限于**分页内存布局（PagedAttention）**、**融合注意力算子（Fused Kernels）**以及**内存带宽限制**。许多复杂方法因引入非规则内存访问或额外的计算开销，导致端到端吞吐量反而下降。
*   **论文目标**：寻找在真实系统约束下，既能保持精度又能实现最高效率的“最小化”4 位量化方案。

#### 2. 方法论：核心思想与关键技术
论文提出了一种名为 **SAW-INT4** 的系统感知量化框架，其核心思想是：**逐 Token 的 INT4 量化 + 块对角哈达玛旋转（Block-Diagonal Hadamard Rotation, BDR）**。

*   **逐 Token INT4 量化**：采用非对称量化，对每个 Token 的每个 Head 进行独立缩放。这种方式天然兼容分页内存布局，避免了跨块的数据依赖。
*   **块对角哈达玛旋转 (BDR)**：
    *   **原理**：针对 INT4 量化中严重的通道离群值（Outliers）问题，在量化前应用正交变换（Hadamard 矩阵）。
    *   **创新点**：不使用全局旋转，而是将 Head 维度划分为较小的块（如 block size=128），在块内进行旋转。这在抑制离群值的同时，显著降低了计算复杂度。
*   **系统协同设计（算子融合）**：
    *   开发了**融合旋转-量化 CUDA 内核**（基于 Triton 实现）。
    *   将旋转操作直接嵌入到 KV 缓存的写入和读取路径中，避免了额外的全局内存读写（Memory Pass），实现了相对于普通 INT4 的“零开销”集成。

#### 3. 实验设计
*   **数据集/场景**：涵盖了推理和代码生成的 5 个核心基准测试：**GPQA-Diamond**、**HumanEval**、**LiveCodeBench (v6)**、**AIME25** 和 **MATH500**。
*   **测试模型**：Qwen3-4B-Thinking、Qwen3-8B、Qwen3-32B 以及超大规模的 GLM-4.7-FP8 (358B)。
*   **对比方法 (Baselines)**：
    1.  **BF16**（全精度基准）。
    2.  **朴素 INT4**（无旋转的逐 Token 量化）。
    3.  **KMeans 矢量量化**（不同质心数量 C=256, 2048）。
    4.  **Hessian 感知量化**（利用二阶信息指导旋转）。
    5.  **Kitty**（现有的高性能 2-bit 压缩方案）。

#### 4. 资源与算力
*   **硬件环境**：主要使用 **NVIDIA H100 (80GB)** GPU。
*   **配置**：
    *   单卡 H100 用于 Qwen3-8B 的吞吐量测试。
    *   2×H100 (TP=2) 用于 Qwen3-4B/8B/32B 的性能评估。
    *   8×H100 (TP=8) 用于 GLM-4.7 (358B) 的大规模推理测试。
*   **训练/校准**：Hessian 和 KMeans 方法需要离线校准（使用 MMLU-Pro 数据集的前 20,000 个 token），而核心方法 BDR 是无需训练/校准的（Training-free）。

#### 5. 实验数量与充分性
*   **实验规模**：论文在不同规模（4B 到 358B）的模型上进行了广泛测试，并针对不同并发数（1 到 256）和上下文长度（短文本及长达 32k 的长文本）进行了压力测试。
*   **消融实验**：详细探讨了块大小（16, 64, 128）、旋转对象（仅 Key 或 Key+Value）以及不同量化复杂度的组合。
*   **客观性**：实验不仅报告了离线精度，还重点报告了**端到端吞吐量（TPS）**和**首字延迟（TTFT）**，这在系统类论文中非常客观且具有说服力。

#### 6. 主要结论与发现
*   **旋转是关键**：在 INT4 量化下，朴素方案在某些模型（如 Qwen3）上精度会彻底崩溃（得分为 0），而加入 BDR 后可恢复至接近 BF16 的水平（差距仅 1-3%）。
*   **复杂度边际递减**：一旦引入了旋转，更复杂的 KMeans 或 Hessian 感知方案带来的精度提升非常有限，但在系统实现上却复杂得多且开销更大。
*   **系统效率**：通过算子融合，BDR-INT4 的端到端吞吐量完全匹配普通 INT4，且显著优于 BF16（尤其在高并发和长上下文场景下，吞吐量提升可达 10%-40%）。
*   **模型差异**：不同模型对量化的敏感度不同，GLM-4.7 对朴素 INT4 较鲁棒，而 Qwen3 系列则必须依赖旋转。

#### 7. 优点
*   **极强的实用性**：直接针对生产级推理引擎（SGLang/vLLM）的约束进行设计，代码已开源。
*   **系统感知设计**：深刻理解了 GPU 内存带宽瓶颈，通过算子融合解决了“精度提升但速度变慢”的常见量化痛点。
*   **大道至简**：证明了简单的线性变换（Hadamard）在块对角化后足以解决复杂的离群值问题。

#### 8. 不足与局限
*   **压缩率上限**：本文主要聚焦于 4 位量化（4x 压缩），对于更极端的 2 位量化（如 KIVI 或 Kitty 尝试的方向），BDR 方案可能需要更复杂的补丁（如残差处理）才能维持精度。
*   **模型覆盖**：虽然测试了 Qwen 和 GLM，但尚未涵盖所有主流架构（如 Llama 3 或 MoE 架构 DeepSeek-V3 的全面评估）。
*   **硬件依赖**：虽然使用了 Triton，但其高度优化的内核性能可能在非 H100 硬件（如旧款 A100 或国产芯片）上需要重新调优。
