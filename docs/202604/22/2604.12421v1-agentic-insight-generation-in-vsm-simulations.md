## 摘要
从复杂的价值流图（VSM）模拟中提取可操作的洞察可能具有挑战性、耗时且容易出错。大语言模型的最新进展为支持用户完成这一任务提供了新途径。虽然现有方法在处理原始数据以获取信息方面表现出色，但在结构上不适合捕捉区分该领域相似数据源所需的细微情境差异。为了解决这一问题，我们提出了一种解耦的两步代理式架构。通过将编排与数据分析分离，该系统利用注入了领域专家知识的渐进式数据发现。这种架构允许编排层智能地选择数据源，并在保持精简内部上下文的同时，跨数据结构执行多跳推理。来自多个最先进大语言模型的结果证明了该框架的可行性：顶级模型实现了高达 86% 的准确率，并在多次评估运行中表现出极高的鲁棒性。

## Abstract
Extracting actionable insights from complex value stream map simulations can be challenging, time-consuming, and error-prone. Recent advances in large language models offer new avenues to support users with this task. While existing approaches excel at processing raw data to gain information, they are structurally unfit to pick up on subtle situational differences needed to distinguish similar data sources in this domain. To address this issue, we propose a decoupled, two-step agentic architecture. By separating orchestration from data analysis, the system leverages progressive data discovery infused with domain expert knowledge. This architecture allows the orchestration to intelligently select data sources and perform multi-hop reasoning across data structures while maintaining a slim internal context. Results from multiple state-of-the-art large language models demonstrate the framework's viability: with top-tier models achieving accuracies of up to 86% and demonstrating high robustness across evaluation runs.

### 2. 方法论：解耦的代理架构
论文提出了一种**解耦的两步代理架构**，核心思想是将“逻辑编排”与“数据分析”分离：
*   **编排代理（Orchestration Agent）**：作为高层规划器，负责理解用户意图并导航模拟数据的分类体系。它不直接接触原始模拟数据，从而保持精简的上下文。
*   **总结子工作流（Summarization Subworkflow）**：当编排代理确定了目标数据源后，触发该子工作流。它专门负责分析特定的数据片段（如某段库存时间序列），并将提炼后的洞察返回给编排代理。
*   **关键技术细节（四种工具）**：
    1.  **节点发现工具**：列出 VSM 中的实体（如超市、加工站）。
    2.  **属性提取工具**：获取特定节点的静态参数（如安全库存水平）。
    3.  **分类导航工具**：展示模拟输出的 11 个主要类别及其包含的数据元素描述。
    4.  **总结工具**：连接编排与分析，执行具体的数据处理任务。
*   **渐进式发现**：通过多跳推理（Multi-hop Reasoning），代理先找节点，再找属性，最后分析动态数据，实现精准定位。

*   **实验规模**：针对 47 个评估样本，每个模型配置均进行了 **4 次独立运行**，以抵消 LLM 的非确定性。
*   **充分性评价**：实验设计较为充分。通过跨模型对比（从 14B 到顶级模型）验证了架构的普适性；引入人类专家基准确保了自动评分的客观性；涵盖了多个工业领域的 VSM 场景，具有较好的代表性。

### 6. 主要结论与发现
*   **性能分层**：模型表现与规模呈正相关。Claude-Opus 4.6 表现最佳（86% 准确率），GLM-5 紧随其后（84%）。
*   **稳定性差异**：顶级模型不仅准确率高，且多次运行的方差极小（Claude 的标准差仅 0.86），而小模型（Ministral）表现出极高的不确定性（标准差 6.64）。
*   **失败分析**：约 15% 的样本所有模型均失败，主要原因是编排代理在导航复杂拓扑时选择了错误的数据源，而非子工作流的总结错误。

### 7. 优点
*   **架构创新**：解耦设计有效解决了长上下文导致的“上下文腐烂”问题，使模型能处理兆字节级别的模拟数据。
*   **知识注入**：将领域专家知识（Domain Expert Knowledge）融入工具描述中，引导代理进行专业化推理。
*   **评估严谨**：建立了可靠的 LLM-as-a-Judge 流程，并提供了详细的碳排放报告，体现了学术严谨性和社会责任。

### 8. 不足与局限
*   **小模型适配性差**：目前的渐进式发现逻辑对 14B 级别的小模型来说过于复杂，导致其导航成功率低。
*   **成本与延迟**：依赖 LLM 进行多步推理和数据总结，导致 Token 消耗量大（平均每样本约 3-4 万 Token），推理延迟较高。
*   **数据长度限制**：虽然解耦了数据，但如果单个时间序列数据量超过模型的上下文窗口，该方法仍会失效（本研究中尚未涉及超长序列）。
*   **闭环缺失**：目前仅实现了“洞察生成”，尚未实现自动修改参数并重新触发模拟的“优化闭环”。
