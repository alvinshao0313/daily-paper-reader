## 摘要
本研究提出了一种代理式、意图驱动的端到端（E2E）编排框架，该框架将意图共同创建与测试驱动的质量保证范式相结合。在该框架中，自主代理迭代地将用户的初始意图细化为经过确认且可审计的规范。此外，系统在部署前自动从这些意图中导出验证测试，直接镜像了软件工程中的测试驱动开发工作流，以确保主动的服务水平协议（SLA）合规性。该架构基于符合标准的知识表示，使用了 TM 论坛（TMF）的信息模型和目录。这实现了从高层产品供应到细粒度服务/资源及测试规范的确定性图遍历。我们通过使用消息驱动的多代理模式扩展 OpenSlice，并集成支持 MCP（模型上下文协议）的工具访问以进行实时知识检索，从而实现了该架构的原型。目前，我们对代理的评估针对意图共同创建阶段，作为迈向全规模编排的基准。基于集成 TMF 知识库的多个开源大语言模型（LLM）后端的实验，我们观察到工具使用可靠性和幻觉模式存在显著差异，这强调了在代理式 6G 系统中进行稳健知识集成的关键重要性。

## Abstract
This work proposes an agentic, intent-driven end-to-end (E2E) orchestration framework that integrates intent co-creation with a Test-Driven Quality Assurance paradigm. In this framework, autonomous agents iteratively refine a user's initial intent into a confirmed, auditable specification. Furthermore, the system automatically derives validation tests from these intents before provisioning, directly mirroring the Test-Driven Development workflow in software engineering to ensure proactive Service Level Agreement (SLA) compliance. The architecture is grounded in a standards-aligned knowledge representation using TM Forum (TMF) information models and catalogs. This enables deterministic graph traversal from high-level Product Offerings down to granular Service/Resource and Test specifications. We prototyped this architecture by extending OpenSlice with a message-driven, multi-agent pattern and integrating MCP-enabled (Model Context Protocol) tool access for real-time knowledge retrieval. Currently, our evaluation of the agents targets the intent co-creation phase as a baseline toward full-scale orchestration. Building on experiments with multiple open-source Large Language Model (LLM) backends integrated with the TMF-based knowledge base, we observe substantial variability in tool-use reliability and hallucination patterns, underscoring the critical importance of robust knowledge integration in agentic 6G systems.

### 2. 方法论
该论文提出了一种**代理式、意图驱动的端到端（E2E）编排框架**，核心思想是将软件工程中的“测试驱动开发（TDD）”引入网络服务生命周期。
*   **意图共创（Intent Co-creation）**：通过“意图共创智能体”与用户进行多轮对话，将模糊需求细化为包含成本、质量和操作约束的显式规范。
*   **测试驱动质量保证（TDQA）**：在资源配置（Provisioning）之前，系统根据确认的意图自动生成 SLO/SLA 验证测试。部署过程必须“通过”这些预设测试才算成功。
*   **基于标准的知识表示（KR）**：利用 TM Forum (TMF) 信息模型（如 TMF620 产品目录、TMF653 测试管理）构建知识图谱。智能体通过**确定性图遍历**（从产品供应到服务/资源规范）来生成执行计划。
*   **技术架构**：
    *   **OpenSlice 扩展**：基于开源 OSS 项目 OpenSlice，引入消息驱动的多智能体模式。
    *   **MCP 协议**：集成模型上下文协议（Model Context Protocol），使智能体能实时检索目录、查询库存和调用编排工具。
    *   **智能体技能（Skills）**：将专家经验封装为模块化技能，减少 Token 消耗并提高推理准确性。

### 3. 实验设计
*   **实验场景**：一个典型的“网络即服务（NaaS）”基准测试。用户请求在希腊帕特雷市提供为期一周的高清体育赛事直播服务（要求低延迟、高并发、有预算限制）。
*   **Benchmark（基准）**：设定了一个“地面真理（Ground Truth）”专家配置，包含 4 个特定产品组合、9000 欧元的预算上限及 1000 名并发用户的性能指标。
*   **对比方法**：对比了 13 种不同的开源大语言模型（LLM），分为两类：

    2.  **工具增强型通用模型**：如 Llama3.1/3.2、Mistral 系列、Granite 等。
*   **评估指标**：产品组合准确率、幻觉率（是否存在虚假产品）、成本/时长计算准确性、对话时长及 Token 消耗。

### 4. 资源与算力
*   **硬件环境**：实验运行在一个包含 3 个节点的 Kubernetes 集群上。
*   **配置**：每个节点配备 16 vCPU 和 64 GB RAM。
*   **加速器**：其中一个节点配备了 **NVIDIA RTX A6000 GPU (48 GB VRAM)** 用于模型推理。
*   **优化技术**：提到可以使用 AirLLM 等内存高效推理技术来优化边缘部署。

### 5. 实验数量与充分性
*   **实验规模**：针对 13 个模型进行了多轮对话测试（Q1-Q5 五个阶段的查询）。
*   **充分性评价**：
    *   **广度**：覆盖了从 1.7b 到 32b 不同参数规模的主流开源模型，具有较好的代表性。
    *   **深度**：实验重点评估了“意图共创”这一关键环节，这是实现 TDQA 的前提。
    *   **局限性**：实验场景相对单一（仅一个 NaaS 案例），且目前主要集中在编排前的意图处理阶段，对于后端资源配置后的实时闭环治理实验描述较少。

### 6. 主要结论与发现
*   **模型表现差异巨大**：**Gpt-oss:20b** 表现最佳，能够准确理解约束、无幻觉且正确处理成本计算。
*   **工具调用可靠性是瓶颈**：许多模型（如 Llama3.1、Mistral）在调用 MCP 工具或处理 TMF 规范时出现失败或产生幻觉（虚构产品 ID）。
*   **推理能力不等于操作能力**：一些具有强推理能力的模型（如 Deepseek-r1）在特定工具调用接口上支持不佳。
*   **知识集成的重要性**：在 6G 代理系统中，稳健的知识库（KRL）集成比单纯的模型规模更重要。

### 7. 优点
*   **范式创新**：首次将 TDD 理念引入 6G 网络编排，将 QA 从被动变为主动。
*   **标准化程度高**：严格遵循 TM Forum 和 ETSI 标准，保证了电信级应用的可行性。
*   **架构解耦**：通过 MCP 和智能体技能实现认知层（AI）与执行层（编排器）的分离，确保了系统的确定性和安全性。

### 8. 不足与局限
*   **实验覆盖面**：目前的评估主要集中在“意图共创”阶段，对于全生命周期的自动化修复（Self-healing）和动态测试演进验证不足。
*   **幻觉风险**：实验显示开源模型仍存在较高的幻觉风险，这在电信生产环境中可能导致严重的配置错误。
*   **实时性挑战**：多轮对话和复杂的图遍历可能带来编排延迟，论文未深入探讨在高动态 6G 场景下的实时响应能力。
