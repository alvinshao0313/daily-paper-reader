## 摘要
具身智能体（Embodied AI agents）日益依赖大语言模型（LLMs）进行规划，但每步调用 LLM 会带来严重的延迟和成本。在本文中，我们展示了具身任务表现出极强的规划局部性（plan locality），即下一个规划在很大程度上可以根据当前规划进行预测。基于此，我们引入了 AgenticCache，这是一个通过重用缓存规划来避免每步 LLM 调用的规划框架。在 AgenticCache 中，每个智能体查询频繁规划转换的运行时缓存，而后台的缓存更新器（Cache Updater）则异步调用 LLM 以验证和优化缓存条目。在四个多智能体具身基准测试中，AgenticCache 在 12 种配置（4 个基准测试 x 3 个模型）下平均将任务成功率提高了 22%，模拟延迟降低了 65%，Token 使用量减少了 50%。因此，基于缓存的规划重用为实现低延迟、低成本的具身智能体提供了一条切实可行的路径。代码可在 https://github.com/hojoonleokim/MLSys26_AgenticCache 获取。

## Abstract
Embodied AI agents increasingly rely on large language models (LLMs) for planning, yet per-step LLM calls impose severe latency and cost. In this paper, we show that embodied tasks exhibit strong plan locality, where the next plan is largely predictable from the current one. Building on this, we introduce AgenticCache, a planning framework that reuses cached plans to avoid per-step LLM calls. In AgenticCache, each agent queries a runtime cache of frequent plan transitions, while a background Cache Updater asynchronously calls the LLM to validate and refine cached entries. Across four multi-agent embodied benchmarks, AgenticCache improves task success rate by 22% on average across 12 configurations (4 benchmarks x 3 models), reduces simulation latency by 65%, and lowers token usage by 50%. Cache-based plan reuse thus offers a practical path to low-latency, low-cost embodied agents. Code is available at https://github.com/hojoonleokim/MLSys26_AgenticCache.

*   **核心问题**：具身智能体（Embodied AI）在执行任务时，通常依赖大语言模型（LLM）进行“感知-规划-动作”循环。然而，**每一步都同步调用 LLM 会导致严重的推理延迟和高昂的 Token 成本**，阻碍了智能体的实时响应能力。
*   **研究动机**：作者观察到具身任务具有强烈的**规划局部性（Plan Locality）**，即下一个规划动作往往可以从当前状态和历史动作中预测（例如：抓取物体后，下一步通常是运输）。受人类直觉反应（非每步都深思熟虑）的启发，作者提出通过缓存机制重用规划，从而减少对 LLM 的频繁调用。

### 2. 方法论
AgenticCache 包含两个核心组件：运行时缓存（Runtime Cache）和异步缓存更新器（Cache Updater）。
*   **核心思想**：将规划过程转变为“缓存查询为主，LLM 异步修正为辅”的模式。
*   **关键技术细节**：
    *   **2-Gram 缓存结构**：存储规划转换对 $\langle P_i \to P_j \rangle$，并附带任务状态元数据（如步骤数、持有物品数、完成进度等）。
    *   **评分与选择机制**：通过公式 $S(P_i \to P_j) = C \times I$ 选择最佳规划。其中 $C$ 是局部转换频率（Count），$I$ 是全局重要性（Importance，即 LLM 确认该规划正确的比例）。
    *   **异步更新器**：LLM 在后台运行，不阻塞智能体动作。它执行两种操作：
        *   **确认（Confirmation）**：若 LLM 预测与缓存一致，则增强该缓存条目的权重并抑制冗余查询。
        *   **修正（Correction）**：若 LLM 发现缓存规划错误，立即替换当前正在执行的动作，并更新缓存。
    *   **预填充（Warm-start）**：可选步骤，利用少量离线成功轨迹初始化缓存，解决冷启动问题。

### 6. 主要结论与发现
*   **性能提升**：AgenticCache 平均将任务成功率提高了 **22%**。
*   **效率优化**：模拟延迟平均降低了 **65%**（最高达 86%），Token 使用量减少了 **50%**（最高达 79%）。

*   **内存友好**：每个智能体的缓存仅占用 **KB 级别** 的空间，且增长曲线随任务熟练度趋于平缓。

### 7. 优点
*   **架构创新**：将计算机体系结构中的“缓存”和“分支预测”思想巧妙引入具身智能体规划，实现了计算效率与任务性能的双赢。
*   **异步解耦**：通过异步更新器解决了 LLM 推理慢的痛点，使智能体能够“边做边想”，而不是“想好再做”。
*   **实用性强**：显著降低了使用昂贵模型（如 GPT-5）的经济成本，且兼容现有的多智能体框架。

### 8. 不足与局限
*   **多智能体冲突**：在需要极高同步性的协作点，缓存重用可能导致智能体动作不一致（如两个智能体同时抢夺一个资源），虽然更新器最终能修正，但会存在短暂的延迟。
*   **局部性依赖**：该方法依赖于任务的结构化程度。在完全开放、无规律的探索任务中，规划局部性可能变弱，导致缓存命中率下降。
*   **2-Gram 限制**：目前的 2-Gram 模型较简单，可能无法捕捉更长程的逻辑依赖（未来可能需要 3-Gram 或分层子程序）。
