## 摘要
大语言模型（LLMs）正越来越多地被部署用于自主解决现实世界的任务。其中的一个关键要素是 LLM 函数调用（Function-Calling）范式，这是一种为 LLMs 配备工具使用能力的广泛采用的方法。然而，LLM 错误地调用函数可能会产生严重后果，特别是当其影响不可逆时，例如转账或删除数据。因此，在执行函数调用之前，考虑 LLM 对该调用能正确解决任务的置信度至关重要。不确定性量化（UQ）方法可用于量化这种置信度，并防止潜在的错误函数调用。在这项工作中，我们展示了据我们所知首个针对 LLM 函数调用（FC）的 UQ 方法评估。虽然多样本 UQ 方法（如语义熵）在自然语言问答任务中表现强劲，但我们发现在 FC 设置下，它相比简单的单样本 UQ 方法并没有明显的优势。此外，我们发现可以利用 FC 输出的特殊性来提高现有 UQ 方法在该设置下的性能。具体而言，多样本 UQ 方法受益于基于抽象语法树解析对 FC 输出进行聚类，而单样本 UQ 方法可以通过在计算基于 logit 的不确定性分数时仅选择具有语义意义的 token 来得到改进。

## Abstract
Large Language Models (LLMs) are increasingly deployed to autonomously solve real-world tasks. A key ingredient for this is the LLM Function-Calling paradigm, a widely used approach for equipping LLMs with tool-use capabilities. However, an LLM calling functions incorrectly can have severe implications, especially when their effects are irreversible, e.g., transferring money or deleting data. Hence, it is of paramount importance to consider the LLM's confidence that a function call solves the task correctly prior to executing it. Uncertainty Quantification (UQ) methods can be used to quantify this confidence and prevent potentially incorrect function calls. In this work, we present what is, to our knowledge, the first evaluation of UQ methods for LLM Function-Calling (FC). While multi-sample UQ methods, such as Semantic Entropy, show strong performance for natural language Q&A tasks, we find that in the FC setting, it offers no clear advantage over simple single-sample UQ methods. Additionally, we find that the particularities of FC outputs can be leveraged to improve the performance of existing UQ methods in this setting. Specifically, multi-sample UQ methods benefit from clustering FC outputs based on their abstract syntax tree parsing, while single-sample UQ methods can be improved by selecting only semantically meaningful tokens when calculating logit-based uncertainty scores.

### 1. 核心问题与整体含义
*   **研究动机**：随着 LLM 被赋予调用外部 API 和执行函数的能力（如转账、删除数据），其行为的不可逆性带来了巨大的安全风险。如果模型在不确定的情况下盲目执行错误指令，后果将非常严重。
*   **核心问题**：现有的 UQ 方法（如语义熵）主要针对自然语言问答（Q&A）设计，在结构化程度极高的“函数调用（FC）”场景下是否依然有效？如何利用 FC 的结构化特性改进 UQ 性能？
*   **整体含义**：这是首个针对 LLM 函数调用不确定性量化的系统性评估工作，旨在为构建更安全、可信的自主 AI 代理提供理论支持。

### 2. 论文提出的方法论
论文评估了现有方法，并针对 FC 场景提出了两项关键改进：
*   **多样本 UQ 方法的改进（AST 聚类）**：
    *   **核心思想**：传统的语义熵（SE）通过 LLM 评估文本含义是否一致来聚类。但在 FC 中，参数顺序不同（如 `f(a=1, b=2)` 与 `f(b=2, a=1)`）在语义上是等价的。
    *   **技术细节**：利用**抽象语法树（AST）**解析函数调用，将输出转换为字典结构进行匹配。如果两个输出的 AST 一致，则归为同一类，从而更准确地计算语义熵（SE-AST）。
*   **单样本 UQ 方法的改进（SMT 筛选）**：
    *   **核心思想**：并非所有 token 都能反映模型对逻辑的把握。语法 token（如括号、等号）通常概率极高，会稀释关键决策 token 的不确定性。
    *   **技术细节**：提出**语义关键标记（SMT）**提取算法。仅针对函数名、参数名、参数值等关键位置的 token 计算负对数似然（NLL），排除掉纯语法性质的 token。
*   **对比基准方法**：包括单样本的 MAX（最大 NLL）、G-NLL（序列 NLL 总和）、AVG（平均 NLL）以及多样本的 PE（预测熵）、SE（语义熵）、Ptrue（提示词自评）。

### 3. 实验设计
*   **数据集**：基于 **Berkeley Function Calling Leaderboard (BFCL)**。
*   **实验场景**：
    *   **Simple**：单函数调用。
    *   **Multiple**：从多个函数中选一。
    *   **Parallel**：并行调用多个函数。
    *   **Parallel-Multiple**：复杂混合场景。
    *   **Irrelevance**：不可回答的请求（测试模型是否会拒绝调用）。
*   **评估模型**：共 8 个开源模型，包括 Qwen2.5 (0.5B/3B/7B)、Qwen3-4B、Ministral-8B、Gemma-2-9B、Gemma-3 (4B/12B)。
*   **评价指标**：**AUROC**（衡量区分正确与错误调用的能力）和 **SmoothECE**（衡量置信度校准水平）。

### 4. 资源与算力
*   **算力设备**：使用单节点 **4 张 NVIDIA A100 GPU** 运行 vLLM 进行推理。
*   **耗时**：
    *   生成贪婪采样（Greedy samples）：8 个模型处理 1240 个请求约需 40 分钟。
    *   生成多样本采样（每请求 10 个样本）：约需 3 小时。
*   **后续计算**：UQ 分数的计算在 MacBook Pro (M1 Max, 64GB RAM) 上完成，每个模型/任务的计算时间不足 1 分钟。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 8 种不同规模和架构的模型，并在 10 种不同的任务组合（包括单一任务和混合任务）上进行了测试。
*   **消融实验**：对多样本方法的温度（T）和采样数量（J）进行了敏感性分析（T 从 0.5 到 3.0，J 从 5 到 40）。
*   **客观性**：通过排除无法解析的 AST 错误（避免干扰 UQ 评分）并使用自助法（Bootstrap）计算标准差，确保了结果的统计显著性。实验设计覆盖了从简单到复杂、从可回答到不可回答的多种现实维度，非常充分。

### 6. 主要结论与发现
*   **单样本胜过复杂方法**：在 FC 场景下，简单的 **G-NLL（单样本序列似然）** 表现优于复杂的语义熵（SE）。这与自然语言问答场景的结论相反。
*   **FC 的特殊性**：FC 场景中 token 的平均概率极高（0.97，而问答场景仅 0.71），模型通常只在极少数关键 token 上产生波动。
*   **结构化改进有效**：
    *   使用 **AST 聚类** 提升了多样本方法的 AUROC。
    *   使用 **SMT 筛选** 提升了单样本方法的 AUROC 和校准度（ECE）。
*   **校准度差异**：MAX 方法在校准度上表现最好，而 AVG 方法普遍存在过度自信的问题。

### 7. 优点
*   **首创性**：填补了 LLM 工具使用领域 UQ 评估的空白。
*   **实用性强**：提出的 SMT 和 AST 改进方法不需要重新训练模型，属于即插即用的后处理方案。
*   **分析深入**：不仅给出了分数，还深入分析了 FC 场景下 token 概率分布的特性，解释了为什么传统方法失效。

### 8. 不足与局限
*   **模型覆盖**：未包含闭源顶尖模型（如 GPT-4, Claude 3.5），因为这些模型通常不提供 token 级别的 logit 概率。
*   **场景局限**：实验主要集中在单轮 Python 函数调用，未涉及多轮对话、长链条 Agent 规划或非 Python 语言（如 Java/JS 仅占极小比例）。
*   **SMT 依赖格式**：SMT 提取算法需要针对不同的输出格式（如 JSON 或 Python 列表）进行微调，虽然论文证明了可以用 GPT 辅助生成算法，但仍存在一定的工程维护成本。
