## 摘要
当前的 LLM 智能体在一个隐含但普遍的假设下运行：执行是一个事务——用户提交请求，智能体独立工作，只有在完成后对话才会恢复。这迫使用户面临二选一的境地：要么等待可能错误的结果，要么中断并丢失所有进度。我们拒绝这一假设，并提出了流式范式（stream paradigm），其中智能体执行和用户干预是共享双向通道的并发、交错过程。我们通过可逆性分类法将这一范式形式化，该分类法将每个智能体动作归类为幂等（Idempotent）、可逆（Reversible）、可补偿（Compensable）或不可逆（Irreversible），并得出一个核心结论：智能体的灵活性受其可逆性的限制。我们证明了冲突的可补偿动作会带来不可避免的适应成本，而冲突的不可逆动作使得完全满足规范变得不可能——这些成本是动作空间的属性，而非算法的属性。在这一见解的指导下，我们提出了 Revision Absorber，这是一种基于最早冲突回滚（Earliest-Conflict Rollback）规则的反应式算法，在温和的假设下具有结构最优性。在 StreamBench 上对真实 LLM 智能体进行的实验验证了所有预测：Absorber 的质量与暴力全重启基线相当，同时减少了一个数量级的已完成工作浪费，将执行过程中的修正从死胡同转变为一等公民式的交互。

## Abstract
Current LLM agents operate under an implicit but universal assumption: execution is a transaction -- the user submits a request, the agent works in isolation, and only upon completion does the dialogue resume. This forces users into a binary choice: wait for a potentially incorrect output, or interrupt and lose all progress. We reject this assumption and propose the stream paradigm, in which agent execution and user intervention are concurrent, interleaved processes sharing a bidirectional channel. We formalize this paradigm through a reversibility taxonomy that classifies every agent action as Idempotent, Reversible, Compensable, or Irreversible, and arrive at a core conclusion: an agent's flexibility is bounded by its reversibility. We prove that conflicting compensable actions impose unavoidable adaptation costs and that conflicting irreversible actions make full specification satisfaction impossible -- these costs are properties of the action space, not of the algorithm. Guided by this insight, we present the Revision Absorber, a reactive algorithm based on the Earliest-Conflict Rollback rule that is structurally optimal under mild assumptions. Experiments on StreamBench with real LLM agents validate all predictions: the Absorber matches the quality of a brute-force full-restart baseline while wasting an order of magnitude fewer steps of already-completed work, turning mid-execution revisions from a dead-end into a first-class interaction.

### 1. 核心问题与整体含义（研究动机和背景）
当前大语言模型（LLM）智能体遵循**“事务性假设”**：用户提交请求后，智能体进入封闭的执行循环，用户只能在任务结束或失败后重新介入。这种模式在长程任务中存在严重弊端：如果用户在中途发现意图偏差或约束变化，只能选择“等待错误结果”或“中断并丢失所有进度”。

本文提出了**流式执行范式（Stream Paradigm）**，旨在打破这种二元对立。研究的核心动机是建立一套理论，使智能体能够通过双向非阻塞通道，在执行过程中实时吸收用户的修正（Revisions），并最小化重复工作的代价。

### 2. 方法论：核心思想与关键技术
*   **状态分解**：将智能体状态分为**认知状态（Epistemic State）**（通过截断上下文即可回滚）和**世界状态（World State）**（外部环境的改变，部分可回滚）。
*   **可逆性分类法（{I, R, K, X}）**：
    *   **I (Idempotent, 幂等)**：不改变世界状态（如读文件），无需回滚。
    *   **R (Reversible, 可逆)**：存在精确逆操作（如创建/删除文件），回滚代价小。
    *   **K (Compensable, 可补偿)**：无法精确撤销但可补救（如发错邮件后发更正邮件），有一定代价。
    *   **X (Irreversible, 不可逆)**：无法撤销或代价极高（如完成金融交易）。
*   **核心理论**：证明了智能体的灵活性边界由其动作空间的可逆性决定。冲突的 K 类动作会产生不可避免的补偿成本，而冲突的 X 类动作会导致任务无法完全满足新规范。
*   **Revision Absorber 算法**：
    *   基于**最早冲突回滚（Earliest-Conflict Rollback）**规则。
    *   **流程**：当收到修正 $\phi$ 时，扫描执行轨迹，找到第一个与新规范冲突的 K 或 X 类动作；保留该动作之前的 I/R 前缀，对冲突点之后的动作进行补偿操作，然后从冲突点开始重新规划并执行。

### 3. 实验设计
*   **数据集与场景（StreamBench）**：构建了三个多步工具调用场景（活动策划、旅行安排、报告发布），每个场景包含 12-15 个步骤。
*   **可逆率控制**：通过调整可用工具中 K/X 类的比例，设置了四个可逆率等级 $\rho \in \{1.0, 0.75, 0.5, 0.25\}$。
*   **对比方法**：
    1.  **Oracle**：上限，从一开始就已知修正。
    2.  **Full-Restart**：暴力重启，补偿所有已执行动作并从头开始。
    3.  **Naive Absorber**：仅将修正追加到上下文，不进行回滚。
    4.  **Ignore**：下限，完全忽略修正。
*   **评估指标**：输出质量（LLM-as-judge 评分 1-5）、浪费的步骤数（Wasted acts）、补偿成本、Token 消耗。

### 6. 主要结论与发现

*   **结构性限制**：实验证实了“灵活性受限于可逆性”的理论，即任务的可逆率 $\rho$ 越低，任何算法都无法避免更高的适应成本。
*   **算法优越性**：最早冲突回滚规则在理论上是结构最优的，实验显示其在各种修正类型下均表现稳健。
*   **能力耦合**：修正的效果与基础模型的能力正相关，更强的模型（如 DeepSeek-V3）能更好地利用回滚后的上下文进行重新规划。

*   **交互范式创新**：将智能体从“黑盒事务”转变为“透明流式过程”，更符合人类协作的直觉。

### 8. 不足与局限
