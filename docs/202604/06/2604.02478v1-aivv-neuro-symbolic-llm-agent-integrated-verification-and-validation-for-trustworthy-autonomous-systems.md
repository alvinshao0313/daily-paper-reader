## 摘要
深度学习模型擅长检测正常数据中的异常模式。然而，它们无法为跨多种控制系统的异常分类和可扩展性提供直接解决方案，且经常无法区分真实故障与由噪声或控制系统大幅瞬态响应引起的干扰故障。因此，由于算法故障确认仍难以扩展，完整的验证与确认（V&V）操作仍由人机回环（HITL）分析管理，导致了不可持续的人工工作量。为了实现这一关键监管的自动化，我们提出了智能体集成验证与确认（AIVV），这是一种将大语言模型（LLM）部署为审慎外环的混合框架。由于严谨的系统验证严格依赖于准确的确认，AIVV 将数学标记的异常上报给一个由特定角色组成的 LLM 委员会。委员会智能体通过基于自然语言（NL）需求对干扰和真实故障进行语义确认，开展协作式确认，以确保高保真度的系统验证基准。在此基础上，委员会随后通过根据自然语言运行容差评估故障后响应来进行系统验证，最终生成可操作的 V&V 产物，例如增益调整建议。在无人水下航行器（UUV）时间序列模拟器上的实验表明，AIVV 成功实现了 HITL V&V 过程的数字化，克服了基于规则的故障分类的局限性，并为时间序列数据领域中由 LLM 介导的监管提供了可扩展的蓝图。

## Abstract
Deep learning models excel at detecting anomaly patterns in normal data. However, they do not provide a direct solution for anomaly classification and scalability across diverse control systems, frequently failing to distinguish genuine faults from nuisance faults caused by noise or the control system's large transient response. Consequently, because algorithmic fault validation remains unscalable, full Verification and Validation (V\&V) operations are still managed by Human-in-the-Loop (HITL) analysis, resulting in an unsustainable manual workload. To automate this essential oversight, we propose Agent-Integrated Verification and Validation (AIVV), a hybrid framework that deploys Large Language Models (LLMs) as a deliberative outer loop. Because rigorous system verification strictly depends on accurate validation, AIVV escalates mathematically flagged anomalies to a role-specialized LLM council. The council agents perform collaborative validation by semantically validating nuisance and true failures based on natural-language (NL) requirements to secure a high-fidelity system-verification baseline. Building on this foundation, the council then performs system verification by assessing post-fault responses against NL operational tolerances, ultimately generating actionable V\&V artifacts, such as gain-tuning proposals. Experiments on a time-series simulator for Unmanned Underwater Vehicles (UUVs) demonstrate that AIVV successfully digitizes the HITL V\&V process, overcoming the limitations of rule-based fault classification and offering a scalable blueprint for LLM-mediated oversight in time-series data domains.

#### 1. 核心问题与研究动机
*   **核心问题**：当前的深度学习模型（如 LSTM、Transformer）在自主系统（如无人水下航行器 UUV）的异常检测中表现出色，但存在**语义盲区**。它们难以区分“真实故障”与“干扰故障”（由环境噪声或系统正常瞬态响应引起），且无法提供故障分类或系统重新设计的建议。
*   **研究背景**：由于数学模型无法进行语义推理，目前的验证与确认（V&V）流程高度依赖“人机回环”（HITL）。然而，人工分析数千个传感器流的速度远低于毫秒级的数学检测，导致监管效率低下且不可持续。
*   **目标**：开发一种能够自动化替代人工专家进行语义推理、故障验证及系统调优建议的自动化框架。

#### 2. 方法论：AIVV 混合架构
AIVV 采用**神经符号（Neuro-Symbolic）**混合架构，将高速数学检测与深度语义推理相结合：
*   **数学引擎层（底层）**：使用带有蒙特卡罗丢弃（MC Dropout）的 LSTM 进行不确定性估计，并结合**共形预测（Conformal Prediction）**建立统计保证的误差边界（$C_\alpha$）。
*   **多智能体系统（MAS）管线（外环）**：
    *   **哨兵（Sentry）**：确定性闸门。当数学误差超过共形边界时，将数据上报给 LLM 委员会。
    *   **委员会（Council）**：由三个角色化智能体组成（需求工程师、故障管理器、系统工程师），基于自然语言需求进行投票（3 选 2 多数制），判断是“真实故障”还是“干扰”。
    *   **自适应管线（Inspector & Tuner）**：若判定为干扰，则触发模型重校准或微调。采用“克隆并晋升”机制，先在临时模型上验证，通过后再更新部署模型，防止灾难性遗忘。
*   **V&V 产物生成**：系统工程师智能体根据故障模式分析，自动生成 PID 控制器的增益调整建议（如调整 $K_p, T_i, T_d$）。

#### 3. 实验设计
*   **实验场景**：基于 Simulink 的 REMUS 100 无人水下航行器（UUV）模拟器。
*   **数据集**：生成了三种航行任务：
    1.  **悬停（Hovering）**：低动态场景。
    2.  **割草机测绘模式（Lawnmower Pattern）**：周期性转向。
    3.  **复杂任务（Complex Mission）**：非周期性航向修正。
*   **故障注入**：注入传感器漂移、电气故障和机械阻尼器故障。
*   **对比基准（Benchmark）**：
    *   纯数学引擎（Baseline）。
    *   数学引擎 + LLM 委员会。
    *   完整 AIVV 框架（包含自适应管线）。
*   **评估指标**：故障验证率（FVR）、检测准确率、延迟及令牌消耗。

#### 4. 资源与算力
*   **硬件**：使用了 **Intel Data Center GPU Max Series** 加速器（具备高带宽内存和 L2 缓存）。
*   **LLM 推理**：通过 Groq 的自定义**语言处理单元（LPU）**调用 API，以降低推理延迟。
*   **模型选择**：使用了异构模型组合，包括 LLaMA-3.3-70B、LLaMA-4-17B、GPT-OSS-120B、Qwen3-32B 等，根据任务复杂度分配不同规模的模型。

#### 5. 实验数量与充分性
*   **实验规模**：针对每个数据集进行了 **75 组独立随机种子**的实验，每组包含 409 个测试点。
*   **消融实验**：通过逐步添加委员会和自适应管线，验证了各组件对降低误报率（FPR）和提高 FVR 的贡献。
*   **模型对齐实验**：进行了系统性的模型角色轮换实验（Cyclic Shift），证明了特定任务（如故障管理）需要特定能力等级的模型，体现了实验的严谨性。

#### 6. 主要结论与发现
*   **验证效能**：AIVV 在悬停任务中达到了 **100% 的故障验证率（FVR）**，在复杂任务中也达到了 93.33%，显著优于纯数学模型。
*   **自适应优势**：在复杂任务中，通过自适应管线微调数学模型，准确率提升了 **23.11%**。
*   **数字化 HITL**：成功将原本需要数分钟甚至数小时的人工 V&V 流程缩短至秒级（平均端到端延迟约 6 秒）。
*   **闭环建议**：系统工程师生成的增益调整建议在 Simulink 中得到了验证，证明了其生成的工程产物具有实际指导意义。

#### 7. 优点
*   **神经符号结合**：利用数学模型保证严谨性，利用 LLM 提供语义上下文，解决了传统异常检测“知其然而不知其所以然”的问题。
*   **角色化协作**：通过多智能体委员会模拟人类专家团队的会诊过程，有效抑制了 LLM 的幻觉。
*   **安全更新机制**：引入“临时克隆-验证-晋升”的逻辑，确保了在线学习过程中系统的稳定性。

#### 8. 不足与局限
*   **实时性挑战**：尽管使用了 LPU 加速，端到端延迟仍达 6 秒左右，对于极高频控制环路（如毫秒级响应要求）可能仍显过慢。
*   **领域局限性**：目前仅在 UUV 的偏航角（Yaw）控制上进行了验证，尚未扩展到多变量耦合的复杂系统。
*   **依赖外部 API**：系统性能高度依赖 LLM API 的可用性和稳定性，在离线或极端环境下部署受限。
*   **成本问题**：多智能体多次调用 LLM 会产生较高的令牌（Token）成本。
