## 摘要
多数投票是将多智能体响应聚合为最终决策的标准方法。然而，传统方法通常要求所有智能体在聚合开始前完成推理，这导致了显著的计算开销，因为一旦达成多数共识，许多响应就变得冗余。在这项工作中，我们将多智能体投票建模为一个可靠性感知的智能体调度问题，并提出了一种高效的“多数即停止”（Efficient Majority-then-Stopping, EMS）方法来提高推理效率。EMS 根据任务感知的可靠性对智能体进行优先级排序，并在通过以下三个关键组件达成多数意见时立即终止推理流水线。具体而言，我们引入了智能体置信度建模（Agent Confidence Modeling, ACM）以利用历史表现和语义相似度评估智能体可靠性，引入了自适应增量投票（Adaptive Incremental Voting, AIV）以顺序选择智能体并实现提前停止，以及引入了个人置信度更新（Individual Confidence Updating, ICU）以动态更新每个参与智能体的可靠性。在六个基准测试上的广泛评估表明，EMS 能够稳定地将调用的智能体平均数量减少 32%。

## Abstract
Majority voting is the standard for aggregating multi-agent responses into a final decision. However, traditional methods typically require all agents to complete their reasoning before aggregation begins, leading to significant computational overhead, as many responses become redundant once a majority consensus is achieved. In this work, we formulate the multi-agent voting as a reliability-aware agent scheduling problem, and propose an Efficient Majority-then-Stopping (EMS) to improve reasoning efficiency. EMS prioritizes agents based on task-aware reliability and terminates the reasoning pipeline the moment a majority is achieved from the following three critical components. Specifically, we introduce Agent Confidence Modeling (ACM) to estimate agent reliability using historical performance and semantic similarity, Adaptive Incremental Voting (AIV) to sequentially select agents with early stopping, and Individual Confidence Updating (ICU) to dynamically update the reliability of each contributing agent. Extensive evaluations across six benchmarks demonstrate that EMS consistently reduces the average number of invoked agents by 32%.

*   **核心问题**：在多智能体系统（MAS）中，**多数投票（Majority Voting）**是聚合决策的标准方法。然而，传统方法遵循“先推理、后聚合”的范式，要求所有智能体完成推理后才开始统计。
*   **研究动机**：在实际场景中，一旦达到多数共识（如 $N$ 个智能体中有 $\lceil(N+1)/2\rceil$ 个达成一致），剩余智能体的推理便成为冗余，造成了巨大的计算资源和时间浪费。本文旨在解决如何在不损失准确性的前提下，通过减少冗余推理来提升多智能体系统的推理效率。

#### 2. 方法论
论文提出了 **EMS (Efficient Majority-then-Stopping)** 框架，将投票过程建模为“可靠性感知”的智能体调度问题。核心组件包括：
*   **智能体置信度建模 (ACM)**：
    *   **历史可靠性**：基于智能体过去参与投票且与最终共识一致的频率来评分。
    *   **语义可靠性**：利用预训练编码器提取当前查询的语义特征，计算其与智能体历史成功解决的查询之间的相似度，从而评估智能体对特定任务的擅长程度。
*   **自适应增量投票 (AIV)**：
    *   根据 ACM 计算的得分对智能体进行降序排列。
    *   **初始投票**：首先调用前 $\tau = \lceil(N+1)/2\rceil$ 个最可靠的智能体。
    *   **增量检查**：若初始投票未达成共识，则按序逐个引入后续智能体，直到满足多数条件或所有智能体均已投票。
*   **个体置信度更新 (ICU)**：
    *   动态更新参与投票智能体的状态（参与次数、共识次数、语义缓冲区），形成闭环反馈，使调度策略随任务进行而进化。

#### 7. 优点与亮点
*   **即插即用**：EMS 作为一个决策框架，不改变底层模型，可直接应用于现有的多智能体协作系统。
*   **可靠性感知**：引入了任务感知的动态调度逻辑，改变了传统 MAS 盲目调用所有智能体的现状。
*   **理论结合**：借鉴了分布式系统中的拜占庭容错（BFT）法定人数原则，具有一定的理论支撑。

#### 8. 不足与局限
*   **推理延迟（Wall-clock Time）**：虽然减少了总计算量（Token 消耗），但由于采用了**顺序/增量**调用模式，在最坏情况下（无法达成早期共识时），其总响应时间可能高于并行调用的传统投票。
*   **排序策略尚浅**：目前主要依赖简单的历史统计和语义相似度，对于极其复杂或分布偏移（Out-of-distribution）的任务，排序的准确性可能受限。
*   **应用限制**：主要适用于有明确答案（如选择题、数学题）的任务，对于开放式生成或主观评价任务，如何定义“多数共识”仍是挑战。
