## 摘要
基于大语言模型（LLM）的自主智能体缺乏持久的程序性记忆：即使之前已经解决过结构相同的任务，它们仍会从头开始推导解决方案。我们提出了 APEX-EM，这是一种非参数化在线学习框架，能够在不修改模型权重的情况下积累、检索和重用结构化程序计划。APEX-EM 引入了：(1) 一种结构化经验表示，编码了每次执行的完整程序-情境轨迹——包括规划步骤、中间产物、带有错误分析的迭代历史以及质量评分；(2) 一种包含任务验证器（Task Verifiers）的“规划-检索-生成-迭代-摄入”（PRGII）工作流，任务验证器提供多维奖励信号；(3) 一种双结果经验记忆，采用结合了语义搜索、结构特征匹配和计划有向无环图（DAG）遍历的混合检索方式——实现了在词汇无重叠但操作结构相似的任务之间的跨领域迁移。成功的经验作为正向上下文示例；失败的经验则作为带有结构化错误标注的负面示例。我们使用 Claude Sonnet 4.5 和 Opus 4.5 在 BigCodeBench、KGQAGen-10k 和 Humanity's Last Exam (HLE) 上进行了评估。在 KGQAGen-10k 上，APEX-EM 达到了 89.6% 的准确率，而无记忆情况下为 41.3%（提升了 48.3 个百分点），超过了真值检索（oracle-retrieval）的上限（84.9%）。在 BigCodeBench 上，其成功率（SR）从 53.9% 的基准提升至 83.3%（提升了 29.4 个百分点），在骨干网络冻结的同等条件下，超过了 MemRL 11.0 个百分点的增益（注：我们在分析中对骨干网络的差异进行了控制）。在 HLE 上，实体图检索从 25.2% 提升至 48.0%（提升了 22.8 个百分点）。消融实验表明，各组件的价值取决于具体任务：丰富的评判反馈对代码生成的影响微乎其微，但对结构化查询至关重要（提升了 10.3 个百分点），而基于二元信号的迭代可以部分补偿较弱的反馈。

## Abstract
LLM-based autonomous agents lack persistent procedural memory: they re-derive solutions from scratch even when structurally identical tasks have been solved before. We present APEX-EM, a non-parametric online learning framework that accumulates, retrieves, and reuses structured procedural plans without modifying model weights. APEX-EM introduces: (1) a structured experience representation encoding the full procedural-episodic trace of each execution -- planning steps, artifacts, iteration history with error analysis, and quality scores; (2) a Plan-Retrieve-Generate-Iterate-Ingest (PRGII) workflow with Task Verifiers providing multi-dimensional reward signals; and (3) a dual-outcome Experience Memory with hybrid retrieval combining semantic search, structural signature matching, and plan DAG traversal -- enabling cross-domain transfer between tasks sharing no lexical overlap but analogous operational structure. Successful experiences serve as positive in-context examples; failures as negative examples with structured error annotations.   We evaluate on BigCodeBench, KGQAGen-10k, and Humanity's Last Exam using Claude Sonnet 4.5 and Opus 4.5. On KGQAGen-10k, APEX-EM achieves 89.6% accuracy versus 41.3% without memory (+48.3pp), surpassing the oracle-retrieval upper bound (84.9%). On BigCodeBench, it reaches 83.3% SR from a 53.9% baseline (+29.4pp), exceeding MemRL's +11.0pp gain under comparable frozen-backbone conditions (noting backbone differences controlled for in our analysis). On HLE, entity graph retrieval reaches 48.0% from 25.2% (+22.8pp). Ablations show component value is task-dependent: rich judge feedback is negligible for code generation but critical for structured queries (+10.3pp), while binary-signal iteration partially compensates for weaker feedback.

### 1. 论文的核心问题与整体含义
**研究动机与背景：**
当前基于大语言模型（LLM）的自主智能体在程序层面本质上是“无状态”的。即使面对结构完全相同的任务，智能体通常也会从零开始重新推导解决方案，无法有效复用过往经验。现有的记忆机制（如 MemRL）多采用扁平的文本摘要或标量奖励，存在三个局限：
1. **缺乏程序结构：** 丢失了规划步骤、实体发现和因果链条。
2. **缺乏负面经验利用：** 未能系统性地从失败中学习。
3. **跨领域迁移困难：** 检索高度依赖表面语义相似性，当任务描述词汇不同但逻辑结构相似时，检索往往失效。

**核心目标：**
开发一种名为 **APEX-EM** 的非参数化在线学习框架，通过构建“程序-情境”记忆，使智能体能够在不修改模型权重的情况下，像强化学习中的经验回放（Experience Replay）一样积累并复用结构化经验。

### 2. 方法论：核心思想与关键技术
APEX-EM 的核心在于将经验转化为**程序化知识图谱（PKG）**，并通过 **PRGII 工作流** 进行管理。

*   **程序化知识图谱 (PKG)：** 存储实体、经验、子任务和操作。经验节点包含五层反射：目标反射、程序反射（可复用的步骤模板）、证据反射、错误注册表（记录为何失败）和补丁反射（记录修复方案）。
*   **PRGII 工作流 (算法流程)：**
    1.  **Plan (规划)：** 分解任务，提取“结构特征（Structural Signature）”（如：实体解析 → 时间过滤 → 聚合）。
    2.  **Retrieve (检索)：** 混合检索机制。结合语义搜索、基于最长公共子序列（LCS）的结构特征匹配以及图遍历。实行**双轨分离**，同时检索成功（正样本）和失败（负样本）的经验。
    3.  **Generate (生成)：** 基于规划和检索到的正负样本生成执行工件。
    4.  **Iterate (迭代)：** 自我修正循环。利用验证器（Validator）提供反馈，进行多次尝试。
    5.  **Ingest (摄入)：** 质量门控。由“老师模型（Teacher LLM）”评分，合格后存入 PKG，更新实体和操作节点。

### 7. 优点
*   **结构化深度：** 不同于以往的扁平记忆，它存储了完整的因果链和错误分析，使“失败”也成为宝贵的资产。
*   **非参数化在线学习：** 无需昂贵的微调，仅通过经验积累即可实现性能的单调增长。
*   **解耦知识与程序：** 通过 PKG 将世界知识（实体）与操作逻辑（程序）分离，当世界知识更新时，程序模板依然有效。
