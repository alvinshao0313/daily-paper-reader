## 摘要
强化学习 (RL) 已被证明能显著提升小型和大型语言模型 (LLMs) 的推理能力，但现有方法通常依赖于可验证的奖励，因此需要地面真值 (ground truth) 标签。我们提出了一种强化学习框架，该框架利用大语言模型作为裁判 (LLM-as-a-Judge) 对大量无标签数据上的模型输出进行评估并提供奖励，从而实现无标签知识蒸馏，并取代了对地面真值监督的需求。值得注意的是，该裁判模型通过单 token 输出进行操作，使得奖励计算非常高效。当与可验证奖励相结合时，我们的方法在数学推理基准测试中取得了显著的性能提升。这些结果表明，基于 LLM 的评估器可以为强化学习微调产生有效的训练信号。

## Abstract
Reinforcement Learning (RL) has been shown to substantially improve the reasoning capability of small and large language models (LLMs), but existing approaches typically rely on verifiable rewards, hence ground truth labels. We propose an RL framework that uses rewards from an LLM that acts as a judge evaluating model outputs over large amounts of unlabeled data, enabling label-free knowledge distillation and replacing the need of ground truth supervision. Notably, the judge operates with a single-token output, making reward computation efficient. When combined with verifiable rewards, our approach yields substantial performance gains across math reasoning benchmarks. These results suggest that LLM-based evaluators can produce effective training signals for RL fine-tuning.

*   **核心问题**：现有的强化学习微调方法（如 RLVR）虽然能显著提升模型的推理能力，但高度依赖“可验证奖励”（Verifiable Rewards），即需要标准答案来判断对错。这限制了模型在缺乏标注的海量数据上的学习能力。
*   **研究背景**：传统的知识蒸馏（SFT）往往只是表层 Token 的模仿，难以捕捉深层推理逻辑。而“LLM-as-a-Judge”此前多用于事后评估，未被充分探索作为 RL 过程中的实时、动态奖励信号。

### 2. 方法论
*   **核心思想**：将高性能的教师 LLM 转化为“动态奖励生成器”。在无标签数据上，学生模型生成推理路径（PoT，即 Python 代码），教师模型作为裁判对路径质量进行评估，并将其转化为连续的标量奖励信号。
*   **关键技术细节**：
    *   **单 Token 奖励计算**：为了提高效率，裁判模型不生成长文本评价，而是针对“该方案是否正确”输出“Yes”或“No”。通过提取这两个 Token 的 Logits（概率值）并进行归一化处理，计算出连续奖励值。
    *   **奖励函数公式**：$s = \sigma(u) \cdot I_{\{\sigma(u) \ge a\}}$，其中 $u = (\log p(\text{Yes}) - \log p(\text{No}))/\tau$。这种方法利用了概率分布的细微差别，提供了比二元对错更丰富的信号。
    *   **多源奖励融合**：最终目标函数结合了可验证奖励（$L_{VR}$）、LLM 裁判奖励（$L_{YoN}$）和预训练重排序模型奖励（$L_{Rerank}$）。
    *   **算法流程**：主要采用 PPO（近端策略优化）算法，同时也验证了 GRPO（群体相对策略优化）的适用性。

### 6. 主要结论与发现
*   **性能飞跃**：在 125M 和 350M 小模型上，该方法比仅使用可验证奖励的方法提升了 **5-10 个百分点**。
*   **无标签学习的有效性**：在完全没有标准答案的情况下，仅靠 LLM 裁判奖励也能达到甚至超过有限标签下的 RL 效果。
*   **规模效应**：该方法在 6.7B 模型上依然有效（提升约 1.5 分），证明了其可扩展性。
*   **PoT 优于 NL-CoT**：对于小模型，使用 Python 代码（PoT）作为推理中介比自然语言（NL）更易于被裁判评估且效果更好。

### 7. 优点
*   **高效性**：通过单 Token Logits 计算奖励，极大地降低了调用大模型作为裁判的推理开销。
*   **标签无关**：解锁了利用海量无标注数据进行复杂推理训练的可能性。
*   **鲁棒性强**：实验显示，经过该方法训练的模型在面对干扰项或复杂变体题目时，表现比传统方法更稳健。

### 8. 不足与局限
*   **奖励作弊（Reward Hacking）**：论文提到，如果完全不使用可验证奖励（即 $\lambda=0$），模型可能会出现“奖励塌陷”，例如生成极短或无意义的代码却骗取了裁判的高分。
*   **依赖裁判质量**：蒸馏效果高度依赖于裁判模型的能力，如果裁判本身存在偏见或错误，会误导学生模型。
*   **任务局限**：目前主要集中在数学和编程等逻辑严密的任务上，在主观性更强的自然语言任务上的表现尚待验证。
