## 摘要
对于多模态大语言模型 (MLLMs)，视觉信息相对于文本而言较为稀疏。因此，为了实现高效推理，视觉剪枝研究应运而生。目前的方法通常基于视觉编码器或 LLM 解码器中的注意力分数来衡量 Token 的重要性，进而选择注意力分数较高的视觉 Token 并剪除其余部分。在本文中，我们采用了一种不同且更为精准的方法。我们不依赖于特定机制的信号，而是在视觉和文本特征交互之前，直接计算两者之间的互信息 (MI)。这使我们能够在特征层面显式地衡量跨模态依赖关系。我们的 MI-Pruner 简单、高效且具有非侵入性，无需访问内部注意力图或进行架构修改。实验结果表明，我们的方法在极低延迟的情况下优于以往基于注意力的剪枝方法。

## Abstract
For multimodal large language models (MLLMs), visual information is relatively sparse compared with text. As a result, research on visual pruning emerges for efficient inference. Current approaches typically measure token importance based on the attention scores in the visual encoder or in the LLM decoder, then select visual tokens with high attention scores while pruning others. In this paper, we pursue a different and more surgical approach. Instead of relying on mechanism-specific signals, we directly compute Mutual Information (MI) between visual and textual features themselves, prior to their interaction. This allows us to explicitly measure crossmodal dependency at the feature levels. Our MI-Pruner is simple, efficient and non-intrusive, requiring no access to internal attention maps or architectural modifications. Experimental results demonstrate that our approach outperforms previous attention-based pruning methods with minimal latency.

### 2. 方法论：核心思想与技术细节
**MI-Pruner** 的核心思想是在投影空间（Projection Space）中，通过最大化跨模态互信息来识别与查询最相关的视觉 Token。

*   **核心流程：**
    1.  **特征提取与归一化：** 在视觉特征经过投影层后、进入 LLM 前，提取视觉特征 $V$ 和文本特征 $T$，并映射到单位超球面上。
    2.  **分布构建：** 基于玻尔兹曼分布计算视觉-文本（跨模态）和视觉-视觉（模态内）的相似度矩阵，进而推导出条件概率分布 $p(t|v)$ 和边缘分布 $p(t)$。
    3.  **逐点互信息（PMI）计算：** 计算每个视觉 Token 与文本 Token 之间的 PMI，用于衡量语义相关性；同时计算视觉 Token 间的 PMI，用于衡量冗余度。
    4.  **贪心搜索与评分：** 定义评分函数 $S_i = \lambda S_{cross} - (1-\lambda) S_{self}$。其中 $S_{cross}$ 捕捉最强跨模态关联，$S_{self}$ 捕捉模态内冗余。
    5.  **高效剪枝：** 采用贪心策略选择得分最高的 $N_{keep}$ 个 Token。
*   **算法优势：** 该方法具有 $O(N_V)$ 的复杂度，无需访问模型内部权重或注意力矩阵，是一种“手术刀式”的模块化插件。

*   **性能领先：** MI-Pruner 在多个基准测试中达到了 SOTA 性能，尤其在极高剪枝率（如仅保留 32 或 64 个 Token）下，性能下降远小于传统方法。

*   **极高效率：** 避免了复杂的内部张量提取，计算开销极小，适合资源受限的部署场景。
*   **非侵入性：** 无需修改模型架构，不影响 FlashAttention 等底层优化算子的使用。
*   **可解释性：** 可视化结果清晰展示了剪枝过程如何根据文本查询动态锁定视觉区域。

### 8. 不足与局限
*   **先验假设：** 算法假设视觉 Token 具有相等的先验概率（均匀分布），这在某些构图（如摄影中的三分法）下可能不完全符合实际。
*   **条件独立性假设：** 为了计算简便，假设了视觉 Token 间的条件独立性，这在处理极度精细的语义关联时可能存在近似误差。
*   **文本剪枝缺失：** 目前仅针对视觉 Token 进行剪枝，未涉及长文本 Prompt 的压缩。
*   **动态预算：** 实验中仍需手动设定剪枝比例（如 50%），尚未实现根据输入内容自动决定最优 Token 数量的完全自适应剪枝。
