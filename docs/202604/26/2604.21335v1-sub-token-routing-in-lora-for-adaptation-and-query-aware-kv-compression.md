## 摘要
与以往工作中使用的词元（tokens）、页面（pages）、注意力头（heads）或层（layers）等粗粒度单元相比，子词元路由（Sub-token routing）为提升 Transformer 效率提供了一个更精细的控制维度。在本文中，我们研究了在经过 LoRA 适配的 Transformer 中，词元表示内部的路由机制。其动机在于，一个相关的词元在内部并不一定是均匀的：在保留预算限制下，被保留的值组（value groups）在词元之间以及词元内部的分布都是不均匀的，这表明 KV 压缩不必是在词元层级上的“全有或全无”的决策。我们在两种场景下研究了这种细粒度路由机制。针对压缩感知的语言建模，我们引入了一种查询无关（query-independent）的设计，将路由子空间 LoRA 与 KV 路径上的值组路由相结合。针对保持下游任务性能的 KV 压缩，我们引入了一种查询感知（query-aware）的设计，其中基于预测器的选择器利用查询条件相关的相关性，在上下文词元/值组对上分配全局保留预算。实验表明，查询无关设计改善了语言建模的质量-压缩权衡，而查询感知设计在减少 KV 预算的情况下仍能保持下游任务的表现。我们进一步探讨了词元级与子词元级查询感知路由之间的关系，并证明它们构成了互补的压缩维度：词元级方法决定了哪些词元在全局范围内得以保留，而子词元路由则决定了保留下来的词元在内部如何被压缩。

## Abstract
Sub-token routing offers a finer control axis for transformer efficiency than the coarse units used in most prior work, such as tokens, pages, heads, or layers. In this paper, we study routing within a token representation itself in LoRA-adapted transformers. The motivation is that a relevant token need not be internally uniform: under a retention budget, preserved value groups are distributed unevenly both across tokens and within tokens, which suggests that KV compression need not be an all-or-nothing decision at token level. We study this fine-grained routing mechanism in two settings. For compression-aware language modeling, we introduce a query-independent design that combines routed subspace LoRA with value-group routing on the KV path. For downstream-task-preserving KV compression, we introduce a query-aware design in which a predictor-based selector allocates a global retention budget over context-token/value-group pairs using query-conditioned relevance. Experiments show that the query-independent design improves the quality-compression tradeoff for language modeling, while the query-aware design preserves downstream behavior under reduced KV budgets. We further examine the relation between token-level and sub-token-level query-aware routing, and show that they form complementary compression axes: token-level methods determine which tokens survive globally, while sub-token routing determines how the surviving tokens are compressed internally.

### 1. 核心问题与整体含义
*   **研究动机**：随着大语言模型（LLM）上下文窗口的增加，KV Cache 的内存开销成为推理瓶颈。现有的压缩方法（如 Token 剔除、量化、分层压缩）通常在 Token、Head 或 Layer 等粗粒度级别进行。
*   **核心发现**：作者观察到，即使是一个“重要”的 Token，其内部的 Value 向量表示也并非每一部分都同等重要。在固定的保留预算下，Token 内部的“值组（Value Groups）”分布呈现高度非均匀性。
*   **核心问题**：能否在 Token 内部进行更细粒度的路由（Sub-Token Routing），从而在不损失性能的前提下实现更深度的 KV 压缩？

### 2. 方法论
论文提出了两种子词元路由设计，分别针对不同的应用场景：

*   **查询无关路由（Query-Independent Routing）**：
    *   **路由子空间 LoRA**：在 LoRA 适配路径上，将低秩更新 $\Delta W$ 替换为多个子空间更新的加权组合，通过一个轻量级路由选择 Top-K 子空间。
    *   **值组路由**：将 Value 向量划分为 $S$ 个组，利用 Token 自身的表示预测各组的重要性，保留 Top-K 组存入 Cache，被丢弃的组通过一个小型重构器（MLP）从保留组中恢复。
*   **查询感知路由（Query-Aware Routing）**：
    *   **核心思想**：根据当前查询（Query）与上下文（Context）的相关性，动态分配每个 Token 内部保留的值组数量。
    *   **预测器机制**：引入一个轻量级预测器（两层 MLP），在模型中间层（Split Layer）根据隐藏状态预测各 Token 及其内部值组的重要性得分。
    *   **全局预算分配**：在全局范围内对所有“Token-值组对”进行排序，按预算 $\rho$ 选取得分最高的部分。这意味着相关性高的 Token 可能保留全部值组，而无关 Token 可能被完全剔除。

    *   适配基准：标准 LoRA、MoE-LoRA。
    *   压缩基准：Expected Attention (EA，Token 级压缩)、Quest（查询感知稀疏注意力）。

### 6. 主要结论与发现
*   **细粒度优势**：子词元路由比 Token 级路由提供了更平滑的压缩曲线。
*   **互补性**：Token 级方法决定“哪些 Token 存活”，子词元路由决定“存活的 Token 内部保留多少信息”。两者结合在 37.5% 的 KV 预算下，MMLU 准确率保持在 99% 以上。
*   **规模效应**：模型规模越大，Value 表示的冗余度越高，子词元路由的压缩效果越显著且越稳健。
*   **效率**：单次前向传递（One-pass）即可完成路由决策，无需像 Quest 那样进行额外的诊断计算。

### 7. 优点
*   **视角新颖**：打破了“Token 是最小压缩单元”的惯性思维，深入到向量内部。
*   **即插即用**：作为 LoRA 的扩展，易于集成到现有的参数高效微调流程中。
*   **高性能低开销**：预测器训练极快，推理几乎不增加延迟，且在长文本任务中表现稳健。

### 8. 不足与局限
*   **场景限制**：目前主要在 LoRA 适配的 Transformer 上验证，对于全参数微调或原生预训练模型的适用性尚未深入探讨。
*   **值组划分简单**：目前的值组是简单的连续维度切片，未考虑维度间的语义关联，未来若采用学习到的语义分组可能效果更好。
*   **压缩深度限制**：虽然在 50%-75% 的 KV 预算下表现优异，但在极低预算（如 <20%）下的表现仍有待观察。
