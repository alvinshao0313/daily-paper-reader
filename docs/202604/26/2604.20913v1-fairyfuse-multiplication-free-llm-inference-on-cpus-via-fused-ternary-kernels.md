## 摘要
大语言模型越来越多地部署在仅 CPU 的平台上，在这些平台上，内存带宽是自回归生成的首要瓶颈。将权重进行 4 位或更低比特的量化可以减轻内存压力，但现有系统仍需对权重进行反量化并执行浮点乘法，这限制了可获得的收益。{-1, 0, +1} 的三值权重提供了一种更高效的替代方案，将乘法替换为条件加法、减法或空操作。虽然 Fairy2i 表明三值 LLM 可以达到 FP16 的质量，但其运行时并未利用这种结构。我们提出了 FairyFuse，这是一个推理系统，通过使用掩码加法和减法将每个宽线性层（widely-linear layer）的八个实值子 GEMV 融合到单个 AVX-512 循环中，从而在通用 CPU 上实现无乘法执行，且浮点乘法次数为零。Roofline 分析表明，16 倍的权重压缩使受内存限制的 GEMV 在带宽受限的 CPU 上向计算密集型转变，产生了 29.6 倍的算子加速，而在 GPU 上几乎没有收益。在端到端性能方面，FairyFuse 在单颗 Intel Xeon 8558P 上达到了每秒 32.4 个 token，性能优于 llama.cpp Q4_K_M 1.24 倍，且具有近乎无损的质量（WikiText-2 困惑度为 5.52，而 FP16 为 5.47；下游任务准确率为 66.0%）。

## Abstract
Large language models are increasingly deployed on CPU-only platforms where memory bandwidth is the primary bottleneck for autoregressive generation. Weight quantization to four bits or below reduces memory pressure, yet existing systems still dequantize weights and perform floating-point multiplications, limiting the achievable gains. Ternary weights in {-1, 0, +1} provide a more efficient alternative, replacing multiplications with conditional additions, subtractions, or no-ops. While Fairy2i shows that ternary LLMs can match FP16 quality, its runtime does not exploit this structure. We present FairyFuse, an inference system that enables multiplication-free execution on commodity CPUs by fusing the eight real-valued sub-GEMVs of each widely-linear layer into a single AVX-512 loop using masked additions and subtractions, with zero floating-point multiplications. Roofline analysis shows that 16x weight compression shifts memory-bound GEMV toward the compute regime on bandwidth-limited CPUs, yielding a 29.6x kernel speedup while offering little benefit on GPUs. End-to-end, FairyFuse achieves 32.4 tokens per second on a single Intel Xeon 8558P, outperforming llama.cpp Q4_K_M by 1.24x with near-lossless quality (WikiText-2 perplexity 5.52 vs. 5.47 FP16; downstream accuracy 66.0%).

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）在边缘侧和隐私敏感场景的部署增加，仅使用 CPU 进行推理的需求日益增长。然而，CPU 推理面临两大瓶颈：
*   **内存带宽受限**：自回归生成过程中，权重流转速度远低于计算速度。
*   **计算冗余**：现有的低比特量化方案（如 4-bit）在计算时仍需将权重反量化为浮点数并执行昂贵的浮点乘法（FMA）。
**研究动机**：三值权重（{-1, 0, +1}）理论上可以将乘法完全替换为加减法，但现有的推理框架（如 llama.cpp）并未在指令集层面真正实现“无乘法”执行。本文旨在开发一种专门针对三值权重的 CPU 推理系统，彻底消除浮点乘法，并解决复数域量化带来的多算子调度开销。

#### 2. 方法论：核心思想与关键技术
FairyFuse 的核心思想是利用现代 CPU 的位处理指令和向量掩码指令，直接在压缩后的三值权重上执行计算。

*   **核心技术细节**：
    *   **并行位解码（Parallel Bit Decode）**：使用 BMI2 指令集的 `_pext_u32` 指令，从 32 位字中快速提取正向（+1）和负向（-1）的掩码。
    *   **掩码累加（Masked Accumulation）**：利用 AVX-512 的 `vaddps` 和 `vsubps` 指令，根据提取的掩码对激活值进行条件加法或减法。整个内层循环不含任何浮点乘法指令。
    *   **融合宽线性内核（Fused Widely-Linear Kernel）**：针对 Fairy2i 提出的复数域量化模型，将原本需要 8 次独立调用的实值子 GEMV 融合进一个循环。
*   **关键优化策略**：
    *   **O1 掩码重用**：解码一次掩码同时用于实部和虚部路径。
    *   **O2 输入共享**：一次性加载激活向量，供 8 个子 GEMV 共享。
    *   **O3 符号交换别名**：通过预计算负激活值，将复共轭运算转化为指针切换，消除运行时取反开销。
    *   **O4 寄存器驻留累加**：中间结果全程保留在向量寄存器中，减少 DRAM 访问。
