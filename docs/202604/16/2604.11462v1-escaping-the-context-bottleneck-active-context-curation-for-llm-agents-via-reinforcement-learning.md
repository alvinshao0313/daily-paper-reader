## 摘要
大语言模型（LLMs）在处理长程任务时面临挑战，这主要归因于“上下文瓶颈”和“迷失在中间”现象，即来自冗长环境的累积噪声会降低多轮交互中的推理能力。为了解决这一问题，我们引入了一种将上下文管理与任务执行解耦的共生框架。我们的架构将一个轻量级、专门化的策略模型 ContextCurator 与一个强大的冻结基础模型 TaskExecutor 配对。通过强化学习训练，ContextCurator 主动降低工作记忆中的信息熵。它在保留推理锚点（即对未来推导至关重要的稀疏数据点）的同时，积极剪枝环境噪声。在 WebArena 基准测试中，我们的框架将 Gemini-3.0-flash 的成功率从 36.4% 提升至 41.2%，同时将 Token 消耗降低了 8.8%（从 47.4K 降至 43.3K）。在 DeepSearch 上，它实现了 57.1% 的成功率（对比组为 53.9%），同时将 Token 消耗降低了 8 倍。值得注意的是，一个 7B 参数规模的 ContextCurator 达到了与 GPT-4o 相当的上下文管理性能，为自主长程智能体提供了一种可扩展且计算高效的范式。

## Abstract
Large Language Models (LLMs) struggle with long-horizon tasks due to the "context bottleneck" and the "lost-in-the-middle" phenomenon, where accumulated noise from verbose environments degrades reasoning over multi-turn interactions. To address this issue, we introduce a symbiotic framework that decouples context management from task execution. Our architecture pairs a lightweight, specialized policy model, ContextCurator, with a powerful frozen foundation model, TaskExecutor. Trained via reinforcement learning, ContextCurator actively reduces information entropy in the working memory. It aggressively prunes environmental noise while preserving reasoning anchors, that is, sparse data points that are critical for future deductions. On WebArena, our framework improves the success rate of Gemini-3.0-flash from 36.4% to 41.2% while reducing token consumption by 8.8% (from 47.4K to 43.3K). On DeepSearch, it achieves a 57.1% success rate, compared with 53.9%, while reducing token consumption by a factor of 8. Remarkably, a 7B ContextCurator matches the context management performance of GPT-4o, providing a scalable and computationally efficient paradigm for autonomous long-horizon agents.

*   **研究背景**：现实环境（如网页 DOM 树）包含超过 90% 的结构噪声。随着交互轮数增加，这些噪声会稀释关键的“推理锚点”，导致模型注意力分散、推理失败且 Token 消耗巨大。
*   **研究动机**：现有的静态检索或单体模型架构无法在保持高性能的同时有效过滤动态噪声。作者希望通过将“记忆管理”与“任务执行”解耦，提升智能体的信噪比（SNR）。

### 2. 论文提出的方法论
*   **核心思想**：采用**共生架构（Symbiotic Architecture）**，将系统分为两个专门组件：
    1.  **ContextCurator（策展人）**：轻量级策略模型（如 Qwen-7B），负责主动压缩、重写和精简历史记录，生成高保真的工作记忆。
    2.  **TaskExecutor（执行者）**：强大的冻结基础模型（如 GPT-4o 或 Gemini），负责基于精简后的记忆执行具体任务。
*   **关键技术细节**：
    *   **主动熵减**：策展人不是简单的摘要，而是通过生成式重写，剔除冗余 DOM 元素和语义噪声，保留对未来决策至关重要的“推理锚点”。
    *   **多轮 GRPO 算法（MT-GRPO）**：使用强化学习训练策展人。其奖励信号（Reward）并非来自策展人本身，而是基于执行者最终任务的成功与否（远端奖励）。
    *   **因果依赖链**：通过优化策展人的输出，使其生成的记忆状态能最大化执行者的成功概率。

### 3. 实验设计

    1.  **WebArena**：模拟真实网页环境（购物、论坛、Gitlab 等），测试处理高熵结构噪声的能力。

*   **客观性**：通过跨模型验证（用 7B 策展人配合不同的闭源执行者），证明了该框架的通用性和公平性。

### 6. 论文的主要结论与发现
*   **性能提升**：在 WebArena 上，将 Gemini-3.0-flash 的成功率从 36.4% 提升至 41.2%；在 DeepSearch 上成功率升至 57.1%。

*   **能力解耦**：实验证明，**7B 规模的专用策展人模型在记忆管理上的表现可以媲美 GPT-4o**。这说明上下文管理是一种可以从通用推理中分离出来的独立认知技能。

### 7. 优点（亮点）
*   **解耦范式**：巧妙地解决了大模型微调成本高与小模型推理能力弱的矛盾，通过“小模型管记忆，大模型做决策”实现了高效协作。
*   **信噪比优化**：从信息论角度出发，将记忆管理视为主动的熵减过程，而非简单的存储检索。
*   **显著的效率增益**：在长程 RAG 任务中实现了极高的压缩比，极大地节省了 API 调用成本。

### 8. 不足与局限
*   **训练成本**：虽然推理高效，但策展人的 RL 训练过程仍需要昂贵的 A100 算力支持。
*   **潜在的过度剪枝风险**：如果策展人误删了关键信息，执行者将无法挽回错误（尽管 RL 旨在最小化此类风险）。
*   **应用限制**：目前主要集中在文本/DOM 驱动的环境，对于多模态（如纯视觉导航）的上下文策展尚未深入探讨。
*   **实时性**：增加了一个策展步骤，虽然减少了 Token 处理时间，但增加了一次模型推理的延迟。
