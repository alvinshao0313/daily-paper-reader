## 摘要
大语言模型（LLM）智能体接收来自多种来源的指令，包括系统消息、用户提示、工具输出、其他智能体等，每种来源都带有不同程度的信任和权威。当这些指令发生冲突时，智能体必须可靠地遵循最高权限的指令，以确保安全和有效。目前的主流范式——指令层级（Instruction Hierarchy, IH）——假设存在一组固定的、少量的权限级别（通常少于五个），并由僵化的角色标签定义（例如：系统 > 用户）。这对于现实世界的智能体场景是不够的，因为在这些场景中，冲突可能源于更多的来源和上下文。在这项工作中，我们提出了多层级指令层级（Many-Tier Instruction Hierarchy, ManyIH），这是一种用于解决具有任意多个权限级别的指令冲突的范式。我们推出了 ManyIH-Bench，这是首个针对 ManyIH 的基准测试。ManyIH-Bench 要求模型处理多达 12 个级别的具有不同权限的冲突指令，包含 853 个智能体任务（427 个代码任务和 426 个指令遵循任务）。ManyIH-Bench 组合了由 LLM 开发并经人工验证的约束条件，创建了涵盖 46 个真实世界智能体的真实且具有挑战性的测试用例。我们的实验表明，当指令冲突规模扩大时，即使是目前的前沿模型表现也较差（准确率约为 40%）。这项工作强调了在智能体场景中，迫切需要专门针对细粒度、可扩展指令冲突解决的方法。

## Abstract
Large language model agents receive instructions from many sources-system messages, user prompts, tool outputs, other agents, and more-each carrying different levels of trust and authority. When these instructions conflict, agents must reliably follow the highest-privilege instruction to remain safe and effective. The dominant paradigm, instruction hierarchy (IH), assumes a fixed, small set of privilege levels (typically fewer than five) defined by rigid role labels (e.g., system > user). This is inadequate for real-world agentic settings, where conflicts can arise across far more sources and contexts. In this work, we propose Many-Tier Instruction Hierarchy (ManyIH), a paradigm for resolving instruction conflicts among instructions with arbitrarily many privilege levels. We introduce ManyIH-Bench, the first benchmark for ManyIH. ManyIH-Bench requires models to navigate up to 12 levels of conflicting instructions with varying privileges, comprising 853 agentic tasks (427 coding and 426 instruction-following). ManyIH-Bench composes constraints developed by LLMs and verified by humans to create realistic and difficult test cases spanning 46 real-world agents. Our experiments show that even the current frontier models perform poorly (~40% accuracy) when instruction conflict scales. This work underscores the urgent need for methods that explicitly target fine-grained, scalable instruction conflict resolution in agentic settings.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）被广泛应用于智能体（Agent）系统，它们需要处理来自系统消息、用户查询、工具输出、记忆文件及其他子智能体等多种来源的指令。现有的**指令层级（Instruction Hierarchy, IH）**范式存在“固定且少量层级”的瓶颈：通常仅依赖于预定义的角色标签（如 System > User > Tool），且层级通常少于 5 层。
在现实的复杂智能体场景中，这种设计无法区分同一角色内不同来源的优先级（例如，两个不同的工具输出可能有不同的信任等级）。因此，论文旨在提出一种能够支持**任意多层级、动态指定优先级**的指令冲突解决范式，以确保智能体在复杂指令冲突下仍能保持安全和有效。

#### 2. 方法论：核心思想与关键技术
论文提出了 **Many-Tier Instruction Hierarchy (ManyIH)** 范式，其核心是将指令的“权限/优先级”与“消息角色标签”解耦。
*   **特权提示接口 (Privilege Prompt Interface, PPI)：** 通过在提示词中直接标记指令的优先级，使模型在推理时动态解析冲突。
    *   **序数接口 (Ordinal Interface)：** 使用数字（如 `[[Privilege 1]]`）表示优先级，数字越小优先级越高。
    *   **标量接口 (Scalar Interface)：** 使用标量值（如 `[[z=82]]`）表示优先级，数值越大优先级越高。
*   **元指令 (Meta-instruction)：** 在提示词开头加入规则说明，告知模型当指令冲突时，应优先遵循哪种权限标记的指令。
*   **冲突解决逻辑：** 模型不再依赖训练时固化的角色顺序，而是根据 PPI 提供的数值进行逻辑推理，决定哪条指令“胜出”。

#### 6. 主要结论与发现
*   **性能鸿沟：** 即使是目前最强的模型，在 ManyIH-Bench 上的准确率也普遍低于 50%（最高仅约 42.7%），远低于它们在简单两层级 IH 测试中 >99% 的表现。
*   **层级扩展性差：** 随着指令层级数量的增加，所有模型的表现均呈现明显的单调下降趋势。
*   **格式敏感性：** 模型对权限的表达方式极其敏感。仅将序数格式改为标量格式，GPT-5.4 和 Claude Opus 4.6 的准确率就下降了超过 8%。
*   **推理能力：** 增加推理努力程度（Reasoning Effort）能提升性能，但无法完全解决多层级冲突问题。

#### 7. 优点
*   **范式创新：** 首次提出了突破固定角色层级限制的 ManyIH 概念，更符合未来复杂智能体的发展趋势。
*   **基准测试质量高：** ManyIH-Bench 结合了自动化验证和人工校验，任务设计既具有挑战性又贴近真实应用（如代码风格管理）。
*   **深入的敏感性分析：** 揭示了当前模型在处理数值逻辑和指令优先级时的脆弱性。

#### 8. 不足与局限
*   **数值敏感性：** 实验发现模型对具体的数值（z 值）存在偏见，而非纯粹基于逻辑排序，这限制了该方法在极细粒度场景下的可靠性。
