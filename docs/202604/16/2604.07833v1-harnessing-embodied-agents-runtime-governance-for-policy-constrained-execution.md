## 摘要
具身智能体正从被动的推理系统演变为与工具、机器人及物理环境交互的主动执行者。一旦被授予执行权限，核心挑战便在于如何在运行时保持动作的可治理性。现有方法将安全和恢复逻辑嵌入智能体循环内部，导致执行控制难以标准化、审计和适配。本文认为，具身智能不仅需要更强大的智能体，还需要更强大的运行时治理。我们提出了一种策略约束执行框架，将智能体的认知与执行监督分离。治理被外化为一个专门的运行时层，负责执行策略检查、能力准入、执行监控、回滚处理和人工干预。我们形式化了具身智能体、具身能力模块（ECMs）与运行时治理层之间的控制边界，并通过三个治理维度的 1000 次随机模拟实验进行了验证。结果显示，该框架拦截了 96.2% 的越权动作，在运行时漂移情况下将不安全持续执行率从 100% 降低至 22.2%，并在完全符合策略的前提下实现了 91.4% 的恢复成功率，显著优于所有基准方法（p<0.001）。通过将运行时治理重新定义为一等系统问题，本文将策略约束执行定位为具身智能体系统的关键设计原则。

## Abstract
Embodied agents are evolving from passive reasoning systems into active executors that interact with tools, robots, and physical environments. Once granted execution authority, the central challenge becomes how to keep actions governable at runtime. Existing approaches embed safety and recovery logic inside the agent loop, making execution control difficult to standardize, audit, and adapt.   This paper argues that embodied intelligence requires not only stronger agents, but stronger runtime governance. We propose a framework for policy-constrained execution that separates agent cognition from execution oversight. Governance is externalized into a dedicated runtime layer performing policy checking, capability admission, execution monitoring, rollback handling, and human override.   We formalize the control boundary among the embodied agent, Embodied Capability Modules (ECMs), and runtime governance layer, and validate through 1000 randomized simulation trials across three governance dimensions. Results show 96.2% interception of unauthorized actions, reduction of unsafe continuation from 100% to 22.2% under runtime drift, and 91.4% recovery success with full policy compliance, substantially outperforming all baselines (p<0.001). By reframing runtime governance as a first-class systems problem, this paper positions policy-constrained execution as a key design principle for embodied agent systems.

*   **研究动机**：随着具身智能体（如机器人、自动化工具）从单纯的对话助手转向物理世界的执行者，其执行权限的扩大带来了巨大的安全和治理挑战。
*   **核心问题**：现有的安全约束通常被硬编码在智能体的提示词（Prompt）或内部逻辑中，导致治理逻辑与认知逻辑耦合。这使得安全策略难以标准化、难以跨环境（如从仿真到实物）迁移，且缺乏运行时的持续监控与恢复机制。
*   **整体含义**：论文主张将“治理”作为具身智能系统的一等公民，提出将智能体的**认知（Cognition）**与**执行监管（Execution Oversight）**分离，构建一个独立的运行时治理层。

### 2. 方法论
*   **核心思想**：建立“智能体-能力-治理”三方控制边界。智能体负责提议（Propose），治理层负责准入与监督（Govern），能力模块负责执行（Execute）。
*   **关键技术组件（六大模块）**：
    1.  **能力准入（Capability Admission）**：检查智能体请求的技能是否在当前环境下被授权。
    2.  **策略守卫（Policy Guard）**：依据环境配置文件（Environment Profile）对具体参数进行检查或修正（如限制移动速度）。
    3.  **执行观察者（Execution Watcher）**：运行时持续监控传感器数据和任务进度，检测异常或策略违规。
    4.  **恢复与回滚管理器（Recovery & Rollback Manager）**：当执行失败或违规时，执行有策略意识的恢复动作或状态回滚。
    5.  **人工覆盖接口（Human Override Interface）**：在涉及高风险任务时引入人类审批或紧急接管。
    6.  **审计与遥测（Audit & Telemetry）**：记录所有决策和干预轨迹，用于事后追责和策略优化。
*   **算法流程**：定义了一个 7 阶段的“策略约束执行流水线”，从目标解释、能力提议、准入评估、受控启动、运行时监控、干预恢复到最终审计。

### 3. 实验设计
*   **实验场景**：使用 Gazebo 仿真环境，配置 UR5e 机械臂和移动底座。
*   **任务类别**：导航（区域限制）、操纵（抓取易碎品）及长程复合任务（取送物）。
*   **环境配置文件**：设置了仿真宽松（Sim-Relaxed）、实物受限（Real-Restricted）、人机共存（Human-Shared）和测试审计（Test-Audit）四种模式。
*   **对比基准（Baselines）**：
    *   **直接执行（Direct Exec）**：无治理层。
    *   **静态规则（Static Rule）**：仅有执行前的预检，无运行时监控。
    *   **能力内部安全（Cap.-Internal）**：安全逻辑嵌入在每个技能内部，但缺乏全局协调。

### 4. 资源与算力
*   **算力说明**：论文主要关注系统架构和运行时逻辑，未涉及大规模深度学习训练。实验在基于 Python 的仿真环境中运行。
*   **性能开销**：文中明确给出了治理层的延迟数据。预执行治理（准入+策略检查）的总延迟小于 **1.5 μs**，每步监控延迟约 **0.55 μs**。这表明该框架对机器人控制频率的影响几乎可以忽略不计。

### 5. 实验数量与充分性
*   **实验规模**：进行了 **1000 次随机试验**（5 个随机种子 × 每组 200 次试验）。
*   **充分性**：
    *   涵盖了越权动作拦截、运行时违规检测、故障恢复三个核心维度。
    *   进行了详细的**消融实验**，逐一移除观察者、恢复管理器等组件，验证了各模块的独立贡献。
    *   引入了统计显著性检验（p < 0.001），证明了结果的客观性。

### 6. 主要结论与发现
*   **高效拦截**：该框架实现了 **96.2%** 的越权动作拦截率。
*   **运行时安全**：在发生运行时漂移（如人类突然进入工作区）时，将不安全持续执行率从 100% 降低至 **22.2%**。
*   **合规恢复**：实现了 **91.4%** 的恢复成功率，且所有恢复动作均符合预设策略。
*   **人工干预**：人工覆盖接口成功拦截了 100% 未经批准的高风险请求。

### 7. 优点
*   **架构解耦**：实现了认知与监管的分离，使得同一套智能体逻辑可以无缝适配不同的安全策略。
*   **动态性**：不同于传统的静态检查，该框架强调“持续治理”，能够应对执行过程中的动态风险。
*   **环境敏感性**：通过环境配置文件（Profiles），实现了策略的灵活性，解决了“仿真与实物策略不一致”的问题。

### 8. 不足与局限
*   **仿真局限**：所有实验均在 Gazebo 仿真中完成，尚未在真实物理机器人上进行大规模验证，真实世界的传感器噪声和通信延迟可能影响监控精度。
*   **能力包依赖**：框架假设执行动作可以被封装为结构化的“能力包”，对于完全端到端（像素到动作）的黑盒策略，该框架的适用性受限。
*   **策略编写成本**：治理的效果高度依赖于人类预设策略的质量，如何自动生成或验证这些策略仍是挑战。
*   **观察者漏报**：实验显示在低灵敏度配置下，仍有约 38.7% 的运行时违规未被检测到，存在灵敏度与误报率的权衡。
