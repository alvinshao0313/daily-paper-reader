## 摘要
推理时计算缩放已成为提高大语言模型（LLM）智能体可靠性的强大技术，但现有方法通常统一应用计算资源：无论决策难度如何，每个决策步骤都分配相同的预算。我们引入了 TrACE（通过一致性实现的轨迹自适应计算），这是一种无需训练的控制器，通过衡量轨迹间动作一致性，在智能体的时间步长中自适应地分配 LLM 调用。在每一步，TrACE 采样一小组候选下一步动作，并衡量模型提交到同一动作的一致性程度。高一致性信号代表决策简单，控制器立即提交；低一致性信号代表不确定性，控制器在提交多数派动作之前会采样额外的轨迹，直到达到可配置的上限。该方法不需要学习组件、外部验证器或人类标签。我们在涵盖单步推理（GSM8K）和多步家庭导航（MiniHouse）的两个基准测试上，使用在 CPU 上运行的 Qwen 2.5 3B Instruct 模型，将 TrACE 与贪婪解码和固定预算自一致性（SC-4, SC-8）进行了评估。在 GSM8K 上，TrACE-4 在减少 33% LLM 调用的情况下达到了与 SC-4 相当的准确率，在 MiniHouse 上减少了 39%。TrACE-8 在 GSM8K 上减少 55% 调用、在 MiniHouse 上减少 65% 调用的情况下，达到了与 SC-8 相当的准确率。我们进一步证明，轨迹间一致性是步骤级成功的可靠信号，验证了核心假设，即模型自身的输出一致性编码了可以在不经过训练的情况下利用的难度信息。TrACE 是首个在多步序列决策任务上进行评估的、针对 LLM 智能体的无需训练、逐时间步自适应计算控制器。

## Abstract
Inference-time compute scaling has emerged as a powerful technique for improving the reliability of large language model (LLM) agents, but existing methods apply compute uniformly: every decision step receives the same budget regardless of its difficulty. We introduce TrACE (Trajectorical Adaptive Compute via agrEement), a training-free controller that allocates LLM calls adaptively across agent timesteps by measuring inter-rollout action agreement. At each step, TrACE samples a small set of candidate next actions and measures how consistently the model commits to the same action. High agreement signals an easy decision; the controller commits immediately. Low agreement signals uncertainty; the controller samples additional rollouts up to a configurable cap before committing to the plurality action. No learned components, no external verifier, and no human labels are required. We evaluate TrACE against greedy decoding and fixed-budget self-consistency (SC-4, SC-8) on two benchmarks spanning single-step reasoning (GSM8K, n=50) and multi-step household navigation (MiniHouse, n=30), using a Qwen 2.5 3B Instruct model running on CPU. TrACE-4 matches SC-4 accuracy while using 33% fewer LLM calls on GSM8K and 39% fewer on MiniHouse. TrACE-8 matches SC-8 accuracy with 55% fewer calls on GSM8K and 65% fewer on MiniHouse. We further show that inter-rollout agreement is a reliable signal of step-level success, validating the core hypothesis that the model's own output consistency encodes difficulty information that can be exploited without training. TrACE is the first training-free, per-timestep adaptive-compute controller for LLM agents to be evaluated on multi-step sequential decision tasks.

### 1. 核心问题与研究动机
*   **研究动机**：当前的 LLM 智能体在执行任务（如导航、编码、推理）时，通常对每一个决策步骤分配固定的计算预算（如贪婪解码或固定次数的并行采样）。然而，任务中不同步骤的难度差异巨大：有些步骤显而易见，而有些则充满歧义。
*   **核心问题**：统一的计算分配导致在简单步骤上浪费资源，而在困难步骤上计算不足。现有的自适应方法通常需要额外的训练、奖励模型或外部验证器，这增加了部署成本和模型依赖性。

### 2. 方法论：TrACE 核心思想与算法
TrACE 的核心思想是利用**模型自身输出的一致性**作为难度的“免费”信号。
*   **核心逻辑**：如果模型在多次随机采样中一致地选择同一个动作，说明该决策简单且可信；如果采样结果分散，则说明模型不确定，需要更多计算资源。
*   **算法流程**：
    1.  **初始采样**：在当前时间步 $t$，以温度 $\tau > 0$ 独立采样 $k_{init}$（默认 2）个候选动作。
    2.  **计算一致性 ($\alpha_t$)**：计算多数派动作在当前采样集合中所占的比例。
    3.  **自适应决策**：
        *   如果 $\alpha_t \ge \tau_{high}$（阈值，默认 0.75），则认为达成共识，立即执行该动作。
        *   如果一致性低于阈值，则逐个增加采样，直到一致性达标或达到预设的最大采样上限 $k_{max}$（如 4 或 8）。
    4.  **规范化处理**：使用动作规范化器（如统一大小写、去除标点）来准确计算文本动作的一致性。

*   **基准方法 (Benchmarks)**：
    *   **Greedy**：贪婪解码（每步 1 次调用）。

*   **对比指标**：任务成功率（准确率）与平均每个任务的 LLM 调用次数。

*   **消融实验**：对一致性阈值 $\tau_{high}$ 进行了消融研究（0.50, 0.75, 1.00），验证了默认参数的合理性。
*   **充分性评价**：实验覆盖了单步推理和多步决策两种典型场景。虽然样本量（n=30~50）相对较小，但通过自助法（Bootstrap）计算了 95% 置信区间，增强了结果的可信度。

### 6. 主要结论与发现
*   **帕累托改进**：TrACE 在保持与固定预算 SC 方法相同准确率的前提下，显著减少了计算开销。
    *   在 GSM8K 上，TrACE-8 比 SC-8 减少了 **55%** 的调用。
    *   在 MiniHouse 上，TrACE-8 比 SC-8 减少了 **65%** 的调用，且墙钟时间缩短了约 65%。
*   **信号有效性**：实验证明，动作一致性与步骤的最终成功率呈正相关，验证了一致性可以作为难度的可靠代理信号。
*   **任务自适应**：TrACE 的调用次数分布呈现双峰态，能够自动识别“简单”和“困难”步骤。

### 7. 优点
*   **无需训练 (Training-free)**：不需要标注数据、奖励模型或微调，可直接应用于任何开源 LLM。
*   **高效节能**：通过动态分配计算量，大幅降低了推理成本和延迟。
*   **简单易行**：算法逻辑简单，易于集成到现有的智能体框架中。
*   **即插即用**：作为一种推理时控制器，它可以与其他技术（如思维链、树搜索）结合使用。

### 8. 不足与局限
*   **模型规模限制**：实验仅基于 3B 参数模型，尚未在大规模模型（如 70B+）上验证其效率增益是否一致。
*   **动作空间限制**：主要针对离散、可枚举的动作空间。在开放式生成（如长代码编写）中，由于文本表面形式多样，计算一致性可能更具挑战。
*   **准确率天花板**：在 MiniHouse 任务中，由于 3B 模型能力的限制，TrACE 虽然提高了效率，但未能突破模型本身的准确率上限。
*   **基准多样性**：MiniHouse 是自定义的简化环境，在更复杂的标准基准（如 ALFWorld 或 WebArena）上的表现仍待观察。
