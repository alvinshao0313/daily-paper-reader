## 摘要
从复杂的价值流图（VSM）模拟中提取具有可操作性的洞察可能具有挑战性、耗时且容易出错。大语言模型的最新进展为支持用户完成这一任务提供了新途径。虽然现有方法擅长处理原始数据以获取信息，但在结构上难以捕捉到区分该领域相似数据源所需的细微情境差异。为解决这一问题，我们提出了一种解耦的两步式代理架构。通过将编排与数据分析分离，该系统利用注入了领域专家知识的渐进式数据发现。这种架构允许编排层智能地选择数据源，并在保持精简内部上下文的同时，跨数据结构执行多跳推理。来自多个最先进大语言模型的结果证明了该框架的可行性：顶尖模型实现了高达 86% 的准确率，并在多次评估运行中表现出极高的鲁棒性。

## Abstract
Extracting actionable insights from complex value stream map simulations can be challenging, time-consuming, and error-prone. Recent advances in large language models offer new avenues to support users with this task. While existing approaches excel at processing raw data to gain information, they are structurally unfit to pick up on subtle situational differences needed to distinguish similar data sources in this domain. To address this issue, we propose a decoupled, two-step agentic architecture. By separating orchestration from data analysis, the system leverages progressive data discovery infused with domain expert knowledge. This architecture allows the orchestration to intelligently select data sources and perform multi-hop reasoning across data structures while maintaining a slim internal context. Results from multiple state-of-the-art large language models demonstrate the framework's viability: with top-tier models achieving accuracies of up to 86% and demonstrating high robustness across evaluation runs.

*   **研究背景**：VSM 模拟会产生海量的 KPI 和多变量时间序列数据。现有的 LLM 方法在处理此类数据时面临两大挑战：
    *   **上下文腐烂（Context Rot）**：将大量原始数据直接塞入上下文窗口会导致模型推理能力下降。
    *   **语义模糊**：模拟输出中存在大量名称相似但情境不同的 KPI，普通模型难以区分。
*   **目标**：开发一种能够像领域专家一样，通过理解 VSM 结构并精准检索数据来回答复杂问题的智能代理系统。

### 2. 方法论
论文提出了一种**解耦的两步代理架构（Decoupled, Two-step Agentic Architecture）**，核心思想是将“逻辑推理”与“数据分析”分离：
*   **编排代理（Orchestration Agent）**：作为“大脑”，负责高层规划和多步推理。它不直接接触原始模拟数据，而是通过四个专用工具进行**渐进式数据发现**：
    1.  **节点发现工具**：识别 VSM 中的实体（如仓库、工位）。
    2.  **属性提取工具**：获取特定实体的静态参数（如安全库存、容量）。
    3.  **分类导航工具**：在模拟数据的 11 个主要类别中定位相关数据项。
    4.  **摘要工具**：触发子工作流。
*   **摘要子工作流（Summarization Subworkflow）**：作为“分析员”，仅在编排代理指定特定数据项时被激活。它接收原始数据、领域专家知识和具体指令，完成分析后将精炼的结论返回给编排代理。
*   **领域知识注入**：在检索过程中引入专家定义的元数据，帮助代理理解不同 KPI 的物理含义和适用场景。

*   **实验规模**：针对 4 种模型在评估集上进行了测试。

### 6. 主要结论与发现
*   **性能分级**：模型表现与参数规模正相关。Claude-Opus 4.6 表现最佳（86/100），GLM-5 紧随其后（83.88/100）。
*   **稳定性差异**：顶级模型不仅准确率高，且多次运行的波动极小（SD < 1.0）；而 14B 小模型表现出极高的不确定性（SD=6.64）。
*   **失败分析**：约 15% 的错误源于编排代理在导航时选择了错误的数据源，而非子工作流的分析错误，说明“理解领域拓扑”比“处理原始数据”更难。

### 7. 优点
*   **架构优雅**：通过解耦设计有效解决了长上下文导致的推理失效问题。
*   **渐进式发现**：模拟了人类专家的分析路径（先看结构，再找参数，最后看趋势），提高了过程的可解释性。
*   **评估严谨**：对“LLM 裁判”进行了人类专家对齐验证，增强了实验结论的可信度。

### 8. 不足与局限
*   **小模型适配性差**：目前的架构在参数量较小的模型上运行不够鲁棒，限制了边缘侧部署的可能。
*   **成本与延迟**：多步代理调用产生了大量的 Token 消耗（平均每样本约 3-4 万 Token），导致分析延迟较高。
*   **窗口限制**：虽然缓解了上下文压力，但对于极长的时间序列数据，如果单项数据超过了子工作流模型的窗口，系统仍会失效。
*   **闭环缺失**：目前仅实现了“洞察生成”，尚未实现自动修改参数并重新触发模拟的“优化闭环”。
