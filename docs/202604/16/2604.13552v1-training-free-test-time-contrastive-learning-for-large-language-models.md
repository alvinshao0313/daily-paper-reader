## 摘要
大语言模型（LLMs）展现出强大的推理能力，但其性能在分布偏移下往往会下降。现有的测试时自适应（TTA）方法依赖于基于梯度的更新，这需要白盒访问权限并产生巨大的开销，而免训练的替代方案要么是静态的，要么依赖于外部引导。在本文中，我们提出了免训练测试时对比学习（TF-TTCL），这是一种免训练的自适应框架，使冻结的 LLM 能够通过从自身的推理经验中提取监督信息来实现在线改进。具体而言，TF-TTCL 通过三个核心模块实现了一个动态的“探索-反思-引导”循环：1）语义查询增强首先通过多智能体角色扮演使问题视角多样化，从而生成不同的推理轨迹；2）对比经验蒸馏随后捕捉优劣轨迹之间的语义差异，并将其蒸馏为显式的文本规则；3）上下文规则检索最后在推理过程中激活这些存储的规则，动态地引导冻结的 LLM 走向鲁棒的推理模式，同时避免已观察到的错误。在闭口推理任务和开口评估任务上的广泛实验表明，在在线评估下，TF-TTCL 始终优于强大的零样本基准和代表性的 TTA 方法。代码可在 https://github.com/KevinSCUTer/TF-TTCL 获取。

## Abstract
Large language models (LLMs) demonstrate strong reasoning capabilities, but their performance often degrades under distribution shift. Existing test-time adaptation (TTA) methods rely on gradient-based updates that require white-box access and need substantial overhead, while training-free alternatives are either static or depend on external guidance. In this paper, we propose Training-Free Test-Time Contrastive Learning TF-TTCL, a training-free adaptation framework that enables a frozen LLM to improve online by distilling supervision from its own inference experiences. Specifically, TF-TTCL implements a dynamic "Explore-Reflect-Steer" loop through three core modules: 1) Semantic Query Augmentation first diversifies problem views via multi-agent role-playing to generate different reasoning trajectories; 2) Contrastive Experience Distillation then captures the semantic gap between superior and inferior trajectories, distilling them into explicit textual rules; and 3) Contextual Rule Retrieval finally activates these stored rules during inference to dynamically steer the frozen LLM toward robust reasoning patterns while avoiding observed errors. Extensive experiments on closed-ended reasoning tasks and open-ended evaluation tasks demonstrate that TF-TTCL consistently outperforms strong zero-shot baselines and representative TTA methods under online evaluation. Code is available at https://github.com/KevinSCUTer/TF-TTCL.

#### 1. 核心问题与整体含义
论文针对大语言模型（LLMs）在面对**分布偏移（Distribution Shift）**或复杂推理任务时性能下降的问题展开研究。现有的测试时自适应（TTA）方法存在两大局限：
*   **依赖梯度更新：** 需要访问模型内部参数（白盒），且在推理阶段产生巨大的计算和内存开销。
*   **依赖外部引导：** 免训练方法通常需要外部知识库或地面真值（Ground Truth）验证器（如代码单元测试），在现实黑盒 API 调用场景中难以实现。

**研究动机：** 借鉴人类从错误中反思的学习机制，提出一种无需训练、无需梯度、无需外部反馈的在线自适应框架，使模型能从自身的推理经历中提取“语义梯度”并自我改进。

#### 2. 方法论：TF-TTCL 框架
TF-TTCL 通过一个动态的**“探索-反思-引导”（Explore-Reflect-Steer）**循环运行，包含三个核心模块：

*   **语义查询增强 (SQA - 探索)：**
    *   采用多智能体角色扮演：**Teacher**（生成高置信度锚点答案）、**Tutor**（重写原始查询以引入语义变体）、**Student**（在变体下探索多样化推理路径）。
    *   通过这种方式模拟对比学习中的数据增强，暴露模型在不同表达下的不确定性。
*   **对比经验蒸馏 (CED - 反思)：**
    *   **候选划分：** 在闭口任务中通过多数投票划分正负样本；在开口任务中通过语义相似度划分。
    *   **样本选择：** 利用**最小困惑度（min-PPL）**作为置信度代理，从正负集合中选出最具代表性的样本。
    *   **规则蒸馏：** 将优劣轨迹之间的语义差异总结为显式的文本规则（“正向规则”：应该做什么；“负向规则”：应避免什么）。
*   **上下文规则检索 (CRR - 引导)：**
    *   维护一个动态更新的经验规则库。
    *   对于新查询，检索最相关的 Top-K 条正负规则，将其注入 Prompt 中，动态引导模型避开历史错误并遵循正确模式。

#### 3. 实验设计
*   **数据集：**
    *   **闭口推理任务 (CRT)：** GSM8k、MATH-500、AIME24、Minerva（涵盖小学数学到竞赛级难度）。
    *   **开口评估任务 (OET)：** DomainBench（包含地理、农业、医学、金融四个专业领域）。
*   **对比方法：**
    *   **基准：** Base LLM (Zero-shot)、Chain-of-Thought (CoT)。
    *   **梯度自适应方法：** Tent、EATA、COME、TLM。
    *   **免训练方法：** TF-GRPO（基于强化学习思想的免训练变体）。
*   **测试模型：** Llama-3.1-8B-Instruct（主实验）、Qwen-Plus、DeepSeek-V3.2（黑盒 API 验证）、以及从 3B 到 235B 不同规模的模型。

#### 4. 资源与算力
*   **硬件：** 论文提到使用 **NVIDIA A800 (80GB)** GPU 进行实验。
*   **软件环境：** CUDA 12.1, PyTorch 2.5.1。
*   **开销优化：** 为了降低延迟，Tutor 和 Student 智能体并行执行，规则蒸馏过程异步处理。实验显示，其用户感知延迟仅为单次调用的 2.01 倍。

#### 5. 实验数量与充分性
实验设计非常充分且具有客观性：
*   **多维度评估：** 涵盖了数学推理和专业领域知识问答两大类任务。
*   **消融实验：** 验证了 SQA、CED、CRR 各模块的必要性，并深入探讨了正负规则的非对称贡献（发现负向规则往往比正向规则更有效）。
*   **规模扩展性：** 测试了从 3B 到 235B 的模型，证明了该方法在不同参数量级下的稳健性。
*   **黑盒验证：** 在 DeepSeek 和 Qwen 的商业 API 上进行了测试，证明了其在实际部署场景中的价值。

#### 6. 主要结论与发现
*   **性能提升：** TF-TTCL 在所有基准测试中均显著优于零样本和其他 TTA 方法。在 GSM8k 上实现了 28.6% 到 54.4% 的相对错误减少率。
*   **负向约束的力量：** 实验发现，显式告诉模型“不要重复过去的某种错误逻辑”比单纯提供正确示例更能有效修正决策边界。
*   **无需外部反馈：** 即使在没有地面真值的情况下，通过模型内部的一致性和困惑度分析，也能提取出高质量的监督信号。
*   **系统效率：** 通过相似度过滤和 FIFO（先进先出）内存剪枝策略，规则库的增长得到了有效控制，且能略微提升准确率。

#### 7. 优点
*   **完全免训练：** 无需梯度访问，适用于所有黑盒 LLM API。
*   **在线自适应：** 能够随测试流实时进化，无需预先准备源域数据。
*   **对比逻辑：** 创新性地将对比学习引入 Prompt 工程，通过“正负对比”提取结构化知识。
*   **部署友好：** 异步更新和并行推理设计解决了多智能体框架常见的延迟瓶颈。

#### 8. 不足与局限
*   **性能天花板：** 随着模型接近其固有能力的上限，通过探索获得的边际收益会递减。
*   **上下文窗口限制：** 尽管有剪枝策略，但一次性注入过多规则仍可能干扰长程推理。
*   **对抗性风险：** 框架对输入配置的灵活性可能增加对提示词注入攻击的敏感性。
*   **计算成本：** 虽然比梯度方法省显存，但多次推理调用（Teacher/Tutor/Student）相比单次推理仍增加了 Token 消耗成本。
