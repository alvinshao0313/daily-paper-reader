## 摘要
由于高昂的计算成本、漫长的迭代周期以及新模型不断发布导致的快速过时，针对智能体任务训练大型基础模型正变得越来越不切实际。与其针对每个新任务或领域对海量模型进行后训练，我们提出了补充生成训练（Supplement Generation Training, SGT），这是一种更高效且可持续的策略。SGT 训练一个较小的 LLM 来生成有用的补充文本，当这些文本附加到原始输入时，可以帮助较大的 LLM 更有效地解决任务。这些轻量级模型可以根据任务需求动态调整补充内容，在不修改底层大模型的情况下提高性能。这种方法将特定任务的优化与大型基础模型解耦，从而在实际应用中实现更灵活、更具成本效益的 LLM 驱动智能体部署。

## Abstract
Training large foundation models for agentic tasks is increasingly impractical due to the high computational costs, long iteration cycles, and rapid obsolescence as new models are continuously released. Instead of post-training massive models for every new task or domain, we propose Supplement Generation Training (SGT), a more efficient and sustainable strategy. SGT trains a smaller LLM to generate useful supplemental text that, when appended to the original input, helps the larger LLM solve the task more effectively. These lightweight models can dynamically adapt supplements to task requirements, improving performance without modifying the underlying large models. This approach decouples task-specific optimization from large foundation models and enables more flexible, cost-effective deployment of LLM-powered agents in real-world applications.

### 2. 方法论：补充生成训练（SGT）
SGT 的核心思想是训练一个小型 LLM（Generator）生成**补充文本（Supplement）**，将其与原始查询拼接后输入给冻结的大型 LLM（Actor）。

2.  **代理奖励信号（Proxy Reward）**：由于补充文本的质量难以直接量化，SGT 将 Actor 模型最终任务的成败（二值化奖励）作为补充文本质量的代理信号。
3.  **训练两阶段**：
    *   **热启动 SFT（Warm-Start SFT）**：让小模型学习补充文本的格式和 8 种预定义类型的基本生成能力。

### 7. 优点
*   **高效解耦**：将任务特定的优化从昂贵的大模型中解耦出来，训练成本极低（1.5 小时 vs 数天/周）。
*   **黑盒友好**：不需要 Actor 模型的梯度，完全适配 API 调用场景。
*   **可解释性**：生成的补充信息是自然语言文本，人类可以直接观察“助理”提供了什么建议，增加了系统的透明度。
*   **动态性**：相比于静态的 Prompt Template，SGT 实现了 Instance-specific（针对每个具体问题的）动态增强。

*   **数据集规模限制**：为了模拟低资源环境，实验使用的数据集规模在千量级，尚未在大规模工业级数据集上验证其扩展上限。

*   **任务范围**：主要集中在逻辑、代码和知识问答，对于多模态或长文档理解等更复杂的智能体任务覆盖不足。
