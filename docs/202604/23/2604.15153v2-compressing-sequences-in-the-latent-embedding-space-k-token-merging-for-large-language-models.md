## 摘要
大语言模型（LLMs）在处理长提示词时会产生显著的计算和内存开销，因为全自注意力机制的复杂度随输入长度呈二次方增长。Token 压缩旨在通过减少表示输入的 Token 数量来应对这一挑战。然而，现有的提示词压缩方法主要在 Token 空间中运行，忽略了潜嵌入空间中的低效问题。在本文中，我们提出了 K-Token Merging，这是一种潜空间压缩框架，通过轻量级编码器将每 K 个连续 Token 嵌入块合并为单个嵌入。压缩后的序列由经过 LoRA 适配的大语言模型处理，而生成过程仍保持在原始词表空间。在结构化推理（Textualized Tree）、情感分类（Amazon Reviews）和代码编辑（CommitPackFT）上的实验表明，K-Token Merging 处于性能与压缩比的帕累托前沿，在性能下降极小的情况下实现了高达 75% 的输入长度缩减。代码可在 https://github.com/shsjxzh/K-Token-Merging 获取。

## Abstract
Large Language Models (LLMs) incur significant computational and memory costs when processing long prompts, as full self-attention scales quadratically with input length. Token compression aims to address this challenge by reducing the number of tokens representing inputs. However, existing prompt-compression approaches primarily operate in token space and overlook inefficiencies in the latent embedding space. In this paper, we propose K-Token Merging, a latent-space compression framework that merges each contiguous block of K token embeddings into a single embedding via a lightweight encoder. The compressed sequence is processed by a LoRA-adapted LLM, while generation remains in the original vocabulary. Experiments on structural reasoning (Textualized Tree), sentiment classification (Amazon Reviews), and code editing (CommitPackFT) show that K-Token Merging lies on the Pareto frontier of performance vs. compression, achieving up to 75% input length reduction with minimal performance degradation. Code is available at https://github.com/shsjxzh/K-Token-Merging.

*   **硬压缩（Hard Compression）**：直接丢弃或总结 Token，但在信息密集型任务（如代码、逻辑推理）中容易丢失关键细节。

### 2. 方法论：K-Token Merging
该方法通过一个轻量级的编码器将每 $K$ 个连续 Token 的嵌入合并为一个。

*   **核心思想**：在输入端（Prefill 阶段）进行压缩，在输出端（Generation 阶段）保持原始词表生成，从而兼顾效率与兼容性。
*   **关键技术细节**：
    *   **编码器 $f$**：一个约 50MB 的三层 MLP。它接收 $K$ 个原始嵌入向量，输出一个压缩后的嵌入向量 $C_i$。
    *   **均值初始化（Average-Initialized Encoder）**：为了加速收敛，编码器被设计为“原始 $K$ 个嵌入的平均值 + 一个残差 MLP 输出”。训练初期 MLP 权重接近零，使模型从语义合理的平均嵌入开始优化。
    *   **训练策略**：使用 LoRA 适配器对 LLM 进行微调，同时训练编码器 $f$。损失函数仅在生成部分的原始 Token 位置计算，确保模型学会理解压缩后的上下文。
    *   **推理流程**：Prefill 阶段序列长度变为 $1/K$，生成阶段新产生的 Token 逐个追加到压缩前缀后。

### 6. 主要结论与发现

*   **语义保留**：即使是代码这种对 Token 极其敏感的任务，潜空间合并也能通过微调找回丢失的信息，表现优于硬丢弃 Token 的方法。

### 7. 优点
*   **突破性视角**：不同于以往只在 Token 数量上做文章，该方法指出了嵌入空间本身的低效并加以利用。
*   **性能强劲**：在极高的压缩率（75%）下仍能保持极高的准确率，P-L F1 指标显著领先。
*   **初始化巧妙**：均值初始化策略解决了潜空间映射难以训练的问题，提供了稳定的起点。
*   **通用性**：不依赖于特定词表扩展，能够泛化到未见过的 $K$-grams。

### 8. 不足与局限
*   **生成阶段未压缩**：目前仅压缩 Prefill 阶段的输入，生成过程中的 KV Cache 仍会随长度线性增长，限制了长文本生成的效率提升。
*   **固定压缩率**：对所有文本统一使用固定的 $K$ 值，未考虑文本内部的信息熵差异（例如，重要信息应少压缩，冗余信息应多压缩）。
*   **模型规模限制**：实验仅在 0.5B 的小模型上完成，尚未在大规模模型（如 70B+）上验证其扩展性（Scaling Law）。
*   **训练成本**：相比于无需训练的硬压缩方法，该方法需要针对特定任务或领域进行 LoRA 微调。
