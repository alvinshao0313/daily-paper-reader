## 摘要
AI 智能体越来越多地通过模型上下文协议（MCP）调用外部工具（文件系统、网络、API）。这些工具调用相当于智能体的系统调用（syscalls）——即对共享状态产生副作用的特权操作。然而，目前的安全性强制执行完全处于用户态，仅需 10 行脚本即可绕过。本文提出了受控 MCP（Governed MCP），这是一种构建在基于 Logit 的安全原语（ProbeLogits，见配套论文 arXiv:2604.11943）之上的内核驻留工具治理网关。该网关通过一个六层流水线拦截每一次 MCP 工具调用：模式验证、信任层级检查、速率限制、对抗性预过滤、ProbeLogits 门控（核心语义检查）以及宪法级策略匹配，并配有基于 Blake3 哈希的审计链。我在 Anima OS（一个约 8.6 万行 Rust 代码编写的裸机 x86_64 操作系统）中实现了受控 MCP。五个非推理层为每次调用增加了 65.3 微秒的开销；在 7B Q4_0 模型上，ProbeLogits 增加了 65 毫秒（每 Token 类别的语义决策）的开销。在包含 101 个提示词的 MCP 领域基准测试中进行的四种配置消融实验表明，移除 ProbeLogits 层会导致 F1 分数从 0.773 骤降至 0.327（ΔF1 = -0.446）——仅靠手动规则防火墙是不够的。运行时中所有 15 个 WASM 到系统的宿主函数都通过该网关路由（实现了对 WASM ABI 表面的完全中介；该声明的范围和注意事项见第 4.6 节）；对于驻留内核的门控，能够击败现有护栏库的 10 行代码用户态绕过在结构上是不可能的。

## Abstract
AI agents increasingly call external tools (file system, network, APIs) through the Model Context Protocol (MCP). These tool calls are the agent's syscalls -- privileged operations with side effects on shared state -- yet today's safety enforcement lives entirely in userspace, where a 10-line script can bypass it. I propose Governed MCP, a kernel-resident tool governance gateway built on a logit-based safety primitive (ProbeLogits, companion paper: arXiv:2604.11943). The gateway interposes on every MCP tool call in a 6-layer pipeline: schema validation, trust tier check, rate limit, adversarial pre-filter, ProbeLogits gate (the load-bearing semantic check), and constitutional policy match, with a Blake3-hashed audit chain.   I implement Governed MCP in Anima OS, a bare-metal x86_64 OS in approximately 86,000 lines of Rust. The five non-inference layers add 65.3 microseconds of overhead per call; ProbeLogits adds 65 ms (per-token-class semantic decision) on 7B Q4_0. A 4-config ablation on a 101-prompt MCP-domain benchmark shows that removing the ProbeLogits layer collapses F1 from 0.773 to 0.327 (Delta F1 = -0.446) -- hand-rule firewalling alone is insufficient. All 15 WASM-to-system host functions in the runtime route through the gateway (complete mediation of the WASM ABI surface; the scope and caveats of this claim are stated in Section 4.6); a 10-LoC userspace bypass that defeats existing guardrail libraries is structurally impossible against the kernel-resident gate.

### 1. 核心问题与整体含义
*   **研究动机：** 随着 AI 智能体（Agents）通过模型上下文协议（MCP）自主调用文件系统、网络和 API，这些“工具调用”实际上等同于操作系统的“系统调用（Syscalls）”。
*   **核心矛盾：** 现有的安全防护措施（如 NeMo Guardrails）大多作为用户态的 Python 库运行。论文指出，这种设计在结构上是不安全的：智能体只需 10 行代码即可通过修改内存、跳过导入或篡改返回值来绕过防护。
*   **整体含义：** 本文主张将 AI 工具治理下沉到操作系统内核（Kernel-level），在智能体（运行在 WASM 沙箱中）与宿主系统之间建立一个不可逾越的特权边界。

### 2. 方法论
*   **核心思想：** 在基于 Rust 编写的 **Anima OS** 内核中实现一个六层治理网关，强制拦截所有 MCP 工具调用。
*   **六层流水线流程：**
    1.  **L1 模式验证（Schema Validation）：** 检查 JSON-RPC 格式是否符合工具定义的 JSON Schema。
    2.  **L2 信任层级（Trust Tier）：** 根据智能体身份（系统级、原生 AI 级等）匹配工具白名单。
    3.  **L3 速率限制（Rate Limit）：** 使用令牌桶算法防止资源耗尽或 DoS 攻击。
    4.  **L4 对抗性预过滤（Adversarial Pre-filter）：** 使用正则 DFA 快速扫描已知的提示词注入模式。
    5.  **L5 ProbeLogits 门控（核心层）：** 调用内核推理引擎，读取“安全/危险”Token 的 Logit 概率值进行语义分类。这是该系统的“承重”层。
    6.  **L6 宪法策略（Constitutional Policy）：** 匹配 12 条预设的硬性安全原则。
*   **关键技术细节：**
    *   **ProbeLogits 原语：** 不进行完整的文本生成，而是通过单次前向传播读取特定 Token 的 Logit 值，大幅降低延迟。
    *   **KV 缓存快照：** 在进行安全检查前备份智能体的上下文状态，检查后恢复，确保安全探测不会污染智能体的对话记忆。
    *   **完全中介（Complete Mediation）：** 确保 WASM 运行时中所有 15 个宿主函数都必须经过网关校验。

### 4. 资源与算力
*   **硬件环境：** 使用 AMD Ryzen 9800X3D CPU (8核, 16线程, 支持 AVX-512)，60 GB DDR5-6000 内存。
*   **模型：** 主要使用 Qwen2.5-7B-Instruct (Q4_0 量化版)，同时也验证了 Llama 3 8B 和 Mistral 7B。

### 5. 实验数量与充分性
*   **实验规模：** 进行了 101 个提示词的端到端分类测试、260 个 OS 动作测试、123 条 WASM 调用路径的完整性验证，以及针对阈值 $\alpha$ 的敏感度扫描。
*   **充分性与客观性：** 实验涵盖了性能（微秒级开销）和安全性（F1 分数）两个维度。消融实验清晰地展示了语义层的重要性。但需要注意，核心基准测试 Custom-101 是作者自建并标注的，虽然覆盖面广，但可能存在一定的分布偏差。

### 6. 主要结论与发现
*   **内核级治理的必要性：** 证明了在内核态部署安全门控可以彻底杜绝用户态绕过攻击。
*   **语义检查是核心：** 移除 ProbeLogits 层后，F1 分数从 0.773 骤降至 0.327。这表明仅靠传统的规则/正则防火墙无法识别复杂的 AI 攻击意图。
*   **高性能可行性：** 非推理层的总开销仅为 65.3 微秒；在 7B 模型上，语义检查延迟约为 65 毫秒，远快于需要多轮生成的传统分类器（通常需 1-5 秒）。
*   **故障关闭（Fail-Closed）：** 系统实现了推理引擎失效时默认拒绝所有工具调用的安全机制。

### 7. 优点
*   **架构安全性：** 首次将 AI 语义安全强制执行置于操作系统特权边界之下，提供了结构性的安全保障。
*   **低延迟：** 通过 Logit 探测而非文本生成，使实时工具调用治理成为可能。
*   **工程实现扎实：** 在 8.6 万行 Rust 代码的自研内核 Anima OS 中完整实现，并提供了防篡改的哈希审计链。

### 8. 不足与局限
*   **硬件瓶颈：** 目前依赖 CPU 推理，在高并发场景下可能存在锁竞争和性能瓶颈（未来需 GPU/NPU 加速）。
*   **数据集局限：** 缺乏大规模、社区公认的 MCP 专用安全基准测试，目前主要依赖作者自建数据。
*   **防御边界：** 无法完全防御极其隐蔽的“对抗性框架”攻击（例如看似良性但组合后有害的操作）。
*   **模型信任：** 尚未实现对内核加载模型本身的签名验证，若模型文件被篡改，整个安全基石将失效。
