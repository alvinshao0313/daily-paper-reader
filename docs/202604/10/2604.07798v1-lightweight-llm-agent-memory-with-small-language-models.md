## 摘要
尽管大语言模型（LLM）智能体能够利用工具处理复杂任务，但它们仍需要记忆来维持跨轮次的一致性，并在长程交互中积累可重用的信息。然而，基于检索的外部记忆系统虽然在线开销较低，但由于查询构建和候选过滤的局限性，其准确性并不稳定。相比之下，许多系统在在线记忆操作中重复调用大模型，虽然提高了准确性，但在长程交互中会不断累积延迟。我们提出了 LightMem，这是一种由小语言模型（SLM）驱动的轻量级记忆系统，旨在提升智能体的记忆能力。LightMem 将记忆检索、写入和长期整合模块化，并将在线处理与离线整合分离，从而在有限的计算资源下实现高效的记忆调用。我们将记忆组织为用于即时对话上下文的短期记忆（STM）、用于可重用交互摘要的中期记忆（MTM）以及用于整合知识的长期记忆（LTM），并利用用户标识符支持多用户场景下的独立检索和增量维护。在线运行时，LightMem 在固定的检索预算下工作，并通过两阶段过程选择记忆：先进行基于向量的粗筛，再进行语义一致性重排序。离线状态下，它提取可重用的交互证据并将其增量整合到 LTM 中。实验表明，该系统在不同规模的模型上均有提升，在 LoCoMo 数据集上的平均 F1 值提升了约 2.5，且表现出更高的效率和较低的中值延迟（检索延迟 83 毫秒；端到端延迟 581 毫秒）。

## Abstract
Although LLM agents can leverage tools for complex tasks, they still need memory to maintain cross-turn consistency and accumulate reusable information in long-horizon interactions. However, retrieval-based external memory systems incur low online overhead but suffer from unstable accuracy due to limited query construction and candidate filtering. In contrast, many systems use repeated large-model calls for online memory operations, improving accuracy but accumulating latency over long interactions. We propose LightMem, a lightweight memory system for better agent memory driven by Small Language Models (SLMs). LightMem modularizes memory retrieval, writing, and long-term consolidation, and separates online processing from offline consolidation to enable efficient memory invocation under bounded compute. We organize memory into short-term memory (STM) for immediate conversational context, mid-term memory (MTM) for reusable interaction summaries, and long-term memory (LTM) for consolidated knowledge, and uses user identifiers to support independent retrieval and incremental maintenance in multi-user settings. Online, LightMem operates under a fixed retrieval budget and selects memories via a two-stage procedure: vector-based coarse retrieval followed by semantic consistency re-ranking. Offline, it abstracts reusable interaction evidence and incrementally integrates it into LTM. Experiments show gains across model scales, with an average F1 improvement of about 2.5 on LoCoMo, more effective and low median latency (83 ms retrieval; 581 ms end-to-end).

*   **研究动机**：LLM 智能体在长程交互中需要记忆来维持一致性。现有的记忆系统面临“效率与准确性”的权衡：
    *   **检索式记忆（如 RAG）**：虽然在线开销低，但由于查询构建简单、候选过滤不足，导致检索噪声大、准确性不稳定。
    *   **LLM 驱动的记忆（如 MemGPT）**：通过反复调用大模型进行记忆管理，虽然准确但会产生巨大的推理延迟和计算成本。
*   **核心目标**：利用**小语言模型（SLM）**驱动高频的在线记忆操作，并将重负载的记忆整合任务移至离线，从而在有限算力下实现高效、精准的智能体记忆管理。

### 2. 方法论
LightMem 的核心思想是**模块化解耦**与**多层存储架构**：
*   **三层存储结构**：
    *   **短期记忆 (STM)**：即当前的对话上下文窗口，不持久化。
    *   **中期记忆 (MTM)**：存储用户特定的情节记忆（摘要、时间戳、用户 ID），支持个性化检索。
    *   **长期记忆 (LTM)**：存储去标识化的、跨用户的语义知识，采用**轻量级图结构**组织，支持多跳推理。
*   **在线处理流程（由三个专门的 SLM 协作）**：
    *   **SLM-1 (Controller)**：意图建模。将用户输入重写为多个“假设性查询”（HQs），并根据意图在 MTM 和 LTM 之间分配检索预算。
    *   **SLM-2 (Selector)**：两阶段检索。第一阶段进行基于向量的粗筛（2K 候选）；第二阶段进行语义一致性重排，过滤噪声并压缩至最终的 Top-K 记忆。
    *   **SLM-3 (Writer)**：交互摘要。每轮对话后提取关键信息，增量写入 MTM，并处理冗余和冲突。
*   **离线整合流程**：
    *   由大模型（LLM）异步执行，将 MTM 中的高价值情节抽象为 LTM 中的知识节点，进行图谱更新和自然遗忘处理，不占用在线交互时间。

### 3. 实验设计
*   **数据集**：
    *   **LoCoMo**：长对话逻辑推理基准，包含单跳、多跳、时间推理、开放域和对抗性任务。
    *   **DialSim**：多方长程对话模拟数据集。
*   **对比基准 (Baselines)**：包括 LoCoMo 原生方法、ReadAgent、MemoryBank、MemGPT 和 A-MEM。
*   **骨干模型 (Backbones)**：测试了 GPT-4o, GPT-4o-mini 以及开源的 Qwen2.5 (1.5B/3B) 和 Llama 3.2 (1B/3B)。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 1B 到 GPT-4o 不同量级的 6 种模型作为骨干，验证了系统的泛化性。
*   **消融实验**：针对语义重排、HQ 生成、MTM 模块、离线整合、图结构等 5 个核心组件分别进行了消融测试，证明了各模块的必要性。
*   **客观性**：每组实验运行 3 次随机种子并报告了标准差，同时进行了配对自助法重采样（Paired Bootstrap Resampling）以确保统计显著性（p-value < 0.01）。

### 6. 主要结论与发现
*   **性能领先**：LightMem 在 LoCoMo 基准上的 F1 分数平均提升了约 **2.5**，在多跳和时间推理任务中表现尤为突出。
*   **高效低时延**：检索中值延迟仅为 **83ms**，端到端中值延迟为 **581ms**，显著优于 A-MEM 和 MemGPT。
*   **语义增强**：在 DialSim 实验中，SBERT 语义相似度显著提升，表明系统能更准确地捕捉对话意图而非仅仅是关键词匹配。
*   **上下文压缩**：通过精准检索，将有效上下文长度从 16K 降低到 1K 左右，大幅节省了推理成本。

### 7. 优点
*   **架构优雅**：在线/离线分离的设计极大地缓解了长程交互中的延迟累积问题。
*   **SLM 潜力挖掘**：成功证明了 1B-3B 规模的小模型在经过特定任务微调后，可以胜任复杂的记忆控制逻辑。
*   **多用户支持**：通过用户标识符实现逻辑隔离，兼顾了隐私与扩展性。

### 8. 不足与局限
*   **意图分解风险**：SLM-1 在处理极度模糊的查询时，可能会生成不准确的假设性查询，导致检索到的信息“焦点稀释”。
*   **离线策略单一**：论文主要关注在线效率，对离线整合的不同策略（如不同的图构建算法）探讨较少。
*   **任务覆盖面**：实验主要集中在对话和逻辑推理，对于需要频繁调用外部工具（Tool-use）或多模态环境下的记忆管理尚未验证。
