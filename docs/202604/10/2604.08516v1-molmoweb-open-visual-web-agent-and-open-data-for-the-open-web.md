## 摘要
网络智能体——代表用户在网络上导航并执行任务的自治系统——具有改变人们与数字世界交互方式的潜力。然而，目前能力最强的网络智能体依赖于具有未公开训练数据和配方的专有模型，这限制了科学理解、可复现性以及社区驱动的进步。我们认为，面向开放网络的智能体应当以开放的方式构建。为此，我们推出了：(1) MolmoWebMix，一个包含浏览器任务演示和网页图形用户界面（GUI）感知数据的大规模多样化混合数据集；(2) MolmoWeb，一系列完全开放的多模态网络智能体。具体而言，MolmoWebMix 将来自多个互补生成流水线的 10 万余条合成任务轨迹，与 3 万余条人类演示、原子级网络技能轨迹以及 GUI 感知数据（包括指代表达定位和截图问答）相结合。MolmoWeb 智能体作为以指令为条件的视觉语言动作策略运行：给定任务指令和网页截图，它们即可预测下一步浏览器动作，无需访问 HTML、可访问性树或专门的 API。MolmoWeb 智能体提供 4B 和 8B 两种规模，在 WebVoyager、Online-Mind2Web 和 DeepShop 等浏览器使用基准测试中取得了最先进的结果，性能优于 Fara-7B、UI-Tars-1.5-7B 和 Holo1-7B 等同规模的仅开放权重模型。MolmoWeb-8B 还超越了基于 GPT-4o 等更大规模闭源前沿模型构建的标记集（SoM）智能体。我们进一步证明，通过带有 N 选优（best-of-N）策略的并行展开进行测试时扩展，可以获得持续的增益，在 WebVoyager 和 Online-Mind2Web 上分别实现了 94.7% 和 60.5% 的 pass@4（相比之下，pass@1 分别为 78.2% 和 35.3%）。我们将发布模型权重、训练数据、代码和统一的评估框架，以实现可复现性并加速网络智能体的开放研究。

## Abstract
Web agents--autonomous systems that navigate and execute tasks on the web on behalf of users--have the potential to transform how people interact with the digital world. However, the most capable web agents today rely on proprietary models with undisclosed training data and recipes, limiting scientific understanding, reproducibility, and community-driven progress.   We believe agents for the open web should be built in the open. To this end, we introduce (1) MolmoWebMix, a large and diverse mixture of browser task demonstrations and web-GUI perception data and (2) MolmoWeb, a family of fully open multimodal web agents. Specifically, MolmoWebMix combines over 100K synthetic task trajectories from multiple complementary generation pipelines with 30K+ human demonstrations, atomic web-skill trajectories, and GUI perception data, including referring expression grounding and screenshot question answering. MolmoWeb agents operate as instruction-conditioned visual-language action policies: given a task instruction and a webpage screenshot, they predict the next browser action, requiring no access to HTML, accessibility trees, or specialized APIs.   Available in 4B and 8B size, on browser-use benchmarks like WebVoyager, Online-Mind2Web, and DeepShop, MolmoWeb agents achieve state-of-the-art results outperforming similar scale open-weight-only models such as Fara-7B, UI-Tars-1.5-7B, and Holo1-7B. MolmoWeb-8B also surpasses set-of-marks (SoM) agents built on much larger closed frontier models like GPT-4o. We further demonstrate consistent gains through test-time scaling via parallel rollouts with best-of-N selection, achieving 94.7% and 60.5% pass@4 (compared to 78.2% and 35.3% pass@1) on WebVoyager and Online-Mind2Web respectively. We will release model checkpoints, training data, code, and a unified evaluation harness to enable reproducibility and accelerate open research on web agents.

#### 2. 方法论：核心思想与关键技术
*   **核心思想**：采用**纯视觉（Vision-only）**的动作策略。模型仅通过网页截图、任务指令和历史动作进行决策，而不依赖于复杂的 HTML 源代码或可访问性树（AxTree）。这种设计模拟了人类使用网页的方式，降低了 Token 消耗，并避免了 DOM 结构变动带来的鲁棒性问题。
*   **MolmoWebMix 数据集构成**：
    *   **合成轨迹**：利用 Gemini-3-Flash 作为“老师”，通过 AxTree 辅助生成 10 万+ 条任务路径，并将其转化为纯视觉格式。
    *   **人类演示**：收集了 3 万+ 条真实人类在浏览器上的操作轨迹，涵盖搜索、填表、多跳导航等。
    *   **原子技能轨迹**：针对特定技能（如“加入购物车”、“应用筛选”）提供专项监督。
    *   **GUI 感知数据**：包含 1000 万+ 条指代表达定位（Grounding）和截图问答（QA）对，强化模型对网页元素的识别能力。
*   **模型架构**：基于 Molmo2 架构（Qwen2.5 语言模型 + SigLIP2 视觉编码器），输出结构化的 JSON 对象，包含“思维（Thought）”和“动作（Action）”。
*   **动作空间**：定义了包括 `mouse_click`、`keyboard_type`、`scroll`、`goto` 等在内的标准浏览器操作集合。

#### 3. 实验设计
*   **评估基准（Benchmarks）**：在四个主流在线网页任务基准上进行测试：**WebVoyager**、**Online-Mind2Web**、**DeepShop** 和 **WebTailBench**。
*   **对比方法**：
    *   **开源模型**：Fara-7B、UI-Tars-1.5-7B、Holo1-7B、GLM-4.1V-9B。

*   **测试时缩放（Test-time Scaling）**：引入了 **Pass@k** 评估，即通过并行运行 $N$ 个智能体并使用 VLM 评判员（Judge）选择最佳结果。

*   **模型规模**：提供了 4B 和 8B 两个参数量版本。

*   **实验规模**：论文进行了多维度的实验，包括 SOTA 性能对比、测试时缩放分析、采样策略（Greedy vs. Nucleus）对比、定位能力专项评估。
*   **消融实验**：详细探讨了数据规模（1% vs 10% vs 100%）的影响，以及**人类数据与合成数据**的贡献度对比。
*   **客观性与公平性**：实验在实时网站环境下运行，并使用了各基准官方推荐的 LLM-as-a-Judge 评估流程。通过多次运行取平均值，并对过时任务指令进行了时间修正，确保了评估的公正性。

#### 6. 主要结论与发现
*   **开源超越闭源**：MolmoWeb-8B 在多个基准上超越了基于 GPT-4o 构建的 SoM 智能体，证明了高质量专用数据可以弥补模型规模的差距。
*   **纯视觉方案可行性**：尽管不使用 HTML，MolmoWeb 依然在 WebVoyager 等任务上达到了 SOTA，证明了视觉特征足以支撑复杂的网页交互。
*   **推理时计算的威力**：通过并行展开（Pass@4），模型在 WebVoyager 上的成功率从 78.2% 飙升至 **94.7%**，显示出巨大的性能潜力。
*   **合成数据质量**：实验发现，由 AxTree 辅助生成的合成轨迹比人类演示更具学习效率，因为人类操作往往包含更多噪声和冗余。

#### 7. 优点
*   **完全透明**：不仅开源权重，还开源了极其珍贵的训练数据集（MolmoWebMix）和生成流水线。
*   **高效性**：纯视觉方案显著减少了上下文窗口的压力（AxTree 动辄消耗数万 Token）。
*   **泛化能力**：通过大规模 GUI 感知预训练，模型在未见过的网站和定位任务（ScreenSpot）上表现强劲。

#### 8. 不足与局限
*   **OCR 瓶颈**：在处理极小文字或复杂长文本阅读时，模型仍可能出现识别错误。
*   **指令遵循偏差**：对于极其模糊或约束条件过多的指令，性能会有所下降。
*   **动作一致性**：有时会出现“思维”与“动作”不匹配的情况（例如思维说要打字，动作却是按回车）。
*   **长尾动作不足**：模型在预测低频动作（如 `scroll_at` 或 `drag_and_drop`）时鲁棒性较差。
