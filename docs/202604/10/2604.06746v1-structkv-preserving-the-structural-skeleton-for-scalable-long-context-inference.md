## 摘要
随着大语言模型（LLMs）扩展到支持超过一百万个 token 的上下文窗口，键值（KV）缓存的线性增长带来了严重的内存容量和带宽瓶颈，限制了长上下文推理的效率。现有的压缩方法通常根据局部显著性指标对 token 进行优先级排序，以将预填充计算与解码内存解耦。然而，这些方法往往依赖于特定层的局部显著性快照，从而系统性地丢弃了那些在整个网络深度中充当全局信息枢纽、但在选定的剪枝层中暂时处于休眠状态的 token。为了解决这一局限性，我们提出了 StructKV，这是一个结构感知的 KV 缓存压缩框架，它引入了三项核心创新：首先，全局入度中心性（Global In-Degree Centrality）通过聚合整个网络深度的注意力模式来识别全局信息枢纽；其次，动态枢轴检测（Dynamic Pivot Detection）利用信息论指标自适应地定位压缩的最佳层；最后，结构传播与解耦（Structural Propagation and Decoupling）将计算预算与内存存储预算分离。在 LongBench 和 RULER 基准测试上的实验结果表明，StructKV 有效地保留了长程依赖关系和检索鲁棒性。

## Abstract
As Large Language Models (LLMs) scale to support context windows exceeding one million tokens, the linear growth of Key-Value (KV) cache imposes severe memory capacity and bandwidth bottlenecks, constraining the efficiency of long-context inference. Existing compression approaches typically prioritize tokens based on local saliency metrics to decouple prefill computation from decoding memory. However, these methods often rely on local saliency snapshots at a specific layer, thereby systematically discarding tokens that act as global information hubs across the network depth but appear temporarily dormant at the specific layer selected for pruning. To address this limitation, we propose StructKV, a structure-aware KV cache compression framework that introduces three core innovations: First, Global In-Degree Centrality aggregates attention patterns across the network depth to identify global information hubs. Second, Dynamic Pivot Detection utilizes information-theoretic metrics to adaptively locate the optimal layer for compression. Finally, Structural Propagation and Decoupling separates the computational budget from the memory storage budget. Experimental results on the LongBench and RULER benchmarks demonstrate that StructKV effectively preserves long-range dependencies and retrieval robustness.

随着大语言模型（LLM）支持的上下文窗口突破百万级别，**KV Cache（键值缓存）** 的线性增长成为了推理效率的致命瓶颈，导致显存占用过高和带宽受限。
*   **现有挑战**：目前的压缩方法（如 SnapKV, FastKV）主要依赖“局部显著性”，即通过特定某一层的注意力快照来决定哪些 token 重要。
*   **研究动机**：作者发现这种“局部快照”具有盲目性。某些 token 在特定层可能表现“休眠”（注意力权重低），但在整个网络深度中却起着全局信息枢纽的作用。一旦被误删，会导致长程依赖丢失和推理质量下降。

### 2. 方法论：核心思想与关键技术
StructKV 提出了一个“结构感知”的压缩框架，旨在识别并保留文本的“结构骨架”。其核心包含三个模块：
*   **全局入度中心性（Global In-Degree Centrality）**：不再依赖单层快照，而是通过递归累加从输入层到“枢轴层”的所有注意力分数。公式上利用衰减因子 $\lambda$ 结合历史层的注意力权重，识别出在多层中持续贡献的“信息枢纽” token。
*   **动态枢轴检测（Dynamic Pivot Detection）**：传统方法通常固定在某一层（如第15层）进行剪枝，但这不适用于不同深度的模型。StructKV 通过在线监测注意力熵（Entropy）、稀疏度（Sparsity）和方差（Variance）的梯度变化，自动定位注意力机制从“广泛探索”转向“聚焦提取”的相位转换点（即最佳压缩层 $L^*$）。
*   **结构化传播与解耦（Structural Propagation & Decoupling）**：
    *   **计算与存储解耦**：在预填充（Prefill）阶段保留较多 token 以维持计算的高保真度（$R_{struct}$）；在解码（Decoding）阶段则使用更严格的预算来压缩 KV Cache 以节省内存（$R_{KV}$）。
    *   **位置对齐**：通过重新排列 Position ID 确保旋转位置嵌入（RoPE）在剪枝后依然保持一致。

### 6. 主要结论与发现
*   **性能优异**：在仅保留 10% KV Cache 的严苛条件下，StructKV 在 LongBench 上的表现接近全上下文（Full-Context）水平，显著优于 FastKV 和 GemFilter。
*   **检索鲁棒性**：在 RULER 128K 任务中，StructKV 能够找回 FastKV 丢失的信息，证明了全局中心性指标能有效保护“休眠”的关键 token。
*   **动态适应性**：实验发现模型越大，注意力稳定的层数越深（如 Qwen-32B 在第 28 层才稳定），StructKV 的动态检测机制成功捕捉到了这一特性。

### 7. 优点（亮点）
*   **全局视角**：解决了局部显著性导致的“近视”剪枝问题，通过累加注意力分数保留了语义骨架。
*   **自动化程度高**：动态枢轴检测消除了人工调参的麻烦，使算法能自动适配不同深度的模型。
*   **高效解耦**：巧妙地平衡了预填充阶段的计算精度与生成阶段的内存压力。

### 8. 不足与局限
*   **长度限制**：受限于计算资源，实验验证的上下文长度上限为 128K，尚未在百万级（1M+）真实场景下验证其稳定性。
*   **架构覆盖**：目前主要针对标准的 Dense Transformer 架构，对于混合专家模型（MoE）或非注意力架构（如 SSM/Mamba）的适用性尚待研究。
*   **硬件依赖**：虽然在 A800 上表现良好，但在显存带宽极度受限的边缘设备上，额外的向量累加操作可能产生更明显的开销。
