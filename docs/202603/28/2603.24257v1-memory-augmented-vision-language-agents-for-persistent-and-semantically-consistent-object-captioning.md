## 摘要
视觉语言模型（VLM）在不同视角下对同一物体的描述往往不一致，这阻碍了具身智能体随时间构建一致语义表示的能力。以往的方法通过离线多视图聚合或多阶段流水线来解决不一致性，这些流水线将探索、数据关联和描述学习解耦，对先前观察到的物体的推理能力有限。在本文中，我们提出了一种统一的记忆增强型视觉语言智能体，它在单个自回归框架内同时处理数据关联、物体描述和探索策略。该模型处理当前的 RGB 观测、自顶向下的探索地图以及序列化为物体级标记（tokens）的物体级情景记忆，从而确保在长序列中保持持久的物体身份和语义一致性。为了以自监督方式训练模型，我们利用基于分歧的策略和伪描述模型在逼真的 3D 环境中收集了一个数据集，该模型强制要求多视图描述历史之间的一致性。在手动标注的物体级测试集上的广泛评估表明，与基准模型相比，标准描述评分提高了高达 11.86%，描述自相似性提高了 7.39%，同时通过紧凑的场景表示实现了可扩展的性能。代码、模型权重和数据可在 https://github.com/hsp-iit/epos-vlm 获取。

## Abstract
Vision-Language Models (VLMs) often yield inconsistent descriptions of the same object across viewpoints, hindering the ability of embodied agents to construct consistent semantic representations over time. Previous methods resolved inconsistencies using offline multi-view aggregation or multi-stage pipelines that decouple exploration, data association, and caption learning, with limited capacity to reason over previously observed objects. In this paper, we introduce a unified, memory-augmented Vision-Language agent that simultaneously handles data association, object captioning, and exploration policy within a single autoregressive framework. The model processes the current RGB observation, a top-down explored map, and an object-level episodic memory serialized into object-level tokens, ensuring persistent object identity and semantic consistency across extended sequences. To train the model in a self-supervised manner, we collect a dataset in photorealistic 3D environments using a disagreement-based policy and a pseudo-captioning model that enforces consistency across multi-view caption histories. Extensive evaluation on a manually annotated object-level test set, demonstrate improvements of up to +11.86% in standard captioning scores and +7.39% in caption self-similarity over baseline models, while enabling scalable performance through a compact scene representation. Code, model weights, and data are available at https://github.com/hsp-iit/epos-vlm

### 1. 核心问题与研究动机
*   **核心问题**：现有的视觉语言模型（VLM）在具身导航过程中，由于视角变化、距离远近或遮挡，对同一个物理物体往往会产生不一致甚至矛盾的描述（语义漂移）。
*   **研究背景**：
    *   **静态局限性**：传统 VLM 多在静态数据集上训练，缺乏物体持久性（Object Permanence）概念。
    *   **现有方案缺陷**：以往解决一致性的方法多采用离线多视图聚合或多阶段流水线（将探索、数据关联和描述生成解耦），导致智能体无法根据当前语义的不确定性主动调整观察视角。
*   **研究目标**：构建一个统一的、具备记忆能力的具身智能体，在单一框架内实现物体识别、一致性描述生成和主动探索。

### 2. 方法论
*   **核心思想**：提出 **EPOS-VLM** 框架，将具身任务转化为一个统一的自回归序列生成问题。模型同时输入当前视觉观测、探索地图和**序列化的物体级情节记忆**。
*   **关键技术细节**：
    *   **物体级情节记忆（Episodic Memory）**：维护一个包含物体 ID、3D 坐标位置和历史描述频率的结构化存储。
    *   **记忆标记化（Tokenization）**：将记忆内容序列化为特定标记（如 `[OBJ]`, `[POS]`, `[CAP-HISTORY]`），直接注入 VLM 的语言上下文，使其能推理长程物体历史。
    *   **统一输出序列**：模型在每个时间步输出一个结构化序列，包含：
        1.  **数据关联（Match）**：将当前检测到的物体与记忆中的 ID 匹配或标记为新物体。
        2.  **物体描述（Caption）**：基于记忆修正后的物体级描述。
        3.  **导航动作（Action）**：决定下一步移动方向。
*   **自监督训练流程**：
    *   **3D-CPS 伪标签生成器**：利用 3D 几何信息筛选最佳视角，并结合历史描述频率，生成视角无关的物体描述作为训练真值。
    *   **基于分歧的探索策略**：在数据收集阶段，智能体优先访问那些描述分歧较大的物体，以获取更多信息。

### 3. 实验设计
*   **数据集与场景**：
    *   使用 **Habitat** 仿真平台，基于 **HM3D**（128 个训练场景）和 **Gibson**（用于跨领域泛化测试）数据集。

*   **对比方法**：

    *   **一致性增强方法**：IC3, ECO-score, 以及作者之前的多阶段方法 LD-CPS。
    *   **探索策略对比**：随机目标（Random Goal）、前沿探索（Frontier Exploration）。
*   **评估指标**：
    *   描述准确性：BLEU-4, METEOR, ROUGE-L, CIDEr, SPICE。
    *   语义一致性：SBERT 嵌入的余弦相似度（Mean CS）及离散度（IQR）。
    *   数据关联：跟踪准确率（Acc）、身份切换（IDSW）等。

### 4. 资源与算力
*   **硬件环境**：使用了 **2 张 NVIDIA A100 GPU**。
*   **训练细节**：
    *   基础模型为 Qwen3-VL-2B。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从伪标签质量评估、物体描述准确性、跨视图一致性到数据关联性能的全方位测试。
*   **消融实验**：专门设计了“无记忆（w/o memory）”版本，验证了情节记忆对维持一致性的核心作用。
*   **泛化性测试**：在 HM3D 上训练并在 Gibson 上测试，证明了模型在未见环境下的鲁棒性。

### 6. 主要结论与发现
*   **性能提升**：EPOS-VLM 在 SPICE 指标上比最强基准提升了 **11.86%**，在描述一致性（Mean CS）上提升了 **7.39%**。
*   **记忆优势**：将记忆序列化为 Token 注入 VLM，比传统的点云密集表征更节省内存（从 GB 级降至 KB 级），且推理时间不随步数线性增长。
*   **主动探索的作用**：实验证明，能够感知语义不确定性并主动复访物体的策略，比单纯追求几何覆盖的探索策略能获得更准确的语义理解。

### 7. 优点
*   **架构统一**：将感知、记忆、推理和动作整合进单一自回归框架，避免了多阶段系统的误差累积。
*   **高效性**：创新的标记化记忆表征解决了具身智能体长程导航中的计算瓶颈。
*   **自监督闭环**：通过 3D 几何引导的伪标签生成，降低了对大规模人工标注数据的依赖。

### 8. 不足与局限
*   **分割依赖**：模型仍依赖外部的实例分割模型（如 Mask2Former），若分割出错会直接影响数据关联和描述质量。
*   **环境假设**：实验主要在静态、逼真的仿真环境中进行，未充分考虑动态场景（物体移动）或传感器噪声严重的真实世界环境。
*   **实时性挑战**：虽然推理时间稳定，但在资源受限的嵌入式机器人平台上，0.7s/步的延迟可能仍需进一步优化。
