## 摘要
最近的联合音视频扩散模型取得了显著的生成质量，但由于其双向注意力依赖性，面临高延迟问题，阻碍了实时应用。我们提出了 OmniForcing，这是第一个将离线双流双向扩散模型蒸馏为高保真流式自回归生成器的框架。然而，由于模态之间极端的时域不对称性以及由此产生的 token 稀疏性，直接将因果蒸馏应用于此类双流架构会引发严重的训练不稳定性。我们通过引入一种带有零截断全局前缀（Global Prefix）的非对称块因果对齐（Asymmetric Block-Causal Alignment）来解决固有的信息密度差异，从而防止多模态同步漂移。因果切换过程中由极端音频 token 稀疏性引起的梯度爆炸，通过配备恒等 RoPE 约束（Identity RoPE constraint）的音频 Sink Token 机制得到了进一步解决。最后，一种联合自强制蒸馏（Joint Self-Forcing Distillation）范式使模型能够在长序列生成过程中动态地自我纠正由曝光偏差引起的累积跨模态误差。得益于模态独立的滚动 KV 缓存推理方案，OmniForcing 在单块 GPU 上实现了约 25 FPS 的最先进流式生成速度，同时保持了与双向教师模型相当的多模态同步和视觉质量。项目主页：https://omniforcing.com

## Abstract
Recent joint audio-visual diffusion models achieve remarkable generation quality but suffer from high latency due to their bidirectional attention dependencies, hindering real-time applications. We propose OmniForcing, the first framework to distill an offline, dual-stream bidirectional diffusion model into a high-fidelity streaming autoregressive generator. However, naively applying causal distillation to such dual-stream architectures triggers severe training instability, due to the extreme temporal asymmetry between modalities and the resulting token sparsity. We address the inherent information density gap by introducing an Asymmetric Block-Causal Alignment with a zero-truncation Global Prefix that prevents multi-modal synchronization drift. The gradient explosion caused by extreme audio token sparsity during the causal shift is further resolved through an Audio Sink Token mechanism equipped with an Identity RoPE constraint. Finally, a Joint Self-Forcing Distillation paradigm enables the model to dynamically self-correct cumulative cross-modal errors from exposure bias during long rollouts. Empowered by a modality-independent rolling KV-cache inference scheme, OmniForcing achieves state-of-the-art streaming generation at $\sim$25 FPS on a single GPU, maintaining multi-modal synchronization and visual quality on par with the bidirectional teacher.\textbf{Project Page:} \href{https://omniforcing.com}{https://omniforcing.com}

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：现有的联合音视频生成模型（如 LTX-2、Veo 3）虽然质量高，但依赖**双向全序列注意力机制**，导致生成首帧/首块的延迟（TTFC）极高，无法满足实时交互或流式传输的需求。
*   **研究背景**：
    *   级联方案（先视频后音频或反之）会割裂音视频的联合分布，且无法实现真正的同步流式生成。
    *   现有的视频流式生成技术（如 CausVid）仅针对单模态。直接将其应用于音视频双流架构时，由于视频（3 FPS）与音频（25 FPS）之间极端的**时间不对称性**和**Token 稀疏性**，会导致训练极度不稳定（梯度爆炸和 Softmax 崩溃）。

### 2. 方法论：核心思想与关键技术
OmniForcing 是首个将重型双向音视频模型蒸馏为高保真流式自回归生成器的框架。其核心技术包括：
*   **非对称块因果对齐（Asymmetric Block-Causal Alignment）**：
    *   引入**1秒宏块（Macro-block）**概念，将 3 帧视频和 25 帧音频 Token 对齐，解决非整数频率比带来的截断问题。
    *   **全局前缀（Global Prefix）**：将 $t=0$ 时刻的 Token 设为全局可见，作为长序列生成的语义锚点。
*   **音频 Sink Token 与恒等 RoPE（Identity RoPE）**：
    *   针对音频 Token 稀疏导致的梯度爆炸，引入 16 个可学习的 Sink Token 扩展注意力分母，提升熵值。
    *   对 Sink Token 使用恒等 RoPE 约束，使其位置无关，稳定因果切换过程。
*   **三阶段蒸馏流水线**：
    1.  **Stage I (双向 DMD)**：赋予模型少步（Few-step）去噪能力。
    2.  **Stage II (因果 ODE 回归)**：使模型权重适应因果掩码，注入流式生成能力。
    3.  **Stage III (联合自强力蒸馏)**：通过自回归展开（Self-Forcing）让模型学习纠正自身的预测偏差，解决长序列生成的曝光偏差（Exposure Bias）。
*   **推理优化**：采用**模态独立的滚动 KV 缓存**，利用音视频分支的计算不对称性实现单 GPU 上的并行推理。

### 3. 实验设计

*   **Benchmark**：
    *   **JavisBench**：评估音视频质量（FVD/FAD）、文本一致性（CLIP/CLAP）、跨模态相干性（AV-IB）和同步性（JavisScore/DeSync）。

*   **对比方法**：

    *   联合模型：MM-Diffusion, JavisDiT++, 以及教师模型 LTX-2。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了主流的音视频生成评价指标，并在 JavisBench 和 VBench 两个权威榜单上进行了测试。
*   **消融实验**：针对 Stage II 的稳定性做了详细消融，对比了不同数量的 Sink Token（1 到 24 个）、RoPE 类型以及其他稳定器（如 QK-Norm、Tanh-Gated）。
*   **充分性评价**：实验设计较为全面，不仅对比了生成质量，还重点量化了推理效率（FPS、TTFC）。对比基线涵盖了从级联到联合生成的多种范式，结果具有客观性。

### 6. 主要结论与发现
*   **性能突破**：OmniForcing 在单 GPU 上达到了 **~25 FPS** 的生成速度，相比教师模型 LTX-2 实现了 **~35倍** 的加速。
*   **超低延迟**：首块生成延迟（TTFC）仅为 **~0.7秒**（教师模型需 197秒）。
*   **质量保持**：在视觉保真度上甚至略优于教师模型（VBench 得分更高），在音视频同步性和跨模态一致性上显著优于所有非教师基线模型。
*   **稳定性发现**：Sink Token 是解决多模态 Token 密度不匹配导致训练崩溃的关键。

### 7. 优点
*   **首创性**：首次实现了联合音视频的实时流式扩散生成。
*   **架构优雅**：通过物理时间对齐（1s 宏块）巧妙解决了音视频采样率不统一的难题。
*   **训练稳定**：提出的 Sink Token + Identity RoPE 方案为多模态自回归模型提供了有效的稳定性保障。
*   **实用性强**：支持单卡实时推理，且具备良好的算力扩展性。

### 8. 不足与局限
*   **因果权衡**：由于将双向注意力改为因果注意力，在跨模态一致性（AV-IB）和同步性指标上较教师模型仍有微小差距。
*   **数据集规模**：蒸馏所用的数据集相对较小（虽然对于蒸馏任务可能足够），在大规模开放域场景下的泛化能力有待进一步验证。
*   **长序列漂移**：尽管有 Self-Forcing 机制，但在极长序列（如数分钟）生成中是否会出现音画去同步现象，文中缺乏超长序列的量化分析。
