## 摘要
大语言模型通过长思维链（CoT）序列在复杂推理方面取得了突破。然而，这往往导致严重的推理膨胀，造成大量的计算冗余。为了最大化单位 Token 的智能（Intelligence per Token），我们引入了一个理论指标：最小充分长度（MSL）。MSL 严格刻画了在保持答案正确性的前提下的最短推理长度。我们提供了一个基于独立采样序列的递归定义，并证明了其极限的存在性，从而为推理链压缩建立了第一个可衡量的下界。基于对主流 CoT 压缩策略的分析，我们识别出了使模型能够接近 MSL 的关键结构因素。基于这些见解，我们提出了 TRiMS，它在训练过程中结合基于 MSL 的估计采用了 GRPO 算法，同时通过动态批次聚合和使用批次级标准差计算优势值，缓解了训练过程中的不稳定性。TRiMS 在所有基准测试中实现了超过 80% 的 CoT Token 缩减，并伴有轻微的准确率提升。

## Abstract
Large language models achieve breakthroughs in complex reasoning via long chain-of-thought sequences. However, this often leads to severe reasoning inflation, causing substantial computational redundancy. To maximize Intelligence per Token, we introduce a theoretical metric, MSL-Minimal Sufficient Length. MSL rigorously characterizes the shortest reasoning length that preserves answer correctness. We provide a recursive definition based on independently sampled sequences and prove the existence of its limit, establishing the first measurable lower bound for reasoning-chain compression. Building on an analysis of mainstream CoT compression strategies, we identify key structural factors enabling a model to approach MSL. Based on these insights, we propose TRiMS which employs the GRPO algorithm in conjunction with MSL-based estimation during training, while mitigating instabilities during the training process through dynamic batch aggregation and advantage computation using batch-level standard deviation. TRiMS achieves over 80% CoT token reduction with a minor accuracy boost across all benchmarks.

*   **核心问题**：大语言模型（如 OpenAI-o1、DeepSeek-R1）虽然通过长链条思维（CoT）提升了推理能力，但也带来了严重的“推理膨胀”（Reasoning Inflation），即在简单问题上消耗过多 Token，导致计算冗余和高延迟。
*   **研究动机**：目前缺乏一个理论指标来界定“准确性”与“压缩率”之间的边界。研究者旨在寻找推理链压缩的理论下界，并开发一种能够在不损失准确率的前提下，最大化“单位 Token 智能”（Intelligence per Token, IPT）的训练框架。

### 2. 方法论
*   **核心思想**：引入**最小充分长度（Minimal Sufficient Length, MSL）**概念，即在保证答案正确的前提下，模型推理所需的最短 Token 数。
*   **关键技术细节**：
    *   **MSL 定义与估计**：通过对模型进行 $k$ 次独立采样，定义最短正确路径长度（SCPT@k）。证明了随着采样数 $k$ 增加，该长度会收敛到一个稳定值（即 MSL）。
    *   **TRiMS 框架**：基于 GRPO（组相对策略优化）算法，在强化学习过程中实时追踪并逼近 MSL。
    *   **动态批次聚合（Dynamic Batch Aggregation）**：在训练早期，过滤掉全对或全错的退化组，以确保梯度信号的有效性。
    *   **自适应截断（Adaptive Truncation）**：根据问题难度，动态选择截断长度（如 2048 或 4096 Token），平衡计算效率与推理完整性。
    *   **批次级优势归一化（Batch-level Advantage Normalization）**：针对奖励稀疏问题（如 16 个样本中仅 1 个正确），使用整个 Batch 的标准差而非组内标准差来计算优势值，防止梯度爆炸，增强训练稳定性。

### 6. 主要结论与发现
*   **高效压缩**：TRiMS 在 1.5B 和 7B 模型上均实现了 **80% 以上的 Token 缩减**。
*   **性能提升**：在大幅压缩长度的同时，准确率不仅没有下降，反而有轻微提升（1.5B 模型平均提升 6.8%）。

*   **泛化性强**：在数学以外的 MMLU 任务上也表现出良好的压缩效果和准确率提升。

### 8. 不足与局限
*   **模型规模限制**：实验主要集中在 1.5B 到 7B 的小参数模型上，尚未在 70B 或更大规模的模型上验证 MSL 的收敛特性。
*   **领域局限**：训练数据主要集中在数学领域，虽然有 MMLU 测试，但在法律、创意写作等非结构化推理领域的表现尚待验证。

*   **提示词敏感性**：尚未探讨不同 Prompt 诱导出的 MSL 差异，即 MSL 是否随 Prompt 变化而波动。
