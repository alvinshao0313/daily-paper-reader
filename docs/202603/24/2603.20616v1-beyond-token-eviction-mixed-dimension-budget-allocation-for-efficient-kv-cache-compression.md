## 摘要
键值（KV）缓存被广泛用于加速 Transformer 推理，但其内存开销随输入长度线性增长，限制了长上下文的部署。现有的 Token 剔除方法通过丢弃次要 Token 来减少内存占用，这可以被视为一种粗粒度的降维形式，即为每个 Token 分配零维度或全维度。我们提出了 MixedDimKV，一种混合维度的 KV 缓存压缩方法，它在更细粒度的层面上为 Token 分配维度；以及 MixedDimKV-H，它进一步整合了多头（head-level）重要性信息。在长上下文基准测试上的实验表明，MixedDimKV 优于以往不依赖多头重要性分析的 KV 缓存压缩方法。在具备相同的多头重要性信息时，MixedDimKV-H 的表现始终优于 HeadKV。值得注意的是，在 LongBench 上，我们的方法仅使用 6.25% 的 KV 缓存就达到了与全注意力（full attention）相当的性能。此外，在“大海捞针”（Needle-in-a-Haystack）测试中，我们的方案在 50K 上下文长度下保持了 100% 的准确率，而使用的缓存仅为 0.26%。

## Abstract
Key-value (KV) caching is widely used to accelerate transformer inference, but its memory cost grows linearly with input length, limiting long-context deployment. Existing token eviction methods reduce memory by discarding less important tokens, which can be viewed as a coarse form of dimensionality reduction that assigns each token either zero or full dimension. We propose MixedDimKV, a mixed-dimension KV cache compression method that allocates dimensions to tokens at a more granular level, and MixedDimKV-H, which further integrates head-level importance information. Experiments on long-context benchmarks show that MixedDimKV outperforms prior KV cache compression methods that do not rely on head-level importance profiling. When equipped with the same head-level importance information, MixedDimKV-H consistently outperforms HeadKV. Notably, our approach achieves comparable performance to full attention on LongBench with only 6.25% of the KV cache. Furthermore, in the Needle-in-a-Haystack test, our solution maintains 100% accuracy at a 50K context length while using as little as 0.26% of the cache.

### 2. 方法论：核心思想与关键技术
*   **混合维度分配（Mixed-Dimension Allocation）**：预设一组候选压缩率（如 {0%, 12.5%, 25%, 100%}），利用 PCA（主成分分析）对 Key 和 Value 进行降维。

*   **约束优化与二分查找**：

*   **MixedDimKV-H**：该变体进一步整合了离线分析得到的“多头重要性”信息，实现跨头、跨 Token 的联合优化。

*   **性能卓越**：在 LongBench 上，仅需 **6.25%** 的 KV 缓存即可达到全注意力水平。
*   **极端压缩**：在 50K 长度的“大海捞针”测试中，仅用 **0.26%** 的缓存（等效 128 Token）即可保持 100% 准确率。

*   **动态适应**：实验证明，相比于静态降维（如 Palu），动态为不同 Token 分配维度的策略在低预算下表现更稳健。

### 7. 优点与亮点
*   **粒度更细**：打破了 Token 剔除的二元限制，通过混合维度实现了更高效的信息保留。
*   **理论支撑**：引入拉格朗日对偶和二分查找，将复杂的离散优化问题转化为高效的可求解过程。
*   **兼容性强**：作为一种即插即用的推理优化技术，不改变模型权重，且能与现有的多头重要性分析方法结合。
*   **效率平衡**：通过逐头 PCA 和 Contiguous Tensor 布局设计，兼顾了压缩精度与底层计算效率。

### 8. 不足与局限
*   **PCA 开销**：虽然 PCA 是在 Prefill 阶段一次性计算的，但对于极短序列或实时性要求极高的场景，PCA 的计算耗时仍是一个额外负担。
*   **候选集依赖**：维度分配依赖于预设的候选压缩率集合（如 12.5%, 25%），如何自动确定最优的候选集仍有待研究。
*   **硬件算子优化**：虽然论文实现了 FlashAttention 集成，但混合维度的计算在某些硬件架构上可能面临内存碎片或 Kernel 调度开销，需要更深度的算子级优化。
