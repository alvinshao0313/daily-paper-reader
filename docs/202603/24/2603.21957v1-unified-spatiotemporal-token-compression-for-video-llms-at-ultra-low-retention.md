## 摘要
视频大语言模型 (Video-LLMs) 由于大量的视觉 token 而面临高昂的计算成本。现有的 token 压缩方法通常采用两阶段的时空压缩策略，依赖于特定阶段的指标以及时空可分离性的隐式假设。然而，在极低的保留率下，此类方法往往会导致分配不均，并丢失对问答至关重要的视觉证据。我们将 token 压缩重新表述为全局 token 保留池中的时空分配任务。我们提出了一种统一的选择机制，整合了注意力权重和语义相似度，以在全局范围内选择具有高贡献度和低冗余度的 token。未被选中的 token 通过聚类进行合并并重新填充，从而保持信息的完整性。在 LLM 内部，我们进一步引入了文本感知合并，根据查询相关性进行二次压缩。该方法无需重新训练，可作为即插即用的模块与现有的 Video-LLMs 兼容。实验表明，仅保留约 2% 的视觉 token 即可在多个基准测试中保持 90.1% 的基准性能，同时将 FLOPs 降低至约 2.6%。这些优势可推广到不同的骨干网络，降低了端到端的推理延迟和内存消耗。我们的统一时空 token 压缩策略在极低 token 保留率下的视频理解领域达到了当前最先进水平。

## Abstract
Video large language models (Video-LLMs) face high computational costs due to large volumes of visual tokens. Existing token compression methods typically adopt a two-stage spatiotemporal compression strategy, relying on stage-specific metrics and an implicit assumption of spatiotemporal separability. Under extremely low retention ratios, however, such approaches often result in unbalanced allocation and loss of visual evidence essential for question answering. We reformulate token compression as a spatiotemporal allocation task within a global token retention pool. We propose a unified selection mechanism that integrates attention weights and semantic similarity to globally select tokens with high contribution and low redundancy. Unselected tokens are merged via clustering and refilled, preserving information integrity. Inside the LLM, we further introduce text-aware merging to perform secondary compression based on query relevance. Without requiring retraining, our method serves as a plug-and-play module compatible with existing Video-LLMs. Experiments show that retaining only about 2% of visual tokens preserves 90.1% of baseline performance across multiple benchmarks, while reducing FLOPs to roughly 2.6%. These benefits generalize across diverse backbones, decreasing end-to-end inference latency and memory consumption. Our unified spatiotemporal token compression strategy establishes the state-of-the-art in video understanding under ultra-low token retention.

### 1. 核心问题与整体含义（研究动机和背景）
视频大语言模型（Video-LLMs）在处理长视频时，由于视觉 Token 数量庞大（如 32 帧视频可产生超过 6000 个 Token），导致推理延迟高、内存消耗大。
*   **现有局限：** 目前的 Token 压缩方法多采用“先空间后时间”或“先时间后空间”的两阶段策略，这基于时空可分离的隐式假设。
*   **痛点：** 在**超低保留率**（如仅保留 1%-5% 的 Token）下，这种分离策略会导致时空资源分配不均，容易丢失关键视觉证据，导致模型性能剧烈下降。
*   **目标：** 实现一种在极低保留率下依然能保持高性能、无需重训练且即插即用的压缩方案。

### 2. 方法论
论文提出了一种统一的时空 Token 压缩框架，包含两个核心阶段：

*   **统一时空压缩（LLM 外部）：**
    *   **全局筛选：** 不再分阶段处理，而是将所有视频帧的 Token 放入全局池。利用视觉编码器的 CLS 注意力分数衡量“贡献度”，利用余弦相似度衡量“冗余度”。
    *   **时空聚类（回收机制）：** 未被选入保留池的 Token 不直接丢弃，而是通过 **DPC-KNN（基于 K 近邻的密度峰值聚类）** 算法进行聚类合并，提取语义中心并重新填回保留池，以确保视觉信息的完整性。
*   **文本感知合并（LLM 内部）：**
    *   **查询引导：** 在 LLM 的中间层（如第 18 层），根据文本查询（Query）与视觉 Token 之间的交叉注意力分数及语义相似度，计算决策分数。
    *   **二次压缩：** 将与查询无关的视觉 Token 合并到最相似的保留 Token 中，进一步减少计算量。

*   **数据集：** 涵盖了 6 个主流视频理解基准测试：**MVBench**（综合任务）、**EgoSchema**（长视频）、**MLVU**（长视频理解）、**LongVideoBench**、**VideoMME** 以及 **ActivityNet-QA**（开放式问答）。

*   **硬件环境：** 实验在单张显卡上完成，使用了 **NVIDIA RTX 4090** 或 **A100**。

*   **效率提升：** 在 2% 保留率下，FLOPs 降低至原始模型的 **2.6%**，吞吐量显著提升。

### 5. 实验数量与充分性
*   **实验规模：** 论文进行了极其详尽的实验，包括在 4 种不同架构的骨干网络上进行测试，覆盖了从短视频到超长视频（1 小时以上）的多种场景。
*   **消融实验：** 针对注意力筛选、相似度剪枝、聚类合并、文本感知权重（$\lambda$）、剪枝层位置（$K$）等关键超参数均做了详细的消融研究。
*   **客观性：** 实验不仅对比了准确率，还对比了 FLOPs、TTFT（首字延迟）和吞吐量，评估维度全面且客观。

### 6. 主要结论与发现
*   **超高压缩比：** 在仅保留 **2%** 视觉 Token 的极端情况下，该方法仍能保持基准模型 **90.1%** 的性能。
*   **打破性能瓶颈：** 相比之前的 SOTA 方法（如 HoliTom），在 1% - 2% 的超低保留率下，本方法的性能优势更加明显（领先约 2-7 个百分点）。
*   **长视频优势：** 通过极低保留率，模型能够处理更多的输入帧（如从 16 帧扩展到 128 帧），在长视频基准测试中表现甚至超过了全量 Token 的短帧模型。

### 7. 优点
*   **全局最优：** 统一时空池化避免了分阶段压缩带来的局部最优问题。
*   **信息无损感：** 引入聚类回收机制，即使是被剪枝的 Token 也能通过聚类中心贡献语义，缓解了信息丢失。
*   **文本感知：** 内部合并机制考虑了用户提问的意图，使压缩具有动态针对性。
*   **通用性强：** 适配多种主流模型，无需任何微调，部署成本极低。

### 8. 不足与局限
*   **实时性限制：** 目前主要针对离线视频处理，对于实时流媒体视频的动态 Token 压缩支持尚不明确。
*   **采样偏差：** 依然依赖于均匀帧采样，对于某些动作极快或关键信息极短的视频，可能会因为采样层面的缺失而影响最终效果。
*   **计算开销：** 虽然减少了 LLM 的计算量，但外部的聚类（DPC-KNN）和相似度计算会带来一定的额外预处理开销（尽管文中称其远小于 LLM 节省的开销）。
