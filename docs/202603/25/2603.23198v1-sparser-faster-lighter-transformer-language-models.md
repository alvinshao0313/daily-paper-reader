## 摘要
扩展自回归大语言模型（LLM）推动了前所未有的进步，但也带来了巨大的计算成本。在这项工作中，我们通过利用 LLM 前馈层中的非结构化稀疏性来应对这些成本，前馈层是占据模型大部分参数和执行 FLOPs 的主要组件。为实现这一目标，我们引入了一种新的稀疏打包格式和一组 CUDA 内核，旨在与现代 GPU 的优化执行流水线无缝集成，从而在 LLM 推理和训练期间实现高效的稀疏计算。为了证实我们的收益，我们对 LLM 稀疏性进行了定量研究，证明简单的 L1 正则化可以诱导超过 99% 的稀疏性，且对下游性能的影响微乎其微。当与我们的内核结合使用时，我们展示了这些稀疏水平可以转化为显著的吞吐量、能效和内存使用优势，且这些优势随模型规模的扩大而增加。我们将以开源许可证发布所有代码和内核，以促进采用并加速相关研究，旨在将稀疏性确立为提高现代基础模型效率和可扩展性的实用维度。

## Abstract
Scaling autoregressive large language models (LLMs) has driven unprecedented progress but comes with vast computational costs. In this work, we tackle these costs by leveraging unstructured sparsity within an LLM's feedforward layers, the components accounting for most of the model parameters and execution FLOPs. To achieve this, we introduce a new sparse packing format and a set of CUDA kernels designed to seamlessly integrate with the optimized execution pipelines of modern GPUs, enabling efficient sparse computation during LLM inference and training. To substantiate our gains, we provide a quantitative study of LLM sparsity, demonstrating that simple L1 regularization can induce over 99% sparsity with negligible impact on downstream performance. When paired with our kernels, we show that these sparsity levels translate into substantial throughput, energy efficiency, and memory usage benefits that increase with model scale. We will release all code and kernels under an open-source license to promote adoption and accelerate research toward establishing sparsity as a practical axis for improving the efficiency and scalability of modern foundation models.

### 1. 论文的核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）规模的不断扩大，其推理和训练的计算成本、能耗及内存需求已成为行业发展的重大瓶颈。前馈网络（FFN）层占据了模型约 2/3 的参数和超过 80% 的计算量（FLOPs）。尽管研究表明 LLM 的 FFN 层存在天然的激活稀疏性，但由于**非结构化稀疏性与现代 GPU 架构（针对稠密计算优化）之间存在严重的硬件不匹配**，导致理论上的计算量减少往往无法转化为实际的运行加速。本研究旨在通过软硬件协同设计，打破“稀疏算子比稠密算子慢”的悖论。

### 2. 论文提出的方法论
论文提出了一套完整的稀疏 LLM 优化方案，核心包括：
*   **TwELL (Tile-wise ELLPACK) 格式**：这是一种新型稀疏数据打包格式。与传统 ELL 格式需要全局同步不同，TwELL 将矩阵划分为 1D 瓦片（Tiles），允许在矩阵乘法内核的收尾阶段（Epilogue）直接生成稀疏索引和值，实现了算子融合并消除了内存同步开销。
*   **推理内核融合**：设计了定制 CUDA 内核，将门控（Gate）、上升（Up）和下降（Down）投影融合进两个内核调用中。利用 TwELL 格式，内核仅处理非零激活对应的权重行/列，大幅减少了 DRAM 访问。
*   **混合训练格式（Hybrid Format）**：针对训练阶段稀疏模式极度不均匀的特点，提出将激活行动态划分为“紧凑 ELL 存储”和“稠密备份存储”。这种设计既能享受稀疏加速，又能处理那些激活数异常高的 Token，保证了训练的稳定性。
*   **L1 正则化诱导稀疏**：在训练目标中加入简单的 L1 损失项，配合 ReLU 激活函数，强制模型在不损失性能的前提下产生极高的稀疏度。

### 6. 主要结论与发现
*   **极高稀疏度**：通过轻微的 L1 正则化，LLM 可以达到 **99% 以上的稀疏度**，而下游任务性能几乎无损（下降在 2% 以内）。
*   **规模效应**：模型规模越大，稀疏化带来的收益越显著。2B 模型在推理时提速 **20.5%**，训练提速 **21.9%**。

*   **输入特性**：模型会自动将计算资源分配给信息量更大的 Token（如特定名词、动词）和序列起始位置的 Token。

### 7. 优点（亮点）
*   **软硬件协同**：不只是改进算法，而是通过底层 CUDA 内核重写（利用了 Hopper 架构的 TMA 等特性）实现了真正的物理加速。
*   **实用性强**：无需改变现有的训练流水线，仅需更换激活函数和增加正则化项，且代码开源，易于集成。
*   **训练/推理双加速**：大多数稀疏化研究仅关注推理，本文证明了稀疏性在训练阶段同样能带来巨大的内存和速度收益。

### 8. 不足与局限
*   **硬件依赖**：部分核心优化（如 TMA 访问）针对 NVIDIA Hopper 架构（H100）设计，在旧款 GPU 或非 NVIDIA 硬件上的加速效果可能受限。
*   **死神经元风险**：高强度 L1 正则化会导致大量神经元永久失活（Dead Neurons），虽然论文提出了缓解策略，但这仍是稀疏模型训练中的潜在不稳定因素。
*   **激活函数限制**：为了获得高稀疏度，必须使用 ReLU 而非目前主流的 SiLU，尽管实验显示性能差异微小，但在追求极致性能的场景下可能存在争议。
