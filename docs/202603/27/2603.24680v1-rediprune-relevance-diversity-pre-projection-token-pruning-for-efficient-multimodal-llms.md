## 摘要
近期的多模态大语言模型计算开销巨大，因为 Transformer 必须处理大量的视觉 Token。我们提出了 ReDiPrune，这是一种在视觉-语言投影器之前应用的无需训练的 Token 剪枝方法，此时视觉特征仍保持着丰富的判别性。与在压缩表示上运行的投影后剪枝方法不同，ReDiPrune 直接从视觉编码器输出中选择信息丰富的 Token，从而保留了细粒度的空间和语义线索。每个 Token 都由一个轻量级规则进行评分，该规则共同考虑了文本条件相关性和最大最小多样性，确保所选 Token 既与查询相关又不冗余。ReDiPrune 完全是即插即用的，不需要重新训练或修改架构，并且可以无缝插入到编码器和投影器之间。在四个视频和五个图像基准测试中，它一致地改善了准确率与效率之间的权衡。例如，在 EgoSchema 基准测试中，使用 LLaVA-NeXT-Video-7B 模型，仅保留 15% 的视觉 Token 即可获得 +2.0% 的绝对准确率提升，同时将 TFLOPs 计算量减少 6 倍以上。代码可在 https://github.com/UA-CVML/ReDiPrune 获取。

## Abstract
Recent multimodal large language models are computationally expensive because Transformers must process a large number of visual tokens. We present \textbf{ReDiPrune}, a training-free token pruning method applied before the vision-language projector, where visual features remain rich and discriminative. Unlike post-projection pruning methods that operate on compressed representations, ReDiPrune selects informative tokens directly from vision encoder outputs, preserving fine-grained spatial and semantic cues. Each token is scored by a lightweight rule that jointly consider text-conditioned relevance and max-min diversity, ensuring the selected tokens are both query-relevant and non-redundant. ReDiPrune is fully plug-and-play, requiring no retraining or architectural modifications, and can be seamlessly inserted between the encoder and projector. Across four video and five image benchmarks, it consistently improves the accuracy-efficiency trade-off. For example, on EgoSchema with LLaVA-NeXT-Video-7B, retaining only 15\% of visual tokens yields a +2.0\% absolute accuracy gain while reducing computation by more than $6\times$ in TFLOPs. Code is available at https://github.com/UA-CVML/ReDiPrune.

#### 1. 核心问题与整体含义（研究动机和背景）
多模态大语言模型（MLLM）在处理图像和视频时，由于 Transformer 架构对序列长度具有二次方计算复杂度，海量的视觉 Token（尤其是高分辨率图像或长视频）导致了巨大的计算开销、内存占用和推理延迟。
*   **现有问题**：目前的 Token 剪枝方法大多在“投影后”（Post-projection）进行，即在视觉特征进入 LLM 嵌入空间后才筛选，这往往会损失细粒度的空间细节；或者需要昂贵的重新训练（如 TRIM）。
*   **核心动机**：在视觉特征进入投影器（Projector）之前的原始特征空间进行剪枝，利用原始特征的丰富判别性，在不进行模型微调的前提下，实现高效且精准的 Token 筛选。

#### 2. 论文提出的方法论
**ReDiPrune** 是一种**免训练（Training-free）**、**即插即用**的投影前剪枝模块。其核心思想是结合“文本相关性”与“视觉多样性”来筛选最具代表性的 Token。
*   **核心流程**：
    1.  **查询向量构建**：从用户输入的 Prompt 中提取文本 Token，通过加权聚合（如指数加权）生成一个文本查询向量 $q$。
    2.  **文本相关性评分**：计算视觉编码器输出的每个 Token 与查询向量 $q$ 的余弦相似度，识别出与问题相关的视觉区域。
    3.  **视觉多样性度量**：计算 Token 之间的余弦不相似度，确保选出的 Token 集合能够覆盖不同的视觉信息，避免冗余。
    4.  **统一目标函数与贪心求解**：定义一个综合目标函数，在满足预设保留比例（Retention Ratio）的条件下，通过贪心算法迭代选择既与文本相关又具有视觉多样性的 Token 子集。
    5.  **投影与解码**：仅将筛选后的少量 Token 送入投影器和 LLM 进行后续推理。

#### 6. 主要结论与发现
*   **性能提升**：在大幅剪枝的情况下，性能不降反升。例如在 EgoSchema 上，仅保留 15% 的 Token，准确率反而提升了 2.0%。
*   **计算加速**：在视频任务中效果尤为显著，计算量（TFLOPs）可降低 6 倍以上，端到端推理延迟显著缩短（如从 0.447s 降至 0.146s）。
*   **去噪作用**：剪枝过程实际上起到了一种“去噪”和“正则化”的作用，通过移除无关和冗余信息，使 LLM 的注意力更加集中在与问题相关的视觉证据上。

#### 7. 优点（亮点）
*   **位置前移**：在投影前剪枝，保留了原始视觉特征的细粒度语义，优于传统的投影后剪枝。
*   **完全免训练**：无需任何参数微调，可直接适配现有的 LLaVA 类模型，部署成本极低。
*   **双重引导**：同时考虑文本相关性和视觉多样性，解决了“仅看文本导致信息丢失”或“仅看多样性导致无关信息过多”的问题。
*   **通用性强**：在图像和视频任务上均表现出稳健的性能提升。

#### 8. 不足与局限
*   **架构限制**：目前主要在 LLaVA 及其变体上验证，对于非 Patch 结构的视觉编码器或其他架构（如 Q-Former 类）的适用性有待进一步探索。
*   **时空优化**：目前的剪枝主要基于单帧内进行（Per-frame selection），尚未充分利用视频全局的时空冗余性进行跨帧联合优化。
*   **超参数敏感性**：虽然提供了默认配置，但针对不同复杂度的任务，最优的保留比例（r）和平衡因子（α）可能需要手动调整。
