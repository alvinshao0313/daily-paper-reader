## 摘要
在本文中，我们提出了一个新颖的见解，即现成的大语言模型（LLM）可以作为出色的 Token 压缩器和解压器。为了证明这一点，我们设计了一个自表达自编码学习框架，该框架微调预训练的 LLM，将长文本翻译成一种由离散、变长潜码组成的紧凑内部语言（称为 Z-tokens），并从中精确地重建原始文本。由此产生的表示是内容自适应的：通过轻量级的基于 LoRA 的适配器头，语义密集的片段会获得更多的 Z-tokens，而冗余或可预测的区域则会被大幅压缩。实验表明，我们的方法在 Wikipedia、CNN/DailyMail、HotpotQA 和 Qulac 风格的长查询数据集上实现了高达 18 倍的 Token 缩减，同时保持了重建保真度和下游性能。这种简单而有效的设计支持包括提示词压缩和直接在 Z-token 空间中进行自回归生成在内的应用，为实现 Token 高效的长上下文推理提供了一条潜在路径。

## Abstract
In this paper, we establish the novel insight that an off-the-shelf LLM can function as an excellent token compressor and decompressor. To demonstrate, we design a self-expressive autoencoding learning framework fine-tunes a pretrained LLM to translate long texts into a compact internal language of discrete, variable-length latent codes, termed Z-tokens, and to reconstruct the original text exactly from them. The resulting representation is content-adaptive: semantically dense segments receive more Z-tokens, while redundant or predictable regions are aggressively compressed, via lightweight LoRA-based adapter heads. Empirically, our method achieves up to 18 times token reduction on Wikipedia, CNN/DailyMail, HotpotQA, and Qulac-style long-query datasets, while preserving reconstruction fidelity and downstream performance. This simple yet effective design supports applications including prompt compression and autoregressive generation directly in the Z-token space, offering a potential pathway toward token-efficient long-context reasoning.

### 2. 方法论：核心思想与关键技术
*   **核心思想**：采用**自表达自编码（Self-expressive Autoencoding）**框架。利用 LoRA 微调预训练模型，使其学习将输入序列 $X$ 映射为更短的离散 Token 序列 $Z$（Z-tokens），并能从 $Z$ 中还原 $X$ 或直接进行推理。
*   **关键技术细节**：
    *   **变长压缩（Autoregressive Compressor）**：不同于以往生成固定长度向量的方法，该压缩器自回归地生成 Z-tokens。语义复杂的片段分配更多 Z-tokens，冗余片段则分配更少。
    *   **离散化处理**：扩展模型词表以包含 Z-tokens。训练时使用 **Gumbel-Softmax** 估计器和直通估计器（Pass-through Estimator）解决离散 Token 采样不可导的问题。
    *   **解压器双模式**：
        1.  **推理模式（Inferencer）**：直接基于 Z-tokens 进行下游任务（如问答、摘要）。
        2.  **重建模式（Reconstructor）**：将 Z-tokens 还原为原始文本，支持在 Z-token 空间进行抽象推理后再还原。
    *   **滑动窗口压缩**：针对超长文本，采用带重叠的滑动窗口，并通过一致性约束（Consistency Constraint）确保语义连贯。
    *   **计划采样（Scheduled Sampling）**：在训练中逐步从教师强制（Teacher-Forcing）过渡到自由解码，以减轻推理时的曝光偏差（Exposure Bias）。

### 6. 主要结论与发现
*   **高压缩比与保真度**：在 Wikipedia 等数据集上实现了最高 **18 倍**的 Token 缩减，且在 4 倍压缩下几乎能完美重建原文（BLEU-4 > 99）。

*   **Z-tokens 的多义性**：发现 Z-tokens 并非简单的 1:1 映射，而是具有上下文相关性的语义单元，类似于人类语言中的成语或隐喻，具有很强的语义抽象能力。

### 7. 优点（亮点）
*   **内容自适应性**：通过自回归生成 Z-tokens，解决了固定压缩比带来的信息瓶颈问题。
*   **端到端集成**：将压缩与解压过程内化在 LLM 自身的推理流中，无需外部复杂的编码器。
*   **可解释性**：离散的 Z-tokens 相比连续向量更具可解释性，且支持直接在压缩空间进行“思考”和生成。
*   **鲁棒性**：通过计划采样有效解决了自回归压缩中的误差累积问题。

### 8. 不足与局限
*   **微调成本**：虽然使用了 LoRA，但仍需要针对特定模型进行微调训练，无法完全“开箱即用”。
*   **长程依赖协调**：在滑动窗口模式下，虽然有重叠约束，但跨超长窗口的全局语义协调机制仍有优化空间。
*   **任务感知局限**：目前的压缩过程主要是通用的语义压缩，尚未实现针对特定任务（如只保留事实或只保留逻辑）的定制化压缩。
*   **模型依赖**：压缩效果高度依赖于底座模型的语义理解能力，在极小规模模型上效果受限。
