## 摘要
视觉语言导航（VLN）日益依赖大型视觉语言模型，但其高昂的推理成本与实时部署的需求相冲突。Token 缓存是一种极具前景的免训练策略，通过在不同帧之间重用稳定的视觉 token 来避免冗余计算。然而，现有方法假设相机是静态的且语义焦点固定，而 VLN 任务从根本上违背了这些假设。我们识别出两种失效模式：（1）视觉动态，即视点偏移导致 token 位置在帧间发生位移，使得基于位置的匹配会关联到未对齐的内容；（2）语义动态，即随着导航的推进，token 的相关性在不同任务阶段发生转移，导致缓存状态过时。为此，我们提出了 VLN-Cache，这是一个具备视觉动态感知和语义动态感知能力的缓存框架，它引入了视角对齐重映射以恢复几何对应关系，并采用任务相关性显著性过滤器在语义转换时否决重用。此外，一种层自适应熵策略进一步平衡了每层的重用预算。在 R2R-CE 仿真基准上的实验表明，该方法在保持竞争性导航成功率的同时，实现了高达 1.52 倍的加速。

## Abstract
Vision-and-Language Navigation (VLN) increasingly relies on large vision-language models, but their inference cost conflicts with real-time deployment. Token caching is a promising training-free strategy that avoids redundant computation by reusing stable visual tokens across frames. However, existing methods assume a static camera and fixed semantic focus, assumptions that VLN fundamentally violates. We identify two failure modes: (1) visual dynamics, where viewpoint shift displaces token positions across frames, causing position-wise matching to pair misaligned content; (2) semantic dynamics, where token relevance shifts across task stages as navigation progresses, making cached states stale. We propose VLN-Cache, a visual-dynamic-aware and semantic-dynamic-aware caching framework that introduces view-aligned remapping to recover geometric correspondences and a task-relevance saliency filter to veto reuse at semantic transitions. A layer-adaptive entropy policy further balances the per-layer reuse budget. Experiments on the R2R-CE simulation benchmark show up to 1.52x speedup while maintaining competitive navigation success rates.

*   **核心问题**：现有的 Token 缓存技术（通过复用相邻帧的稳定视觉 Token 来加速）主要针对静态相机或静态场景设计。在 VLN 任务中，这些方法会失效，原因有二：
    1.  **视觉动态（Visual Dynamics）**：智能体在移动和旋转，导致同一物理对象在图像坐标系中的位置发生偏移，传统的“按位置匹配”会导致错误的 Token 复用。
    2.  **语义动态（Semantic Dynamics）**：随着导航任务的推进，同一区域的语义重要性会随指令阶段改变（例如：走过沙发后，沙发就不再是关键地标），导致缓存的语义状态过时。
*   **研究目标**：开发一种能够感知视觉位移和语义变化的免训练 Token 缓存框架，以实现 VLN 的实时加速。

### 2. 方法论
VLN-Cache 提出了一套双感知（Dual-aware）的缓存框架，核心技术包括：
*   **视觉动态感知（视角对齐重映射）**：利用 RGB-D 信息和相机位姿变换，将当前帧的 Token 中心点反投影到 3D 空间，再投影回前一帧。通过这种几何对应关系（而非简单的坐标位置）来检索缓存的 Key-Value (KV) 状态。
*   **语义动态感知（任务相关性显著性过滤器）**：引入一个“否决机制”。计算每个 Token 与导航指令的相关性得分。如果某个 Token 的相关性极高，或者相关性随时间发生了剧烈变化（意味着进入了新的任务阶段），则强制重新计算该 Token，不使用缓存。
*   **层自适应熵策略（Layer-Adaptive Policy）**：基于 Transformer 各层注意力分布的熵值来动态调整复用比例。浅层（处理低级特征）通常更稳定，允许更高比例的复用；深层（处理决策逻辑）则更保守。
*   **双感知融合**：只有当 Token 在几何上稳定且在语义上未发生剧烈转换时，才允许复用缓存。
