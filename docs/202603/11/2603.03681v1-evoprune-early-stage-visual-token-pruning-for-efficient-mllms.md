## 摘要
多模态大语言模型 (MLLMs) 在视觉语言任务中表现出强大的性能，但其推理效率受到高分辨率图像和视频等复杂场景中视觉 Token 指数级增长的严重限制。现有的视觉 Token 剪枝方法主要在视觉编码之后进行，忽略了编码阶段产生的巨大计算开销。为了解决这一问题，我们提出了 EvoPrune，这是一种用于 MLLMs 的早期视觉 Token 剪枝方法，直接在视觉编码过程中执行剪枝。具体而言，EvoPrune 采用了一种由 Token 相似度、多样性和基于注意力的重要性引导的逐层剪枝策略，以在选定的编码层保留最具信息量的视觉 Token。在图像和视频基准测试上的广泛实验验证了 EvoPrune 的有效性。特别是在 VideoMME 数据集上，EvoPrune 在性能下降不到 1% 的情况下实现了 2 倍的推理加速，展示了其在延迟敏感型 MLLM 部署中的潜力。

## Abstract
Multimodal Large Language Models (MLLMs) have shown strong performance in vision-language tasks, but their inference efficiency is severely limited by the exponential growth of visual tokens in complex scenarios such as high-resolution images and videos. Existing visual token pruning methods mainly operate after visual encoding, overlooking the substantial computational cost incurred during the encoding stage. To address this issue, we propose EvoPrune, an early-stage visual token pruning method for MLLMs that performs pruning directly during visual encoding. Specifically, EvoPrune employs a layer-wise pruning strategy guided by token similarity, diversity, and attention-based importance to retain the most informative visual tokens at selected encoding layers. Extensive experiments on image and video benchmarks validate the effectiveness of EvoPrune. In particular, on the VideoMME dataset, EvoPrune achieves 2$\times$ inference speedup with less than 1% performance degradation, demonstrating its potential for latency-sensitive MLLM deployment.

### 1. 论文的核心问题与整体含义
多模态大语言模型（MLLM）在处理高分辨率图像和长视频时，视觉 Token 的数量呈指数级增长，导致推理效率低下。现有的视觉 Token 剪枝方法大多在**视觉编码之后**（即进入 LLM 之前）进行，这虽然减少了 LLM 的计算量，但忽略了视觉编码器（Vision Encoder）本身在高分辨率输入下的巨大计算开销。论文的核心动机是提出一种**早期剪枝（Early-Stage Pruning）**方案，通过在视觉编码过程中直接减少 Token 数量，同时加速编码器和后续 LLM 的推理过程。

### 2. 论文提出的方法论
EvoPrune 提出了一种在视觉编码器内部执行的**层级引导 Token 合并策略**，其核心思想包括：
*   **逐层剪枝预算分配（Layer-wise Budget Allocation）**：在视觉编码器的不同层之间分配剪枝配额。实验发现“Skip”策略（每隔一层进行一次合并）效果最稳健。
*   **得分引导的 Token 合并（Score-Guided Merging）**：通过一个综合得分矩阵来决定哪些 Token 对应该被合并。得分由三个维度构成：
    1.  **相似度吸引（Similarity Attraction）**：利用余弦相似度识别语义冗余的 Token。
    2.  **多样性惩罚（Diversity Penalty）**：通过局部密度估计，保护位于稀疏区域、具有独特信息的 Token 不被合并。
    3.  **注意力保留（Attention Preservation）**：利用编码器内部的注意力权重识别关键 Token，并设置“保护区”，确保这些对下游推理至关重要的信息不被破坏。
*   **算法流程**：在选定的编码层，将 Token 集分为两组，计算跨组的综合得分，选择得分最高的 Top-r 对进行合并，从而实现 Token 数量的逐步缩减。

### 6. 主要结论与发现
*   **效率提升**：在 VideoMME 基准上，EvoPrune 实现了 **2 倍的推理加速**，而性能下降不到 1%。
*   **端到端加速**：与仅在 LLM 阶段减负的方法不同，EvoPrune 显著降低了视觉编码器的耗时（加速约 1.8 倍）和中间模块耗时（加速约 5 倍）。
*   **性能保持**：在极端剪枝（保留极少 Token）的情况下，EvoPrune 的准确率依然优于现有的后置剪枝方法，证明了其多因子引导策略能更有效地保留关键视觉特征。

### 7. 优点
*   **全链路加速**：首次系统性地解决了视觉编码阶段的计算瓶颈，适合处理超长视频和超高分辨率图像。
*   **即插即用**：无需任何微调或重新训练，可直接应用于现有的预训练 MLLM。
*   **综合评估机制**：结合了相似度、多样性和注意力，比单一依赖注意力权重的剪枝方法更具鲁棒性。

### 8. 不足与局限
*   **层级策略依赖经验**：虽然提出了多种分配策略，但最佳策略（如 Skip）仍带有一定的启发式色彩，可能需要针对不同架构微调。

*   **模型覆盖面**：实验主要集中在 7B 规模的模型上，尚未在更大规模（如 70B+）或不同架构（如 MoE）的 MLLM 上验证其扩展性。
