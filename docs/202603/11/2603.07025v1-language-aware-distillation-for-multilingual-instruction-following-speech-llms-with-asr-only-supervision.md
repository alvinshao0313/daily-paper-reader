## 摘要
能够理解并遵循多种语言指令的语音大语言模型（Speech LLMs）在现实交互中非常有用，但通过监督微调进行训练十分困难，因为这需要大规模且特定于任务的语音语料库。虽然最近基于蒸馏的方法仅使用带标注的 ASR 数据，通过轻量级投影器对齐文本和语音，训练出了性能良好的纯英文语音 LLM，但由于共享投影器中的语言干扰，这些模型在扩展到多语言设置时表现不佳。我们通过引入语言感知蒸馏来解决这一问题，该方法使用查询库（query bank）和门控网络，通过 Q-Former 投影器选择或混合查询标记（query tokens）。我们的方法在指令遵循方面比匹配的多语言蒸馏基线提高了 14%。我们进一步合成了 Audio-MLQA，这是一个基于 MLQA 并使用高质量 TTS 问题构建的多语言口语问答基准。我们的最佳模型在 Audio-MLQA 上比现有的语音 LLM 基线提高了 32%。

## Abstract
Speech Large Language Models (LLMs) that understand and follow instructions in many languages are useful for real-world interaction, but are difficult to train with supervised fine-tuning, requiring large, task-specific speech corpora. While recent distillation-based approaches train performant English-only Speech LLMs using only annotated ASR data by aligning text and speech using only a lightweight projector, these models under-perform when scaled to multilingual settings due to language interference in the shared projector. We address this by introducing language-aware distillation using a query bank and a gating network that selects or mixes query tokens using a Q-Former projector. Our approach shows gains of 14% over matched multilingual distillation baselines on instruction following. We further synthesize Audio-MLQA, a multilingual spoken QA benchmark built on MLQA with high-quality TTS questions. Our best model improves over existing Speech LLM baselines by 32% on Audio-MLQA.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：现有的语音大模型通常依赖大规模的监督微调（SFT），这需要昂贵的任务特定语音数据。虽然基于蒸馏的方法（如 DiVA）可以仅利用 ASR 数据将语音与冻结的 LLM 对齐，但在多语言场景下，**共享的投影器（Projector）会产生语言干扰**。
*   **研究动机**：当模型处理多种语言（尤其是差异巨大的语言，如英语和中文）时，单一的静态查询序列（Query Tokens）难以捕捉不同语言的音素和语义细微差别，导致低资源或弱势语言的性能下降。
*   **整体含义**：本文旨在通过改进架构，在不微调语音编码器和 LLM 的前提下，实现高效、低成本且具备多语言指令遵循能力的语音大模型。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：引入**查询库（Query Bank）**和**门控网络（Gating Network）**，根据输入语音动态选择或混合特定语言的查询标记，从而解耦不同语言的信息。
*   **关键技术细节**：
    *   **模型架构**：由冻结的 Whisper-large-v3 编码器、冻结的 Llama-SEA-LION-v3-8B-IT（针对东南亚语言优化）和可训练的 Q-Former 投影器组成。
    *   **查询库与门控**：为 $K$ 种语言维护 $K$ 组可学习的查询标记。门控网络（卷积或注意力池化结构）根据语音特征输出语言概率，通过**硬选择（Hard Selection）**或**软混合（Soft Mixing）**生成最终进入投影器的查询。
    *   **训练目标（三部分损失）**：
        1.  **LID 损失**：监督门控网络进行准确的语言识别。
        2.  **输入蒸馏损失 ($L_{IN}$)**：使投影后的语音嵌入在特征空间上接近对应文本的 LLM 输入嵌入。
        3.  **输出蒸馏损失 ($L_{OUT}$)**：使 LLM 在语音输入下的末层隐藏状态接近其在文本输入下的状态，实现行为对齐。
    *   **调度教师强制（Scheduled Teacher Forcing）**：在训练初期强制使用正确的语言标签引导门控，随后逐渐过渡到模型自主预测。

### 6. 主要结论与发现
*   **语言感知路由有效**：硬门控（Hard-gating）版本在指令遵循任务上比多语言基线提升了 14%，在 Audio-MLQA 上提升了 32%。
