## 摘要
当答案依赖于即时信息或冲突信息时，大语言模型（LLMs）在事实性问答任务中仍然表现得不够可靠。尽管检索增强和使用工具的 LLM 减少了幻觉，但它们通常依赖隐式规划，导致工具使用效率低下。我们提出了一个模块化框架，将规划与事实检索及答案合成明确分离。通过教师-学生框架训练一个轻量级学生规划器，以生成由抽象推理步骤和可搜索事实请求组成的结构化分解。监督信号仅包含规划轨迹和事实请求，不提供事实性答案或检索到的证据。在推理阶段，规划器生成计划，而经过提示工程优化的模块则执行检索和响应合成。我们在 SEAL-0 上评估了所提框架，这是一个针对搜索增强型 LLM 的极具挑战性的基准测试。结果表明，与单体推理模型和基于提示的工具增强框架相比，有监督规划提高了准确性并降低了延迟，证明了显式学习的规划结构对于构建可靠的事实性 LLM 至关重要。

## Abstract
Fact-seeking question answering with large language models (LLMs) remains unreliable when answers depend on up-to-date or conflicting information. Although retrieval-augmented and tool-using LLMs reduce hallucinations, they often rely on implicit planning, leading to inefficient tool usage. We propose a modular framework that explicitly separates planning from factual retrieval and answer synthesis. A lightweight student planner is trained via a teacher-student framework to generate structured decompositions consisting of abstract reasoning steps and searchable fact requests. The supervision signals contain only planning traces and fact requests, without providing factual answers or retrieved evidence. At inference, the planner produces plans, while prompt-engineered modules perform retrieval and response synthesis. We evaluate the proposed framework on SEAL-0, an extremely challenging benchmark for search-augmented LLMs. Results show that supervised planning improves both accuracy and latency compared to monolithic reasoning models and prompt-based tool-augmented frameworks, demonstrating that explicitly learned planning structures are essential for reliable fact-seeking LLMs.

### 2. 方法论：核心思想与关键技术
*   **核心思想**：通过教师-学生架构，将教师模型的“问题分解与规划能力”蒸馏给小型学生模型，但在训练数据中剔除所有具体的事实答案和检索证据，使学生模型仅掌握抽象的推理结构。
*   **模块化流水线**：
    1.  **规划（Planning）**：学生规划器将问题 $q$ 分解为结构化计划 $P = (R, F)$。其中 $R$ 是抽象推理步骤，$F$ 是原子级事实请求（如搜索查询或计算指令）。
    2.  **检索（Retrieval）**：通过外部工具（如 Google Search API）执行事实请求。
    3.  **事实提取（Extraction）**：从冗长的搜索结果中提取简洁的原子事实。
    4.  **答案聚合（Aggregation）**：根据推理步骤和提取的事实合成最终答案。
*   **关键技术细节**：
    *   **教师模型**：使用 GPT-5.2 生成高质量的规划轨迹，严格禁止其直接回答问题。
    *   **学生模型**：对 Qwen 3-8B 进行微调，使其模仿教师的 JSON 格式输出（包含 `required_information` 和 `reasoning_steps`）。
    *   **依赖解析**：支持占位符（如 `<RESULT_1>`），允许后续查询依赖于前序步骤的结果。

### 6. 主要结论与发现
*   **准确性提升**：在 SEAL-0 上，本文框架（学生规划器）达到了 **10.8%** 的准确率，显著优于单体模型的 1.8% 和 Prompted 框架的 6.3%。
*   **延迟降低**：学生规划器的平均延迟为 **27.8秒**，远低于单体推理模型的 159.9秒。这得益于规划器生成的查询更精准，且避免了冗长的内部思维链（CoT）循环。
*   **推理与工具的冲突**：实验发现，在工具增强框架中开启 LLM 的内部推理反而会降低准确率（从 6.3% 降至 3.6%），因为模型容易陷入错误的推理循环或生成格式错误的计划。

### 7. 优点
*   **知识解耦**：通过“无知识蒸馏”，解决了模型参数化知识过时的问题，使模型专注于逻辑调度。
*   **高效轻量**：证明了 8B 规模的小模型通过专门的规划训练，在复杂任务上可以超越未经过针对性训练的巨型模型。
*   **高可靠性**：强制要求所有事实必须来源于外部证据，极大降低了幻觉风险。
*   **结构化输出**：采用 JSON 格式计划，增强了 AI Agent 工作流的可解释性和可维护性。

### 8. 不足与局限
*   **检索依赖**：系统的最终表现高度依赖于外部搜索引擎（如 SerpAPI）的质量。如果搜索不到相关信息，系统将无法回答。
*   **错误传播**：流水线式的架构意味着如果规划器生成的第一个查询有误，后续所有步骤都会失效。
*   **应用范围限制**：该框架主要针对事实寻求型任务，对于创意写作、纯逻辑推导或不需要外部信息的任务可能并不适用。
*   **实时性挑战**：尽管比单体模型快，但多次外部 API 调用和多步迭代仍存在一定的端到端延迟，难以满足极高实时性的场景。
