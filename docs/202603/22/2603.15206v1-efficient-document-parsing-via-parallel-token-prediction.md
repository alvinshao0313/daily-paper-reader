## 摘要
文档解析作为一项基础且关键的视觉任务，正受到视觉语言模型（VLM）的变革性影响。然而，VLM 固有的自回归（AR）解码方式造成了严重的瓶颈，极大地限制了解析速度。在本文中，我们提出了并行 Token 预测（PTP），这是一种可插拔、模型无关且简单有效的方法，使 VLM 能够并行生成多个未来 Token，并提高样本效率。具体而言，我们在输入序列中插入一些可学习的 Token，并设计相应的训练目标，使模型具备用于文档解析的并行解码能力。此外，为了支持有效训练，我们开发了一个全面的数据生成流水线，能够为 VLM 高效生产大规模、高质量的文档解析训练数据。在 OmniDocBench 和 olmOCR-bench 上的广泛实验表明，我们的方法不仅显著提高了解码速度（1.6 倍至 2.2 倍），还减少了模型幻觉，并展现出强大的泛化能力。

## Abstract
Document parsing, as a fundamental yet crucial vision task, is being revolutionized by vision-language models (VLMs). However, the autoregressive (AR) decoding inherent to VLMs creates a significant bottleneck, severely limiting parsing speed. In this paper, we propose Parallel-Token Prediction (PTP), a plugable, model-agnostic and simple-yet-effective method that enables VLMs to generate multiple future tokens in parallel with improved sample efficiency. Specifically, we insert some learnable tokens into the input sequence and design corresponding training objectives to equip the model with parallel decoding capabilities for document parsing. Furthermore, to support effective training, we develop a comprehensive data generation pipeline that efficiently produces large-scale, high-quality document parsing training data for VLMs. Extensive experiments on OmniDocBench and olmOCR-bench demonstrate that our method not only significantly improves decoding speed (1.6x-2.2x) but also reduces model hallucinations and exhibits strong generalization abilities.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：视觉语言模型（VLM）在文档解析（如 OCR、公式识别、表格提取）中表现卓越，但其固有的**自回归（AR）解码机制**导致推理速度极慢。由于需要逐个生成 Token，处理高密度、复杂的文档时延迟巨大，成为大规模实际应用的瓶颈。
*   **研究背景**：文档解析本质上是高确定性的“转录”任务，而非开放式生成。作者认为文档图像可以被视为多个 Patch 的组合，理论上具备并行识别的潜力。因此，研究如何打破串行生成限制、实现多 Token 并行预测具有重要意义。

### 2. 方法论：并行 Token 预测（PTP）
*   **核心思想**：通过在输入序列中插入可学习的**寄存器 Token（Register Tokens）**，并修改训练目标与注意力掩码，使模型在每一个解码步骤中能够同时预测当前及未来的多个 Token。
*   **关键技术细节**：
    *   **寄存器 Token 插入**：在训练序列的每个常规 Token 后附加 $n$ 个连续的寄存器 Token。这些 Token 共享相同的 ID 和嵌入，但具有不同的位置编码。
    *   **训练目标**：定义了 PTP 损失函数 $L_{PTP} = \alpha L_{NTP} + (1-\alpha) L_{reg}$。其中 $L_{reg}$ 训练寄存器 Token 根据其相对位置预测未来的 Token（例如，紧随 $x_i$ 的第一个寄存器预测 $x_{i+2}$）。
    *   **注意力掩码修改**：设计了特殊的因果掩码，确保常规 Token 不受寄存器 Token 干扰，而寄存器 Token 仅能关注之前的常规 Token 和同组内的寄存器 Token。
    *   **推理加速**：推理时，每步附加 $n$ 个寄存器 Token，实现 $n+1$ 倍的理论并行度。每步结束后，丢弃寄存器产生的 KV Cache，并用预测出的真实 Token 替换，以维持长文本生成的准确性。
*   **数据引擎**：构建了包含 1.8M 样本的高质量布局级数据集，通过多模型协作标注（GPT-4o, Gemini 等）、LLM 后处理纠错及基于 Embedding 和 pHash 的双重去重策略确保数据质量。

### 6. 主要结论与发现
*   **显著加速**：PTP-1（并行预测 2 个 Token）实现 **1.6 倍**加速，PTP-2 实现 **2.2 倍**加速。
*   **精度不降反升**：在 OmniDocBench 上，PTP-1 的文本识别精度甚至优于基准 NTP 模型。这归功于 PTP 强化了模型对上下文的利用能力。
*   **抑制幻觉**：PTP 减少了模型对序列上下文的过度依赖，更多地关注视觉证据，从而降低了在模糊或噪声图像中的幻觉率。
*   **强泛化性**：该方法在 Qwen2.5-VL、InternVL3 等不同架构上均有效，且能与投机采样（Speculative Decoding）结合，在 VLU 任务中达到 82% 的接受率。

### 7. 优点（亮点）
*   **模型无关与即插即用**：不需要修改模型核心架构（如增加额外的 Head 或 Block），仅通过插入 Token 和调整掩码即可实现。
*   **训练高效**：相比 MTP，PTP 收敛更快，且对超参数（如预测头数量）不敏感。
*   **数据质量高**：论文详细描述了 layout-level 数据的生成流水线，这对文档解析社区具有独立贡献。
*   **推理策略巧妙**：通过 KV Cache 替换机制，完美平衡了并行速度与自回归的准确性。

### 8. 不足与局限
*   **外推性能损失**：当推理时使用的寄存器 Token 数量超过训练时的数量（如训练 $n=2$，推理 $n=4$）时，性能会出现一定程度的下降。
*   **任务局限性**：虽然在确定性高的 OCR 任务中表现完美，但在逻辑推理要求极高的开放式任务（如复杂数学证明）中，仍需配合投机采样机制才能确保精度不丢失。
*   **硬件依赖**：虽然是内存受限任务，但并行预测仍会增加瞬时计算量，在算力极度受限的边缘设备上加速效果可能打折扣。
