## 摘要
软上下文压缩通过将长上下文编码为较少数量的隐式 token，减少了大型语言模型（LLMs）处理长上下文的计算工作量。然而，现有框架采用统一的压缩率，未能考虑到自然语言信息密度的极大差异。虽然采用密度感知的动态压缩率似乎很直观，但实证研究表明，模型在处理由输入相关的连续结构超参数参数化的操作时存在内在困难。为了解决这一缺陷，我们引入了半动态上下文压缩框架。我们的方法包含一个离散率选择器，它根据内在信息密度预测压缩目标，并将其量化为一组预定义的离散压缩率。该选择器在合成数据上与压缩器高效地进行联合训练，并以摘要长度作为代理来创建压缩率预测的标签。广泛的评估证实，我们以均值池化为骨干的密度感知框架始终优于静态基准，为上下文压缩技术建立了稳健的帕累托前沿。我们的代码、数据和模型权重可在 https://github.com/yuyijiong/semi-dynamic-context-compress 获取。

## Abstract
Soft context compression reduces the computational workload of processing long contexts in LLMs by encoding long context into a smaller number of latent tokens. However, existing frameworks apply uniform compression ratios, failing to account for the extreme variance in natural language information density. While adopting a density-aware dynamic compression ratio seems intuitive, empirical investigations reveal that models struggle intrinsically with operations parameterized by input dependent, continuous structural hyperparameters. To resolve this pitfall, we introduce Semi-Dynamic Context Compression framework. Our approach features a Discrete Ratio Selector, which predicts a compression target based on intrinsic information density and quantizes it to a predefined set of discrete compression ratios. It is efficiently jointly trained with the compressor on synthetic data, with the summary lengths as a proxy to create labels for compression ratio prediction. Extensive evaluations confirm that our density-aware framework, utilizing mean pooling as the backbone, consistently outperforms static baselines, establishing a robust Pareto frontier for context compression techniques. Our code, data and model weights are available at https://github.com/yuyijiong/semi-dynamic-context-compress

*   **核心问题**：现有的软上下文压缩方法（将长文本编码为少量潜在标记）通常采用**固定压缩率**，忽略了自然语言信息密度的巨大差异（例如，技术报告比对话记录更稠密）。
*   **研究动机**：
    *   **静态压缩的局限**：固定比例会导致冗余文本压缩不足或稠密文本信息丢失。
    *   **完全动态压缩的陷阱**：作者发现，如果让模型预测并执行完全连续的动态压缩率（如任意长度的输出），会导致模型性能崩溃。这是因为 LLM 难以处理由输入决定的、具有无限变化的“连续结构超参数”。
*   **整体含义**：通过引入“半动态”机制，在感知信息密度的同时，将压缩操作限制在有限的离散集合内，从而兼顾灵活性与训练稳定性。

### 2. 方法论
*   **核心思想**：预测连续的信息密度值，但将其**量化**为预定义的离散压缩率（如 2x, 4x, 8x, 16x, 32x）。
*   **关键技术细节**：
    *   **离散比例选择器 (DRS)**：在编码器末端添加一个回归头，预测文本的对数压缩率 $\hat{y}$。通过一个用户可调的 `scale` 参数（偏置项）来控制全局压缩的激进程度，最后将预测值映射到最近的离散候选比例。
    *   **特征提取骨干**：研究对比了“末端标记”、“压缩标记”和“均值池化（Mean-pooling）”，最终选用**均值池化**作为骨干，因为它在无需大规模预训练的情况下表现最稳健。
    *   **单阶段架构**：在一次编码过程中同时完成密度预测和特征压缩，提高推理效率。
    *   **密度代理标签**：利用教师模型生成的“极简摘要”长度作为信息密度的代理标签，通过监督微调（SFT）而非强化学习（RL）来训练预测头。

### 6. 主要结论与发现
*   **结构超参数陷阱**：证实了 LLM 无法有效学习无限变化的连续结构操作，而离散化的“半动态”操作能保持高精度。
*   **均值池化占优**：在纯 SFT 场景下，简单的均值池化效果显著优于目前流行的“压缩标记（Compression Tokens）”方法。
*   **帕累托前沿提升**：半动态框架在相同的平均压缩率下，准确率始终优于静态基准，尤其在 4x 到 16x 的中等压缩区间提升最明显。
*   **双向注意力必要性**：对于高倍率压缩（≥16x），编码器使用双向注意力比因果注意力更有助于捕捉全局关键特征。

### 7. 优点
*   **创新性地识别了陷阱**：指出了完全动态压缩在工程实现上的本质困难（连续结构超参数问题）。
*   **用户可控性**：通过简单的 `scale` 参数，用户可以在推理时无需重新训练即可平滑调节压缩强度。
*   **高效训练**：采用纯 SFT 和合成数据代理标签，规避了复杂的强化学习过程，易于复现和部署。

### 8. 不足与局限
*   **上下文长度限制**：实验评估的文本长度主要在 2048 token 以内，对于真正“超长”上下文（如 100k+）的泛化能力尚待验证。

*   **模型依赖**：实验主要集中在 Qwen3 系列模型上，其结论在其他架构（如 Llama 或 MoE 模型）上的普适性有待进一步确认。
