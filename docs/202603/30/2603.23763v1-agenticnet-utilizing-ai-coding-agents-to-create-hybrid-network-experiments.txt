Abstract —Traditional network experiments focus on validation through either simulation or emulation. Each approach has its own advantages and limitations. In this work, we present a new tool for next-generation network experiments created through Artificial Intelligence (AI) coding agents. This tool facilitates hy-brid network experimentation through simulation and emulation capabilities. The tool supports three main operation modes: pure simulation, pure emulation, and hybrid mode. AgenticNet pro-vides a more flexible approach to creating experiments for cases that may require a combination of simulation and emulation. In addition, AgenticNet supports rapid development through AI agents. We test Python and C++ versions. The results show that C++ achieves more accurate results and better performance than the Python version. 

Index Terms —Network Simulation, Emulation 

I. I NTRODUCTION 

Existing network experiment tools focus either on simula-tion or emulation. For instance, Mininet provides emulation to run experiments for SDN networks. OMNET++ is useful for simulations. There is a need for tools that combine both simulation and emulation. Those tools are essential for the research community. In this paper, we provide a proof of concept with the following benefits:  

> •

Hybrid Experiments: built-in support and co-existence of simulated and emulated nodes within the same network experiment.  

> •

Fine-grained Control: per-link parameters (delay, band-width, loss), per-node failure, traffic isolation, and custom code.  

> •

Rapid development through Large Language Model (LLM)-based agents: an interactive LLM-based agent to provide the ability to rapidly create network experiments using natural-language descriptions. II. R ELATED -WORK 

Currently, several network simulators and emulators are used to create network experiments. This includes network simulators such as OMNET++ [1] and ns-3 [2], in addition to emulators such as Mininet [3]. However, these works either focus on simulation or emulation. In this work, we consider hybrid network experiments, in which both simulation and emulation can co-exist. In Table I, we compare AgenticNet with other existing tools. III. A GENTIC NET 

Initially, we asked Claude [4] to design a simulator that combines the benefits of simulator and emulator frameworks (See Prompt 1). 

Prompt 1: Creating AgenticNet 

We need to design a simulator that combines the flexibility of ns-3 or OMNET++ (i.e., simulation) and the realistic approach of Mininet (i.e., emulation). 

First, we explore the main design suggested by the coding agent (See Fig. 1). This design consists of four main compo-nents:  

> •

Simulation Domain: pure simulation.  

> •

Emulation Domain: pure emulation.  

> •

Bridge Link: connects the simulation domain to the emulation domain.  

> •

Synchronization: necessary to keep the simulated and emulated domains synchronized. The tool supports the following three main working modes as follows. Pure simulation mode supports only simulations. 

Pure emulation mode supports only emulations. Hybrid mode supports both simulation and emulation. In this case, the simulation engine will be waiting for the emulator to keep the simulated and emulated domains synchronized. A code snippet for hybrid mode case is shown in Listing 1.

> s = Scenario("Exp1", mode="hybrid") src =s.node("src", kind="emulated") sw =s.node("sw", kind="switch", forwarder=OVSSwitch()) dst =s.node("dst", kind="emulated") br1 =s.link(src, sw, bw="1Gbps", delay="5ms") br2 =s.link(sw, dst, bw="1Gbps", delay="5ms") s.start_all()

B. Exp2: Testing the performance of pure simulation mode 

We measure wall-clock time in pure simulation mode across varying virtual durations. We use the same topology in Fig. 2(a). A traffic source sends UDP packets at 100 pps (i.e., one packet every 10ms) through a simulated switch to a destination. The virtual time starts from 1 second up to 1 full day. The results show that a full day of network traffic is completed in 100 seconds (See Fig 4) using the Python version. Furthermore, the C++ version shows that 1 day of vir-tual experiment can be completed in 4.3 seconds. To measure the throughput of AgenticNet, we use the same topology but with different link parameters (100 Gbps, 1 ms delay). The MAC addresses are pre-populated in the switch forwarding table to avoid flooding. The packet rate increases from 100 to 1,000,000 pps. The results for throughput measurements are given in Table II. The C++ version shows better throughput compared to the Python version (8.6-9.5 × higher). E. Conclusion 

In this paper, we explore the usage of AI coding agents for creating hybrid network experiments. The tool supports three modes: pure simulation, pure emulation, and hybrid mode. AgenticNet provides a more flexible approach to creating experiments for cases that may require a combination of sim-ulation and emulation. We explore Python and C++ versions. The results show that C++ offers more accurate results and better performance than the Python version. We also run several experiments to test the performance and applicability of AgenticNet.
