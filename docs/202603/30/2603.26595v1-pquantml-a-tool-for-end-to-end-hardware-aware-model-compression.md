## 摘要
PQuantML 是一个全新的开源、硬件感知神经网络模型压缩库，专为端到端工作流量身定制。出于在具有严格延迟限制的环境中部署高性能模型的需求，PQuantML 通过提供统一的接口来联合或单独应用剪枝和量化，从而简化了压缩模型的训练。该库实现了多种不同粒度的剪枝方法，以及支持高粒度量化（HGQ）的定点量化。我们在代表性任务上评估了 PQuantML，例如喷注子结构分类（即所谓的喷注标记），这是一个与大型强子对撞机（LHC）实时数据处理相关的边缘计算问题。通过结合多种剪枝方法与定点量化，PQuantML 在保持准确性的同时实现了显著的参数量和位宽缩减。此外，我们还将所得的压缩结果与 QKeras 和 HGQ 等现有工具进行了对比。

## Abstract
PQuantML is a new open-source, hardware-aware neural network model compression library tailored to end-to-end workflows. Motivated by the need to deploy performant models to environments with strict latency constraints, PQuantML simplifies training of compressed models by providing a unified interface to apply pruning and quantization, either jointly or individually. The library implements multiple pruning methods with different granularities, as well as fixed-point quantization with support for High-Granularity Quantization. We evaluate PQuantML on representative tasks such as the jet substructure classification, so-called jet tagging, an on-edge problem related to real-time LHC data processing. Using various pruning methods with fixed-point quantization, PQuantML achieves substantial parameter and bit-width reductions while maintaining accuracy. The resulting compression is further compared against existing tools, such as QKeras and HGQ.

#### 1. 核心问题与整体含义（研究动机和背景）
在高能物理（HEP）实验（如大型强子对撞机 LHC）中，数据产生速率极高（40 MHz），必须在微秒级延迟内通过硬件触发系统（L1T）决定数据的存留。传统的深度学习模型因参数量大、计算复杂，难以直接部署在资源受限的 FPGA 硬件上。
**研究动机：** 现有的模型压缩工具（如 QKeras、HGQ）往往功能单一（侧重于量化），且缺乏统一的剪枝与量化协同优化接口。PQuantML 旨在提供一个**开源、硬件感知、端到端**的库，简化在严格延迟约束环境下训练和部署压缩模型的过程。

#### 2. 方法论：核心思想与关键技术
PQuantML 的核心思想是将**剪枝（Pruning）**、**量化（Quantization）**与**硬件资源评估**集成在一个统一的训练框架中。
*   **核心架构：** 支持 PyTorch 和 Keras/TensorFlow 双后端，通过 YAML 配置文件定义压缩策略。
*   **剪枝技术：**
    *   **结构化剪枝：** 如激活剪枝（AP）和基于 MDMM（修正微分乘子法）的约束优化剪枝。
    *   **非结构化剪枝：** 包括 AutoSparse、连续稀疏化（CS）、动态稀疏训练（DST）、参数无关微分剪枝（PDP）和 Wanda。
    *   **N:M 剪枝：** 兼顾稀疏度与硬件执行效率。
*   **量化技术：**
    *   **定点量化：** 支持张量级、通道级和权重级的位宽配置。
    *   **高粒度量化（HGQ）：** 通过梯度下降学习每个权重的最优位宽，并引入 EBOPs（有效位操作）指标来平衡精度与硬件成本。
*   **算法流程：** 采用三阶段训练模式：**预训练（Pre-training）** -> **训练/掩码学习（Training）** -> **微调（Fine-tuning）**。

*   **高效压缩：** PQuantML 在保持与 QKeras 相当的准确率（约 76.3%）时，显著降低了硬件资源。例如，使用 DST 剪枝可将 LUT 使用量从 5504 降至 3895，延迟从 105 ns 缩短至约 47 ns。

#### 7. 优点与亮点
*   **端到端集成：** 首次将多种先进剪枝算法与高粒度量化集成在同一接口下，极大降低了物理学家的使用门槛。
*   **硬件感知：** 引入 EBOPs 指标，使模型优化目标直接指向硬件资源消耗，而非单纯的数学稀疏度。
*   **灵活性：** 支持自动层替换（Layer Replacement），用户无需重写模型代码即可注入压缩算子。

#### 8. 不足与局限
*   **后端功能不均：** 部分高级功能（如 FITCompress）目前仅支持 PyTorch 后端。
*   **架构限制：** TensorFlow 后端的自动层替换功能目前仅支持简单的顺序（Sequential）架构，不支持包含残差连接或注意力机制的复杂分支模型。
*   **编译链支持：** 目前主要针对 `hls4ml` 编译器优化，对其他硬件后端（如编译器工具链）的支持尚待开发。
*   **预训练风险：** 在某些压缩模式下，长时间预训练可能导致权重范围过大，增加量化难度，需引入额外的正则化手段。
