## 摘要
由大语言模型（LLM）驱动的自主智能体通过长程推理和工具交互执行复杂任务，其中在执行效率和推理鲁棒性之间存在着根本性的权衡。处于不同能力-成本水平的模型具有互补优势：低成本模型能够实现快速执行，但在困难的推理环节可能表现不佳；而更强大的模型则以更高的计算成本提供更稳健的推理。我们提出了 AgentCollab，这是一种自驱动的协作推理框架，在智能体执行过程中动态协调具有不同推理能力的模型。该框架不依赖于外部路由模块，而是利用智能体自身的自我反思信号来判断当前的推理轨迹是否取得了实质性进展，并仅在必要时将控制权提升至更强的推理层级。为了进一步稳定长程执行，我们引入了一种难度感知的累积提升策略，该策略根据最近的失败信号分配额外的推理预算。在实验中，我们使用“小-大”两级模型设置来实例化该框架。在多种多步智能体基准测试上的实验表明，AgentCollab 持续改进了大语言模型智能体的准确率-效率帕累托前沿。

## Abstract
Autonomous agents powered by large language models (LLMs) perform complex tasks through long-horizon reasoning and tool interaction, where a fundamental trade-off arises between execution efficiency and reasoning robustness. Models at different capability-cost levels offer complementary advantages: lower-cost models enable fast execution but may struggle on difficult reasoning segments, while stronger models provide more robust reasoning at higher computational cost. We present AgentCollab, a self-driven collaborative inference framework that dynamically coordinates models with different reasoning capacities during agent execution. Instead of relying on external routing modules, the framework uses the agent's own self-reflection signal to determine whether the current reasoning trajectory is making meaningful progress, and escalates control to a stronger reasoning tier only when necessary. To further stabilize long-horizon execution, we introduce a difficulty-aware cumulative escalation strategy that allocates additional reasoning budget based on recent failure signals. In our experiments, we instantiate this framework using a two-level small-large model setting. Experiments on diverse multi-step agent benchmarks show that AgentCollab consistently improves the accuracy-efficiency Pareto frontier of LLM agents.

### 2. 方法论
AgentCollab 提出了一种自驱动的协作推理范式，核心包含以下三个关键环节：
*   **模型升级机制（Model Escalation）**：
    *   **预热阶段**：任务开始时由大模型（$M_L$）执行，负责全局规划和目标分解。
    *   **常规阶段**：随后转交给小模型（$M_S$）进行低延迟的常规推理和工具调用。
    *   **自评估触发**：在每一步推理中，模型会生成一个结构化的“进度检查”（Progress Check），包含理由和布尔值（TRUE/FALSE）。若检测到停滞（FALSE），则临时将控制权移交给大模型。
*   **难度感知的累积预算分配策略**：
    *   引入**升级等级 $l_t$**：记录连续失败的次数。
    *   **动态预算**：大模型的干预步数 $B_l$ 随失败次数增加而增长（线性或 Sigmoid 增长），公式为 $f(B_0, l) = \min\{B_{max}, B_0 + kl\}$。这能确保在持续困难的环节给予更深度的修正，同时减少频繁切换带来的计算开销（如 Prefill 状态丢失）。
*   **算法流程**：系统在小模型运行期间持续自评，一旦失败则按动态预算调用大模型，解决问题后或预算耗尽后再回退至小模型。

### 6. 主要结论与发现
*   **帕累托前沿优化**：AgentCollab 在所有基准测试中均显著提升了准确率-效率边界，性能接近纯大模型，而速度接近纯小模型。
*   **轨迹效率**：虽然小模型单步快，但出错后会导致推理轨迹变长（绕弯路）；AgentCollab 通过关键时刻的大模型干预，缩短了总推理步数，从而实现了更好的端到端加速。
*   **动态策略优势**：动态预算分配能有效减少模型间的频繁切换，提高缓存复用率，稳定长程执行。
*   **能力差距影响**：协作的效果取决于模型间的“能力差”，当大模型能显著修正小模型错误时，增益最明显。

### 7. 优点与亮点
*   **无需额外训练**：完全依赖模型自身的反思能力，部署成本低，通用性强。
*   **轨迹级优化**：跳出了单步路由的局限，从整个推理轨迹的视角来分配计算资源。
*   **自适应性**：通过累积失败信号动态调整干预强度，能够灵活应对不同难度的任务片段。

### 8. 不足与局限
*   **指令遵循依赖**：该方法高度依赖模型的指令遵循能力。如果模型在“进度检查”中出现假阳性（误报有进展），会导致升级机制失效（实验中 WebSailor 框架效果较差即归因于此）。
*   **同构模型限制**：实验主要在同系列不同规模的模型间进行，尚未探索跨架构、互补性异构模型之间的协作。
*   **闭源模型缺失**：由于 API 延迟不稳定，实验未包含 GPT-4 或 Gemini 等闭源模型，限制了在云端服务场景下的验证。
