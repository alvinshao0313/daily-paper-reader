## 摘要
KV 缓存压缩对于高效的长上下文大语言模型（LLM）推理至关重要。减少每对占用空间的方法（如量化和低秩分解）与减少缓存序列长度的方法是正交的。在序列长度维度上，现有方法涵盖了从纯逐出（选择保留哪些 KV 对）到合并（将相似对组合成更少的对）的范围。两者都仍然锚定在原始缓存条目上。我们提出了 KVSculpt，它转向了这一频谱的另一端：我们不再选择或组合原始对，而是在连续嵌入空间中优化一组更小的、不受约束的 KV 对，以保留每一层的注意力行为。Key 通过 L-BFGS 进行优化，Value 通过最小二乘法以闭式解求解，每隔几步交替进行。在此基础上，我们引入了自适应预算分配，利用低成本的试点压缩运行，根据每个组件的难度在各层和 KV 头之间重新分配压缩预算。在具有 2048 个 token 上下文的 Qwen2.5-1.5B-Instruct 上，在压缩率 r 为 {0.3, 0.5, 0.7} 的情况下，与 Select+Fit（基于注意力分数的逐出结合最小二乘 Value 拟合）相比，KVSculpt 将 KL 散度降低了 3.5-4.1 倍。自适应分配在不增加额外推理成本的情况下，使 KL 散度进一步降低了 1.3 倍。分析表明，压缩难度高度不均匀：各层试点均方误差（MSE）的差异可达 100 倍，而单层内的两个 KV 头之间的差异可达 467 倍，这证明了细粒度的预算分配至关重要。

## Abstract
KV cache compression is critical for efficient long-context LLM inference. Approaches that reduce the per-pair footprint -- quantization and low-rank decomposition -- are orthogonal to those that reduce the sequence length of the cache. Along the sequence-length dimension, existing methods range from pure eviction -- selecting which KV pairs to keep -- to merging, which combines similar pairs into fewer ones. Both remain anchored to the original cache entries. We propose KVSculpt, which moves to the other end of this spectrum: instead of selecting or combining original pairs, we optimize a smaller set of unconstrained KV pairs in continuous embedding space to preserve each layer's attention behavior. Keys are optimized via L-BFGS and values are solved in closed form via least squares, alternating every few steps. On top of this, we introduce adaptive budget allocation, which uses a cheap pilot compression run to redistribute the compression budget across layers and KV heads based on per-component difficulty. On Qwen2.5-1.5B-Instruct with 2048-token contexts, KVSculpt reduces KL divergence by 3.5-4.1x compared to Select+Fit -- attention-score eviction with least-squares value fitting -- across compression ratios r in {0.3, 0.5, 0.7}. Adaptive allocation provides an additional 1.3x KL reduction at no extra inference cost. Analysis reveals that compression difficulty is highly non-uniform: per-layer pilot MSE varies by up to 100x across layers, and the two KV heads within a single layer can differ by up to 467x -- demonstrating that fine-grained budget allocation is essential.

### 1. 核心问题与研究动机
*   **核心问题**：在长文本推理中，KV 缓存随序列长度线性增长，成为显存瓶颈。
*   **研究背景**：现有的序列长度压缩方法主要分为“逐出”（Eviction，选择保留哪些原始对）和“合并”（Merging，将相似对加权组合）。
*   **动机**：作者认为现有方法受限于“原始缓存条目”的离散选择，过于死板。由于旋转位置编码（RoPE）已将位置信息嵌入向量，压缩后的 KV 对不一定非要是原始 token 的子集。通过在连续嵌入空间中优化 KV 对，可以更精确地模拟原始注意力的行为。

### 2. 方法论
*   **核心思想**：将 KV 缓存压缩建模为一个蒸馏问题。给定 $N$ 个原始 KV 对，优化 $k$ 个不受限的虚拟 KV 对（$k < N$），使得压缩后的缓存产生的注意力输出与原始缓存尽可能一致。
*   **关键技术细节**：
    *   **损失函数**：包含两部分：输出均方误差（Output MSE）和对数总和指数（LSE）匹配。LSE 匹配确保了注意力权重的分布比例在合并时保持正确。
    *   **交替优化**：
        *   **Key 优化**：使用 **L-BFGS** 算法（一种拟牛顿法）在连续空间中寻找最优的 Key 向量。
        *   **Value 求解**：在 Key 固定时，利用**岭回归（最小二乘法）**以闭式解的形式直接计算最优的 Value 向量。
    *   **训练查询构建**：使用“保留区”的真实查询，并通过“去 RoPE”和“重 RoPE”技术生成合成的未来查询，以增强泛化性。
    *   **自适应预算分配**：引入“试点运行”（Pilot Run）机制。先进行短时间的统一压缩，测量各层和各 KV 头的压缩难度（MSE），然后根据难度动态分配 $k$ 的数量（难度大的层/头分配更多预算）。

### 3. 实验设计
*   **模型与数据**：使用 **Qwen2.5-1.5B-Instruct** 模型；测试集采用 **PG19**（长文档数据集）；上下文长度设为 2048 token，评估后续 128 token 的生成质量。
*   **评估指标**：主要使用 **KL 散度**（衡量压缩模型与原始模型输出分布的差异）。
*   **对比基准（Benchmarks）**：
    *   **Random**：随机保留。
    *   **Attention Score**：基于注意力分数累加的逐出（类似 H2O）。
    *   **Select+Fit**：先筛选 token，再用最小二乘法拟合 Value（目前最强的逐出变体）。
    *   **Joint Opt**：使用离散门控函数进行联合优化。

*   **说明**：该方法属于离线/摊销压缩，适用于那些会被多次查询的长文档场景。

### 5. 实验数量与充分性
*   **实验规模**：主实验涵盖了 20 条长序列，详细分析和消融实验针对 5 条代表性序列展开。
*   **压缩率测试**：测试了 $r \in \{0.1, 0.3, 0.5, 0.7\}$ 多种压缩比例。
*   **消融实验**：对比了 L-BFGS 与 Adam 优化器的效果、不同的查询采样策略、以及自适应分配中不同平滑系数的影响。
*   **充分性评价**：实验设计较为严谨，通过对比多种基准方法证明了“连续空间优化”的优越性。但样本量（20 条序列）相对较小，且仅针对单一模型系列进行了深入测试。

### 6. 主要结论与发现
*   **性能飞跃**：在相同压缩率下，KVSculpt 的 KL 散度比最强的逐出基准（Select+Fit）低 **3.5-4.1 倍**。
*   **自适应分配的价值**：通过在层间和头间动态分配预算，KL 散度可进一步降低 **1.3 倍**，且不增加推理成本。
*   **难度不均性**：发现不同层之间的压缩难度差异可达 100 倍，同一层内不同 KV 头的难度差异甚至高达 467 倍。
*   **优化器关键性**：L-BFGS 在处理注意力机制的非凸景观时远优于 Adam，能达到接近理论最优的单层压缩效果。

### 7. 优点
*   **突破限制**：打破了“必须保留原始 token”的思维定式，利用连续空间的自由度实现了极高的压缩保真度。
*   **数学严谨**：结合了 L-BFGS 的二阶优化特性和最小二乘法的解析解，优化效率高。
*   **细粒度控制**：自适应预算分配机制充分利用了模型内部结构的非均匀性。

### 8. 不足与局限
*   **计算开销**：压缩过程（170秒）远慢于简单的逐出算法，不适合单次使用的实时对话，更适合离线处理长文档或系统提示词。
*   **模型覆盖**：实验主要集中在 Qwen2.5-1.5B，尚未在更大规模（如 70B+）或其他架构（如 Llama 3）上广泛验证。
*   **误差累积**：虽然单层压缩接近完美，但层与层之间的误差会累积，论文指出这是未来的改进方向（级联感知优化）。
*   **任务评估**：主要依赖 KL 散度，缺乏在具体下游任务（如摘要、问答、MMLU）上的准确率评估。
