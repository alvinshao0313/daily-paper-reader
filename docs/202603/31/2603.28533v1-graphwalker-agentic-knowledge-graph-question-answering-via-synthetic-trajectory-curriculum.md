## 摘要
智能体知识图谱问答 (KGQA) 要求智能体与知识图谱 (KGs) 进行迭代交互，这在训练数据稀缺性和推理泛化性方面都带来了挑战。具体而言，现有方法通常限制了智能体的探索：基于提示 (prompting) 的方法缺乏自主导航训练，而当前的训练流程通常将推理局限于预定义的轨迹。为此，本文提出了 GraphWalker，这是一个新颖的智能体 KGQA 框架，通过自动化轨迹合成 (Automated Trajectory Synthesis) 和分阶段微调 (Stage-wise Fine-tuning) 来解决这些挑战。GraphWalker 采用两阶段有监督微调 (SFT) 训练范式：首先，在从受限随机游走路径合成的结构多样化轨迹上训练智能体，从而在知识图谱上建立广泛的探索先验；其次，在少量专家轨迹上进一步微调智能体，以培养其反思和错误恢复能力。大量实验表明，我们的分阶段 SFT 范式为轻量级强化学习 (RL) 阶段解锁了更高的性能上限，使 GraphWalker 在 CWQ 和 WebQSP 上达到了最先进的性能。在 GrailQA 和我们构建的 GraphWalkerBench 上的额外结果证实，GraphWalker 增强了对分布外 (out-of-distribution) 推理路径的泛化能力。代码已在 https://github.com/XuShuwenn/GraphWalker 公开。

## Abstract
Agentic knowledge graph question answering (KGQA) requires an agent to iteratively interact with knowledge graphs (KGs), posing challenges in both training data scarcity and reasoning generalization. Specifically, existing approaches often restrict agent exploration: prompting-based methods lack autonomous navigation training, while current training pipelines usually confine reasoning to predefined trajectories. To this end, this paper proposes \textit{GraphWalker}, a novel agentic KGQA framework that addresses these challenges through \textit{Automated Trajectory Synthesis} and \textit{Stage-wise Fine-tuning}. GraphWalker adopts a two-stage SFT training paradigm: First, the agent is trained on structurally diverse trajectories synthesized from constrained random-walk paths, establishing a broad exploration prior over the KG; Second, the agent is further fine-tuned on a small set of expert trajectories to develop reflection and error recovery capabilities. Extensive experiments demonstrate that our stage-wise SFT paradigm unlocks a higher performance ceiling for a lightweight reinforcement learning (RL) stage, enabling GraphWalker to achieve state-of-the-art performance on CWQ and WebQSP. Additional results on GrailQA and our constructed GraphWalkerBench confirm that GraphWalker enhances generalization to out-of-distribution reasoning paths. The code is publicly available at https://github.com/XuShuwenn/GraphWalker

### 论文总结：GraphWalker —— 基于合成轨迹课程的智能体知识图谱问答

#### 1. 核心问题与整体含义（研究动机和背景）
传统的知识图谱问答（KGQA）主要依赖信息检索（IR）或语义解析（SP），这些方法通常在预定义的静态子图内运行，难以应对大规模、多噪声及长尾实体的真实知识图谱环境。
**智能体 KGQA**（Agentic KGQA）通过让大模型（LLM）与知识图谱进行多轮交互来解决上述问题，但面临两大瓶颈：
*   **训练数据稀缺**：缺乏高质量、涵盖复杂推理路径的交互轨迹数据。
*   **推理泛化性差**：现有模型往往局限于模仿预定义的简单工作流，缺乏在复杂图结构中自主导航、反思和纠错的能力。
**GraphWalker** 的核心目标是通过构建一个“合成轨迹课程”，让轻量级模型通过分阶段学习，掌握从广度探索到深度反思的完整推理技能。

#### 2. 方法论：核心思想与关键技术
GraphWalker 提出了一个包含**数据合成**与**三阶段训练**的框架：

*   **核心思想**：先通过大规模合成数据建立“探索先验”，再通过专家数据培养“反思能力”，最后通过强化学习（RL）突破性能上限。
*   **关键技术细节**：
    1.  **GraphSynth（15k 轨迹）**：利用**约束随机游走（CRW）**在全局 KG 上生成多样化的推理路径（链式、汇聚式等），并利用 LLM 合成对应的自然语言问题，经过语义过滤后生成推理思维链（CoT）。
    2.  **GraphRoll（6k 轨迹）**：通过专家模型（如 GPT-4o-mini）在真实数据集上生成包含“死胡同回溯”和“错误恢复”的高质量轨迹，并利用基于结果的拒绝采样确保正确性。
    3.  **交互工具箱**：智能体配备 `get_relations`（获取邻接关系）和 `get_triples`（获取特定三元组）两个原子工具，直接操作全局 KG。
*   **训练流程**：
    *   **阶段 1 (SFT)**：在 GraphSynth 上训练，学习基础的图导航和工具调用。
    *   **阶段 2 (SFT)**：在 GraphRoll 上微调，学习反思和错误恢复。
    *   **阶段 3 (RL)**：使用 **GRPO（群体相对策略优化）** 算法，以精确匹配（EM）作为稀疏奖励，进一步优化推理策略。

#### 3. 实验设计
*   **数据集**：
    *   **主流基准**：CWQ (Complex WebQuestions) 和 WebQSP。
    *   **零样本泛化测试**：GrailQA（零样本划分）。
    *   **自定义基准**：**GraphWalkerBench**，包含 GraphSynth 中未出现的复杂图结构（如 IP、PI 结构），用于测试 OOD（分布外）泛化性。
*   **对比方法**：
    *   **提示词方法**：ToG, GoG, ToG-2.0。
    *   **训练方法**：RoG, KG-Agent, KBQA-o1, 以及最新的强化学习方法 **KG-R1**。
    *   **闭源模型**：GPT-4o-mini, DeepSeek-V3.2（作为 Agent 框架下的基座对比）。

#### 4. 资源与算力
*   **硬件环境**：使用了 **8 张 NVIDIA A100 (80GB) GPU**。
*   **训练细节**：
    *   SFT 阶段：学习率 1e-5，Batch Size 32，训练 2 个 Epoch。
    *   RL 阶段：使用 GRPO 算法，学习率 1e-6，训练 100 个 Step，全局 Batch Size 为 512。
    *   推理设置：最大交互轮数 $T=10$。

#### 5. 实验数量与充分性
*   **实验规模**：论文在 3 个开源基座模型（Qwen2.5-3B/7B, Llama-3.1-8B）上进行了验证，涵盖了从 3B 到 8B 的参数量。
*   **消融实验**：非常充分。分别验证了阶段 1、阶段 2、RL 阶段、训练顺序（分阶段 vs 混合训练）、数据规模以及关系重排序模块对最终性能的影响。
*   **客观性**：通过 Pass@k 分析和 OOD 结构测试（GraphWalkerBench），客观地展示了模型在搜索空间扩张和结构泛化上的优势，而非仅仅依赖记忆训练集。

#### 6. 主要结论与发现
*   **性能领先**：GraphWalker 在 CWQ 和 WebQSP 上均取得了 SOTA 性能，显著超过了包括 GPT-4 驱动的智能体方法。
*   **分阶段训练的必要性**：先学探索（Stage 1）再学反思（Stage 2）的效果远好于将数据混合在一起训练。
*   **RL 的增益**：两阶段 SFT 建立的强大先验为 RL 提供了更高的起点，使其能够通过自主探索突破 SFT 的性能天花板。
*   **强大的迁移能力**：仅在 CWQ 上训练的模型，在 WebQSP 和 GrailQA 上表现出极强的零样本迁移能力。

#### 7. 优点
*   **数据合成方案科学**：通过约束随机游走解决了 KGQA 训练数据不足的问题，且通过语义过滤保证了合成质量。
*   **课程学习设计巧妙**：将复杂的推理任务拆解为“导航”和“反思”两个阶段，符合人类认知规律。
*   **全局 KG 交互**：不依赖预先提取的子图，更贴近真实应用场景，展示了极强的鲁棒性。
*   **泛化性验证深入**：通过构建 GraphWalkerBench 专门测试未见过的图结构，证明了模型掌握的是推理逻辑而非路径记忆。

#### 8. 不足与局限
*   **对基座模型依赖**：虽然在轻量级模型上表现优异，但合成数据和专家轨迹仍高度依赖更强大的模型（如 GPT-4o-mini）进行标注和生成。
*   **计算开销**：多轮交互和 RL 训练对算力有一定要求，且在推理时多次调用 KG 接口可能存在延迟。
*   **Schema 依赖**：实验主要基于 Freebase，对于 Schema 极度稀疏或完全不同的知识图谱，其泛化效果尚待进一步验证。
