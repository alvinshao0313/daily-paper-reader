## 摘要
键值（KV）缓存对于大语言模型（LLMs）的高效推理至关重要，但过长的上下文会显著增加 KV 缓存的内存占用。现有的 KV 缓存压缩方法通常依赖于预填充阶段 Prompt 观察窗口内的输入侧注意力模式来估计 Token 的重要性。由于这些评估并非源自解码过程，它们无法为未来的生成保留关键 Token。直观上，一个有效的观察窗口应该镜像解码阶段的查询，以准确反映生成过程将关注哪些 Token。然而，在推理过程中，真实的解码查询本质上是不可用的。为了构建伪查询来近似它们，我们发现位置信息比语义内容起着更关键的作用。基于这一见解，我们提出了通过位置感知伪查询实现的解码对齐 KV 缓存压缩（DapQ），这是一种新颖且轻量级的逐出框架，它利用位置感知伪查询来模拟输出 Token，从而建立一个有效的观察窗口进行重要性评估。它与实际的生成上下文紧密对齐，并实现了精确的 Token 逐出。在多个基准测试和大语言模型上的广泛评估表明，DapQ 取得了卓越的性能，特别是在严格的内存限制下（例如，在仅 3% 的 KV 缓存预算下，在 NIAH 上实现了高达 99.5% 的近乎无损性能）。

## Abstract
The Key-Value (KV) cache is crucial for efficient Large Language Models (LLMs) inference, but excessively long contexts drastically increase KV cache memory footprint. Existing KV cache compression methods typically rely on input-side attention patterns within a prompt observation window to estimate token importance during the prefill stage. They fail to preserve critical tokens for future generation since these assessments are not derived from the decoding process. Intuitively, an effective observation window should mirror the decoding-stage queries to accurately reflect which tokens the generation process will attend to. However, ground-truth decoding queries are inherently unavailable during inference. For constructing pseudo queries to approximate them, we find that positional information plays a more critical role than semantic content. Motivated by this insight, we propose decoding-aligned KV cache compression via position-aware pseudo queries (DapQ), a novel and lightweight eviction framework that leverages position-aware pseudo queries to simulate the output tokens, thereby establishing an effective observation window for importance assessment. It aligns closely with the actual generation context and enables precise token eviction. Extensive evaluations across multiple benchmarks and LLMs demonstrate that DapQ achieves superior performance, particularly under strict memory constraints (e.g., up to nearly lossless performance 99.5% on NIAH with 3% KV cache budgets).

*   **研究动机**：现有的 KV Cache 压缩方法（如 SnapKV、PyramidKV）主要依赖预填充（Prefill）阶段最后几个 Token 的注意力模式来评估 Token 重要性。这种“输入中心”的方法与实际“解码阶段”的动态查询存在失配，导致无法准确保留对未来生成至关重要的 Token。
*   **核心发现**：作者发现，在构建用于模拟未来解码的“伪查询（Pseudo Queries）”时，**位置信息（Positional Information）的影响远大于语义内容（Semantic Content）**。只要位置 ID 对齐，即使伪查询的内容是随机的，其产生的注意力模式也与真实解码查询高度相似。

### 2. 论文提出的方法论：DapQ
**DapQ (Decoding-aligned KV Cache Compression via Position-aware Pseudo Queries)** 的核心思想是在预填充阶段通过模拟未来解码位置的查询来精准识别重要的 KV 对。
*   **核心思想**：利用未来位置编码构造“位置感知伪查询”，建立一个与解码过程对齐的观察窗口。
*   **技术流程**：
    1.  **构造伪序列**：在原始 Prompt（长度为 $L_p$）后附加 $N$ 个人工构造的 Token。实验发现使用 Prompt 的前缀和后缀拼接效果最好。
    2.  **位置分配**：关键步骤是给这 $N$ 个 Token 分配**未来的位置 ID**（即 $L_p$ 到 $L_p + N - 1$），而非紧随其后的逻辑位置。
    3.  **获取伪查询 ($Q_{pseudo}$)**：模型处理该扩展序列，提取这 $N$ 个 Token 对应的 Query 向量。
    4.  **重要性评估**：计算 $Q_{pseudo}$ 与 Prompt 中所有 Key 的注意力分数并进行累加，得到每个 Token 的重要性得分 $S(j)$。
    5.  **逐出与解码**：保留得分最高的 Top-K 个 KV 对，立即丢弃所有伪 Token 及其缓存，随后从位置 $L_p$ 开始正式解码。

### 6. 主要结论与发现
*   **位置主导性**：位置编码（如 RoPE）在决定查询向量表征中起主导作用，语义内容是次要的。
*   **性能卓越**：DapQ 在所有基准测试中均优于现有基线。特别是在 LLaMA-3-8B 上，仅保留 256 个 KV 预算（约 3%）时，NIAH 任务准确率高达 **99.5%**。
*   **压缩极限**：在极低预算（如 64 个 Token）下，DapQ 相比 SnapKV 等方法表现出更强的鲁棒性，能有效保留复杂推理任务（如 HotpotQA）中的关键信息。

### 7. 优点（亮点）
*   **解码对齐**：首次系统性地揭示并利用了位置信息在模拟解码查询中的核心作用，解决了预填充与解码阶段的失配问题。
*   **无需训练**：作为一种 Plug-and-play（即插即用）的方法，不需要对模型进行任何微调。
*   **高效率**：通过一次性的伪查询计算替代了昂贵的预生成（Pre-generation）过程，避免了内存峰值问题。
*   **理论支撑**：提供了关于查询相似度约束注意力误差的理论推导。

### 8. 不足与局限
*   **语义仍有微弱影响**：虽然位置是主导，但实验显示语义内容完全无关时（如随机 Token），性能仍有轻微下降，如何低成本地优化伪查询的语义仍有提升空间。
*   **层间差异处理**：目前 DapQ 在所有层使用统一的伪查询策略，未考虑 LLM 不同层对位置/语义敏感度的差异。
*   **超参数敏感性**：伪查询窗口长度 $N$ 需要根据任务平衡（太小评估不准，太大引入噪声），虽然论文给出了建议值，但在极端任务下可能需要微调。
