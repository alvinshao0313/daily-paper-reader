## 摘要
视觉语言导航（VLN）日益依赖大型视觉语言模型，但其推理成本与实时部署的需求相冲突。Token 缓存是一种极具前景的免训练策略，通过在帧间重用稳定的视觉 Token 来避免冗余计算。然而，现有方法假设摄像机是静态的且语义焦点固定，而 VLN 从根本上违背了这些假设。我们识别出两种失效模式：（1）视觉动态，即视点偏移导致 Token 位置在帧间发生位移，使得基于位置的匹配将不一致的内容配对；（2）语义动态，即随着导航的进行，Token 的相关性在不同任务阶段发生转变，导致缓存状态失效。我们提出了 VLN-Cache，这是一个具备视觉动态感知和语义动态感知能力的缓存框架，它引入了视角对齐重映射以恢复几何对应关系，并引入了任务相关性显著性过滤器以在语义转换时否决重用。一种层自适应熵策略进一步平衡了每层的重用预算。在 R2R-CE 仿真基准上的实验表明，该方法在保持具有竞争力的导航成功率的同时，实现了高达 1.52 倍的加速。

## Abstract
Vision-and-Language Navigation (VLN) increasingly relies on large vision-language models, but their inference cost conflicts with real-time deployment. Token caching is a promising training-free strategy that avoids redundant computation by reusing stable visual tokens across frames. However, existing methods assume a static camera and fixed semantic focus, assumptions that VLN fundamentally violates. We identify two failure modes: (1) visual dynamics, where viewpoint shift displaces token positions across frames, causing position-wise matching to pair misaligned content; (2) semantic dynamics, where token relevance shifts across task stages as navigation progresses, making cached states stale. We propose VLN-Cache, a visual-dynamic-aware and semantic-dynamic-aware caching framework that introduces view-aligned remapping to recover geometric correspondences and a task-relevance saliency filter to veto reuse at semantic transitions. A layer-adaptive entropy policy further balances the per-layer reuse budget. Experiments on the R2R-CE simulation benchmark show up to 1.52x speedup while maintaining competitive navigation success rates.

### 1. 核心问题与研究动机
*   **核心问题**：大型视觉语言模型（VLM）在视觉语言导航（VLN）任务中表现优异，但其巨大的推理开销导致高延迟，难以满足机器人实时部署的需求。
*   **研究背景**：传统的 Token 缓存技术（Token Caching）通过重用相邻帧中相似的视觉 Token 来加速推理，但它们大多基于“静态相机”假设。
*   **失效原因**：在 VLN 任务中，该假设因两种动态性而失效：
    1.  **视觉动态（Visual Dynamics）**：智能体在移动和旋转，导致同一物理对象在图像坐标系中的位置发生偏移，简单的位置匹配会导致内容错位。
    2.  **语义动态（Semantic Dynamics）**：随着导航任务的推进，即使某些区域视觉上没变，其对当前指令的语义重要性也会发生变化（例如：转弯前的地标在转弯后变得无关紧要）。

### 2. 方法论
VLN-Cache 是一个免训练、即插即用的推理加速框架，核心思想是**“感知动态，按需重用”**：
*   **视角对齐重映射（View-aligned Remapping）**：利用深度信息和相机位姿变换，将当前帧的 Token 中心点投影回上一帧的坐标系，找到物理上对应的 Token 索引，从而解决视角偏移导致的匹配失效。
*   **双重感知门控机制**：
    *   **视觉门控**：通过计算重映射后的 Token 相似度，确保物理内容确实一致且未被遮挡。
    *   **语义门控（任务相关性显著性过滤器）**：计算 Token 与指令的相关性得分。如果某个 Token 的相关性很高或发生剧烈变化（意味着进入了新的任务阶段），则强制重新计算，防止语义过期。
*   **层自适应熵策略（Layer-adaptive Policy）**：根据 Transformer 各层注意力分布的熵值动态调整重用比例。不确定性高（熵大）的层减少重用，稳定的层增加重用。
*   **算法流程**：在每一步导航推理前，生成重用掩码（Reuse Mask），对需要重用的 Token 直接从缓存中提取 KV 状态，仅对发生变化的 Token 进行前向计算。

*   **评估指标**：成功率（SR）、路径长度加权成功率（SPL）、每步延迟（Latency）、加速比（Speedup）。

### 6. 主要结论与发现
*   **加速效果显著**：VLN-Cache 实现了高达 **1.52 倍** 的步级和情节级加速。
*   **性能保持良好**：在大幅提速的同时，导航成功率（SR）仅下降了约 1.2%（从 64.3% 降至 63.1%），SPL 下降 0.9%。

### 7. 优点
*   **免训练与通用性**：不需要对模型进行微调或修改架构，可直接应用于任何基于 Transformer 的视觉语言动作（VLA）模型。
*   **解决核心痛点**：首次系统性地分析并解决了 VLN 中由于智能体运动导致的 Token 缓存失效问题。
*   **系统实现高效**：通过 GPU HBM 存储缓存，避免了 CPU-GPU 之间的数据传输开销。

*   **依赖深度信息**：视角对齐重映射需要准确的深度图和相机位姿，在缺乏深度传感器的设备上难以应用。
