## 摘要
思维链（CoT）提高了推理的可靠性，但也增加了 token 成本，这促使人们对显式推理轨迹进行训练后压缩。然而，最短的充分推理并非普适的：它取决于难度、模型能力和训练状态，这使得固定长度的目标变得脆弱。在实践中，朴素的基于强化学习（RL）的压缩也可能导致面向用户的答案被意外缩短，因为单一的生成级（completion-level）学习信号会跨越“思考/答案”边界发生泄漏。我们提出了难度缩放的分段式 GRPO（DSS-GRPO），它将回报分解为思考和答案两个部分，计算每个分段的组内相对优势，并通过硬 token 掩码进行路由，使得压缩更新仅作用于思考部分，而答案对齐仅作用于答案部分。DSS-GRPO 使用基于提示的组内塑造和难度感知缩放，在不破坏答案行为的情况下鼓励简洁的推理。

## Abstract
Chain-of-thought (CoT) improves reasoning reliability but increases token cost, motivating post-training compression of explicit reasoning traces. However, the shortest sufficient reasoning is not universal: it depends on difficulty, model capacity, and training state, making fixed length targets brittle. In practice, naive RL-based compression can also undesirably shorten the user-facing answer, because a single completion-level learning signal leaks across the think/answer boundary. We propose Difficulty-Scaled Segment-Wise GRPO (DSS-GRPO), which decomposes returns into think and answer components, computes group-relative advantages per segment, and routes them with hard token masks so compression updates act only on think while answer alignment acts only on answer. DSS-GRPO uses prompt-wise within-group shaping and difficulty-aware scaling to encourage concise reasoning without collapsing answer behavior.

### 1. 核心问题与研究动机
*   **研究背景**：思维链（CoT）虽能显著提升大模型的推理能力，但其生成的冗长中间推理轨迹（Thinking Trace）带来了高延迟和高 Token 成本。
*   **核心问题**：
    1.  **信号泄露（Credit Assignment）**：传统的强化学习（RL）压缩方法通常将单一的标量奖励应用于整个生成序列。这导致旨在压缩“思考”部分的奖励信号误伤了“最终答案”部分，使得答案变得过于简短或信息不足。
    2.  **长度目标的僵化性**：最短的充分推理长度并非固定值，而是取决于题目难度、模型能力和训练状态。统一的长度压力在简单题目上有效，但在难题上会导致推理不足。
*   **研究目标**：实现“更短的思考，同样的答案”——在压缩推理过程的同时，保持推理准确性并维持原有的回答风格（尤其是答案长度）。

### 2. 方法论：DSS-GRPO
论文提出了**难度缩放的分段式 GRPO（DSS-GRPO）**，核心思想是通过结构化解耦来精细化控制学习信号。
*   **分段掩码与路由（Segment-Wise Routing）**：
    *   利用模型生成的结构化标记（如 `<think>` 和 `</think>`）将输出划分为“思考段”和“答案段”。
    *   引入硬标记掩码（Hard Token Masks），确保压缩奖励（Efficiency Reward）仅作用于思考段 Token，而答案对齐奖励（Length Alignment Reward）仅作用于答案段 Token。
*   **难度感知缩放（Difficulty-Aware Scaling）**：
    *   定义难度权重 $W_{diff}(x) = 2 - \hat{p}_{succ}(x)$，其中 $\hat{p}_{succ}$ 是该提示词在组内的成功率。
    *   **非对称缩放**：在难题上（成功率低），放大正向优势信号（Positive Advantages），引导模型向少数正确的轨迹学习；在简单题上则保持正常压力。
*   **质量门控（Quality Gate）**：
    *   只有当模型输出格式正确且答案准确时，才激活长度相关的奖励，防止模型通过跳过推理步骤或破坏格式来“刷分”（Reward Hacking）。
*   **奖励设计**：
    *   **思考压缩奖励**：采用组内最小-最大缩放（Min-Max Scaling），使压缩目标具有提示词相关性，而非全局固定目标。
    *   **答案长度对齐**：设置一个容差带（Tolerance Band），惩罚过短的答案，允许适度冗余，以锚定原始模型的回答行为。
