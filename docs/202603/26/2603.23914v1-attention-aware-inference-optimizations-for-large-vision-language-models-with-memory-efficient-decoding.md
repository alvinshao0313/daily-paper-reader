## 摘要
大型视觉语言模型（VLM）在多模态推理方面取得了显著成功，但由于解码过程中的内存开销，其推理时间效率仍面临重大挑战，特别是当 VLM 的查询和回答包含长序列的视觉和文本标记（tokens）时。本文提出了 AttentionPack，这是一个专为大型视觉语言模型量身定制的自适应且注意力感知的优化框架，旨在提高解码过程中的内存效率，重点解决由于视觉输入和交互数量增加而带来的挑战，特别是在涉及多张高分辨率图像或视频的长上下文任务中。AttentionPack 的创新之处体现在两个方面：(i) 我们引入了一种多头注意力压缩方法，通过利用隐式低秩结构来经济地存储键（key）和值（value）矩阵；(ii) 我们开发了一种特定于标记（token-specific）的注意力感知解压缩机制，以减少延迟开销。在多个基准测试上的实验结果表明，AttentionPack 将内存效率提高了多达 8 倍，在保持模型输出质量或支持更长上下文长度以实现卓越检索性能的同时，实现了更高的批处理大小和更快的批处理推理。我们还报告了 AttentionPack 与逐出（eviction）、量化和算子融合（kernel fusion）相结合的有效性，展示了在资源受限环境下进一步的效率提升。

## Abstract
Large Vision-Language Models (VLMs) have achieved remarkable success in multi-modal reasoning, but their inference time efficiency remains a significant challenge due to the memory overhead during decoding, especially when the query and answer of VLMs consist of long sequences of visual and text tokens. This paper presents AttentionPack, an adaptive and attention-aware optimization framework tailored for large vision-language models with improving memory-efficiency during decoding, focusing on addressing the challenges due to the increased high number of visual inputs and interactions, particularly in long-context tasks with multiple high-resolution images or videos. AttentionPack is novel in two aspects: (i) We introduce a multi-head attention compaction method for economically storing key and value matrices by exploiting the implicit low-rank structure, and (ii) we develop a token-specific attention-aware decompression mechanism to reduce latency overhead. Experimental results on multiple benchmarks demonstrate that AttentionPack improves memory efficiency by up to 8x, enabling higher batch sizes and faster batch inference while preserving the model output quality or longer context lengths for superior retrieval performance. We also report the effectiveness of AttentionPack combined with eviction, quantization and kernel fusion, showing further efficiency gains for resource-limited environments.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：大型视觉语言模型（VLM）在处理包含多张高分辨率图像或长视频的任务时，推理过程中的 **KV Cache（键值缓存）** 内存开销巨大。这不仅限制了可处理的上下文长度，还由于频繁的内存读写（I/O 瓶颈）导致推理延迟高、批处理能力（Batch Size）受限。
*   **研究背景**：现有的优化方法如 Token 逐出（Eviction）或量化（Quantization）虽然能减少内存，但前者可能丢失关键信息，后者在处理离群值时存在精度挑战。本文旨在通过挖掘视觉 Token 在隐藏维度上的**低秩结构**，实现更高效的压缩与检索。

### 2. 方法论：AttentionPack 框架
AttentionPack 的核心思想是利用矩阵分解减少 KV Cache 的存储维度，并根据注意力权重动态调整解压精度。
*   **多头注意力压缩（Multi-head Compression）**：
    *   **低秩分解**：分析发现 KV 矩阵具有隐式的低秩特性。通过奇异值分解（SVD），将原始 KV 矩阵分解为压缩后的缓存矩阵和解压缩矩阵。
    *   **跨头合并**：不同于逐头压缩，该方法将多个注意力头的向量合并后再进行 SVD，利用头之间的冗余信息实现更高的压缩率。
    *   **模态分离**：对视觉 Token 和文本 Token 分别应用不同的压缩策略，以保持多模态信息的完整性。
*   **注意力感知解压缩（Attention-aware Decompression）**：
    *   **重要性评分**：利用滑动平均（Moving Average）追踪每个 Token 的累积注意力分数。
    *   **部分解压**：在解码阶段，仅对注意力分数高的“重要”Token 使用全秩（Full Rank）解压，而对不重要的 Token 使用极低秩解压，从而大幅降低计算延迟（FLOPs）。
*   **算法流程**：在 Prefill（预填充）阶段后执行压缩；在每个 Decoding（解码）步骤中，先进行部分解压，计算注意力得分，更新 Token 重要性，并周期性地重新触发压缩以整合新生成的 Token。

### 3. 实验设计
*   **数据集与场景**：

    *   **视频 QA**：MSVD-QA、MSRVTT-QA（短视频理解）。
    *   **长文本**：LongBench（验证在纯文本领域的泛化性）。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 5 个多模态数据集和 1 个长文本基准，测试了从 7B 到 13B 不同规模的模型。
*   **消融实验**：详细分析了压缩秩（Rank）的大小、解压比例（r1/r2）、多头合并的影响、以及与量化/逐出技术的兼容性。
*   **客观性与公平性**：实验对比了多种主流的 KV Cache 压缩技术，并在相同的硬件和 Batch Size 条件下报告了吞吐量和精度，结果具有较强的说服力。

### 6. 主要结论与发现
*   **内存效率**：在 LLaVA 和 VideoLLaVA 上实现了 **5.1x 至 8.1x** 的内存缩减。
*   **推理加速**：通过减少内存占用，支持更大的 Batch Size，使批处理推理吞吐量提升了 **50% - 74%**。
*   **精度保持**：在压缩率极高的情况下，模型在 QA 任务上的精度损失微乎其微（通常在 1% 以内），甚至在某些 OCR 任务中因过滤了噪声而略有提升。
*   **兼容性**：AttentionPack 可以与 4-bit 量化、Token 逐出以及 FlashAttention 算子融合技术完美结合，进一步压榨硬件性能。

### 7. 优点（亮点）
*   **维度压缩而非数量逐出**：相比直接丢弃 Token，通过低秩分解保留了所有 Token 的全局信息，鲁棒性更强。
*   **注意力感知机制**：巧妙地利用解码过程中的注意力分布来动态分配计算资源，解决了 SVD 解压带来的额外延迟问题。
*   **底层优化**：提供了融合内核（Fused Kernel）实现，将解压与注意力计算合并，减少了 HBM 与 SRAM 之间的数据交换。

### 8. 不足与局限
*   **解压敏感性**：实验显示 Key 向量对部分解压比 Value 向量更敏感，过度降低 Key 的解压秩会导致注意力机制失效。
*   **计算开销**：虽然减少了内存 I/O，但 SVD 压缩和解压增加了额外的计算步骤，在极短序列或计算受限（而非内存受限）的场景下，加速效果可能不明显。
*   **文本 Token 压缩率较低**：该方法在视觉 Token 上效果显著，但在纯文本长序列上，由于文本信息的稠密性，压缩率提升空间相对有限。
