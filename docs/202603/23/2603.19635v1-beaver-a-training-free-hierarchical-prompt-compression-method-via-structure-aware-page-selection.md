## 摘要
大语言模型（LLM）上下文窗口的指数级扩张开启了长文档理解的能力，但也带来了推理延迟和信息利用方面的严重瓶颈。现有的压缩方法通常面临高昂的训练成本，或者由于激进的 Token 剪枝而导致语义碎片化。在本文中，我们提出了 BEAVER，这是一个全新的无需训练的框架，它将压缩从线性的 Token 移除转变为结构感知的分层选择。BEAVER 通过双路径池化将变长上下文映射为稠密的页面级张量，从而最大化硬件并行性，并通过结合语义和词法双分支选择与句子平滑的混合规划器来保持话语完整性。在四个长上下文基准测试上的广泛评估表明，BEAVER 达到了与 LongLLMLingua 等最先进（SOTA）方法相当的性能。值得注意的是，在 RULER 基准测试中，BEAVER 在基线方法性能退化的多针检索任务中保持了高保真度。在效率方面，BEAVER 在 128k 上下文中将延迟降低了 26.4 倍，为高吞吐量应用提供了一个可扩展的解决方案。我们的代码可在 https://cslikai.cn/BEAVER/ 获取。

## Abstract
The exponential expansion of context windows in LLMs has unlocked capabilities for long-document understanding but introduced severe bottlenecks in inference latency and information utilization. Existing compression methods often suffer from high training costs or semantic fragmentation due to aggressive token pruning. In this paper, we propose BEAVER, a novel training-free framework that shifts compression from linear token removal to structure-aware hierarchical selection. BEAVER maximizes hardware parallelism by mapping variable-length contexts into dense page-level tensors via dual-path pooling, and preserves discourse integrity through a hybrid planner combining semantic and lexical dual-branch selection with sentence smoothing. Extensive evaluations on four long-context benchmarks demonstrate that BEAVER achieves comparable performance to state-of-the-art (SOTA) methods like LongLLMLingua. Notably, on the RULER benchmark, BEAVER maintains high fidelity in multi-needle retrieval where baselines deteriorate. Regarding efficiency, BEAVER reduces latency by 26.4x on 128k contexts, offering a scalable solution for high-throughput applications. Our code is available at https://cslikai.cn/BEAVER/.

### BEAVER：一种通过结构感知页面选择实现的无需训练的分层提示词压缩方法

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）上下文窗口扩展至百万级，长文档理解能力显著增强，但也带来了两大挑战：
*   **计算墙（Computation Wall）：** 自注意力机制的 $O(L^2)$ 复杂度导致推理延迟（尤其是首字延迟）随长度激增。
*   **收益递减（Diminishing Returns）：** 模型常出现“迷失在中间”（Lost in the Middle）现象，无法有效利用长文本中的关键信息。
现有的提示词压缩方法要么依赖高昂的**监督训练**（难以跨模型迁移），要么采用**细粒度的 Token 剪枝**（破坏了语义和语法的连贯性）。BEAVER 的动机在于提出一种无需训练、硬件友好且能保持语义结构完整性的层级化压缩框架。

#### 2. 核心方法论
BEAVER 将压缩范式从“线性 Token 移除”转变为“结构感知的页面选择”，包含三个核心组件：
*   **分段器（Segmenter）：** 利用自然分隔符（如换行符）将长文本切分为逻辑段落，并采用贪婪分页策略将其映射为二维页面张量（Page Tensor），以便于 GPU 并行处理并保留局部语义边界。
*   **页面编码器（PageEncoder）：** 
    *   **双路径池化：** 同时使用加权平均池化（捕获全局语义）和最大池化（捕获显著局部特征，如稀有关键词）。
    *   **上下文自适应权重（In-Context ITF）：** 计算 Token 在当前上下文中的逆词频，降低高频无意义词的权重，突出关键实体。
*   **查询规划器（QueryPlanner）：**
    *   **混合匹配：** 结合语义相似度（稠密向量检索）和词法重叠（精确 Token 匹配）进行评分。
    *   **结构化先验：** 引入**锚点页**（保留开头元数据）、**流动页**（保留查询前的连续窗口以维持工作记忆）和**闪速页**（根据评分选择的高相关页面）。
    *   **句子平滑：** 将选中的页面扩展至最近的句子边界，确保输出文本的语法完整性。

*   **数据集/基准测试：** 涵盖了四个主流长上下文 Benchmark：

    2.  **Zero-SCROLLS：** 评估零样本长文本理解。

*   **训练时长：** 该方法为 **Training-Free（无需训练）**，因此没有训练阶段的算力消耗，仅涉及推理时的编码和选择开销。

#### 5. 实验数量与充分性
*   **实验规模：** 论文在 4 个基准测试的数十个子任务上进行了评估，样本量充足（如 LongBench 包含 4750 个样本）。
*   **消融实验：** 针对页面大小、池化方式、ITF 权重、混合评分比例、结构先验策略等进行了详尽的消融研究，验证了各组件的必要性。
*   **可扩展性分析：** 测试了从 0.6B 到 32B 不同参数规模的模型，验证了方法的稳健性。

#### 6. 主要结论与发现
*   **性能优异：** 在 LongBench 和 Zero-SCROLLS 上达到或超过了 SOTA 方法（如 LLMLingua-2）。
*   **检索能力极强：** 在 RULER 的多针检索任务中，BEAVER 几乎保持了 100% 的准确率，而 LongLLMLingua 等基线在相同条件下性能大幅崩溃。

#### 7. 优点
*   **无需训练：** 插件化设计，无需昂贵的微调即可适配各种模型。
*   **硬件友好：** 通过二维张量化操作最大化 GPU 并行度，显著降低推理延迟。
*   **结构感知：** 相比 Token 级剪枝，页面级选择和句子平滑有效避免了语义碎片化，生成的压缩文本更易于 LLM 理解。
*   **抗干扰能力：** ITF 权重和混合匹配机制使其在处理包含大量噪声的长文档时非常稳健。

#### 8. 不足与局限
*   **粒度限制：** 页面级粒度虽然保护了结构，但相比 Token 级剪枝，在极度苛刻的压缩比下可能保留少量冗余信息。
*   **多步推理挑战：** 检索机制主要基于语义和词法相似度，对于证据与查询之间无直接重叠、需深度多步演绎的复杂推理场景，可能存在挑战。
*   **超参数依赖：** 作为非学习方法，某些超参数（如混合评分的权重 $\lambda$）在极端不同的领域间可能需要手动微调以达最优。
