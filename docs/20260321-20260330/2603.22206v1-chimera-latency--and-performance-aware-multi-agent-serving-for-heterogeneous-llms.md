## 摘要
多智能体应用通常将复杂任务执行为多阶段工作流，其中每个阶段都是一次大语言模型（LLM）调用，其输出会成为后续步骤上下文的一部分。现有的 LLM 推理服务系统在很大程度上假设集群是同构的，即拥有相同的模型副本。这种设计忽视了异构部署的潜力，在异构部署中，不同规模和能力的模型可以在延迟和性能之间实现更精细的权衡。然而，异构性在跨具有不同吞吐量和性能的模型进行调度时引入了新的挑战。我们提出了 Chimera，这是一个用于异构 LLM 集群上多智能体工作流推理服务的预测性调度系统，旨在同时优化端到端延迟和任务性能。Chimera 应用语义路由来估计每个请求针对各模型的置信度分数，预测工作流的总剩余输出长度，并利用处理中的预测 token 量来估计各模型的拥塞情况以实现负载均衡。我们在代码生成和数学推理的代表性智能体工作流上，使用多种异构 LLM 配置对 Chimera 进行了评估。在可比设置下，Chimera 描绘了最佳的延迟-性能前沿，与包括 vLLM 在内的竞争基准相比，端到端延迟降低了 1.2-2.4 倍，任务性能平均提高了 8.0-9.5 个百分点。

## Abstract
Multi-agent applications often execute complex tasks as multi-stage workflows, where each stage is an LLM call whose output becomes part of context for subsequent steps. Existing LLM serving systems largely assume homogeneous clusters with identical model replicas. This design overlooks the potential of heterogeneous deployments, where models of different sizes and capabilities enable finer trade-offs between latency and performance. However, heterogeneity introduces new challenges in scheduling across models with diverse throughput and performance. We present Chimera, a predictive scheduling system for multi-agent workflow serving on heterogeneous LLM clusters that jointly improves end-to-end latency and task performance. Chimera applies semantic routing to estimate per-model confidence scores for each request, predicts the total remaining output length of the workflow, and estimates per-model congestion using in-flight predicted token volumes for load balancing. We evaluate Chimera on representative agentic workflows for code generation and math reasoning using multiple heterogeneous LLM configurations. Across comparable settings, Chimera traces the best latency-performance frontier, reducing end-to-end latency by 1.2--2.4$\times$ and improving task performance by 8.0-9.5 percentage points on average over competitive baselines including vLLM.

### Chimera：面向异构 LLM 集群的多智能体推理服务系统总结

#### 1. 核心问题与研究动机
*   **背景**：大语言模型（LLM）正从独立对话向多智能体（Multi-agent）协作工作流转变。这类任务通常涉及多个依赖阶段，前一阶段的输出是后一阶段的上下文。
*   **核心问题**：现有的推理服务系统（如 vLLM）多假设集群是同构的（运行相同模型），忽视了**异构部署**（混合不同规模和能力的模型）在性能与延迟权衡上的潜力。
*   **挑战**：在异构环境下，模型选择与系统拥塞高度耦合。将请求路由到更强但更慢的模型虽能提升性能，但会加剧 GPU 资源竞争，导致整体延迟增加。如何在异构集群中调度多智能体工作流，以同时优化端到端延迟和任务准确率，是本文的核心动机。

#### 2. 方法论：Chimera 系统架构
Chimera 是一个位于应用层与推理引擎之间的中间件，包含四个核心模块：
*   **语义路由器 (Semantic Router)**：基于 ModernBert-large 的转换器编码器，为每个请求预测各候选模型的“置信度分数”（即解决该任务的概率）。
*   **长度预测器 (Length Predictor)**：使用轻量级 CPU 端的分位数随机森林（QRF），预测**整个工作流剩余的总输出长度**（当前阶段 + 未来所有阶段），用于确定调度优先级。
*   **活动监控器 (Activity Monitor)**：实时跟踪每个引擎中“处理中”（in-flight）的预测 token 总量，以此估算各模型的负载诱导延迟。
*   **负载均衡器 (Load Balancer)**：
    *   **调度策略**：采用“最短总任务优先”（STJF）原则。
    *   **分发规则**：在满足“延迟松弛度”（Latency Slack，即比最快模型慢的比例在可接受范围内）的模型中，选择置信度最高的模型。
    *   **抗饥饿机制**：引入基于老化的提升机制，防止长任务被无限期推迟。

#### 3. 实验设计
*   **数据集/场景**：
    *   **APPS**：代码生成任务，模拟 Planner -> Coder 等 1-4 阶段的工作流。
    *   **MATH**：数学推理任务，模拟 Planner -> Solver -> Verifier 等工作流。
*   **模型配置**：混合了 Qwen2.5 (1.5B/3B/7B/14B)、Llama3.2-3B 和 Ministral-8B 等异构组合。
*   **对比基准 (Baselines)**：
    *   **vLLM**：标准的负载均衡调度。
    *   **MLFQ**：多级反馈队列调度。
    *   **LTR (Learning to Rank)**：基于学习排序的 SJF 调度。
    *   **vLLM (STJF)**：使用真实长度（Oracle）的参考上限。

#### 4. 资源与算力
*   **硬件**：实验运行在 **NVIDIA RTX A6000 GPUs (49GB VRAM)** 上。
*   **训练细节**：
    *   语义路由器：微调 5 个 epoch，学习率 1e-5，Batch Size 16。
    *   长度预测器：基于 CPU 运行，推理开销极低。
*   **调度开销**：论文明确指出 Chimera 的调度开销极小，仅占端到端延迟的 **0.2% - 2.2%**。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了 2 种典型任务（代码、数学）、4 种异构模型组合、3 种不同的请求频率（RPS=8, 12, 16）。
*   **消融实验**：对比了“有无语义路由器”的效果，并与“Oracle 预测器”和“Oracle 路由器”进行了差距分析。
*   **充分性评价**：实验设计较为充分，通过 Pareto 前沿曲线展示了延迟与性能的权衡，对比方法涵盖了当前主流的调度优化思路，结果具有较强的说服力。

#### 6. 主要结论与发现
*   **性能提升**：相比 vLLM，Chimera 将端到端延迟降低了 **1.2–2.4 倍**，同时将任务准确率平均提升了 **8.0–9.5 个百分点**。
*   **调度优势**：STJF（最短总任务优先）比传统的 SJF（最短作业优先）在多智能体场景下更有效，能额外减少 15-34% 的排队时间。
*   **权衡能力**：通过调整“延迟松弛度”参数，服务商可以灵活地在“极低延迟”和“极高准确率”之间切换。

#### 7. 优点
*   **协同优化**：首次将模型路由（选哪个模型）与系统调度（何时运行）在异构环境下结合。
*   **工作流感知**：利用多智能体阶段间的依赖关系进行长度预测，而非孤立看待单个请求。
*   **轻量高效**：核心组件（QRF 预测器、异步路由）设计精巧，几乎不增加推理负担。
*   **实用性强**：作为 vLLM 的插件层实现，易于集成到现有生产环境。

#### 8. 不足与局限
*   **模型依赖性**：语义路由器和长度预测器需要针对特定模型和任务进行离线训练/微调，对于全新任务的泛化能力未深入探讨。
*   **KV 缓存假设**：虽然提到了 KV 缓存重用，但在高度异构且频繁切换模型的极端情况下，缓存命中率的波动可能影响实际表现。
*   **硬件环境**：实验主要在单机多卡环境（A6000）完成，在大规模分布式集群（跨节点通信延迟）中的表现尚待验证。
