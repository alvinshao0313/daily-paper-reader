## 摘要
基于大语言模型（LLM）的智能体依赖记忆机制来复用过去解决问题经验中的知识。现有方法通常以每个智能体为单位构建记忆，将存储的知识与单个模型的推理风格紧密耦合。在异构智能体的现代部署中，一个自然的问题随之而来：单个记忆系统能否在不同模型之间共享？我们发现，在智能体之间简单地迁移记忆往往会降低性能，因为此类记忆将任务相关知识与智能体特定的偏置纠缠在一起。为了应对这一挑战，我们提出了 MemCollab，这是一个协作记忆框架，通过对比不同智能体在同一任务上生成的推理轨迹来构建与智能体无关的记忆。这种对比过程蒸馏出了抽象的推理约束，这些约束捕捉了共享的任务级不变性，同时抑制了智能体特定的伪影。我们进一步引入了一种任务感知检索机制，该机制根据任务类别调节记忆访问，确保在推理时仅使用相关的约束。在数学推理和代码生成基准测试上的实验表明，MemCollab 在包括跨模型家族设置在内的多种智能体中，一致地提高了准确率和推理效率。我们的结果表明，协作构建的记忆可以作为多种基于 LLM 的智能体的共享推理资源。

## Abstract
Large language model (LLM)-based agents rely on memory mechanisms to reuse knowledge from past problem-solving experiences. Existing approaches typically construct memory in a per-agent manner, tightly coupling stored knowledge to a single model's reasoning style. In modern deployments with heterogeneous agents, a natural question arises: can a single memory system be shared across different models? We found that naively transferring memory between agents often degrades performance, as such memory entangles task-relevant knowledge with agent-specific biases. To address this challenge, we propose MemCollab, a collaborative memory framework that constructs agent-agnostic memory by contrasting reasoning trajectories generated by different agents on the same task. This contrastive process distills abstract reasoning constraints that capture shared task-level invariants while suppressing agent-specific artifacts. We further introduce a task-aware retrieval mechanism that conditions memory access on task category, ensuring that only relevant constraints are used at inference time. Experiments on mathematical reasoning and code generation benchmarks demonstrate that MemCollab consistently improves both accuracy and inference-time efficiency across diverse agents, including cross-modal-family settings. Our results show that the collaboratively constructed memory can function as a shared reasoning resource for diverse LLM-based agents.

#### 1. 核心问题与整体含义（研究动机和背景）
现有的 LLM 智能体记忆机制通常是“单兵作战”模式：记忆是从特定模型的推理轨迹中提取的，并仅供该模型自身复用。这种方式导致记忆与特定模型的推理风格、偏好和偏见深度耦合。在实际应用中，往往存在由不同规模或架构组成的异构智能体集群，直接在不同智能体间迁移记忆（例如将 32B 模型的记忆直接给 7B 模型用）往往会导致性能下降。
**核心问题**：能否构建一种**与智能体无关（Agent-agnostic）**的共享记忆系统，使不同模型能够协同利用过去的经验？

#### 2. 方法论：核心思想与关键技术
MemCollab 提出了一个协作记忆框架，通过对比不同智能体在同一任务上的表现来提取“任务级不变量”。其流程分为两个阶段：

*   **对比驱动的记忆构建（Contrast-Derived Memory Construction）**：
    *   **轨迹配对**：让两个异构智能体（如一强一弱）处理同一任务，收集推理轨迹。
    *   **对比分析**：利用强模型作为分析器，对比正确轨迹（Preferred）与错误轨迹（Unpreferred）。
    *   **蒸馏约束**：识别出导致失败的“违规模式”（Violation patterns）和确保成功的“推理不变量”（Reasoning invariants）。
    *   **结构化存储**：记忆条目表示为 `mk = (enforce ik; avoid vk)`，即“执行[正确准则]；避免[错误模式]”。这种抽象描述避开了具体的数值或实例，更具通用性。
*   **任务感知检索与推理（Task-Aware Retrieval and Inference）**：
    *   **任务分类**：在检索前，先用 LLM 将当前任务分类（如代数、数论、代码逻辑等）。
    *   **条件检索**：仅在相同类别的记忆子集中进行相似度检索，减少无关噪声的干扰。
    *   **引导推理**：将检索到的抽象约束作为 Prompt 注入智能体，引导其推理过程。

    *   **Vanilla**：无记忆增强的基准。
    *   **Buffer of Thoughts (BoT)**：基于思维模板的记忆。
    *   **Dynamic Cheatsheet (DC)**：动态备忘录机制。
    *   **Naive Transfer**：直接使用其他模型生成的记忆。
    *   **Self-Contrast**：模型自身多次尝试的对比记忆。

*   **实验规模**：在 4 个主流基准数据集上进行了详尽测试，涵盖了同系列模型（Qwen 7B/32B）和跨系列模型（Qwen 与 Llama）的协作。
*   **消融实验**：

*   **评价**：实验设计较为全面，不仅关注准确率，还分析了推理效率（推理轮数），验证了跨架构的泛化能力，实验结果具有较强的说服力和客观性。

#### 6. 主要结论与发现
*   **打破耦合**：MemCollab 成功实现了记忆的跨模型共享，弱模型（7B）在使用协作记忆后，性能显著提升，甚至接近或超过了更强的基准模型。
*   **双向受益**：不仅弱模型受益，强模型（32B）在引入协作记忆后，准确率也得到了进一步提升。
*   **效率提升**：配备协作记忆的智能体在解决问题时所需的推理轮数（Trial-and-error）显著减少，推理过程更加直接。
*   **跨架构有效性**：即使在完全不同的模型家族（如 Llama 和 Qwen）之间，对比蒸馏出的推理约束依然有效。

#### 7. 优点（亮点）
*   **对比学习的巧妙应用**：通过“找不同”的方式过滤掉模型特有的风格偏见，只保留逻辑层面的核心约束。
*   **任务感知检索**：引入任务分类预处理，有效解决了大规模记忆库检索时的噪声干扰问题。
*   **抽象化表达**：记忆内容不是原始轨迹，而是高度概括的准则，这使得知识更易于在不同规模的模型间迁移。

#### 8. 不足与局限
*   **对强模型的依赖**：记忆的构建过程（对比分析阶段）高度依赖于一个性能较强的 Backbone 模型（如 32B 或更高），若分析器本身存在逻辑缺陷，生成的记忆质量会受限。

*   **领域局限性**：目前实验主要集中在数学和代码等逻辑严密的领域，在文学创作、常识问答等开放域任务上的表现尚待验证。
