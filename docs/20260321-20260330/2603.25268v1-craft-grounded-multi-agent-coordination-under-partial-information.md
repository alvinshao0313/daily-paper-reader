## 摘要
我们介绍了 CRAFT，这是一个用于评估大语言模型在严格部分信息条件下语用通信（pragmatic communication）的多智能体基准测试。在该设定中，多个拥有互补但非完整视野的智能体必须通过自然语言进行协作，以构建一个任何单个智能体都无法完全观察到的共享 3D 结构。我们将该问题形式化为一个多发送者语用推理任务，并提供了一个诊断框架，将失败原因分解为空间具身（spatial grounding）、信念建模和语用通信错误，并对前沿模型和开源权重模型中的行为失败概况进行了分类。通过对包括 8 个开源权重模型和 7 个前沿模型（含推理模型）在内的多种模型进行测试，我们发现更强的推理能力并不一定能转化为更好的协作能力：较小的开源权重模型往往能达到甚至超过前沿系统的表现，且个体通信能力的提升并不保证协作的成功。这些结果表明，多智能体协作对于当前的语言模型来说仍然是一个从根本上尚未解决的挑战。我们的代码可在 https://github.com/csu-signal/CRAFT 找到。

## Abstract
We introduce CRAFT, a multi-agent benchmark for evaluating pragmatic communication in large language models under strict partial information. In this setting, multiple agents with complementary but incomplete views must coordinate through natural language to construct a shared 3D structure that no single agent can fully observe. We formalize this problem as a multi-sender pragmatic reasoning task and provide a diagnostic framework that decomposes failures into spatial grounding, belief modeling and pragmatic communication errors, including a taxonomy of behavioral failure profiles in both frontier and open-weight models. Across a diverse set of models, including 8 open-weight and 7 frontier including reasoning models, we find that stronger reasoning ability does not reliably translate to better coordination: smaller open-weight models often match or outperform frontier systems, and improved individual communication does not guarantee successful collaboration. These results suggest that multi-agent coordination remains a fundamentally unsolved challenge for current language models. Our code can be found at https://github.com/csu-signal/CRAFT

#### 2. 方法论：CRAFT 框架与理论模型
*   **核心思想**：构建一个 3D 结构搭建任务，由三名“主管（Director）”和一名“建筑师（Builder）”协作完成。
*   **关键技术细节**：
    *   **信息不对称设计**：三名主管分别只能看到 3D 目标结构的一个 2D 侧面（左墙、远墙、右墙），且视野仅在极少数锚点重合。建筑师完全看不到目标，只能根据主管的指令行动。
    *   **多发送者有界语用说话者（BPS）框架**：将任务形式化为概率推理问题。每个主管 $D_i$ 的策略 $\pi$ 需平衡其基础生成分布（基于私有观测）和对建筑师（联合听者）理解能力的建模。
    *   **智能体架构**：主管输出包含私有推理块（`<think>`）和公共消息块（`<message>`），强制模型进行从内部空间推理到外部语用表达的转化。
    *   **Oracle 辅助建筑师**：为消除建筑师执行能力的干扰，建筑师从一组经 Oracle 验证的合法候选动作中进行选择，从而将失败归因精准定位在主管的沟通质量上。

#### 3. 实验设计
*   **场景与基准**：
    *   **CRAFT 基准**：包含 20 个程序生成的 3D 结构，分为简单、中等、复杂三个等级（21-25 个方块）。
    *   **任务流程**：每场游戏持续 20 轮，主管轮流发言，建筑师执行 `PLACE`（放置）、`REMOVE`（移除）或 `CLARIFY`（澄清）。
*   **对比方法（15 种模型）**：
    *   **前沿/闭源模型**：GPT-4o, GPT-4o-mini, GPT-4.1-mini, Claude-3.5-Sonnet, Gemini-1.5-Flash 等。
    *   **开源权重模型**：Qwen-2.5 (7B/14B/32B/72B), Llama-3-8B, Mistral-7B, Gemma-2-9B, DeepSeek-V2-Lite。

#### 6. 主要结论与发现
*   **能力倒挂**：前沿模型并不总是在协作中胜出。Mistral-7B 和 Qwen-2.5-7B 的表现优于多数前沿系统（如 Claude-3.5-Sonnet 和部分 Gemini 模型）。
*   **推理不等于协作**：个体沟通质量（SG 和 MM 评分）与最终任务成功率往往呈**负相关**。前沿模型虽然在私有推理和消息校准上得分更高，但往往因过度纠错导致失败。
*   **纠错螺旋（Correction Spirals）**：低效协作的典型特征是智能体陷入反复移除方块的死循环，无法通过沟通克服信息不对称，白白消耗轮次。
*   **语用失效**：主管往往无法根据其他成员已提供的信息来精简自己的表达，导致信息冗余或冲突。

#### 7. 论文优点
*   **任务设计精巧**：通过 3D 搭建任务完美模拟了现实中的空间定位、视角切换和信息互补需求。
*   **理论支撑深厚**：将语用学理论（RSA/BPS）成功引入多智能体 LLM 评估，提供了严谨的数学形式化。
*   **诊断深入**：不仅关注成功率，还通过 LLM Judge 拆解了失败的具体维度（空间、心理、语用），提供了行为层面的深度分析。

#### 8. 不足与局限
*   **模态限制**：目前仅限于纯文本设定，未测试具备原生视觉能力的多模态模型（VLM）在相同视角下的表现。
*   **建筑师固定**：实验中建筑师是固定的且有 Oracle 辅助，未探讨当建筑师也具备不确定性或不同能力水平时的异构协作。
*   **同质化假设**：每场游戏中的主管均使用同一模型，未测试不同模型混合（如 GPT 搭配 Llama）时的协作鲁棒性。
*   **应用范围**：任务场景较为特定（方块搭建），其结论在更复杂的开放域协作任务中的泛化性有待验证。
