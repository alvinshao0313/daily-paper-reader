## 摘要
在企业环境中部署 AI 智能体需要平衡能力、数据主权和成本约束。虽然小语言模型为前沿模型提供了保护隐私的替代方案，但其专业化受到碎片化开发流水线的阻碍，这些流水线将工具集成、数据生成和训练分离开来。我们推出了 EnterpriseLab，这是一个将这些阶段统一到闭环框架中的全栈平台。EnterpriseLab 提供：(1) 一个通过模型上下文协议（Model Context Protocol）暴露企业应用程序的模块化环境，实现专有工具和开源工具的无缝集成；(2) 自动轨迹合成，可根据环境模式以编程方式生成训练数据；(3) 具有持续评估功能的集成训练流水线。我们通过 EnterpriseArena 验证了该平台，这是一个包含 IT、人力资源、销售和工程领域的 15 个应用程序和 140 多个工具的实例。我们的结果表明，在 EnterpriseLab 中训练的 8B 参数模型在复杂的企业工作流中达到了与 GPT-4o 相当的性能，同时将推理成本降低了 8-10 倍，并且在包括 EnterpriseBench（+10%）和 CRMArena（+10%）在内的各种企业基准测试中保持稳健。EnterpriseLab 为企业提供了一条在不损害运营能力的情况下部署高性能、隐私保护型智能体的切实可行路径。

## Abstract
Deploying AI agents in enterprise environments requires balancing capability with data sovereignty and cost constraints. While small language models offer privacy-preserving alternatives to frontier models, their specialization is hindered by fragmented development pipelines that separate tool integration, data generation, and training. We introduce EnterpriseLab, a full-stack platform that unifies these stages into a closed-loop framework. EnterpriseLab provides (1) a modular environment exposing enterprise applications via Model Context Protocol, enabling seamless integration of proprietary and open-source tools; (2) automated trajectory synthesis that programmatically generates training data from environment schemas; and (3) integrated training pipelines with continuous evaluation. We validate the platform through EnterpriseArena, an instantiation with 15 applications and 140+ tools across IT, HR, sales, and engineering domains. Our results demonstrate that 8B-parameter models trained within EnterpriseLab match GPT-4o's performance on complex enterprise workflows while reducing inference costs by 8-10x, and remain robust across diverse enterprise benchmarks, including EnterpriseBench (+10%) and CRMArena (+10%). EnterpriseLab provides enterprises a practical path to deploying capable, privacy-preserving agents without compromising operational capability.

在企业环境中部署 AI 智能体面临三大挑战：**数据主权（隐私）**、**高昂的推理成本**以及**API 延迟**。虽然前沿大模型（如 GPT-4o）能力强劲，但企业更倾向于在本地部署 8B-32B 参数的小语言模型（SLM）。然而，SLM 的专业化过程受到开发流程碎片化的阻碍——工具集成、数据生成和模型训练往往是脱节的。
论文提出了 **EnterpriseLab**，旨在通过一个闭环框架统一这些阶段，使企业能够利用自身的专有工具和业务逻辑，高效地训练出性能媲美顶级模型且成本低廉的专用智能体。

#### 2. 方法论：核心思想与关键技术
EnterpriseLab 采用闭环设计，包含三个核心组件：
*   **模块化工具环境（Modular Tool Environment）：** 采用 **Model Context Protocol (MCP)** 协议，实现工具的即插即用。它包含动态工具注册表（规范化参数模式）、状态化执行容器（Docker 隔离）和观察结果规范化器（处理异构 API 输出）。
*   **自动化轨迹合成流水线（Task Synthesis Pipeline）：** 
    *   **工具图构建：** 根据 API 的输入输出依赖关系构建有向图。
    *   **约束感知采样：** 通过深度优先遍历（DFS）生成可执行的工具序列，利用内存缓冲区确保数据流的一致性。
    *   **分层任务合成：** 使用 LLM 先生成底层思维链，再合成高层用户意图。
    *   **验证与过滤：** 通过去重、多样性筛选（MMR 算法）和环境执行验证，确保数据的高质量。
*   **集成训练基础设施：** 支持监督微调（SFT）、偏好优化（DPO）以及 **Agentic GRPO（智能体组相对策略优化）**。Agentic GRPO 通过在环境中进行 ReAct 风格的采样，利用环境反馈（工具调用正确性、任务完成度等）作为奖励信号进行在线强化学习。

#### 7. 优点
*   **全栈闭环：** 解决了从工具接入到数据生成再到模型训练的全流程自动化，降低了企业开发门槛。
*   **环境对齐：** 任务合成直接源于环境模式（Schema），确保了训练数据与实际生产环境的高度一致性。
*   **协议标准化：** 引入 MCP 协议，极大增强了平台对各种企业软件的兼容性。
*   **强化学习创新：** 将 GRPO 应用于智能体轨迹优化，有效利用了环境的真实反馈。

*   **交互模态限制：** 目前主要侧重于基于文本 API 的工具调用，尚未覆盖基于图形用户界面（GUI）的视觉交互。

*   **领域覆盖：** 虽然涵盖了 15 个应用，但对于某些极度垂直或复杂的企业专有逻辑，可能仍需更细致的工具建模。
