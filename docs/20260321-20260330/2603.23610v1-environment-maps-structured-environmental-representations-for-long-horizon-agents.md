## 摘要
尽管大语言模型（LLMs）发展迅速，但复杂软件工作流的鲁棒自动化仍是一个悬而未决的问题。在长程（long-horizon）场景中，智能体经常受到级联错误和环境随机性的困扰；动态界面中的一次失误就可能导致任务失败，进而引发幻觉或陷入反复试错。本文介绍了“环境地图”（Environment Maps）：一种持久的、与智能体无关的表示方式，通过将屏幕录像和执行轨迹等异构证据整合到结构化图中，来缓解这些失败。该表示由四个核心组件组成：(1) 上下文（抽象位置），(2) 动作（参数化可供性），(3) 工作流（观测到的轨迹），以及 (4) 隐性知识（领域定义和可重用程序）。我们在 WebArena 基准测试的五个领域中评估了该框架。配备环境地图的智能体实现了 28.2% 的成功率，几乎是受限于会话上下文的基线模型（14.2%）的两倍，且优于能够访问用于生成环境地图的原始轨迹数据的智能体（23.3%）。通过在模型和环境之间提供结构化接口，环境地图为长程规划奠定了持久的基础，且具有可解释性、可编辑性和增量完善性。

## Abstract
Although large language models (LLMs) have advanced rapidly, robust automation of complex software workflows remains an open problem. In long-horizon settings, agents frequently suffer from cascading errors and environmental stochasticity; a single misstep in a dynamic interface can lead to task failure, resulting in hallucinations or trial-and-error. This paper introduces $\textit{Environment Maps}$: a persistent, agent-agnostic representation that mitigates these failures by consolidating heterogeneous evidence, such as screen recordings and execution traces, into a structured graph. The representation consists of four core components: (1) Contexts (abstracted locations), (2) Actions (parameterized affordances), (3) Workflows (observed trajectories), and (4) Tacit Knowledge (domain definitions and reusable procedures). We evaluate this framework on the WebArena benchmark across five domains. Agents equipped with environment maps achieve a 28.2% success rate, nearly doubling the performance of baselines limited to session-bound context (14.2%) and outperforming agents that have access to the raw trajectory data used to generate the environment maps (23.3%). By providing a structured interface between the model and the environment, Environment Maps establish a persistent foundation for long-horizon planning that is human-interpretable, editable, and incrementally refinable.

#### 1. 核心问题与研究动机
尽管大语言模型（LLM）在短期任务上表现出色，但在处理**长程（Long-horizon）**复杂软件工作流时仍面临巨大挑战。核心痛点在于：
*   **级联错误**：长路径中的一次微小失误会导致后续步骤全部失效。
*   **缺乏持久认知**：智能体往往只具备当前页面的局部视野，无法理解自己在全局流程中的位置、哪些动作有效以及动作的长期后果。
*   **环境动态性**：UI 的微调或领域特定语义的差异会导致预训练模型的知识失效。
*   **现有方案局限**：现有的 UI 图结构或技能库通常是零散的、非持久的，且与原始观测数据（如 DOM）耦合过紧，难以维护和复用。

#### 2. 方法论：环境地图框架
论文提出了一种名为 **Environment Maps (M)** 的持久化、与智能体无关的结构化表示方法。其核心是一个四元组 $M = (C, A, W, K)$：
*   **Contexts (C, 上下文)**：对抽象位置（如特定 URL 模式的页面）的定义，包含该页面可用的动作集。
*   **Actions (A, 动作)**：参数化的可供性（Affordances）。例如将“点击 John”抽象为“点击 {用户名}”，并记录观察到的取值范围。
*   **Workflows (W, 工作流)**：从历史轨迹中提取的成功操作序列。
*   **Tacit Knowledge (K, 默会知识)**：包含领域术语定义（如“MR”代表“Merge Request”）和常用操作规程。

**构建流水线（5个阶段）：**
1.  **统一步骤序列化**：解析原始观测（屏幕录制、轨迹），生成包含动作名、描述和结果的步骤。
2.  **动作提取**：区分“已执行动作”和通过 DOM 分析发现的“潜在动作”。
3.  **动作泛化**：利用 LLM 识别模式，将具体动作抽象为带占位符的参数化模板。
4.  **上下文与知识提取**：通过 URL 正则化对页面进行聚类，并提取领域定义和操作规程。
5.  **整合与合并**：去重并建立实体间的追踪链接，形成最终的 JSON 结构图。

#### 3. 实验设计
*   **基准测试**：使用 **WebArena**，涵盖 5 个真实交互环境（电子商务、CMS 管理后台、GitLab、Reddit 论坛、地图）。
*   **任务规模**：共 812 个自然语言指令任务，要求跨页面、跨站点的多步交互。
*   **对比实验设置**：
    *   **Baseline (No Map)**：仅提供当前页面观测（可访问性树）和任务目标。
    *   **Trajectory Access**：允许智能体通过工具查询原始的人类操作轨迹文件（JSONL 格式）。
    *   **Environment Map Access**：允许智能体访问基于相同轨迹数据生成的结构化环境地图。
*   **智能体配置**：统一使用 `claude-3-5-sonnet` 驱动的智能体，配备文件读取、搜索和 Bash 工具。

#### 4. 资源与算力
*   **构建成本**：环境地图的构建成本极低。针对 5 个环境，每个环境仅需 19-45 条轨迹，构建耗时约 **13-31 分钟**，成本仅为 **1-4 美元**。
*   **算力需求**：文中未提及具体的 GPU 训练算力，因为该方法属于**推理侧的知识增强**，而非模型微调。相比于需要数百 GPU 小时的微调（SFT），该方法通过结构化外部知识实现了高效增强。
*   **运行环境**：实验在云端 Docker 容器中运行，确保环境可重置和可复现。

#### 5. 实验数量与充分性
*   **实验规模**：在 812 个任务上进行了完整测试，并提供了 95% 的威尔逊置信区间（Wilson CIs），数据具有统计学意义。
*   **消融与对比**：通过对比“原始轨迹”和“结构化地图”，成功证明了**知识的组织形式（Representation）**比数据本身更重要。
*   **覆盖度**：实验覆盖了从简单导航到复杂数据修改（MUTATE）的多种任务类型。
*   **客观性**：使用了 `WebArena Verified` 评估框架，通过 HAR 追踪文件进行确定性验证，避免了 LLM 自我评分的幻觉问题。

#### 6. 主要结论与发现
*   **性能翻倍**：配备环境地图的智能体成功率达到 **28.2%**，远高于基线的 14.2%。
*   **优于原始数据**：即使拥有相同的数据源，结构化地图（28.2%）也显著优于原始轨迹访问（23.3%）。
*   **复杂环境增益更大**：在 GitLab 和 CMS 等分支多、UI 复杂的环境中，地图带来的提升最明显。
*   **泛化能力**：在没有人类演示过的任务上，环境地图依然能提供显著的性能提升（从 12.6% 提升至 26.9%）。
*   **交互模式改变**：使用地图后，智能体更倾向于主动查询外部知识（平均每任务 12.1 次工具调用），表现出更强的规划性。

#### 7. 优点
*   **智能体无关性**：地图是独立的 JSON 构件，可以适配不同的模型（如 GPT 或 Claude）而无需重新训练。
*   **人类可读与可编辑**：地图结构清晰，人类专家可以轻松修正其中的错误或更新 UI 变化。
*   **低成本高收益**：相比微调，构建地图的成本几乎可以忽略不计，且易于增量维护。
*   **减少回溯**：HAR 分析显示，使用地图显著减少了智能体在页面间的无效回溯和网络请求。

#### 8. 不足与局限
*   **跨站点限制**：目前的地图主要针对单一环境，对于需要跨多个复杂站点协作的任务（如跨站数据同步）支持有限。
*   **状态验证不足**：目前的动作前置条件和效果主要是描述性的，尚未实现与实时环境状态的硬性绑定验证。
*   **时效性风险**：如果 Web 界面发生重大重构，地图可能失效，需要自动化的检测和修复机制（未来研究方向）。
*   **任务覆盖**：虽然在 WebArena 表现良好，但在极度动态或个性化极强的私有软件环境中的表现尚待验证。
