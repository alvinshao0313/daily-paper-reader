## 摘要
在综合基准上评估 AI 智能体成本高昂，因为每次评估都需要涉及工具使用和多步推理的交互式展开。我们研究了较小的任务子集是否能在大幅降低成本的情况下保持智能体排名。与静态语言模型基准不同，智能体评估受脚手架驱动的分布偏移影响，因为性能取决于包装底层模型的框架。通过对 8 个基准、33 个智能体脚手架和 70 多个模型配置的研究，我们发现绝对分数的预测在这种偏移下会退化，而排名预测则保持稳定。利用这种不对称性，我们提出了一种简单的无需优化的协议：仅在历史通过率处于中间水平（30-70%）的任务上评估新智能体。这种受项目反应理论启发的难度中等过滤器，在保持脚手架和时间偏移下的高排名保真度的同时，将评估任务数量减少了 44-70%。它比随机采样提供了更可靠的排名（随机采样在不同种子间表现出高方差），并且在分布偏移下优于贪婪任务选择。这些结果表明，可靠的排行榜排名并不需要全基准评估。

## Abstract
Evaluating AI agents on comprehensive benchmarks is expensive because each evaluation requires interactive rollouts with tool use and multi-step reasoning. We study whether small task subsets can preserve agent rankings at substantially lower cost. Unlike static language model benchmarks, agent evaluation is subject to scaffold-driven distribution shift, since performance depends on the framework wrapping the underlying model. Across eight benchmarks, 33 agent scaffolds, and 70+ model configurations, we find that absolute score prediction degrades under this shift, while rank-order prediction remains stable. Exploiting this asymmetry, we propose a simple optimization-free protocol: evaluate new agents only on tasks with intermediate historical pass rates (30-70%). This mid-range difficulty filter, motivated by Item Response Theory, reduces the number of evaluation tasks by 44-70% while maintaining high rank fidelity under scaffold and temporal shifts. It provides more reliable rankings than random sampling, which exhibits high variance across seeds, and outperforms greedy task selection under distribution shift. These results suggest that reliable leaderboard ranking does not require full-benchmark evaluation.

*   **研究动机**：现有的基准测试缩减研究多针对静态 NLP 任务，而智能体性能高度依赖于“脚手架”（Scaffold，即包装模型的框架）。这种依赖导致了严重的分布偏移（Distribution Shift）。
*   **整体含义**：研究探讨了是否可以通过选择一小部分任务子集，在大幅降低成本（减少 44-70% 任务）的同时，依然能够准确预测智能体的排名。

### 2. 论文提出的方法论
*   **核心思想**：利用**排名预测（Rank Prediction）**与**分数预测（Score Prediction）**之间的不对称性。实验发现，在脚手架或时间偏移下，绝对分数很难预测，但智能体之间的相对排名非常稳定。
*   **关键技术：中等难度过滤器（Mid-Range Difficulty Filter, MR）**：
    *   **筛选规则**：仅选择历史通过率在 **30% 到 70%** 之间的任务。
    *   **理论依据**：借鉴**项目反应理论（Item Response Theory, IRT）**。根据 Fisher 信息准则，通过率接近 50% 的题目对于区分不同能力的个体具有最大的信息量。
*   **算法流程**：
    1.  收集初始的一组智能体在全量基准上的表现数据。
    2.  计算每个任务的平均通过率。
    3.  保留 30-70% 通过率的任务作为“缩减版基准”。
    4.  对新智能体仅运行这些任务，通过简单平均或 Ridge 回归预测其排名。

### 3. 实验设计
*   **数据集/场景**：涵盖了 8 个主流智能体基准，包括 **Terminal-Bench 2.0**（终端环境）和 **HAL**（包含 SWE-bench Verified, GAIA, Mind2Web, SciCode, USACO 等）。
*   **对比方法**：

    *   **Random-k**：随机抽取相同数量的任务（100 个随机种子）。
    *   **Easiest/Hardest-k**：选择最简单或最难的任务。
    *   **Stratified-k**：按难度分层抽样。

### 6. 主要结论与发现
*   **排名稳定性**：即使绝对分数预测失效（$R^2$ 降至 0.5 左右），排名相关性（Spearman $\rho$）依然能保持在 0.9 以上。
*   **成本大幅降低**：MR 协议平均减少了 **63%** 的任务量，在某些基准（如 USACO）上甚至减少了 70%。
*   **MR 优于贪婪算法**：在存在分布偏移（如新脚手架出现）时，贪婪选择容易过拟合，而基于 IRT 理论的 MR 更加鲁棒。
*   **任务透明度呼吁**：作者强调，只有公开每项任务的原始得分矩阵（而非仅总分），才能实现这种高效评估。

### 7. 优点
*   **简单且无需优化**：MR 是一种确定性的规则，不需要复杂的训练或超参数调节。
*   **理论支撑强**：将心理测量学中的 IRT 理论成功应用于 AI 智能体评估。
*   **实战导向**：直接解决了独立研究者和小实验室因成本问题无法参与智能体研究的痛点。

### 8. 不足与局限
*   **冷启动问题**：MR 需要初始的 5-15 个智能体运行全量基准来估计任务难度。
*   **难度分布敏感**：如果基准测试的任务难度极度不均（例如 SciCode，大部分题都极难），MR 可能会因为可选任务太少而失效。
*   **绝对能力评估受限**：该方法主要用于“排名”，如果目标是衡量智能体的绝对成功率（如“能解决多少百分比的现实问题”），缩减版基准会产生偏差。
