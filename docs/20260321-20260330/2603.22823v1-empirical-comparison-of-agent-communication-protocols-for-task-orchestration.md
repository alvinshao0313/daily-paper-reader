## 摘要
背景：如今，人工智能智能体系统正从单一工具交互向复杂的多智能体编排转型。因此，出现了两种竞争性的通信协议：一种是规范智能体如何调用外部工具的工具集成协议，另一种是使自主智能体能够相互发现并委派任务的智能体间委派协议。尽管这些协议已被数十家企业合作伙伴在工业界广泛采用，但现有文献中尚缺乏对这些协议的实证比较。目标：本研究旨在开发首个系统性基准，在三个复杂度级别的标准化查询下，对纯工具集成、多智能体委派及混合架构进行比较，并量化其在响应时间、上下文窗口消耗、货币成本、错误恢复和实现复杂度方面的权衡。

## Abstract
Context. Nowadays, artificial intelligence agent systems are transforming from single-tool interactions to complex multi-agent orchestrations. As a result, two competing communication protocols have emerged: a tool integration protocol that standardizes how agents invoke external tools, and an inter-agent delegation protocol that enables autonomous agents to discover and delegate tasks to one another. Despite widespread industry adoption by dozens of enterprise partners, no empirical comparison of these protocols exists in the literature. Objective. The goal of this work is to develop the first systematic benchmark comparing tool-integration-only, multi-agent delegation, and hybrid architectures across standardized queries at three complexity levels, and to quantify the trade-offs in response time, context window consumption, monetary cost, error recovery, and implementation complexity.

随着大语言模型（LLM）智能体系统的演进，架构设计面临一个核心抉择：智能体应如何与工具及其他智能体通信？目前行业内存在两种主流但设计哲学不同的协议：
*   **Model Context Protocol (MCP)**：由 Anthropic 提出，侧重于“智能体-工具”集成，将外部工具标准化为服务器，由单个智能体在统一的上下文窗口中调用。
*   **Agent-to-Agent Protocol (A2A)**：由 Google 提出，侧重于“智能体-智能体”委派，允许自主智能体之间通过 JSON-RPC 发现并分配任务，每个智能体拥有独立的上下文。

**研究动机**：尽管这两种协议在工业界已被广泛采用，但学术界此前仅有理论分析，缺乏实证数据来指导开发者在不同复杂度的任务中如何进行协议选型。

### 2. 论文提出的方法论
论文构建了一个名为“开源健康分析器（Open Source Health Analyzer）”的实验应用，并实现了三种对比架构：
*   **架构 A (MCP-Only)**：单智能体架构。通过 stdio 传输连接四个 MCP 工具服务器（GitHub, npm, OSV.dev, StackOverflow），所有工具返回的数据都累积在同一个上下文窗口中。
*   **架构 B (A2A Multi-Agent)**：多智能体架构。由一个协调智能体（Coordinator）通过 HTTP/JSON-RPC 将子任务委派给四个专家智能体，每个专家智能体维护独立的上下文。
*   **架构 C (Hybrid)**：混合架构。引入一个轻量级的启发式路由层，根据查询关键词预判复杂度。简单查询走 MCP，复杂查询走 A2A。
