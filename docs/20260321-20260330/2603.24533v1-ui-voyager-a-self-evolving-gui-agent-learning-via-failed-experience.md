## 摘要
随着多模态大语言模型（MLLMs）的发展，自主移动 GUI 智能体引起了越来越多的关注。然而，现有方法在长程 GUI 任务中仍面临从失败轨迹中学习效率低下以及在稀疏奖励下信用分配模糊的问题。为此，我们提出了 UI-Voyager，一种新型的两阶段自进化移动 GUI 智能体。在第一阶段，我们采用了拒绝微调（RFT），实现了数据和模型在完全自主循环中的持续协同进化。第二阶段引入了群体相对自蒸馏（GRSD），该方法识别群体展开（rollouts）中的关键分叉点，并从成功轨迹中构建密集的步骤级监督，以纠正失败的轨迹。在 AndroidWorld 上的广泛实验表明，我们的 4B 模型实现了 81.0% 的 Pass@1 成功率，优于众多近期的基准模型，并超过了人类水平的表现。消融实验和案例研究进一步验证了 GRSD 的有效性。我们的方法在无需昂贵的人工数据标注的情况下，向高效、自进化且高性能的移动 GUI 自动化迈出了重要一步。

## Abstract
Autonomous mobile GUI agents have attracted increasing attention along with the advancement of Multimodal Large Language Models (MLLMs). However, existing methods still suffer from inefficient learning from failed trajectories and ambiguous credit assignment under sparse rewards for long-horizon GUI tasks. To that end, we propose UI-Voyager, a novel two-stage self-evolving mobile GUI agent. In the first stage, we employ Rejection Fine-Tuning (RFT), which enables the continuous co-evolution of data and models in a fully autonomous loop. The second stage introduces Group Relative Self-Distillation (GRSD), which identifies critical fork points in group rollouts and constructs dense step-level supervision from successful trajectories to correct failed ones. Extensive experiments on AndroidWorld show that our 4B model achieves an 81.0% Pass@1 success rate, outperforming numerous recent baselines and exceeding human-level performance. Ablation and case studies further verify the effectiveness of GRSD. Our method represents a significant leap toward efficient, self-evolving, and high-performance mobile GUI automation without expensive manual data annotation.

### 1. 核心问题与整体含义（研究动机和背景）
随着多模态大语言模型（MLLM）的发展，自主移动 GUI 智能体（Agent）成为研究热点。然而，现有方法在处理复杂的长程（Long-horizon）移动端任务时面临两大挑战：
*   **失败经验利用率低**：在困难任务中，智能体会产生大量失败轨迹，这些数据在传统训练中往往被直接丢弃，导致数据效率低下。
*   **信用分配（Credit Assignment）模糊**：GUI 任务通常只提供稀疏的轨迹级奖励（成功或失败）。在长达数十步的操作中，智能体难以判断究竟是哪一步操作导致了最终的失败，导致强化学习（RL）优化困难。

**UI-Voyager** 的核心意义在于提出了一种**无需人工标注**的自进化框架，通过挖掘失败轨迹中的“关键转折点”，利用成功的经验来精准纠正错误决策。

### 2. 方法论
UI-Voyager 采用两阶段自进化优化流水线：

*   **第一阶段：拒绝微调（Rejection Fine-Tuning, RFT）**
    *   **核心思想**：通过迭代的“生成-过滤-微调”循环提升模型基础能力。
    *   **流程**：利用种子任务生成器合成新任务 -> 模型生成多条轨迹 -> 规则验证器（基于 ADB 状态检查）过滤出成功轨迹 -> 对成功轨迹进行有监督微调（SFT） -> 更新模型并进入下一轮。
*   **第二阶段：群体相对自蒸馏（Group Relative Self-Distillation, GRSD）**
    *   **核心思想**：在同一任务的多个尝试（Group Rollouts）中，识别失败轨迹与成功轨迹的“分叉点”，并进行步骤级的自我修正。
    *   **关键技术：分叉点检测（Fork Point Detection）**：
        *   使用 **SSIM（结构相似性指数）** 比较失败轨迹与成功轨迹的屏幕截图。
        *   当发现失败轨迹在某一步 $j$ 的状态与成功轨迹某步 $i$ 的状态高度相似（SSIM 达标），但后续动作导致结果背离时，该点即为“分叉点”。
    *   **自蒸馏过程**：将成功轨迹在分叉点处的正确动作作为标签，替换失败轨迹中的错误动作，构建密集的步骤级监督样本进行训练。这种方法规避了传统 RL（如 PPO/GRPO）在稀疏奖励下的不稳定性。
