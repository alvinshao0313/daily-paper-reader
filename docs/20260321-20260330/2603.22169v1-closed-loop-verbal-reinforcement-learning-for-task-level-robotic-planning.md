## 摘要
我们提出了一种新的言语强化学习（VRL）框架，用于在执行不确定性下运行的移动机器人系统中进行可解释的任务级规划。该框架采用闭环架构，通过与物理环境的交互实现策略的迭代改进。在我们的框架中，可执行的行为树由大语言模型（LLM）执行器（actor）进行重复优化，该执行器使用由视觉语言模型（VLM）评论器（critic）生成的结构化自然语言反馈，而该评论器负责观察物理机器人和执行轨迹。与传统的强化学习不同，VRL 中的策略更新直接发生在符号规划层面，无需基于梯度的优化。这实现了透明的推理、显式的因果反馈以及人类可解释的策略演化。我们在一个真实的移动机器人上验证了所提出的框架，该机器人执行在执行不确定性下的多阶段操作和导航任务。实验结果表明，该框架支持可解释的策略改进、对执行失败的闭环自适应以及在物理机器人系统上的可靠部署。

## Abstract
We propose a new Verbal Reinforcement Learning (VRL) framework for interpretable task-level planning in mobile robotic systems operating under execution uncertainty. The framework follows a closed-loop architecture that enables iterative policy improvement through interaction with the physical environment. In our framework, executable Behavior Trees are repeatedly refined by a Large Language Model actor using structured natural-language feedback produced by a Vision-Language Model critic that observes the physical robot and execution traces. Unlike conventional reinforcement learning, policy updates in VRL occur directly at the symbolic planning level, without gradient-based optimization. This enables transparent reasoning, explicit causal feedback, and human-interpretable policy evolution. We validate the proposed framework on a real mobile robot performing a multi-stage manipulation and navigation task under execution uncertainty. Experimental results show that the framework supports explainable policy improvements, closed-loop adaptation to execution failures, and reliable deployment on physical robotic systems.

在现实世界中，移动机器人面临显著的**执行不确定性**（如定位误差、控制失效、环境变化）。传统的任务级规划（如行为树 BT）虽然具有模块化和透明性，但通常需要大量人工调优来应对失败。虽然强化学习（RL）能自主改进策略，但其优化过程通常在亚符号空间进行，缺乏可解释性。
本文旨在解决**如何利用大模型实现既具备自主改进能力、又保持高度可解释性的机器人任务规划**。研究动机是将大语言模型（LLM）的推理能力与视觉语言模型（VLM）的感知反馈相结合，构建一个无需梯度下降、直接在符号层面（行为树）进行迭代优化的闭环系统。

#### 2. 方法论：核心思想与算法流程
论文提出了 **言语强化学习（Verbal Reinforcement Learning, VRL）** 框架，其核心思想是“以自然语言作为反馈信号进行策略演化”：
*   **策略表示**：使用**行为树（Behavior Tree, BT）**作为任务策略。BT 结构清晰、可直接执行且易于被 LLM 修改。
*   **LLM 执行器 (Actor)**：负责修改 BT 的结构和参数。它接收任务定义、环境规范、节点库、当前 BT 以及来自评论家的反馈，生成更新后的 BT 代码。
*   **VLM 评论家 (Critic)**：观察机器人执行过程中的视觉快照和 BT 执行轨迹。它不仅提供自然语言形式的诊断，还输出两个标量：**警报分数 (Alarm Score)**（衡量问题的严重性）和**置信度 (Confidence)**。
*   **闭环流程 (Algorithm 1)**：
    1.  执行当前 BT。
    2.  VLM 评论家在执行前、中、后阶段进行观察，识别故障原因（如方块放反、位置偏移）。
    3.  如果警报分数过高，通知人工干预。
    4.  LLM 执行器根据评论家的反馈和实际得分（RealScore）更新 BT。
    5.  进入下一轮迭代（Episode）。

*   **感知瓶颈**：引入符号化感知信息（BlockInfo）显著提升了学习速度和稳定性，表明 VRL 的主要瓶颈在于视觉感知的准确性，而非符号规划能力。
*   **协同进化**：执行器会隐式地适应评论家的局限性（例如，当评论家无法可靠识别颜色时，执行器会选择更保守的策略）。

#### 7. 优点
*   **高度可解释性**：策略更新是人类可读的代码（BT），失败诊断是自然语言，整个进化过程透明可查。
*   **无需模拟器或梯度**：直接在物理机器人上通过少量交互实现策略改进，避开了复杂的奖励函数设计和“模拟到现实”的鸿沟。
*   **安全性**：引入了人工警报机制，结合警报分数和置信度，使系统在不确定时能主动寻求人类帮助。

#### 8. 不足与局限
*   **时间窗口限制**：评论家基于快照观察，难以处理具有长程依赖关系的潜在故障（即错误在几步之后才显现）。
*   **感知依赖风险**：系统性能高度依赖 VLM 的视觉接地（Grounding）能力。如果评论家产生错误的因果假设，执行器可能会采取保守甚至错误的修正。
*   **效率问题**：虽然比传统 RL 快，但对于极其复杂的任务，纯符号层面的迭代可能仍需较多轮次才能达到最优。
