## 摘要
大规模视觉语言模型（LVLMs）在需要多模态交互的一系列下游任务中取得了显著成功，但其强大的能力也带来了巨大的计算和内存开销，阻碍了实际部署。在众多加速技术中，训练后量化（PTQ）是降低内存成本和加速推理的一种流行且有效的策略。然而，现有的 LVLM 量化方法通常在模态级别衡量 token 敏感度，这无法捕捉复杂的跨 token 交互，也无法在 token 级别定量衡量量化误差。随着 token 在模型内部进行交互，模态之间的区别逐渐减弱，这表明需要进行细粒度校准。受机械可解释性中公理化归因的启发，我们提出了一种基于量化感知积分梯度（QIG）的细粒度量化策略，该策略利用积分梯度定量评估 token 敏感度，并将粒度从模态级别推向 token 级别，从而反映模态间和模态内的动态变化。在 W4A8 和 W3A16 设置下对多个 LVLM 进行的大量实验表明，我们的方法在几乎不增加延迟开销的情况下，提高了不同模型和基准测试的准确率。例如，在 3-bit 仅权重（weight-only）量化下，我们的方法将 LLaVA-onevision-7B 的平均准确率提高了 1.60%，将其与全精度模型的差距缩小至仅 1.33%。代码可在 https://github.com/ucas-xiang/QIG 获取。

## Abstract
Large Vision Language Models (LVLMs) have achieved remarkable success in a range of downstream tasks that require multimodal interaction, but their capabilities come with substantial computational and memory overhead, which hinders practical deployment. Among numerous acceleration techniques, post-training quantization is a popular and effective strategy for reducing memory cost and accelerating inference. However, existing LVLM quantization methods typically measure token sensitivity at the modality level, which fails to capture the complex cross-token interactions and falls short in quantitatively measuring the quantization error at the token level. As tokens interact within the model, the distinction between modalities gradually diminishes, suggesting the need for fine-grained calibration. Inspired by axiomatic attribution in mechanistic interpretability, we introduce a fine-grained quantization strategy on Quantization-aware Integrated Gradients (QIG), which leverages integrated gradients to quantitatively evaluate token sensitivity and push the granularity from modality level to token level, reflecting both inter-modality and intra-modality dynamics. Extensive experiments on multiple LVLMs under both W4A8 and W3A16 settings show that our method improves accuracy across models and benchmarks with negligible latency overhead. For example, under 3-bit weight-only quantization, our method improves the average accuracy of LLaVA-onevision-7B by 1.60%, reducing the gap to its full-precision counterpart to only 1.33%. The code is available at https://github.com/ucas-xiang/QIG.

### 论文总结：基于量化感知积分梯度的 LVLM 细粒度训练后量化

#### 1. 核心问题与整体含义（研究动机和背景）
随着大规模视觉语言模型（LVLMs）规模的不断扩大，其巨大的计算和内存开销成为实际部署的瓶颈。**训练后量化（PTQ）**是解决该问题的有效手段。然而，现有的 LVLM 量化方法（如 MBQ）主要在**模态级别**（如区分视觉标记和文本标记）衡量量化敏感度。
*   **研究动机**：作者发现，在模型深层，模态间的界限变得模糊，且同一模态内部的不同标记（token）对量化误差的贡献差异巨大（即存在“标记变异性”）。
*   **核心问题**：模态级的粗粒度量化无法捕捉复杂的跨标记交互，导致量化后的性能损失较大。因此，需要一种能够定量衡量**标记级别（Token-level）**敏感度的细粒度量化策略。

#### 2. 核心方法论
论文提出了 **量化感知积分梯度（Quantization-aware Integrated Gradients, QIG）**，将机械可解释性中的归因理论引入量化领域。

*   **核心思想**：通过计算从“量化参考输入”到“原始输入”路径上的积分梯度，定量评估每个 token 对最终模型量化误差的贡献。
*   **关键技术细节**：
    1.  **QIG 定义**：不同于传统的积分梯度（解释预测结果），QIG 解释的是**全精度模型与量化模型之间的输出差异**。它通过在量化基准 $x_q$ 和原始输入 $x$ 之间进行线性插值并累积梯度来计算。
    2.  **IQR 裁剪（IQR-based Clipping）**：由于原始 QIG 值可能存在极端离群值，采用四分位距（IQR）方法对敏感度得分进行裁剪，以稳定量化校准过程。
    3.  **细粒度校准目标**：将计算出的 token 敏感度系数 $\lambda_i$ 引入通道等化（CWE）的优化目标中，对重构误差进行加权。公式表现为最小化加权均方误差（MSE）：
        $$\min_{E} \sum_{i=1}^{T} \lambda_i \| \text{Quantized\_Output}_i - \text{Original\_Output}_i \|^2$$
*   **流程**：输入校准数据 -> 计算 QIG 敏感度 -> IQR 裁剪与归一化 -> 执行带权重的通道等化与量化参数搜索。

#### 3. 实验设计
*   **数据集/场景**：涵盖了视觉推理、真实世界感知、图表理解等多个维度。
    *   **Benchmark**：MMMU, ScienceQA, VizWiz, ChartQA, AI2D。

    *   **LLM 量化方法**：GPTQ, AWQ, SmoothQuant (SQ)。

*   **量化设置**：W4A8（权重 4-bit，激活 8-bit）和 W3A16（权重 3-bit，仅权重）。

*   **实验规模**：在 LLaVA-onevision-7B、Qwen2-VL-7B、InternVL2-8B 和 InternVL2-26B 四个主流模型上进行了验证。

#### 6. 主要结论与发现
*   **标记级敏感度至关重要**：同一模态内的 token 敏感度差异显著，细粒度建模能有效提升量化精度。
*   **性能提升显著**：在 3-bit 权重下，LLaVA-onevision-7B 的平均准确率提升了 1.60%，与全精度模型的差距缩小至 1.33%。
*   **可扩展性强**：该方法在 26B 规模的模型上依然有效，且能作为插件提升 GPTQ 等现有框架的性能。
*   **传统方法局限性**：直接将 LLM 的量化方法（如 SQ）应用于 LVLM 效果往往不如简单的 RTN，证明了多模态特性的特殊性。

#### 7. 优点（亮点）
*   **理论创新**：首次将“公理化归因”引入 LVLM 量化，为衡量量化敏感度提供了坚实的理论基础。
*   **高效率**：在提供 token 级精度的同时，保持了极低的计算开销，具有很强的实用部署价值。
*   **即插即用**：不改变模型结构，不依赖特定算子，且能与现有的 PTQ 优化器（如 GPTQ）结合。

#### 8. 不足与局限
*   **极低比特挑战**：虽然在 3-bit 下表现优异，但在更极端的 2-bit 量化下性能仍有待观察。
*   **校准集依赖**：虽然仅需 128 个样本，但量化性能仍可能受校准数据分布的影响（尽管论文在 OCR 任务上做了鲁棒性测试）。
*   **任务覆盖**：主要集中在判别式和短文本生成任务，对于长文本多模态对话的量化影响探讨较少。
