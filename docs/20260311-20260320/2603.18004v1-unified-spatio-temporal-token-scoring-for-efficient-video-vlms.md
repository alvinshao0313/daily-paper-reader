## 摘要
Token 剪枝对于提高视觉语言模型 (VLMs) 的计算效率至关重要，特别是在时间冗余普遍存在的视频任务中。先前的方法通常要么 (1) 仅在视觉 Transformer (ViT) 内部针对动作识别和物体分割等单模态感知任务进行 Token 剪枝，而不适应下游视觉语言任务；要么 (2) 仅在大语言模型 (LLM) 内部进行剪枝，同时保持 ViT 输出完整，这通常需要复杂的文本条件 Token 选择机制。在本文中，我们引入了时空 Token 评分 (STTS)，这是一个简单且轻量级的模块，它可以在不依赖文本条件或 Token 合并的情况下，同时在 ViT 和 LLM 中剪枝视觉 Token，并且完全兼容端到端训练。通过辅助损失学习时间评分，并通过 LLM 下游梯度学习空间评分，在高效打包算法的辅助下，STTS 在整个架构中剪枝了 50% 的视觉 Token，在 13 个长短视频问答 (QA) 任务中，使训练和推理效率提高了 62%，而平均性能仅下降了 0.7%。随着每个视频采样帧数的增加，效率提升更加显著。对长视频问答应用测试时缩放 (test-time scaling) 进一步使性能比基线提高了 0.5-1%。总之，STTS 代表了一种新颖、简单且有效的技术，用于统一的、全架构范围的视觉 Token 剪枝。

## Abstract
Token pruning is essential for enhancing the computational efficiency of vision-language models (VLMs), particularly for video-based tasks where temporal redundancy is prevalent. Prior approaches typically prune tokens either (1) within the vision transformer (ViT) exclusively for unimodal perception tasks such as action recognition and object segmentation, without adapting to downstream vision-language tasks; or (2) only within the LLM while leaving the ViT output intact, often requiring complex text-conditioned token selection mechanisms. In this paper, we introduce Spatio-Temporal Token Scoring (STTS), a simple and lightweight module that prunes vision tokens across both the ViT and the LLM without text conditioning or token merging, and is fully compatible with end-to-end training. By learning how to score temporally via an auxiliary loss and spatially via LLM downstream gradients, aided by our efficient packing algorithm, STTS prunes 50% of vision tokens throughout the entire architecture, resulting in a 62% improvement in efficiency during both training and inference with only a 0.7% drop in average performance across 13 short and long video QA tasks. Efficiency gains increase with more sampled frames per video. Applying test-time scaling for long-video QA further yields performance gains of 0.5-1% compared to the baseline. Overall, STTS represents a novel, simple yet effective technique for unified, architecture-wide vision token pruning.

### 1. 核心问题与研究动机
*   **核心问题**：视频视觉语言模型（Video VLMs）在处理长视频时，由于帧数多且每帧包含大量 Patch Token，导致计算开销随序列长度呈二次方增长，严重限制了推理速度和显存效率。
*   **研究背景**：现有的 Token 剪枝方法存在局限性：
    *   **ViT 内部剪枝**：多用于单模态任务（如动作识别），未针对下游多模态推理进行优化。
    *   **LLM 内部剪枝（Post-ViT）**：虽然减少了 LLM 的负担，但 ViT 仍需处理所有原始帧，ViT 阶段的计算瓶颈依然存在。
    *   **复杂性**：许多方法依赖复杂的文本条件选择或 Token 合并算法，增加了系统复杂性。

### 2. 方法论：STTS 核心思想与技术细节
STTS（时空 Token 评分）是一个轻量级、端到端可训练的模块，旨在 ViT 和 LLM 中同步实现视觉 Token 剪枝。
*   **双轴评分机制**：
    *   **空间显著性（Spatial）**：通过将评分作为“注意力偏置（Attention Bias）”注入 ViT 层，利用下游 LLM 的梯度回传，使模型自动学会识别对推理任务关键的空间区域。
    *   **时间冗余性（Temporal）**：引入辅助损失函数（Auxiliary Loss），计算相邻帧对应 Patch 的余弦相似度。相似度越高，评分越低，从而引导模型剔除时间上的重复信息。
*   **关键技术组件**：
    *   **评分器（Scorer）**：由简单的 Token 池化层和 3 层 MLP 组成，输入当前帧与前一帧的拼接特征。
    *   **打包算法（Packing Algorithm）**：由于不同帧被剪枝的比例不同（非均匀剪枝），论文采用“首次适应递减算法（First-fit Descending）”将稀疏 Token 重新打包成紧凑的张量，确保在硬件上实现真实的计算加速。
    *   **硬剪枝（Hard Pruning）**：在 ViT 的中间层（实验中为第 3 层）直接丢弃评分最低的 $k\%$ Token。

### 3. 实验设计
*   **数据集与场景**：涵盖了 13 个视频问答（QA）基准测试，包括短视频（如 NextQA, MVBench）和长视频（如 VideoMME, MLVU, LVBench）。

    *   **Heuristic**：基于纯相似度的启发式剪枝。

### 5. 实验数量与充分性
*   **实验规模**：论文进行了多维度的详尽实验，包括：

    *   **测试时缩放（TTS）**：验证了在相同计算预算下，通过剪枝增加采样帧数是否能提升长视频理解能力。
*   **评价**：实验设计客观、公平，通过设置“100% 剪枝（纯文本推理）”作为底线，证明了 STTS 带来的性能增益确实源于有效的视觉信息保留。

### 6. 主要结论与发现
*   **高效性**：在丢弃 50% 视觉 Token 的情况下，模型在 13 个任务上的平均性能仅下降 **0.7%**，而效率提升了 **62%**。
*   **30% 剪枝“甜点位”**：在 30% 剪枝率下，部分任务性能甚至超过了基线，说明剪枝起到了“去噪”作用。
*   **长视频优势**：通过测试时缩放（TTS），在保持计算量不变的前提下增加采样帧数，长视频 QA 性能可提升 0.5-1%。
*   **层深影响**：在 ViT 极早期（如第 0 层）剪枝会显著损害性能，在第 3 层左右剪枝是效率与效果的最佳平衡点。

### 7. 优点与亮点
*   **架构统一**：首次实现了跨 ViT 和 LLM 的统一剪枝，解决了 ViT 端的计算瓶颈。
*   **无需文本条件**：不依赖复杂的文本-图像交叉注意力，降低了推理延迟和实现难度。
*   **真实加速**：通过 Packing 算法解决了稀疏张量在 GPU 上难以加速的痛点。
*   **端到端可训练**：评分器直接受下游任务目标驱动，比静态启发式方法更具语义感知能力。

### 8. 不足与局限
*   **超参数依赖**：剪枝层 $l$ 的选择需要针对不同 ViT 架构进行微调，缺乏完全自适应的机制。
*   **细粒度损失风险**：虽然在 QA 任务表现良好，但在需要极高空间分辨率的任务（如细微物体检测或复杂 OCR）中，激进的剪枝可能会导致关键细节丢失。
*   **训练成本**：虽然推理高效，但 STTS 仍需要一定规模的微调训练，并非完全的“即插即用（Plug-and-play）”方案。
