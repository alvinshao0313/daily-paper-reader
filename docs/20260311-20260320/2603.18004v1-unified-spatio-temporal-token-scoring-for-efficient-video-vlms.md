## 摘要
Token 剪枝对于提高视觉语言模型（VLM）的计算效率至关重要，特别是在时间冗余普遍存在的视频任务中。先前的方法通常要么（1）仅在视觉 Transformer（ViT）内部针对动作识别和物体分割等单模态感知任务进行 Token 剪枝，而不适应下游的视觉语言任务；要么（2）仅在大语言模型（LLM）内部进行剪枝，同时保持 ViT 输出完整，这通常需要复杂的文本条件 Token 选择机制。在本文中，我们引入了时空 Token 评分（STTS），这是一个简单且轻量级的模块，它可以在不依赖文本条件或 Token 合并的情况下，跨 ViT 和 LLM 剪枝视觉 Token，并且与端到端训练完全兼容。通过辅助损失学习时间评分，并通过 LLM 下游梯度学习空间评分，在高效打包算法的辅助下，STTS 在整个架构中剪枝了 50% 的视觉 Token，在 13 个长短视频问答（QA）任务中，训练和推理效率提高了 62%，而平均性能仅下降了 0.7%。效率提升随着每个视频采样帧数的增加而增加。对长视频问答应用测试时缩放（test-time scaling）与基线相比进一步获得了 0.5-1% 的性能提升。总之，STTS 代表了一种新颖、简单且有效的技术，用于统一的、全架构范围的视觉 Token 剪枝。

## Abstract
Token pruning is essential for enhancing the computational efficiency of vision-language models (VLMs), particularly for video-based tasks where temporal redundancy is prevalent. Prior approaches typically prune tokens either (1) within the vision transformer (ViT) exclusively for unimodal perception tasks such as action recognition and object segmentation, without adapting to downstream vision-language tasks; or (2) only within the LLM while leaving the ViT output intact, often requiring complex text-conditioned token selection mechanisms. In this paper, we introduce Spatio-Temporal Token Scoring (STTS), a simple and lightweight module that prunes vision tokens across both the ViT and the LLM without text conditioning or token merging, and is fully compatible with end-to-end training. By learning how to score temporally via an auxiliary loss and spatially via LLM downstream gradients, aided by our efficient packing algorithm, STTS prunes 50% of vision tokens throughout the entire architecture, resulting in a 62% improvement in efficiency during both training and inference with only a 0.7% drop in average performance across 13 short and long video QA tasks. Efficiency gains increase with more sampled frames per video. Applying test-time scaling for long-video QA further yields performance gains of 0.5-1% compared to the baseline. Overall, STTS represents a novel, simple yet effective technique for unified, architecture-wide vision token pruning.

以下是对该论文的结构化总结：

### 1. 核心问题与研究背景
*   **核心问题**：视频 VLM 需要处理大量视频帧，导致视觉 Token 序列极长，在 Transformer 的注意力机制下产生巨大的计算开销（平方级增长）。
*   **研究动机**：现有的剪枝方法存在局限性。要么仅在 ViT 内部针对单模态任务剪枝，不适应下游多模态推理；要么仅在 LLM 之前剪枝，导致 ViT 编码器依然满负荷运行。此外，许多方法依赖复杂的文本条件选择，增加了推理延迟。
*   **整体含义**：STTS 提出了一种跨架构（ViT + LLM）的统一剪枝方案，通过简单的评分机制同时解决空间和时间上的信息冗余。

### 2. 方法论：STTS 核心技术
*   **核心思想**：在 ViT 的早期层插入一个轻量级评分模块，通过“空间显著性”和“时间冗余度”双重维度对 Token 进行打分，并永久剔除低分 Token。
*   **关键技术细节**：
    *   **评分模块 (Scorer)**：由 Token 池化层和 3 层 MLP 组成。它接收当前帧与前一帧的特征，预测每个区域的重要性。
    *   **偏置注入 (Bias Injection)**：将预测的评分作为 Attention Bias 注入到 ViT 的下一层，使得模型能够通过下游任务的梯度回传学习空间重要性。
    *   **辅助损失 (Auxiliary Loss)**：引入基于相邻帧“余弦相似度”的 MSE 损失函数，强制评分模块识别时间上的重复信息（相似度越高，评分越低）。
    *   **打包算法 (Packing Algorithm)**：由于不同视频帧剪枝后的 Token 数量不一，论文采用 **First-Fit Descending (首次下降适配)** 算法将稀疏 Token 重新打包成致密张量，从而在硬件层面实现真实的计算加速。

### 3. 实验设计
*   **数据集/场景**：涵盖了 13 个视频问答（QA）基准测试，包括：
    *   **短视频**：NextQA, Perception-Test, MVBench, Tomato, MotionBench, Temp-Compass。
    *   **长视频**：VideoMME, LongVideoBench, MLVU, LVBench, VideoEvalPro。
*   **对比方法**：
    *   **SOTA 模型**：Qwen3-VL-4B, InternVL3.5-8B, PLM-8B, 以及基线模型 Molmo2。
    *   **剪枝策略**：随机剪枝 (Random)、基于相似度的启发式剪枝 (Heuristic)、Token 合并 (ToMe)。

### 4. 资源与算力
*   **硬件设备**：实验在单节点 **8 张 NVIDIA H100 GPU** 上进行。

*   **效率数据**：在 256 帧的极端设置下，50% 的剪枝率带来了 **2.25 倍的训练加速** 和 **2.22 倍的推理加速**。

### 5. 实验数量与充分性
*   **实验规模**：论文进行了极其详尽的实验，包括：
    *   不同剪枝比例（30% 到 90%）的性能退化曲线分析。
    *   **消融实验**：验证了辅助损失的必要性、剪枝插入层深度（$l=0,1,2,3$）的影响。
    *   **测试时缩放 (TTS)**：验证了在相同计算预算下，通过剪枝换取更多采样帧数带来的性能增益。
*   **客观性**：通过设置 $k=100\%$（完全不看视频，仅靠语言模型盲猜）作为底线，客观展示了剪枝对视觉理解的真实保留程度。实验覆盖了长短视频、图像 QA 等多个维度，具有较强的说服力。

### 6. 主要结论与发现
*   **高效性**：在剪枝 50% 视觉 Token 的情况下，模型在 13 个任务上的平均性能仅下降 **0.7%**，而整体效率提升了 **62%**。
*   **长视频突破**：在长视频任务中，利用剪枝节省的算力采样更多帧（如从 64 帧增加到 128 帧），性能反而比未剪枝的基线高出 **0.5-1%**。
*   **层深影响**：剪枝不宜过早。在 ViT 第 3 层之后剪枝效果最佳，过早（如第 0 层）会导致视觉特征尚未充分建模就被破坏。

### 7. 优点与亮点
*   **架构统一**：首次实现了在 ViT 内部剪枝并直接惠及后续 LLM 的端到端流程。
*   **简单且无需文本**：不依赖复杂的文本-视觉交叉注意力，降低了系统复杂度和推理延迟。
*   **真实加速**：通过 Packing 算法解决了深度学习框架处理变长序列的效率问题，而非仅仅是理论上的 FLOPs 减少。

### 8. 不足与局限
*   **架构局限**：实验主要基于 Molmo2 (SigLIP + Qwen3) 架构，虽然理论上通用，但在其他类型的视觉编码器（如卷积网络或不同变体的 ViT）上的泛化性有待验证。
*   **静态剪枝率**：目前每个视频使用固定的剪枝比例 $k$，未能实现根据视频内容的动态复杂程度（如动作剧烈程度）自适应调整剪枝率。
*   **图像任务潜在风险**：虽然附录显示图像任务性能未降，但大规模剪枝在极高精度的视觉定位任务中可能仍存在细微的信息损失。
