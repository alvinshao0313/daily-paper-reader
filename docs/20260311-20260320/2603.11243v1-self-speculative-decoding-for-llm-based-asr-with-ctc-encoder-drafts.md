## 摘要
我们提出了一种针对语音感知大语言模型（LLM）的自投机解码方法，通过将 CTC 编码器作为草稿模型来加速自回归（AR）推理并提高 ASR 准确率。我们的三步流程如下：（1）如果 CTC 输出分布的帧熵低于阈值，则接受贪婪 CTC 假设作为最终结果；（2）否则，在单次 LLM 前向传递中，使用基于 token 似然度的松弛接受标准对 CTC 假设进行验证；（3）如果验证失败，则从已接受的 CTC 前缀开始恢复自回归解码。在 9 个语料库和 5 种语言上的实验表明，该方法可以同时加速解码并降低词错误率（WER）。在 HuggingFace Open ASR 基准测试中，使用 10 亿参数的 LLM 和 4.4 亿参数的 CTC 编码器，我们实现了创纪录的 5.58% WER，并将逆实时因子提高了 4.4 倍，而相比自回归搜索，WER 仅相对增加了 12%。代码和模型权重已在宽松许可下公开。

## Abstract
We propose self-speculative decoding for speech-aware LLMs by using the CTC encoder as a draft model to accelerate auto-regressive (AR) inference and improve ASR accuracy. Our three-step procedure works as follows: (1) if the frame entropies of the CTC output distributions are below a threshold, the greedy CTC hypothesis is accepted as final; (2) otherwise, the CTC hypothesis is verified in a single LLM forward pass using a relaxed acceptance criterion based on token likelihoods; (3) if verification fails, AR decoding resumes from the accepted CTC prefix. Experiments on nine corpora and five languages show that this approach can simultaneously accelerate decoding and reduce WER. On the HuggingFace Open ASR benchmark with a 1B parameter LLM and 440M parameter CTC encoder, we achieve a record 5.58% WER and improve the inverse real time factor by a factor of 4.4 with only a 12% relative WER increase over AR search. Code and model weights are publicly available under a permissive license.

*   **核心问题**：基于大语言模型（LLM）的语音识别系统（SLM）虽然准确率极高，但其**自回归（AR）推理机制**导致速度缓慢，每生成一个 token 都需要经过庞大的 LLM 前向计算，限制了并行处理能力。
*   **研究背景**：现有的投机解码（Speculative Decoding）通常需要额外的轻量级草稿模型。本文旨在利用 SLM 架构中现有的 **CTC 编码器**作为天然的“草稿模型”，在不增加额外模型负担的情况下，实现推理加速并提升识别精度。

### 2. 方法论：核心思想与技术细节
论文提出了一种名为**自投机解码（Self-Speculative Decoding, SSD）**的三步走策略：
1.  **CTC 解码与初步验证**：
    *   利用编码器自带的 CTC 头进行贪婪解码，得到初步假设。
    *   计算 CTC 输出分布的**帧熵（Frame-level Entropy）**。如果所有帧的熵均低于设定阈值 $\tau_{CTC}$（表示模型非常自信），则直接接受该结果，跳过 LLM 处理。
2.  **LLM 并行验证**：
    *   若初步验证未通过，将 CTC 假设输入 LLM。
    *   利用 LLM 的因果掩码特性，通过**单次前向传递**并行计算所有 token 的似然度。
    *   如果所有 token 的似然度均高于阈值 $\tau_{SLM}$，则接受该 CTC 假设。
3.  **自回归回退（Fallback）**：
    *   若 LLM 验证失败，找到最长且通过验证的 CTC 前缀，从该位置开始恢复标准的自回归解码。
*   **关键公式/逻辑**：最终输出 $y_f$ 取决于熵验证或似然验证的通过情况；若均失败，则采用“已验证前缀 + AR 生成”的组合。

### 6. 主要结论与发现
*   **速度大幅提升**：在 Open ASR 基准上，相比全 AR 解码，该方法实现了 **4.4 倍**的逆实时因子（RTFx）提升。

### 7. 优点
*   **无需额外模型**：巧妙复用了 SLM 架构中已有的 CTC 编码器，实现了“零成本”草稿生成。

*   **开源友好**：模型权重和代码以许可协议公开，具有很高的实用价值。

### 8. 不足与局限
*   **任务局限性**：该方法目前仅适用于语音识别（ASR），无法直接应用于翻译或口语问答等需要大幅改变输出结构的语音任务。
*   **回退开销**：验证失败时需要从失败点重新进行 AR 解码，对于长句或 CTC 质量极差的语料，加速效果会显著打折。
*   **架构依赖**：必须要求 SLM 拥有一个经过 CTC 训练且冻结的编码器，对于纯 AED 架构或其他非 CTC 架构的 SLM 不适用。
