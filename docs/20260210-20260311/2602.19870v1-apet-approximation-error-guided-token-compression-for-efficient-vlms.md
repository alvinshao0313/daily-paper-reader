## 摘要
近期的视觉语言模型（VLMs）展示了卓越的多模态理解能力，但冗余的视觉 token 带来了极高的计算开销并降低了推理效率。先前的研究通常依赖 [CLS] 注意力或文本-视觉交叉注意力来识别并丢弃冗余的视觉 token。尽管取得了不错的效果，但此类方案容易引入位置偏差，更关键的是，它们与 FlashAttention 等高效注意力算子不兼容，限制了其在 VLM 加速方面的实际部署。在本文中，我们摆脱了对注意力的依赖，从信息论的角度重新审视视觉 token 压缩，旨在不涉及任何注意力机制的情况下最大限度地保留视觉信息。我们提出了 ApET，一种基于近似误差引导的 Token 压缩框架。ApET 首先通过线性近似，利用一小组基准 token（basis tokens）重构原始视觉 token，然后利用近似误差来识别并丢弃信息量最少的 token。在多个 VLM 和基准测试上的广泛实验表明，ApET 在图像理解任务上保留了 95.2% 的原始性能，在视频理解任务上甚至达到了 100.4%，同时分别压缩了 88.9% 和 87.5% 的 token 预算。得益于其无注意力（attention-free）的设计，ApET 可以与 FlashAttention 无缝集成，从而实现进一步的推理加速，使 VLM 的部署更具实用性。代码可在 https://github.com/MaQianKun0/ApET 获取。

## Abstract
Recent Vision-Language Models (VLMs) have demonstrated remarkable multimodal understanding capabilities, yet the redundant visual tokens incur prohibitive computational overhead and degrade inference efficiency. Prior studies typically relies on [CLS] attention or text-vision cross-attention to identify and discard redundant visual tokens. Despite promising results, such solutions are prone to introduce positional bias and, more critically, are incompatible with efficient attention kernels such as FlashAttention, limiting their practical deployment for VLM acceleration. In this paper, we step away from attention dependencies and revisit visual token compression from an information-theoretic perspective, aiming to maximally preserve visual information without any attention involvement. We present ApET, an Approximation-Error guided Token compression framework. ApET first reconstructs the original visual tokens with a small set of basis tokens via linear approximation, then leverages the approximation error to identify and drop the least informative tokens. Extensive experiments across multiple VLMs and benchmarks demonstrate that ApET retains 95.2% of the original performance on image-understanding tasks and even attains 100.4% on video-understanding tasks, while compressing the token budgets by 88.9% and 87.5%, respectively. Thanks to its attention-free design, ApET seamlessly integrates with FlashAttention, enabling further inference acceleration and making VLM deployment more practical. Code is available at https://github.com/MaQianKun0/ApET.

### 1. 核心问题与研究动机
*   **核心问题**：视觉语言模型（VLM）在处理高分辨率图像或长视频时，会产生海量的视觉 Token，导致计算开销呈平方级增长，推理效率低下。
*   **研究背景**：现有的 Token 压缩方法（如基于注意力权重的剪枝或合并）存在两大痛点：
    1.  **位置偏差（Positional Bias）**：LLM 的注意力机制往往倾向于关注序列后端的 Token，导致重要的视觉信息因位置靠前而被误删。
    2.  **硬件不兼容性**：这些方法依赖注意力权重，而高效的算子（如 FlashAttention）为了加速并不显式存储权重，导致压缩算法与硬件加速手段无法同时使用。
*   **研究目标**：开发一种不依赖注意力机制、能消除位置偏差且兼容 FlashAttention 的高效 Token 压缩框架。

### 2. 核心方法论
ApET 提出从**信息论**角度评估 Token 的重要性，核心思想是：**如果一个 Token 可以被其他 Token 线性重构，则其包含的独特信息较少，属于冗余信息。**

*   **理论基础**：通过数学证明，最小重构均方误差（MSE）是条件熵的下界。最小化重构误差等同于最大化保留的信息量。
*   **算法流程**：
    1.  **Token 选择（Token Selection）**：从原始视觉 Token 集合中通过最远点采样（FPS）等策略选出一小部分作为“基准 Token”（Basis Tokens）。
    2.  **线性近似（Linear Approximation）**：利用基准 Token 集合，通过求解线性方程组来尝试重构所有的原始 Token。
    3.  **误差计算（Error Computation）**：计算每个原始 Token 与其重构版本之间的欧氏距离（即近似误差 $\xi$）。误差越大，说明该 Token 越不可替代，信息量越高。
    4.  **Token 合并（Token Merging）**：保留误差最大的 Token 和基准 Token，将误差较小的冗余 Token 合并到与其最相似的保留 Token 中，以减少信息损失。
*   **集成方式**：该方法是**无训练（Training-free）**的，可直接插入 VLM 的视觉编码器输出端或 LLM 的中间层。

*   **实验规模**：涵盖了从 7B 到更先进的动态分辨率模型，测试了图像和视频两大领域，实验组数非常多。
*   **消融实验**：针对采样策略（随机、DPC、FPS）、超参数 $M$（基准 Token 数量）进行了详细测试。
*   **客观性**：通过在不同压缩率（如保留 192, 128, 64 个 Token）下进行对比，证明了 ApET 在高压缩比下的领先优势。实验设计考虑了与 FlashAttention 的结合，对比了实际推理耗时而非仅仅是理论 FLOPs，具有很强的实战参考价值。

### 6. 主要结论与发现
*   **高性能保留**：在图像任务中，压缩 88.9% 的 Token 后仍保留了 95.2% 的性能；在视频任务中，压缩 87.5% 的 Token 后性能反而提升至 100.4%（说明压缩起到了去噪作用）。
*   **消除偏差**：可视化证明 ApET 能有效保留图像中关键但处于序列前端的物体，避免了注意力机制带来的位置偏见。
*   **硬件友好**：ApET 是目前少数能与 FlashAttention 无缝结合并产生实际加速效果的 Token 压缩方案。

### 7. 亮点
*   **视角新颖**：跳出了“注意力权重即重要性”的思维定式，从线性重构和信息论角度解决问题。
*   **兼容性极强**：不改变模型结构，不依赖特定算子输出，适用于各种 Transformer 架构的 VLM。
*   **视频去噪效果**：发现 Token 压缩在视频理解中不仅能加速，还能通过移除冗余帧/块来提升模型对核心动作的关注。

### 8. 不足与局限
*   **线性假设**：使用线性近似来衡量信息量虽然高效，但可能无法完全捕捉深度特征空间中复杂的非线性语义冗余。
*   **超参数依赖**：虽然实验证明 $M$ 的敏感度较低，但在不同分辨率或不同任务下，可能仍需微调基准 Token 的数量以达到最优平衡。
*   **极端场景**：对于极度依赖微小细节的 OCR 或精密检测任务，80% 以上的压缩率可能仍会带来不可忽视的精度下降。
