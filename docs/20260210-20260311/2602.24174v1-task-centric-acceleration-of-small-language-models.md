## 摘要
小语言模型 (SLM) 已成为针对特定任务应用的大语言模型的高效替代方案。然而，它们通常被应用于高吞吐量、低延迟的场景，在这些场景中效率至关重要。我们提出了 TASC（任务自适应序列压缩），这是一个用于 SLM 加速的框架，包含两个用例：在进行 SLM 微调时，我们提出了 TASC-ft，它通过高频输出 n-gram 迭代地丰富分词器词表，然后微调模型以利用扩展后的词表。接下来，我们提出了一种推理阶段的方法，称为 TASC-spec。TASC-spec 是一种轻量级、无需训练的投机解码方法，它从任务的输出语料库中构建 n-gram 草稿模型，并融合了任务和上下文的 n-gram 信息。TASC-spec 避免了任何额外的训练，同时绕过了草稿模型与目标模型之间的词表对齐限制。我们在多个低输出变异性的生成任务中证明了这两种方法的有效性。我们的方法在保持任务性能的同时，在推理效率方面表现出持续的提升。

## Abstract
Small language models (SLMs) have emerged as efficient alternatives to large language models for task-specific applications. However, they are often employed in high-volume, low-latency settings, where efficiency is crucial. We propose TASC, Task-Adaptive Sequence Compression, a framework for SLM acceleration comprising two use-cases: When performing SLM fine-tuning, we propose TASC-ft, which iteratively enriches the tokenizer vocabulary with high-frequency output n-grams and then fine-tunes the model to utilize the expanded vocabulary. Next, we propose an inference-time method, termed TASC-spec. TASC-spec is a lightweight, training-free speculative decoding method that constructs an n-gram draft model from the task's output corpus, mixing task and context n-gram information.TASC-spec avoids any additional training, while bypassing draft-target vocabulary alignment constraints. We demonstrate the effectiveness of both methods across multiple low output-variability generation tasks. Our methods show consistent improvements in inference efficiency while maintaining task performance.

### 1. 核心问题与整体含义（研究动机和背景）
论文关注的是**小语言模型（SLM）**在特定任务场景下的推理效率问题。虽然 SLM 相比大模型（LLM）更轻量，但在高吞吐、低延迟的实际应用（如端侧部署、多智能体系统）中，自回归生成的逐 token 步数仍是主要瓶颈。
作者发现，特定任务（如医疗 QA、法律分类）的输出文本往往具有**低变异性**（Low Output Variability），即输出中存在大量高频重复的 n-gram（短语）。基于这一观察，论文提出了 **TASC 框架**，旨在通过“任务自适应序列压缩”来减少生成步数，从而实现加速。

### 2. 方法论：核心思想与关键技术
TASC 框架包含两个针对不同用例的子方法：

*   **TASC-ft（微调阶段加速）：**
    *   **核心思想：** 扩展分词器（Tokenizer）词表，将任务中高频出现的 n-gram 合并为单个新 token，从而缩短序列长度。
    *   **关键技术：**
        *   **合并奖励（Merge Reward）：** 计算 $f_{merge}(t^n) = \text{freq}(t^n) \cdot (n-1)$，优先合并能最大程度减少总 token 数的 n-gram。
        *   **前缀冲突得分（PCS）：** 为了防止新 token 干扰贪婪解码（例如合并了 "in the" 可能导致模型难以生成以 "the" 开头的其他词），引入 PCS 阈值过滤掉可能引起歧义的合并。
        *   **嵌入初始化：** 新 token 的嵌入向量初始化为其组成 token 嵌入的均值，随后通过 QLoRA 进行微调。
*   **TASC-spec（推理阶段加速）：**
    *   **核心思想：** 一种无需训练的**投机解码（Speculative Decoding）**方法。
    *   **关键技术：**
        *   **混合 n-gram 草稿模型：** 结合了从全局任务语料库提取的 n-gram 统计信息（$p_{corp}$）和从当前 Prompt 上下文中提取的局部信息（$p_{prompt}$）。
        *   **词表无关性：** 与传统的投机解码不同，TASC-spec 不需要草稿模型与目标模型共享词表，它直接在 token 级别操作，具有更强的通用性。

### 3. 实验设计
*   **数据集：**
    *   **EUR-LEX：** 法律政策大规模多标签分类。
    *   **Asclepius：** 医疗领域阅读理解与 QA。
    *   **MASSIVE：** 意图分类与槽位填充（ICSF）。
    *   **MedQuaD：** 医疗开放域 QA。
*   **Benchmark 与对比方法：**

*   **评估指标：** 推理加速比（Speedup）、任务特定指标（F1、BERT-score、联合准确率）、LLM Judge 评分。

### 5. 实验数量与充分性
*   **实验规模：** 作者进行了 **124 组模拟实验**，涵盖了 8 种不同的配置（不同模型大小、数据集、n-gram 长度等）。
*   **消融实验：** 针对新增 token 数量、训练轮数、投机解码的混合参数 $\lambda$ 进行了详细的消融研究。
*   **客观性：** 实验不仅关注速度，还引入了 **LLM Judge（使用 Claude-4.5-Sonnet）** 来评估生成质量，确保加速没有以牺牲性能为代价。实验设计较为客观，考虑了批处理（Batch size=8）等实际生产环境因素。

### 6. 主要结论与发现
*   **加速效果显著：** TASC-ft 在保持性能的同时实现了高达 **2.1 倍** 的运行时加速；TASC-spec 在无需训练的情况下实现了高达 **3.15 倍** 的加速。
*   **性能保持：** 扩展词表后的模型在任务指标（如 F1）上甚至略有提升，这可能归功于更高效的表示。
*   **可预测性：** 论文发现 token 分布的 **2-Rényi 熵** 是预测 TASC-ft 加速潜力的可靠指标（相关性 > 0.9）。
*   **UID 假设验证：** 实验证明 TASC-ft 使 token 的信息分布更均匀，符合统一信息密度（UID）假设。

### 7. 优点
*   **任务驱动的针对性：** 敏锐捕捉到特定任务输出变异性低的特点，将“领域自适应”细化为“任务自适应”。
*   **TASC-spec 的灵活性：** 无需额外训练模型，且解决了投机解码中常见的词表对齐难题。
*   **理论与实践结合：** 利用信息论（典型集、熵）为启发式算法提供了坚实的理论支撑。

### 8. 不足与局限
*   **依赖代表性数据：** 方法的效果高度依赖于训练语料库是否能准确反映测试时的输出分布。
*   **通用性受损风险：** 针对特定任务优化词表可能会降低模型在其他通用任务上的表现（尽管论文认为这在 SLM 专用化场景中是可以接受的）。
*   **任务范围限制：** 对于高变异性、开放式生成任务（如创意写作），该方法的加速效果会显著下降。
*   **Token 预算权衡：** 增加过多 token 会增加训练参数量，存在边际收益递减问题。
