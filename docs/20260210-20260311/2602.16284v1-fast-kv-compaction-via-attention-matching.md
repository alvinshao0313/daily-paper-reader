## 摘要
将语言模型扩展到长上下文通常受限于键值（KV）缓存的大小。在部署场景中，长上下文通常通过摘要在令牌空间进行压缩来管理。然而，摘要可能是高度有损的，会显著损害下游性能。最近关于 Cartridges 的研究表明，在潜空间中训练高度压缩的 KV 缓存是可能的，其性能与全上下文相当，但代价是缓慢且昂贵的端到端优化。本研究描述了一种通过注意力匹配（Attention Matching）在潜空间中进行快速上下文压缩的方法，该方法构建压缩的键和值，以在每个 KV 头级别重现注意力输出并保留注意力权重。我们展示了这种公式化方法可以自然地分解为简单的子问题，其中一些子问题具有高效的闭式解。在此框架内，我们开发了一系列方法，显著提升了压缩时间与质量之间的帕累托前沿，在某些数据集上仅需数秒即可实现高达 50 倍的压缩，且质量损失极小。

## Abstract
Scaling language models to long contexts is often bottlenecked by the size of the key-value (KV) cache. In deployed settings, long contexts are typically managed through compaction in token space via summarization. However, summarization can be highly lossy, substantially harming downstream performance. Recent work on Cartridges has shown that it is possible to train highly compact KV caches in latent space that closely match full-context performance, but at the cost of slow and expensive end-to-end optimization. This work describes an approach for fast context compaction in latent space through Attention Matching, which constructs compact keys and values to reproduce attention outputs and preserve attention mass at a per-KV-head level. We show that this formulation naturally decomposes into simple subproblems, some of which admit efficient closed-form solutions. Within this framework, we develop a family of methods that significantly push the Pareto frontier of compaction time versus quality, achieving up to 50x compaction in seconds on some datasets with little quality loss.

*   **核心目标**：开发一种在潜空间中既能保持高压缩比性能，又能实现秒级处理速度的 KV 压缩技术。

### 2. 方法论：注意力匹配（Attention Matching）
*   **核心思想**：将 KV 压缩视为一个数学优化问题，即寻找一组紧凑的键（$C_k$）、值（$C_v$）和标量偏置（$\beta$），使得它们在参考查询（Reference Queries）下的**注意力输出**和**注意力质量（Mass）**尽可能接近原始 KV。
*   **关键技术细节**：
    1.  **参考查询采样**：通过“上下文预填充（Context-prefill）”、“重复预填充（Repeat-prefill）”或“自我学习（Self-study）”生成模型在推理时可能产生的查询向量。
    2.  **Key 选择 ($C_k$)**：从原始 Key 中筛选子集。方法包括简单的“最高注意力分数”或更精确的“正交匹配追踪（OMP）”算法。
    3.  **标量偏置 ($\beta$)**：为每个保留的 Token 引入一个标量，通过**非负最小二乘法（NNLS）**求解，用于补偿因 Token 减少而损失的注意力权重总和（Mass）。
    4.  **Value 拟合 ($C_v$)**：在固定 $C_k$ 和 $\beta$ 后，利用**普通最小二乘法（OLS）**求解闭式解，以最小化注意力输出的 $L_2$ 误差。
    5.  **非均匀分配**：识别不同注意力头对压缩的敏感度，预先计算并为每个头分配不同的 KV 预算（例如，某些头保留更多 Token，某些则更少）。

### 6. 主要结论与发现
*   **帕累托前沿**：注意力匹配（AM）方法在“压缩耗时 vs 压缩质量”上达到了最优平衡，是目前最有效的快速压缩方案。
*   **高压缩比下的韧性**：在 **50 倍压缩**下，AM 方法的性能远超传统的 Token 剔除和摘要方法，且在 Qwen3 等模型上接近无损。
*   **非均匀性至关重要**：实验发现不同注意力头对 KV 缓存的需求差异巨大，采用预计算的非均匀预算分配能显著提升整体性能。
*   **在线压缩可行性**：证明了在长程推理中多次触发压缩（Online Compaction）是可行的，能有效解耦推理深度与物理内存限制。

### 7. 优点
*   **高效性**：将复杂的优化问题转化为线性代数子问题，利用闭式解避免了昂贵的梯度下降。
*   **通用性**：作为一种后处理方法，无需对原始模型进行任何微调或修改。
*   **系统兼容性**：引入的标量偏置（Bias）对推理延迟几乎无影响，且兼容现有的高效注意力算子（如 FlashAttention）。

### 8. 不足与局限
*   **极端压缩瓶颈**：在 100 倍以上的极端压缩比下，其性能仍略逊于 Cartridges，说明在极低预算下，非线性优化（梯度下降）仍有其搜索空间优势。
*   **Query 依赖性**：压缩效果依赖于参考查询（Reference Queries）的质量，如果推理时的 Query 分布与采样时严重偏离，性能可能会受损。
*   **预计算开销**：虽然比训练快，但相比于完全不需计算的简单剔除方法，AM 仍需一定的预填充时间来生成参考查询。
