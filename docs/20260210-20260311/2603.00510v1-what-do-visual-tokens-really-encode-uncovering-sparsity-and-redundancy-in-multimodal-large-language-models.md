## 摘要
多模态大语言模型（MLLMs）将视觉 Token 投影到语言模型的嵌入空间中，但视觉语义的内部结构化和处理过程仍不为人所知。在这项工作中，我们引入了一个双重分析框架，并配备了一种名为 EmbedLens 的新型探测工具，以进行细粒度分析。我们在输入层发现了显著的语义稀疏性：视觉 Token 始终被划分为汇聚（sink）、死亡（dead）和存活（alive）三类。值得注意的是，只有约占总输入 60% 的存活 Token 携带了特定于图像的含义。此外，通过针对性的 Patch 压缩基准测试，我们证明了这些存活 Token 在进入 LLM 之前就已经编码了丰富的细粒度线索（例如物体、颜色和 OCR）。对于大多数标准任务，内部视觉计算（如视觉注意力和前馈网络）是冗余的。对于少数真正受益于内部处理的高度以视觉为中心的任务，我们发现存活 Token 自然地与 LLM 的中间层对齐，而非初始嵌入空间，这表明浅层处理是不必要的，直接进行中层注入就已足够。最终，我们的发现为视觉 Token 处理提供了一个统一的机理视角，通过选择性 Token 剪枝、最小化视觉计算和中层注入，为构建更高效、更具可解释性的 MLLM 架构铺平了道路。代码已发布于：https://github.com/EIT-NLP/EmbedLens。

## Abstract
Multimodal large language models (MLLMs) project visual tokens into the embedding space of language models, yet the internal structuring and processing of visual semantics remain poorly understood. In this work, we introduce a two-fold analytical framework featuring a novel probing tool, $\textbf{EmbedLens}$, to conduct a fine-grained analysis. We uncover a pronounced semantic sparsity at the input level: visual tokens consistently partition into sink, dead, and alive categories. Remarkably, only the alive tokens, comprising $\approx60\%$ of the total input, carry image-specific meaning. Furthermore, using a targeted patch-compression benchmark, we demonstrate that these alive tokens already encode rich, fine-grained cues (e.g., objects, colors, and OCR) prior to entering the LLM. Internal visual computations (such as visual attention and feed-forward networks) are redundant for most standard tasks. For the small subset of highly vision-centric tasks that actually benefit from internal processing, we reveal that alive tokens naturally align with intermediate LLM layers rather than the initial embedding space, indicating that shallow-layer processing is unnecessary and that direct mid-layer injection is both sufficient. Ultimately, our findings provide a unified mechanistic view of visual token processing, paving the way for more efficient and interpretable MLLM architectures through selective token pruning, minimized visual computation, and mid-layer injection. The code is released at: https://github.com/EIT-NLP/EmbedLens.

### 2. 论文提出的方法论
*   **核心思想**：通过宏观聚类分析和微观语义探测相结合的手段，剖析视觉 Token 的功能分区。
*   **关键技术 - EmbedLens**：
    *   这是一种模型无关的探测工具，直接在**输入嵌入空间**测量目标向量与模型词表嵌入矩阵（$W_E$）之间的余弦相似度。
    *   通过检索最相似的 Top-K 个文本 Token，直接识别视觉嵌入中蕴含的离散语义，而无需经过 LLM 的复杂变换。
*   **Token 三分类框架**：
    1.  **汇聚 Token (Sink Tokens)**：图像无关，主要起结构化作用（如稳定注意力分布），不携带图像语义。
    2.  **死亡 Token (Dead Tokens)**：图像无关，且几乎不参与模型计算，属于计算残余。
    3.  **活跃 Token (Alive Tokens)**：靠近文本语义中心，是携带图像特定语义（物体、颜色、OCR 等）的核心载体。
*   **补丁压缩诊断基准**：设计了一个轻量级基准，测试单个视觉 Token 编码多维属性（类别、颜色、计数）的能力。

### 3. 实验设计
*   **模型对象**：LLaVA-1.5 (7B/13B)、Qwen2.5-VL-8B、InternVL-3-8B。
*   **数据集与任务**：
    *   **基础分析**：使用 MS COCO 2017（1万个样本）进行聚类和语义匹配分析。
    *   **下游任务 (Benchmark)**：涵盖通用 VQA (GQA, MME, MMBench, MMStar)、OCR (TextVQA, OCRBench, DocVQA)、视觉中心任务 (RefCOCO/+) 以及幻觉评估 (POPE, HallusionBench)。
*   **对比实验**：
    *   **Token 剪枝**：分别移除 Sink、Dead 和 Alive Token 观察性能变化。
    *   **层解耦 (Layer Decoupling)**：通过训练手段禁用视觉子层（vMHA 或 vFFN）。
    *   **浅层跳过 (Shallow Layer Skipping)**：将视觉 Token 直接注入 LLM 的中间层。

### 6. 主要结论与发现
*   **语义稀疏性**：约 40% 的视觉 Token 是冗余的（Sink 或 Dead），移除它们甚至能提升性能。
*   **预置语义**：活跃 Token 在进入 LLM 之前就已高度“类语言化”，单个 Token 即可编码复杂的组合语义。
*   **计算冗余**：对于大多数任务，LLM 内部的视觉自注意力（vMHA）和前馈网络（vFFN）是多余的，只有 OCR 和精细定位任务需要这些计算。
*   **中层对齐**：视觉 Token 的范数远高于文本，这使得它们在浅层几乎不发生变换，实际上是直接与 LLM 的中间层表示对齐。

### 7. 优点
*   **工具创新**：EmbedLens 提供了一种直观且高效的手段来观察视觉-语言的底层对齐。
*   **机理清晰**：首次明确划分了视觉 Token 的三类功能角色，解释了为什么简单的投影层就能奏效。
*   **架构启示**：为未来的 MLLM 设计提供了明确方向，如视觉 Token 剪枝、跳过浅层计算、直接中层注入等。

### 8. 不足与局限
*   **诊断基准规模较小**：自定义的多轨迹诊断基准仅包含 70 张图像，样本量略显不足。
*   **上下文颜色偏差**：研究发现模型在处理颜色时仍存在严重的上下文偏差（倾向于预测背景色而非物体色），这表明当前的对齐仍不完美。
*   **架构局限**：主要针对基于投影（Projector-based）的 MLLM，对于采用交叉注意力（Cross-attention）或其他融合机制的模型，结论是否适用尚待验证。
