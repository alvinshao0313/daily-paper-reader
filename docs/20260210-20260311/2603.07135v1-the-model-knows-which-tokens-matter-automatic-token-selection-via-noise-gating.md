## 摘要
在视觉语言模型（VLM）中，视觉 token 占据了主要的推理成本，但其中许多 token 携带的是冗余信息。现有的剪枝方法虽然缓解了这一问题，但通常依赖于注意力权重大小或相似度得分。我们将视觉 token 剪枝重新定义为受限容量通信问题：在给定固定预算 K 的情况下，模型必须分配有限的带宽以最大限度地保留视觉信息。我们提出了 AutoSelect，它在冻结的 VLM 上附加了一个轻量级的评分器（Scorer）和去噪器（Denoiser），且仅使用标准的下一个 token 预测损失进行训练，无需辅助目标或额外标注。在训练过程中，一个方差保持噪声门控根据预测的重要性调节每个 token 的信息流，从而使梯度能够传播到所有 token；随后，一个对角注意力去噪器恢复受扰动的表示。在推理时，仅保留评分器和硬性 top-K 选择，增加的延迟微乎其微。在十个 VLM 基准测试中，AutoSelect 在仅增加 0.69 毫秒开销的情况下，保留了完整模型 96.5% 的准确率，同时将 LLM 预填充速度提升了 2.85 倍，并且无需针对特定架构进行调整即可迁移到不同的 VLM 骨干网络。代码可在 https://github.com/MedHK23/AutoSelect 获取。

## Abstract
Visual tokens dominate inference cost in vision-language models (VLMs), yet many carry redundant information. Existing pruning methods alleviate this but typically rely on attention magnitude or similarity scores. We reformulate visual token pruning as capacity constrained communication: given a fixed budget K, the model must allocate limited bandwidth to maximally preserve visual information. We propose AutoSelect, which attaches a lightweight Scorer and Denoiser to a frozen VLM and trains with only the standard next token prediction loss, without auxiliary objectives or extra annotations. During training, a variance preserving noise gate modulates each token's information flow according to its predicted importance so that gradients propagate through all tokens; a diagonal attention Denoiser then recovers the perturbed representations. At inference, only the Scorer and a hard top-K selection remain, adding negligible latency. On ten VLM benchmarks, AutoSelect retains 96.5% of full model accuracy while accelerating LLM prefill by 2.85x with only 0.69 ms overhead, and transfers to different VLM backbones without architecture-specific tuning. Code is available at https://github.com/MedHK23/AutoSelect.

#### 1. 核心问题与整体含义（研究动机和背景）
视觉语言模型（VLM）在处理高分辨率图像或多图/视频场景时，会产生海量的视觉标记（Visual Tokens）。由于大语言模型（LLM）自注意力机制的计算复杂度随序列长度呈平方级增长，这些视觉标记成为了推理延迟和内存占用的主要瓶颈。
*   **研究动机**：现有剪枝方法多依赖于局部启发式信号（如注意力权重、相似度），未能从全局优化角度考虑如何分配有限的计算资源。
*   **核心问题**：在给定固定计算预算 $K$ 的情况下，如何自动识别并保留对下游推理最关键的视觉信息，同时保持极低的推理额外开销？

#### 2. 方法论：核心思想与关键技术
论文将视觉标记剪枝重新定义为**受限容量的表示学习（Capacity-Constrained Representation Learning）**，将视觉编码器与 LLM 之间的接口视为一个带宽受限的信道。

*   **核心思想**：在训练阶段不直接丢弃标记，而是通过注入噪声来限制低分标记的信息通量，使模型在标准预测损失下自发学习识别重要标记。
*   **关键组件**：
    *   **可学习评分器（Scorer）**：由轻量级 Transformer 块组成，为每个标记生成重要性分数。
    *   **软 Top-K 算子（Soft Top-K）**：一种可微的算子，在训练时将分数极化，使其接近二进制掩码，同时允许梯度回传。
    *   **方差保持噪声门控（VP Noise Gate）**：根据分数 $\alpha_i$ 对标记 $x_i$ 注入噪声：$\tilde{x}_i = \sqrt{\alpha_i} x_i + \sqrt{1 - \alpha_i} \epsilon_i$。高分标记保留原信号，低分标记被各向同性高斯噪声淹没。
    *   **对角注意力去噪器（Denoiser）**：在训练中将受扰动的标记映射回 LLM 的特征空间。使用“对角注意力”是为了防止高分标记的信息泄露给低分标记，确保容量限制的有效性。
*   **流程**：训练时通过噪声模拟剪枝；推理时移除去噪器和噪声，直接进行硬性 Top-K 选择。

*   **加速效果**：相比完整模型，LLM 预填充阶段实现了 **2.85 倍** 的加速。

#### 7. 优点（亮点）
*   **端到端优化**：无需手动设计剪枝规则或辅助损失函数，模型通过语言建模目标自发学习重要性。
*   **极低开销**：0.69ms 的处理时间几乎可以忽略不计，是目前已知最高效的端到端剪枝方案之一。
*   **即插即用**：不需要修改 VLM 的原始权重，也不依赖特定的位置编码或 [CLS] 标记。
*   **理论完备**：将剪枝建模为带宽受限的通信问题，并利用 VP 噪声提供平滑的梯度流。

#### 8. 不足与局限
*   **训练依赖**：虽然推理开销低，但仍需要一个专门的训练阶段（在 ImageNet-VL 上），对于完全不想进行任何训练的用户来说存在门槛。
*   **文本无关性**：目前的 Scorer 仅基于视觉特征，不考虑文本提示（Prompt）。虽然这提高了效率（分数可复用），但在某些极端依赖特定文本指令的任务中，可能会误删指令相关的视觉细节。
*   **静态预算**：推理时需要预设固定的 $K$ 值，未能实现根据图像复杂度动态调整保留标记的数量。
